# Governed AI Operations Doctrine

An open doctrine for operating AI in business workflows with source
clarity, human approval, runtime governance, evidence, and proof of
value.

Created and maintained by **Dealix**.

This repository provides doctrine, controls, and adoption checklists.
It does **not** provide commercial Dealix implementation code.

This is **not** legal advice, regulatory certification, or compliance
approval.

## Contents

- `11_NON_NEGOTIABLES.md` — the eleven commitments any team adopting
  this doctrine must accept.
- `CONTROL_MAPPING.md` — how each commitment maps to a verifiable
  technical or operational control.

## License & Use

The doctrine is published openly so that buyers, partners, regulators,
and auditors can verify it. Commercial Dealix products, scoring
algorithms, customer data, partner pipelines, pricing schedules, and
investor materials are **not** part of this repository.

## Status

This is an evolving doctrine. Changes are tracked in git history.
Production claims should reference a specific commit SHA.
