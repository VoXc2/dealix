# Governed AI Operations Doctrine

An open doctrine for operating AI in business workflows with source clarity, human approval, runtime governance, evidence, and proof of value.

Created and maintained by Dealix.

This repository provides doctrine, controls, and adoption checklists.
It does not provide commercial Dealix implementation code.

This is not legal advice, regulatory certification, or compliance approval.

## What This Doctrine Covers
- source clarity before AI use
- workflow ownership
- human approval before external action
- runtime governance before client-facing output
- proof of value before claims
- agent identity before agent autonomy
- responsible AI operations across regulated and B2B services contexts

## What This Doctrine Does Not Cover
- prescriptive legal interpretation of any jurisdiction's AI regulation
- model selection guidance
- vendor endorsement
- compliance certification

## How To Use
1. Read `11_NON_NEGOTIABLES.md`.
2. Map each non-negotiable to a control in your environment using `CONTROL_MAPPING.md`.
3. Run an internal review with your governance, security, and delivery owners.
4. Document gaps in your own risk register.

## License Posture
This doctrine is shared openly so that buyers, partners, and operators can verify what governed AI operations actually mean in practice. It is not a substitute for an internal AI policy.

## Source of Truth
Dealix maintains the canonical version of this doctrine and may update it as practice evolves. Verification endpoints are provided publicly by Dealix for the items where machine-readable verification is possible.
