# 06 — Subagents: التفويض الذكي للمهام الثقيلة

## المبدأ

Subagents تعمل في context window منفصل.
كل ما يفعله الـ Subagent (قراءة ملفات، بحث، استكشاف) يبقى في context منفصل.
فقط **الملخص** يعود للمحادثة الرئيسية.

## متى تستخدم Subagents؟

```
هل المهمة تتطلب قراءة > 5 ملفات كبيرة؟  → Subagent
هل المهمة استكشافية (بحث، تحليل)؟        → Subagent
هل المهمة يمكن تقسيمها بالتوازي؟          → Subagents متعددة
هل المهمة بسيطة ومحددة؟                   → بقاء في المحادثة الرئيسية
```

## أمثلة استخدام

### التوازي للاستكشاف

```
ابحث بالتوازي عن:
1. كل مواضع استخدام ClientModel في المشروع
2. كل API endpoints المتعلقة بـ clients
3. كل اختبارات clients

ثم أعطني ملخصاً موحداً في 10 سطور.
```

### تفويض المهمة الثقيلة

```
قبل أن تقرأ أي ملف، فوّض subagent لـ:
- قراءة كل ملفات api/routes/ وتلخيصها
- العودة بقائمة: [endpoint, method, auth_required]

لا أحتاج تفاصيل الكود، فقط القائمة.
```
