# أنماط Subagents الأكثر فاعلية

## النمط 1: استكشاف متوازٍ

```
استكشف بالتوازي في subagents منفصلة:
- Subagent A: افحص core/ وأعطني ملخص كل module (5 سطور لكل module)
- Subagent B: افحص api/ وأعطني قائمة كل endpoints مع types
- Subagent C: افحص tests/ وأعطني نسبة التغطية لكل module

اجمع النتائج في ملخص واحد.
```

## النمط 2: مراجعة الكود الضخمة

```
فوّض subagent لمراجعة هذا الـ PR:
- اقرأ كل الملفات المتغيرة
- أعطني: [ملف] → [مشاكل محتملة] فقط
- لا تشرح الكود الجيد
- ملاحظات فقط، بدون context غير ضروري
```

## النمط 3: البحث عن أنماط

```
في subagent منفصل:
- ابحث عن كل مواضع استخدام raw SQL في المشروع
- أعطني: [ملف:سطر] → [الاستعلام] فقط
- لا تقرأ أي ملف غير ضروري
```

## النمط 4: التوثيق من Subagent

```
فوّض subagent لـ:
- قراءة api/routes/clients.py
- توليد OpenAPI schema بـ YAML
- العودة بالـ YAML فقط، لا شرح
```

## النمط 5: مهام مستقلة متوازية

```
شغّل 3 subagents بالتوازي:
1. "أضف error handling لـ api/routes/deals.py"
2. "أضف type hints لـ core/services/client.py"
3. "أنشئ اختبارات لـ core/utils.py"

كل واحد يعمل على ملفات مختلفة، لا تعارض.
```

## القيود

- لا تُفوّض مهاماً تعتمد على نتائج بعضها (Subagents لا تتواصل)
- كل Subagent يحتاج context كافٍ في برومبته (لا يرث المحادثة الرئيسية)
- اجعل برومبت Subagent مكتفياً بذاته

## مثال برومبت Subagent مكتفٍ

```
أنت تعمل على مشروع Python FastAPI اسمه Dealix.
المطلوب: اقرأ الملف `api/routes/clients.py` وأعطني:
1. قائمة كل endpoints (method, path, auth_required)
2. أي مشاكل واضحة (بدون شرح الكود الجيد)

أعطني النتيجة كـ JSON فقط، بدون نص إضافي.
```
