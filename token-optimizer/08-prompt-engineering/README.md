# 08 — Prompt Engineering: صياغة البرومبت بكفاءة

## المبدأ

برومبت كفؤ = نتيجة في رسالة واحدة = توكنز أقل.
برومبت غامض = 3-5 رسائل توضيحية = 3-5x التكلفة.

## الملفات في هذا المجلد

| الملف | الوصف |
|-------|--------|
| `prompt-templates.md` | قوالب برومبت جاهزة لمهام شائعة |
| `anti-verbose.md` | كيف تمنع الشرح المُطوَّل |
| `reference-by-line.md` | الإشارة للكود بالسطر لا بالنسخ |
| `diff-style-output.md` | طلب مخرجات بأسلوب diff |
| `batching-questions.md` | تجميع الأسئلة في رسالة واحدة |

## القواعد الأساسية

### 1. امنع الشرح المُطوَّل
```
# أضف هذا في نهاية برومبتك:
No explanations. Code only. Minimal output.
```
**التوفير**: 30-50% من توكنز الاستجابة

### 2. اطلب diff لا rewrite كاملة
```
أعطني التغييرات فقط بأسلوب diff.
لا تُعيد كتابة الملف كله.
```
**التوفير**: 95% من الاستجابة لكود كبير

### 3. أشر بالسطر، لا تنسخ الكود
```
# ❌ مُكلف
هذا هو الكود:
[100 سطر من الكود]
أصلح...

# ✅ فعّال
في `auth.py` السطر 45-67، أصلح...
```

### 4. جمّع الأسئلة
```
# ❌ 3 رسائل منفصلة = 3x التكلفة
رسالة 1: ما هو Pattern الصحيح لـ async?
رسالة 2: كيف أتعامل مع errors?
رسالة 3: هل يجب استخدام try/except؟

# ✅ رسالة واحدة
أجب على الثلاثة معاً:
1. Pattern الصحيح لـ async في FastAPI
2. أفضل طريقة لـ error handling
3. متى يجب try/except vs HTTPException
```

## جدول التوفير

| التقنية | توكنز الاستجابة قبل | بعد | التوفير |
|---------|-------------------|----|---------|
| منع الشرح | 1,000 | 500 | 50% |
| diff بدل rewrite | 3,000 | 150 | 95% |
| إشارة بالسطر | 500 | 50 | 90% |
| تجميع الأسئلة | 3,000 | 1,200 | 60% |
