# منع الشرح المُطوَّل — توفير 30-50% من الاستجابة

## عبارات "لا شرح" الجاهزة

أضف إحدى هذه العبارات في نهاية برومبتك:

```
No explanations. Code only.
لا شرح. الكود فقط.
Minimal output. Just the code.
No comments in the code. No explanations.
Only the changed lines. No context.
Diff format only.
Return only the function/class. No surrounding text.
```

---

## أنواع الاستجابات وكيف تختصرها

### استجابة تلقائية طويلة (مُكلفة)
```
قبل البدء، دعني أفهم المشكلة...
المقاربة التي سأستخدمها هي...
أولاً، سأراجع الكود الحالي...
[شرح مطوَّل]
هذا هو الحل:
[الكود]
وسبب نجاح هذا النهج هو...
```

### كيف تمنعها
```
[برومبتك] — No preamble. No explanation. Start directly with the solution.
```

---

## طلب مخرجات مضغوطة

### بدلاً من قائمة شارحة:
```
❌ أعطني قائمة بمشاكل الكود مع شرح كل مشكلة وكيفية حلها

✅ أعطني: [سطر]: [مشكلة] — بدون شرح
```

### بدلاً من refactor كامل:
```
❌ أعد كتابة الملف مع إضافة التحسينات

✅ أعطني diff فقط للتغييرات المطلوبة
```

### بدلاً من توضيح مطوَّل:
```
❌ اشرح كيف يعمل هذا الكود

✅ لخّص في 3 سطور: ما الذي يفعله هذا الكود
```

## نماذج طلب مضغوط حسب المهمة

| المهمة | الطريقة المُكلفة | الطريقة الفعّالة |
|--------|-----------------|-----------------|
| شرح كود | "اشرح هذا الكود" | "لخّص في 3 سطور" |
| مراجعة PR | "راجع هذا الـ PR" | "اذكر المشاكل فقط، 1 سطر لكل مشكلة" |
| اقتراحات | "ما اقتراحاتك؟" | "أعطني top 3 فقط، بدون شرح" |
| debugging | "ساعدني في debugging" | "في السطر [X]، ما سبب [الخطأ المحدد]؟" |
| تصميم | "كيف أُصمم X؟" | "أعطني النهج الأمثل في جملة واحدة والمقايضة الرئيسية" |

## تأثير عبارة واحدة

```python
# برومبت بدون "No explanations"
"أضف error handling لهذه الدالة"
# نتيجة: 500-800 توكن (شرح + كود + توضيح)

# برومبت مع "No explanations"
"أضف error handling لهذه الدالة. No explanations. Code only."
# نتيجة: 100-200 توكن (الكود فقط)
```

**التوفير: 60-75% من توكنز الاستجابة بإضافة 5 كلمات.**
