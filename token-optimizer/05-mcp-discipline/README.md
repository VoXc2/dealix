# 05 — MCP Discipline: ضبط خوادم MCP

## المشكلة

كل خادم MCP مُتصل يضيف **100-500 توكن** لكل رسالة، حتى لو لم تستخدمه.
في جلسة 100 رسالة مع 5 خوادم غير مستخدمة = **50,000 توكن مُهدرة**.

## الملفات في هذا المجلد

| الملف | الوصف |
|-------|--------|
| `mcp-audit.md` | كيف تُدقق خوادم MCP الحالية |
| `mcp-profiles.md` | ملفات تشغيل لكل نوع مهمة |
| `mcp-minimal-config.json` | تكوين MCP بالحد الأدنى الضروري |
| `cli-alternatives.md` | بدائل CLI لكل MCP شائع |

## الأوامر الأساسية

```bash
/mcp              # قائمة بكل الخوادم المتصلة
/mcp disable X    # تعطيل خادم غير مستخدم
/mcp enable X     # تفعيل خادم عند الحاجة
```

## المبدأ

> **CLI tool يُعيد 10 سطور = 10 توكن**
> **نفس MCP request قد يُعيد 1,000 توكن**

## القاعدة

- اعمل بالحد الأدنى من خوادم MCP
- فعّل الخادم عند الحاجة فقط
- عطّله فوراً بعد الانتهاء
- استخدم CLI بديلاً كلما أمكن

## مثال: GitHub MCP vs gh CLI

```bash
# ❌ MCP (300-500 توكن لكل طلب)
[MCP] list open PRs in dealix-sa/dealix

# ✅ CLI (5-20 توكن)
gh pr list --state open
```

## خوادم MCP ضرورية لـ Dealix

| الخادم | ضروري؟ | البديل |
|--------|--------|--------|
| github | نعم (للـ PR) | `gh` CLI لمعظم العمليات |
| filesystem | لا | `read/write` tools مباشرة |
| postgres | أحياناً | `psql` CLI |
| notion | حسب المهمة | `notion` CLI |
