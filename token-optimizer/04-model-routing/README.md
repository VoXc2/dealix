# 04 — Model Routing: اختيار النموذج الصحيح

## الأثر على التكلفة

| النموذج | التكلفة النسبية | متى تستخدمه |
|---------|-----------------|-------------|
| Haiku | 1x (الأرخص) | مهام ميكانيكية، تنسيق، إعادة تسمية |
| Sonnet | 5x | الغالبية العظمى من المهام (80%) |
| Opus | 25x | تحليل معقد، refactor ضخم |

**معظم المطورين يُبالغون في استخدام Opus عندما Sonnet يكفي 80% من الوقت.**

## الملفات في هذا المجلد

| الملف | الوصف |
|-------|--------|
| `decision-tree.md` | شجرة القرار: أي نموذج لأي مهمة؟ |
| `haiku-tasks.md` | قائمة مهام Haiku مع أمثلة |
| `sonnet-tasks.md` | قائمة مهام Sonnet مع أمثلة |
| `opus-tasks.md` | قائمة مهام Opus مع أمثلة |
| `subagent-model-config.md` | إعداد نماذج Subagents |

## التبديل السريع

```bash
/model haiku     # للمهام البسيطة
/model sonnet    # العودة للافتراضي
/model opus      # للمهام المعقدة فقط
```

## قاعدة الـ 80/15/5

- **80% من المهام**: Sonnet يكفي
- **15% من المهام**: Haiku أسرع وأرخص
- **5% من المهام فقط**: تستحق Opus

## توفير مثال حقيقي

فريق يعمل 100 ساعة شهرياً على Claude Code:
- **قبل (كل شيء بـ Opus)**: $400/شهر
- **بعد (توجيه صحيح)**: $80/شهر
- **التوفير**: $320/شهر (80%)
