# سير عمل الجلسة المثالية

## قبل بدء الجلسة ✅

```markdown
[x] حدد المهمة الواحدة التي ستعمل عليها
[x] اعرف الملفات المعنية (2-3 ملفات كحد أقصى)
[x] اكتب brief المهمة باستخدام القالب
[x] تأكد من .claudeignore موجود ومحدّث
[x] تأكد من CLAUDE.md < 500 توكن
```

## فتح الجلسة — الرسالة الأولى

```
أحتاج مساعدتك في [وصف محدد للمهمة].

الملفات المعنية:
- `api/routes/clients.py` (endpoint الذي يحتاج تعديل)
- `core/models/client.py` (model)

المطلوب: [وصف دقيق]
القيود: [أي قيود تقنية]
معيار الاكتمال: [كيف نعرف أنه انتهى]
```

## أثناء الجلسة

### مراقبة السياق
```bash
/cost    # تحقق بعد كل مهمة كبرى
```

### عند 70% من السياق
```
/compact

قبل الضغط، احتفظ بهذه النقاط الأساسية:
- ما أنجزناه: [...]
- القرار الرئيسي: [...]
- الخطوة التالية: [...]
```

### تبديل المهمة
```bash
/clear   # مسح كامل للمحادثة
```

## إغلاق الجلسة

```
قبل الانتهاء، اعطني session handoff يتضمن:
1. ما أنجزناه
2. القرارات المهمة مع أسبابها
3. الخطوات التالية
4. أي مشاكل قد تظهر لاحقاً
```

احفظ الـ handoff في `session-notes.md` لتبدأ به الجلسة القادمة.

## قواعد لا تُكسر

1. **مهمة واحدة في كل جلسة** — لا "وبعدين عدّل كذا أيضاً"
2. **لا تُعيد لصق الكود** — اذكر الملف والسطر
3. **لا تطلب شرحاً** ما لم تحتجه — "No explanations, just the code"
4. **لا تسأل عما تعرفه** — Claude يحتفظ بالسياق
5. **اضغط عند 70%** — لا تنتظر حتى النهاية
