# 03 — إدارة الجلسات بكفاءة

## المبدأ الأساسي

Claude يُعيد قراءة كامل المحادثة من البداية في **كل رسالة**.
الرسالة رقم 50 تُكلف أكثر من الرسالة رقم 5 بكثير.

## الأوامر الأساسية

| الأمر | متى تستخدمه | التأثير |
|-------|------------|---------|
| `/compact` | السياق وصل 70% | يُلخص المحادثة ويُقلصها |
| `/clear` | تغيير المهمة كلياً | يُصفّي كل شيء |
| `/cost` | بعد كل مهمة كبيرة | يُظهر الاستهلاك |
| `/model haiku` | مهام ميكانيكية | يُقلل التكلفة 5x |
| `/model sonnet` | مهام عامة | الافتراضي الأمثل |
| `/resume <name>` | استئناف جلسة مسماة | بدون إعادة الشرح |

## قاعدة الـ 70%

```
استخدم /compact عندما يصل السياق 70%، لا 95%
```

**لماذا 70% وليس 95%؟**
عند 95%، Claude يبدأ في "نسيان" التعليمات الأولى.
عند 70%، الضغط يحتفظ بكل السياق المهم.

## سير الجلسة المثالية

```
1. بدء الجلسة: مهمة واحدة فقط
   ↓
2. تحديد النطاق: 2-3 ملفات محددة، لا مجلدات
   ↓
3. العمل: مراقبة /cost بعد كل خطوة كبرى
   ↓
4. عند 70%: تشغيل /compact مع "memory anchor"
   ↓
5. إتمام المهمة: طلب session handoff
   ↓
6. مهمة جديدة: /clear أو جلسة جديدة
```
