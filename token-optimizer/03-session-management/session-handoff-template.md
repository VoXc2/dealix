# قالب Session Handoff

## الاستخدام

في نهاية كل جلسة، اطلب من Claude:

```
اكتب session handoff باستخدام القالب التالي
```

---

## القالب

```markdown
# Session Handoff — [التاريخ] — [اسم المهمة]

## ما أُنجز
- [إنجاز 1]
- [إنجاز 2]

## الملفات التي تغيرت
- `path/to/file.py` — [وصف التغيير]
- `path/to/other.py` — [وصف التغيير]

## القرارات المهمة
| القرار | السبب | البديل المرفوض |
|--------|-------|----------------|
| [قرار] | [لماذا] | [البديل] |

## الخطوات التالية
1. [خطوة 1] — مستوى الأولوية: عالي/متوسط/منخفض
2. [خطوة 2]
3. [خطوة 3]

## تحذيرات ومشاكل محتملة
- [تحذير 1]: [التفاصيل]
- [تحذير 2]: [التفاصيل]

## بدء الجلسة القادمة
أقرأ هذا الملف ثم قل:
"أكمل من حيث توقفنا: [الخطوة التالية الأولى]"
```

---

## مثال حقيقي

```markdown
# Session Handoff — 2026-05-30 — إضافة نقطة نهاية تصدير العملاء

## ما أُنجز
- أضفت `GET /api/v1/clients/export` يُعيد CSV
- الاختبار الأساسي في `tests/test_clients.py::test_export_clients`

## الملفات التي تغيرت
- `api/routes/clients.py` — أضفت export endpoint (السطر 145-178)
- `core/services/export.py` — **ملف جديد** - خدمة التصدير
- `tests/test_clients.py` — أضفت test_export_clients

## القرارات المهمة
| القرار | السبب | البديل المرفوض |
|--------|-------|----------------|
| استخدام streaming response | الملفات الكبيرة > 10MB | تحميل كامل في الذاكرة |
| UTF-8 BOM في CSV | Excel يحتاجه للعربية | UTF-8 عادي |

## الخطوات التالية
1. إضافة rate limiting على endpoint التصدير — عالي
2. اختبار مع بيانات > 10,000 سجل — متوسط
3. توثيق API في OpenAPI schema — منخفض

## تحذيرات
- export يستهلك ذاكرة عند البيانات الضخمة، راجع streaming
- Excel 2016 لا يدعم CSV بدون BOM

## بدء الجلسة القادمة
"أكمل من حيث توقفنا: أضف rate limiting على GET /api/v1/clients/export"
```
