# علامات تحذيرية: أين تذهب التوكنز؟

## 🔴 علامات حرجة

### تكلفة 10x لنفس المهمة
**العلامة**: نفس المهمة التي كانت تكلف 1,000 توكن أصبحت 10,000.
**السبب**: مشكلة caching أو CLAUDE.md تضخّم.
**الحل**:
```bash
/cost             # تحقق من التكلفة
wc -l CLAUDE.md   # تحقق من الحجم
```

### "Chat has reached its limit"
**العلامة**: رسالة النهاية.
**السبب**: وصلت نهاية سياق الجلسة.
**الحل**: `/clear` ثم بدء جلسة جديدة مع Handoff.

### 500 errors / ECONNRESET
**العلامة**: أخطاء متكررة من الـ server.
**السبب**: الطلب كبير جداً (context فائض).
**الحل**: `/compact` فوراً.

## 🟡 علامات تحذيرية

### رسائل توضيحية متكررة
**العلامة**: Claude يطرح 3+ أسئلة توضيحية لكل مهمة.
**السبب**: البرومبت غير محدد.
**الحل**: استخدام Task Brief Template.

### قراءة ملفات كبيرة غير ضرورية
**العلامة**: `/context` يُظهر ملفات لا علاقة لها بالمهمة.
**السبب**: طلب "اقرأ المشروع" أو "افهم الكود".
**الحل**: حدد الملفات والسطور بدقة.

### مخرجات bash طويلة
**العلامة**: أوامر مثل `npm install` أو `docker logs` تملأ السياق.
**السبب**: لا hooks لتقليص المخرجات.
**الحل**: تفعيل hooks من `07-hooks/`.

### جلسة طويلة بمهام متعددة
**العلامة**: في جلسة واحدة تعمل على features مختلفة.
**السبب**: تراكم السياق من مهام سابقة غير مرتبطة.
**الحل**: `/clear` بين كل مهمة مختلفة.

## كيفية تشخيص المشكلة

```bash
# 1. افحص ما يملأ السياق
/context

# 2. افحص التكلفة الحالية
/cost

# 3. افحص حجم CLAUDE.md
wc -l CLAUDE.md
wc -w CLAUDE.md

# 4. افحص .claudeignore
cat .claudeignore | grep -c "^[^#]"  # عدد الأنماط الفعّالة

# 5. افحص خوادم MCP
/mcp
```
