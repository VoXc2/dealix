# 10 — Tools & Monitoring: أدوات القياس والمراقبة

## لماذا المراقبة ضرورية؟

"ما لا يُقاس لا يُحسَّن."
بدون مراقبة، لن تعرف أين تذهب التوكنز ولا تأثير أي تحسين.

## الملفات في هذا المجلد

| الملف | الوصف |
|-------|--------|
| `built-in-commands.md` | أوامر Claude المُدمجة للمراقبة |
| `external-tools.md` | أدوات خارجية موصى بها |
| `cost-tracking.md` | كيفية تتبع التكاليف |
| `red-flags.md` | علامات تدل على هدر التوكنز |
| `optimization-checklist.md` | قائمة تحقق شاملة |

## الأوامر المُدمجة الضرورية

```bash
/cost      # الاستهلاك الحالي للجلسة
/context   # ما الذي يشغل السياق؟
/compact   # ضغط عند 70%
/clear     # مسح كامل
/model     # النموذج الحالي
```

## أدوات خارجية (موصى بها)

| الأداة | الوصف | التثبيت |
|--------|--------|---------|
| `ccusage` | تتبع الاستهلاك بالمشروع والوقت | `npm i -g ccusage` |
| `claude-hud` | عرض توكنز حي في الـ terminal | marketplace |
| `claude-token-lens` | تحليل توزيع التوكنز | marketplace |
| `claude-mem` | ذاكرة دائمة بين الجلسات | `npx claude-mem install` |

## مؤشرات الخطر

🔴 **حرج**: تكلفة نفس المهمة ارتفعت 10x → مشكلة caching
🔴 **حرج**: "Chat has reached its limit" → وصلت نهاية السياق
🟡 **تحذير**: السياق > 70% → شغّل /compact الآن
🟡 **تحذير**: رسائل توضيحية متكررة → أعد صياغة البرومبت
🟢 **جيد**: /cost يُظهر تكلفة ثابتة أو متناقصة

## روتين المراقبة اليومي

```bash
# 1. قبل الجلسة: فحص الإعدادات
cat .claudeignore | wc -l    # تأكد من وجود .claudeignore
wc -l CLAUDE.md               # تأكد < 150 سطر

# 2. أثناء الجلسة: راقب عند كل مهمة كبرى
/cost

# 3. عند 70% سياق
/compact

# 4. نهاية الجلسة: سجّل الاستهلاك
/cost > session-log/$(date +%Y%m%d).txt
```
