# قائمة التحقق الشاملة لتحسين التوكنز

## قائمة الإعداد (مرة واحدة)

### ملفات الاستبعاد
- [ ] `.claudeignore` موجود في جذر المشروع
- [ ] `node_modules/` مُستبعدة
- [ ] `__pycache__/` مُستبعدة
- [ ] `dist/` و `build/` مُستبعدة
- [ ] `*.lock` مُستبعدة
- [ ] `coverage/` مُستبعدة
- [ ] `*.log` مُستبعدة
- [ ] ملفات `.env.*` الحساسة مُستبعدة

### CLAUDE.md
- [ ] الحجم < 150 سطر (< 500 توكن)
- [ ] لا تاريخ مشروع
- [ ] لا schema قاعدة بيانات كاملة
- [ ] لا أمثلة كود طويلة
- [ ] Skills في ملفات منفصلة
- [ ] Quick Reference موجود (< 50 سطر)

### الإعدادات
- [ ] Hooks للتحكم في حجم المخرجات
- [ ] `CLAUDE_CODE_SUBAGENT_MODEL=haiku` مضبوطة
- [ ] `MAX_THINKING_TOKENS` محدودة للمهام البسيطة
- [ ] خوادم MCP غير المستخدمة مُعطَّلة

## قائمة تحقق الجلسة (كل جلسة)

### البداية
- [ ] المهمة واضحة ومحددة (مهمة واحدة)
- [ ] الملفات المعنية محددة مسبقاً (2-3 ملفات)
- [ ] استخدام Task Brief Template
- [ ] النموذج الصحيح مختار (Haiku/Sonnet/Opus)

### أثناء العمل
- [ ] مراقبة /cost بعد كل مهمة كبرى
- [ ] الإشارة للكود بالسطر لا بالنسخ
- [ ] "No explanations" في نهاية البرومبت
- [ ] طلب diff بدلاً من rewrite كامل
- [ ] تجميع الأسئلة في رسالة واحدة

### عند 70% سياق
- [ ] تشغيل /compact مع memory anchor
- [ ] التحقق من الإبقاء على السياق المهم

### نهاية الجلسة
- [ ] طلب Session Handoff
- [ ] حفظ الـ Handoff في `session-notes.md`
- [ ] /clear أو جلسة جديدة للمهمة التالية

- [ ] مراجعة ccusage للمشروع (أين ذهبت التوكنز؟)
- [ ] تحديث .claudeignore إذا ظهرت أنماط جديدة
- [ ] تقليم CLAUDE.md إذا تضخّم
- [ ] مراجعة خوادم MCP المتصلة (تعطيل غير المستخدمة)
- [ ] مراجعة session-notes.md وضغطها

## معادلة التوفير الشاملة

```
التوفير = 
  تحسين .claudeignore (40-90%)
+ تحسين CLAUDE.md (30-60%)
+ إدارة جلسة صحيحة (20-50%)
+ اختيار نموذج مناسب (50-80% على المهام المناسبة)
+ hooks للمخرجات (60-90% على أوامر bash)
+ subagents للمهام الثقيلة (30-70%)
+ prompt engineering (30-50%)
--------------------------------------
المجموع: 60-90% توفير إجمالي
```

## هدف الأداء

| المقياس | الهدف |
|---------|-------|
| حجم CLAUDE.md | < 500 توكن |
| توكنز/جلسة (مهمة صغيرة) | < 10,000 |
| توكنز/جلسة (مهمة متوسطة) | < 30,000 |
| توكنز/جلسة (مهمة كبيرة) | < 80,000 |
| نسبة جلسات تستخدم /compact | > 50% |
