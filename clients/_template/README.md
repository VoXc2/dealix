# Client Delivery Lifecycle

This template defines the standard delivery path for every Dealix client engagement.

## Delivery Phases

```
01_intake → 02_diagnosis → 03_blueprint → 04_sprint_plan → 05_training → 06_proof_pack
```

### Phase 1: Intake
Collect the essential facts about the client's business, team, current tools, and pain points.
No promises. No scope. Just listening and documenting.

### Phase 2: Diagnosis
Analyze the intake data against Dealix's operating models.
Identify where revenue leaks, decision gaps, follow-up failures, and visibility problems exist.
Output: a written operating diagnosis the client can review.

### Phase 3: Blueprint
Design the specific operating systems to build for this client.
Map which Dealix modules apply, what order to build them in, and what success looks like.
Output: a reviewable operating blueprint tied to specific outcomes.

### Phase 4: Sprint Plan
Break the blueprint into concrete sprints (typically 1-2 week cycles).
Define what gets delivered, who owns what, and how progress is tracked.
Output: a sprint-by-sprint delivery plan.

### Phase 5: Training
Hand off the built systems to the client's team.
Train on daily operations: Command Room usage, Brain review cadence, approval workflows.
Output: documented SOPs and recorded walkthroughs.

### Phase 6: Proof Pack
Compile evidence of what was built, what changed, and what the client can now do.
No fabricated ROI. No fake metrics. Only observable, verifiable operational improvements.
Output: a proof pack the client can share internally or with stakeholders.

## Usage

1. Copy `_template/` to `clients/{client_name}/`
2. Fill each template in order
3. Use the templates as living documents — update as the engagement progresses
4. Archive completed engagements for future reference

## Operating Principles

- Start with the smallest credible scope
- Keep the client's founder or decision-maker in the loop at every phase
- Do not promise outcomes that depend on external factors
- Document everything that was actually delivered
- Keep compliance and governance considerations visible throughout
