# Outreach Review Workflow (Dealix)

## The 4 states of every draft
1. `draft_pending_human_review` — generated, waiting for review
2. `approved` — founder approved; can be sent manually
3. `rejected` — founder rejected with reason
4. `not_started` — no draft yet (account not yet drafted)

## Reviewer (always)
- Founder only. No exceptions.

## Review steps
1. Open the draft
2. Read the opener + follow-ups
3. Check the source note (URL or quote)
4. Decide: approve / reject / edit
5. Record: reviewer + reviewed_at + reason (if reject)

## Approved ≠ sent
- Approved means: "I would send this"
- It does NOT mean: "the system will send this"
- Sending is always manual

## When to reject
- Too generic
- Mentions private info
- Promises something we can't measure
- Wrong segment
- Wrong language
- Wrong tone
