# Data Handling Checklist — PDPL Compliant

---

## Before Receiving Client Data

- [ ] Confirm client owns the data or has rights to share
- [ ] Verify no third-party PII included
- [ ] Document data types being shared
- [ ] Set retention period (default: 90 days)
- [ ] Confirm deletion process post-engagement

## Data Receipt

- [ ] Transfer via secure method (encrypted email / secure drive)
- [ ] Do NOT use personal email or public file sharing
- [ ] Scan for sensitive PII
- [ ] Anonymize where possible before analysis
- [ ] Store in client-specific folder

## During Analysis

- [ ] Do NOT paste client data into public AI tools
- [ ] Use local/self-hosted tools when possible
- [ ] Mask names and identifying information
- [ ] Work with aggregated data where possible
- [ ] Log all AI actions in ai_action_ledger

## After Engagement

- [ ] Delete all raw client data after 90 days
- [ ] Keep only anonymized insights with consent
- [ ] Confirm deletion with client
- [ ] Archive proof packs (redacted)

## SDAIA PDPL Requirements

| Requirement | Status | Notes |
|-------------|--------|-------|
| Lawful basis for processing | ✅ Consent | Client engagement agreement |
| Purpose limitation | ✅ Defined | Revenue intelligence analysis only |
| Data minimization | ✅ Active | Only request necessary data |
| Retention limit | ✅ 90 days | Auto-deletion scheduled |
| Security measures | ✅ Encrypted storage | Local processing |
| Breach notification | ✅ Documented | 72-hour notification process |
| Data subject rights | ✅ Supported | Client can request deletion anytime |
