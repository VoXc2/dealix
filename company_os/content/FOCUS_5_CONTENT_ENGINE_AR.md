# محرك المحتوى للأنظمة الخمسة — Focus 5 Content Engine

لا تعتمد على الإيميل وحده. كل نظام له محتوى أسبوعي ثابت. المصدر:
`company_os/commercial/content_calendar.json`. القائمة المولّدة:
`reports/content/FOCUS_5_CONTENT_QUEUE.md` عبر `npm run commercial:brief`.

---

## المحتوى الأسبوعي لكل نظام

| النظام | المحور الأسبوعي |
|--------|------------------|
| نظام تشغيل الإيرادات | لماذا تضيع الفرص قبل العرض؟ |
| نظام القيادة التنفيذية | التقارير لا تكفي إذا لا تعطي قرارًا |
| نظام استرجاع المتابعات | آخر متابعة لم تحدث قد تكون أغلى فرصة |
| نظام عملاء واتساب | واتساب ليس Inbox؛ هو workflow |
| نظام العروض والإثبات | العرض المقنع يحتاج Proof وليس كلامًا أكثر |

---

## المحتوى اليومي الخفيف

```
1 founder insight
1 system pain post
1 example workflow
1 objection answer
1 mini case-style post
```

---

## قاعدة الإثبات: لا case studies وهمية

ممنوع اختلاق دراسات حالة أو نسب نتائج لعملاء بأسمائهم دون موافقة. المسموح فقط:

```
مثال توضيحي
نموذج افتراضي
Workflow sample
Before/after template
```

هذه القاعدة متسقة مع منع الادعاءات المضمونة في بوابة الجودة وسياسة الأمان.
