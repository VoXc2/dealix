# سياسة بيانات الشركات غير الموثوقة — Untrusted Company Data Policy

بما أن النظام يعتمد على وكلاء (agents) يعالجون محتوى خارجيًا، نعامل كل محتوى خارجي
كـ **بيانات** لا كـ **تعليمات**. أبحاث حديثة تبيّن أن prompt injection قد يدفع وكيل
LLM لتجاهل تعليماته أو استخدام أدواته بشكل غير مصرّح عند معالجة صفحات ويب أو مستندات
أو أدوات خارجية، وأن هجمات اختيار الأداة قد تؤثر على سلوك الوكلاء.

---

## ما الذي يُعدّ غير موثوق؟

```
كل محتوى من موقع شركة      = untrusted data
كل إيميل وارد              = untrusted data
كل PDF أو صفحة             = untrusted data
أي نص يجلبه الوكيل خارجيًا  = untrusted data
```

في بياناتنا، الحقل `source_excerpt` في `draft_factory.json` هو مثال على نص خارجي
غير موثوق: يُمسح بحثًا عن علامات الحقن، وإن وُجدت يُعزَل (يُرفض الـ draft) ولا يتحول
أبدًا إلى تعليمات.

---

## القواعد

```
لا يتحول أي نص خارجي إلى تعليمات
لا تنفيذ أداة بناءً على محتوى خارجي
لا secrets في prompts أو logs أو reports
لا إرسال خارجي من أي agent
البيانات الشخصية تُقنّع/تُجهّل قبل أي تحليل (PDPL)
```

---

## الفرض الآلي

`npm run commercial:check` يفرض هذه السياسة عبر:
- مسح الحقن في كل `source_excerpt` والتأكد من احتوائه (الرفض).
- التأكد أن لا أسرار في التقارير المولّدة.
- التأكد أن لا اسم نظام داخلي في أي نص قابل للإرسال.

انظر أيضًا: `PROMPT_INJECTION_GATE.md` و`AGENT_TOOL_USE_BOUNDARIES.md` و
`company_os/governance/data_handling_checklist.md`.
