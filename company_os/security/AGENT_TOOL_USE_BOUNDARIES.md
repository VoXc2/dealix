# حدود استخدام الأدوات للوكلاء — Agent Tool-Use Boundaries

ما الذي يُسمح للوكيل بفعله وما لا يُسمح. متوافق مع
`company_os/governance/agent_permissions.md` (مصفوفة الصلاحيات والخطوط الحمراء).

---

## مسموح (Observe / Advise / Draft)

```
البحث في بيانات عامة
تحليل وتسجيل النقاط
كتابة مسودات الإيميل والعروض
ترتيب القوائم والطوابير
توليد التقارير
```

## غير مسموح (خطوط حمراء)

```
لا إرسال خارجي من أي agent (إيميل/واتساب/مكالمة)
لا cold WhatsApp automation
لا automated calling
لا تنفيذ أداة بناءً على نص خارجي
لا قرارات تسعير
لا حذف بيانات
لا تعديل أسرار الإنتاج
لا معالجة PII خام في أدوات عامة
لا قوائم مشتراة (purchased lists)
لا ادعاءات إيرادات مضمونة
```

---

## مبدأ التنفيذ

```
AI يجهّز → المؤسس يعتمد → تنفيذ بشري → تسجيل في ai_action_ledger
```

كل فعل خارجي يمر ببوابة اعتماد بشري (انظر `FOUNDER_DECISION_GATES_AR.md`). الوكلاء
ينتجون مسودات وقوائم وتقارير فقط؛ التنفيذ الخارجي يبقى بيد الإنسان.

---

## الفرض الآلي

`npm run commercial:check` يؤكّد هذه الحدود ويكتب التأكيدات في
`reports/security/DAILY_AGENT_SECURITY_REVIEW.md`:
- لا إرسال خارجي (المسودات تبقى في طابور الاعتماد).
- لا اتصال آلي ولا أتمتة واتساب باردة.
- احترام قائمة الحجب (لا قوائم مشتراة).
- لا أسرار في المخرجات.
