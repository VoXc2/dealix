# Data Handling Checklist — Dealix

## Purpose
Ensure all client data is handled in compliance with:
- Saudi Personal Data Protection Law (PDPL)
- SDAIA AI Ethics Principles
- Gartner AI Governance recommendations
- Dealix internal governance standards

---

## Pre-Engagement Checklist

### Client Data Assessment
- [ ] Identify all data types client will share
- [ ] Classify sensitivity level (Low / Medium / High)
- [ ] Confirm data ownership and consent
- [ ] Verify client has authority to share data
- [ ] Document data retention requirements

### Consent Verification
- [ ] Signed intake form with data privacy consent
- [ ] Explicit consent for AI analysis
- [ ] Explicit consent for data storage
- [ ] Right to deletion acknowledged
- [ ] Data sharing scope defined

---

## During Engagement

### Data Collection
- [ ] Only collect data listed in intake form
- [ ] No unnecessary personal data collected
- [ ] Sensitive data (IDs, financial details) excluded
- [ ] Data collected through secure channels only
- [ ] Collection logged in AI action ledger

### Data Processing
- [ ] Anonymize before AI analysis
- [ ] Remove: names, phone numbers, emails, IDs, financial details
- [ ] Keep only: conversation patterns, timing, sentiment, outcomes
- [ ] All AI processing actions logged
- [ ] Human review for any sensitive findings

### Data Storage
- [ ] Store only in approved locations
- [ ] Encrypt at rest
- [ ] Access limited to assigned team members
- [ ] No data in personal devices
- [ ] Regular access audit

---

## AI-Specific Governance

### Prompt Safety
- [ ] No real client names in AI prompts
- [ ] No sensitive identifiers in prompts
- [ ] Use anonymized examples only
- [ ] Prompts reviewed before sending
- [ ] Prompt templates pre-approved

### Output Review
- [ ] AI outputs reviewed before client delivery
- [ ] No sensitive data in outputs
- [ ] Claims verified with evidence
- [ ] No exaggerated promises
- [ ] Recommendations are actionable

### Action Authorization
- [ ] AI drafts only — no autonomous sending
- [ ] Human approval required for all external actions
- [ ] Approval queue used for all messages
- [ ] All actions logged in ledger
- [ ] Weekly governance audit

---

## Post-Engagement

### Data Retention
- [ ] Client data retained per agreement
- [ ] Default retention: 90 days post-engagement
- [ ] Extended retention only with written consent
- [ ] Secure deletion after retention period
- [ ] Deletion confirmed and logged

### Data Deletion (Upon Request)
- [ ] Acknowledge request within 24 hours
- [ ] Delete all client data within 72 hours
- [ ] Confirm deletion in writing
- [ ] Remove from backups (if applicable)
- [ ] Log deletion in governance ledger

---

## Compliance Checklist

### PDPL Requirements
- [ ] Lawful basis for processing established
- [ ] Consent documented
- [ ] Purpose limitation respected
- [ ] Data minimization practiced
- [ ] Accuracy maintained
- [ ] Storage limitation enforced
- [ ] Integrity and confidentiality protected
- [ ] Accountability demonstrated

### SDAIA AI Ethics
- [ ] Fairness: No biased analysis
- [ ] Privacy and Security: Data protected
- [ ] Humanity: AI assists, humans decide
- [ ] Social and Environmental Benefits: Positive impact
- [ ] Reliability and Safety: Tested outputs
- [ ] Transparency: Clear AI involvement
- [ ] Accountability: Actions logged
- [ ] Explainability: Recommendations justified

## Escalation

**If any compliance issue found:**
1. Stop all AI processing immediately
2. Notify founder within 1 hour
3. Document the issue
4. Assess scope of impact
5. Notify affected clients if required
6. Implement corrective action
7. Review and update procedures
