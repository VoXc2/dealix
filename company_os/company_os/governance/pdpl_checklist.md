# PDPL Compliance Checklist — Dealix

## Overview
This checklist ensures Dealix operations comply with the Saudi Personal Data Protection Law (PDPL) and SDAIA implementing regulations.

**Last Updated**: 2026-05-31
**Next Review**: 2026-06-30
**Responsible**: Dealix Compliance Officer

---

## 1. Legal Basis for Processing

### Consent
- [ ] Explicit consent obtained from all clients
- [ ] Consent documented and stored securely
- [ ] Consent scope clearly defined
- [ ] Right to withdraw consent communicated
- [ ] Withdrawal process established

### Contractual Necessity
- [ ] Processing necessary for Sprint delivery
- [ ] Processing scope limited to contract purpose
- [ ] No processing beyond agreed scope

### Legitimate Interest
- [ ] Legitimate interest assessment completed
- [ ] Client rights balanced against business needs
- [ ] Opt-out provided where applicable

---

## 2. Data Controller Obligations

### Transparency
- [ ] Privacy policy published and accessible
- [ ] Processing purposes clearly stated
- [ ] Data categories specified
- [ ] Retention periods disclosed
- [ ] Third-party sharing disclosed (if any)

### Data Subject Rights
- [ ] Right to access — process established
- [ ] Right to rectification — process established
- [ ] Right to erasure — process established
- [ ] Right to restrict processing — process established
- [ ] Right to data portability — process established
- [ ] Right to object — process established
- [ ] Right to lodge complaint — SDAIA contact provided

### Data Protection Officer (DPO)
- [ ] DPO appointed (Founder acts as DPO for now)
- [ ] DPO contact published
- [ ] DPO trained on PDPL requirements

---

## 3. Data Processing Principles

### Purpose Limitation
- [ ] Data collected only for specified purposes
- [ ] No secondary processing without consent
- [ ] Purpose documented for each dataset

### Data Minimization
- [ ] Only necessary data collected
- [ ] Regular review of data relevance
- [ ] Unused data deleted promptly

### Accuracy
- [ ] Data accuracy verified at intake
- [ ] Clients can update their data
- [ ] Inaccurate data corrected promptly

### Storage Limitation
- [ ] Retention schedule defined
- [ ] Data deleted per retention schedule
- [ ] Deletion verified and logged

### Integrity and Confidentiality
- [ ] Access controls implemented
- [ ] Encryption at rest and in transit
- [ ] Security incident response plan
- [ ] Regular security assessments

### Accountability
- [ ] Processing records maintained
- [ ] Compliance monitoring active
- [ ] Regular audits conducted

---

## 4. Cross-Border Data Transfers

### Current Status
- [ ] All data stored within KSA
- [ ] No cross-border transfers planned
- [ ] If needed: SDAIA adequacy decision obtained
- [ ] If needed: Appropriate safeguards implemented

---

## 5. AI-Specific PDPL Considerations

### Automated Decision Making
- [ ] No fully automated decisions affecting clients
- [ ] Human review for all AI recommendations
- [ ] Clients informed of AI involvement
- [ ] Right to human intervention provided

### AI Training Data
- [ ] No client data used for model training
- [ ] Any training data is anonymized and aggregated
- [ ] Client consent obtained if data used for improvement

### Transparency in AI
- [ ] Clients informed when AI is used
- [ ] AI capabilities and limitations disclosed
- [ ] Human oversight clearly communicated

---

## 6. Incident Management

### Breach Notification
- [ ] Breach detection process
- [ ] 72-hour notification to SDAIA
- [ ] Client notification process
- [ ] Documentation requirements

### Incident Response
- [ ] Response team defined
- [ ] Containment procedures
- [ ] Investigation process
- [ ] Remediation steps
- [ ] Post-incident review

---

## 7. Documentation

### Required Records
- [ ] Processing activities log
- [ ] Consent records
- [ ] Data subject requests log
- [ ] Security incident log
- [ ] Audit reports
- [ ] Training records

### Record Retention
- [ ] Processing records: 5 years
- [ ] Consent records: Duration of relationship + 2 years
- [ ] Security incidents: 5 years
- [ ] Audit reports: 5 years

---

## 8. Regular Compliance Review

### Monthly
- [ ] Consent records up to date
- [ ] Access logs reviewed
- [ ] No unauthorized processing
- [ ] Data deletion on schedule

### Quarterly
- [ ] Full compliance review
- [ ] Risk assessment update
- [ ] Policy review
- [ ] Training refresh

### Annually
- [ ] Comprehensive audit
- [ ] Policy updates
- [ ] SDAIA regulation review
- [ ] DPO training update
