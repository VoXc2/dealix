# Co-Sell Color Tokens

Dealix logo / wordmark is permitted in co-sell contexts only when:

1. The partner has signed `DOCTRINE_ADOPTION_CHECKLIST.md`.
2. The partner's name appears more prominently than Dealix's
   (this is co-sell, not white-label).
3. The Dealix wordmark uses the official tokens below.

## Token Set

| Token        | Value      | Use                            |
|--------------|-----------|--------------------------------|
| `--dx-fg`    | `#e6e9ef` | Dealix wordmark foreground     |
| `--dx-bg`    | `#0b0d12` | Dealix wordmark background     |
| `--dx-pass` | `#2dd4a4` | Doctrine PASS state            |
| `--dx-warn` | `#f5b86b` | Doctrine WARN state            |
| `--dx-fail` | `#ef4d5c` | Doctrine FAIL state            |
| `--dx-accent` | `#6aa6ff` | Links + accent text          |

Use these tokens as-is. Do not recolor or restyle.

## Forbidden

- Stretching, rotating, or warping the wordmark.
- Pairing with a similar-sized partner wordmark (Dealix must be the
  secondary mark by visual weight).
- Removing or replacing the doctrine version label when displaying
  the doctrine status badge.
