# Dealix Partner Kit

The kit that lets an anchor partner co-sell governed AI operations to
their clients without rebuilding Dealix's doctrine, proof, or trust
infrastructure.

## What This Kit Is

A self-contained set of fill-in-the-blanks templates + adoption
checklist + co-branding rules, derived from the open Dealix doctrine.

## What This Kit Is NOT

- It is **not** legal advice, regulatory certification, or compliance
  approval.
- It is **not** commercial Dealix implementation code.
- It is **not** a permission to make claims Dealix has not verified.

## Contents

| File                                      | Purpose                                         |
|-------------------------------------------|-------------------------------------------------|
| `TRUST_PACK_TEMPLATE.md`                  | Fill-in-the-blanks Trust Pack for a partner.    |
| `PROOF_PACK_TEMPLATE.md`                  | Anonymized Proof Pack structure to write into.  |
| `DOCTRINE_ADOPTION_CHECKLIST.md`          | The 11 non-negotiables + partner sign-off.      |
| `branding/COLORS.md`, `branding/LOGO_USAGE.md` | Co-sell branding rules.                    |
| `landing-template/index.html`             | Static co-branded landing template.              |

## How to Use

1. Read `DOCTRINE_ADOPTION_CHECKLIST.md`. Sign or stop.
2. Fork this kit into your own partner repo.
3. Replace every `<PARTNER>` and `<COUNTRY>` token with your details.
4. Run the doctrine adoption test against your fork (see the checklist
   for the snippet).
5. Co-sell within the doctrine bounds.

## What Dealix Provides Live

- Public doctrine: `https://dealix.example.com/api/v1/dealix-promise`
- Per-customer readiness: `/api/v1/customer/{handle}/readiness/public`
- Trust badges + sitemap: see Dealix landing.
