# Enterprise Sales and Procurement Readiness

## Objective
Prepare Dealix to sell to serious B2B and enterprise buyers with trust, evidence, security posture, procurement readiness, and repeatable deal execution.

## Enterprise buyer map

| Role | What they care about | Required asset |
|---|---|---|
| Economic buyer | ROI, risk, speed, budget | Business case and ROI snapshot |
| Champion | Pain relief, internal credibility | Demo script and proof pack |
| Technical evaluator | Integrations, security, reliability | Technical overview and architecture note |
| Procurement | Terms, pricing, vendor info | Contract kit and pricing sheet |
| Legal | Data, liability, privacy, terms | MSA, DPA, NDA, security notes |
| Executive sponsor | Strategic value and confidence | Executive narrative and case proof |

## Enterprise deal stages

| Stage | Exit criteria | Owner |
|---|---|---|
| Target account | ICP fit confirmed | Sales Ops |
| Discovery | Pain, urgency, authority, impact documented | Sales |
| Diagnostic | Current state and success criteria mapped | Founder/Sales |
| Demo | Relevant workflow shown with customer-specific context | Product/Sales |
| Business case | ROI and risk reduction documented | Founder/Finance |
| Security/procurement | Required docs submitted | Ops/Legal |
| Proposal | Scope, price, timeline, success metric defined | Sales |
| Close | Signed agreement and kickoff date | Founder/Sales |
| Onboarding | First value path started | CS/Product |

## Procurement checklist

- Company profile
- Product overview
- Pricing and package sheet
- Security overview
- Data processing overview
- Support SLA summary
- MSA/SOW/NDA/DPA templates
- Insurance or risk notes if applicable
- Tax and billing details
- Implementation plan
- Customer success plan

## Enterprise proof pack

Every enterprise deal should include:

1. Problem statement.
2. Current cost of inaction.
3. Proposed workflow.
4. Expected measurable outcome.
5. Evidence or benchmark.
6. Risk controls.
7. Implementation timeline.
8. Success criteria.
9. Renewal/expansion path.

## Qualification rules

A deal is high quality only when:

- Buyer pain is urgent.
- Success metric is measurable.
- Economic buyer is identified.
- Procurement path is known.
- Security/legal blockers are visible.
- Timeline is real.
- Dealix has a clear reason to win.

## Red flags

- No economic buyer.
- Unclear pain.
- Custom-only request.
- No timeline.
- Security/legal asks beyond company maturity.
- Buyer wants free strategy without commitment.
- Deal requires features outside roadmap.

## Required artifacts

- Enterprise deal desk checklist
- Business case template
- Security overview
- Technical architecture note
- Procurement packet
- Proposal template
- Deal risk scorecard
- Mutual action plan
