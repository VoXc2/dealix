# Source and License Notes

## Important rule
This pack is written as original Dealix operating documentation. It is not a verbatim copy of external repositories.

## Why not copy random GitHub files directly
Many useful GitHub templates are under different licenses, some have no clear license, and some are legal/security documents that should not be reused blindly.

Dealix should only import external files when:

1. The license is clear.
2. Attribution is preserved.
3. The file is useful without creating legal or compliance confusion.
4. The content is reviewed for Saudi/GCC context.
5. The file is adapted into Dealix operations instead of pasted blindly.

## External resources to review

| Area | Resource | Use |
|---|---|---|
| AI risk | NIST AI Risk Management Framework | AI governance, risk mapping, evaluation controls |

| Legal templates | Series Seed documents | Funding document reference only, legal review required |
| Open source governance | Project Harmony | Contributor agreement reference only |

## Dealix adaptation rule
Use external resources as reference material, then create Dealix-specific artifacts with:

- Clear owner
- Operational checklist
- Decision rules
- Audit trail
- Review date
- Saudi/GCC applicability review

## License posture
Until a file is reviewed and approved, treat it as reference-only.

Do not copy legal templates, investor documents, policy text, or third-party playbooks into Dealix without license and legal review.
