# Company Research Policy — سياسة بحث الشركات

**الهدف:** ضبط مصدر المعلومات التي يُبنى عليها Company Intelligence Pack، بحيث يبقى البحث قانونيًا ومحترمًا للخصوصية ومتوافقًا مع PDPL.

---

## 1. ما المسموح؟

```txt
✅ معلومات الشركة العامة (موقع، قنوات تواصل معلنة، قطاع، حجم تقريبي)
✅ بيانات يقدّمها المؤسس أو العميل صراحةً
✅ أدوار وظيفية (Head of Sales, Operations Manager …)
✅ إشارات عامة (وجود واتساب معلن، تعدد البرامج، إعلانات منشورة)
```

## 2. ما الممنوع؟

```txt
❌ قوائم مشتراة أو مكشوطة بطرق غير مصرّح بها
❌ أسماء أشخاص أو بيانات تواصل شخصية في الملفات/التقارير
❌ بيانات حساسة (أسرار، حسابات، مفاتيح، ملفات خاصة)
❌ تخزين بيانات عملاء العميل النهائيين بلا أساس قانوني
```

## 3. مستوى الدليل (evidence_level)

| القيمة | المعنى |
|--------|--------|
| `public_only` | من مصادر عامة فقط |
| `founder_provided` | قدّمها المؤسس/العميل |
| `inferred` | استنتاج/فرضية — يُعامَل بحذر ولا يُقدَّم كحقيقة |

> الافتراضي للملفات المولّدة هو `public_only`، والألم دائمًا **فرضية** (`likely_pain` / `pain_hypothesis`).

## 4. الخصوصية و PDPL

```txt
- أدوار فقط، لا أسماء أشخاص.
- لا PII لعملاء العميل داخل الملفات أو التقارير أو السجلّات.
- احترام قائمة عدم التواصل (do_not_contact) وقوائم الكبح (suppression).
- البريد التسويقي يلتزم بمتطلبات المرسلين: SPF/DKIM (وDMARC للمرسلين الكبار)،
  وإبقاء معدل الـ spam منخفضًا، وإتاحة إلغاء اشتراك بنقرة واحدة للرسائل التسويقية المجمّعة.
```

## 5. الفحص الآلي

- **C07** يمنع وجود أي أسرار (api key / token / password / private key) في التقارير والبيانات.
- **C01/C06/C08/C09/C10** تضمن سلامة المخرجات (نظام صحيح، بلا ضمانات، مسودة، بلا Re:/Fwd:، بلا اتصال آلي).

> هذه السياسة وصفية للضبط؛ التنفيذ القانوني النهائي (عقود، أسس معالجة البيانات) قرار بشري لا يُؤتمت.
