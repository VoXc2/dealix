# نظام العرض المصغّر (Mini Proposal System)

> الغرض: قالب عرض مصغّر يُجهَّز بعد ظهور اهتمام، يحدّد أول Sprint ومخرجاته وسعره الابتدائي — ولا يُرسَل إلا بعد موافقة المؤسس.

المخطط المرجعي: `schemas/mini_proposal.schema.json`.

## قاعدة الموافقة

- `approval_required` دائمًا `true`. لا يُرسَل أي عرض مصغّر قبل موافقة المؤسس.
- الحالة تبدأ `pending` ثم تصبح `approved` / `rejected` / `hold` بقرار المؤسس.
- السعر يُذكر بصيغة «يبدأ من X SAR»، لا كسعر نهائي مقفل.
- ممنوع أي وعد بإيراد مضمون؛ نصف المخرجات لا النتائج المالية.

## شرح الحقول

| الحقل | المعنى | إلزامي |
|------|--------|:------:|
| `title` | عنوان العرض | نعم |
| `company` | اسم الشركة | نعم |
| `recommended_system` | معرّف أحد الأنظمة الخمسة | نعم |
| `why_this_system` | لماذا هذا النظام يناسب وضع الشركة | نعم |
| `current_likely_pain` | الألم المرجّح الحالي (صياغة احتمالية) | لا |
| `first_sprint` | وصف أول Sprint قصير | نعم |
| `deliverables` | قائمة المخرجات (عنصر واحد على الأقل) | نعم |
| `timeline` | المدة الزمنية | نعم |
| `starter_price` | السعر الابتدائي بصيغة «يبدأ من X SAR» | نعم |
| `required_inputs` | المدخلات المطلوبة من الشركة | لا |
| `expected_first_proof` | أول إثبات متوقع تسليمه | لا |
| `next_step` | الخطوة التالية | نعم |
| `approval_required` | دائمًا `true` | نعم |
| `approval_status` | `pending` / `approved` / `rejected` / `hold` | لا |

## الأسعار الابتدائية للأنظمة

| النظام | يبدأ من |
|--------|--------:|
| نظام تشغيل الإيرادات | 4,500 SAR |
| نظام القيادة التنفيذية | 5,500 SAR |
| نظام استعادة المتابعة | 3,500 SAR |
| نظام عملاء واتساب | 4,500 SAR |
| نظام العروض والإثبات | 3,000 SAR |

## مثال مُعبّأ (نظام استعادة المتابعة — 7 أيام)

```json
{
  "title": "عرض مصغّر: استعادة متابعة الاستفسارات",
  "company": "SkillUp Arabia",
  "recommended_system": "follow-up-recovery-os",
  "why_this_system": "اعتماد واضح على استفسارات واردة عبر واتساب والنموذج بلا قائمة متابعة موحدة — وهو جوهر نظام استعادة المتابعة",
  "current_likely_pain": "في شركات التدريب المشابهة غالبًا تُترك بعض الاستفسارات دون متابعة في وقتها، فتذهب لمنافس أسرع",
  "first_sprint": "Sprint من 7 أيام: بناء قائمة متابعة منظمة ونموذج حالة للعميل ومجموعة رسائل تعيد تفعيل الاستفسارات المتروكة",
  "deliverables": [
    "قائمة المتابعة (Follow-up Queue)",
    "نموذج حالة العميل (Lead Status Model)",
    "مجموعة رسائل المتابعة (Message Set)",
    "إيقاع التذكير (Reminder Rhythm)",
    "أول تقرير استعادة (Recovery Report)"
  ],
  "timeline": "7 أيام",
  "starter_price": "يبدأ من 3,500 SAR",
  "required_inputs": [
    "قائمة الاستفسارات أو العملاء المحتملين الحاليين",
    "القنوات التي تصل منها (واتساب، نموذج، اتصال)",
    "من يتابع حاليًا وكم مرة"
  ],
  "expected_first_proof": "حصر أوّلي لعدد الاستفسارات المتروكة بلا متابعة، وأيّها قابل للاستعادة",
  "next_step": "تأكيد البدء وتزويدنا بالمدخلات الأساسية",
  "approval_required": true,
  "approval_status": "pending"
}
```

> الإرسال للعميل يتم فقط بعد أن يصبح `approval_status = approved`.
