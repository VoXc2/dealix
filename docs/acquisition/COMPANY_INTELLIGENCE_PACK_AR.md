# حزمة استخبارات الشركة (Company Intelligence Pack)

> الغرض: ملف بحثي موحّد لكل شركة مستهدفة، يُبنى من معلومات عامة أو بيانات يوفّرها المؤسس فقط، ويُستخدم لتخصيص الرسالة والمكالمة والعرض المصغّر.

المخطط المرجعي: `schemas/company_intelligence_pack.schema.json`.

## القواعد الأساسية

- المصدر معلومات عامة فقط (موقع الشركة، صفحة وظيفة، خبر منشور) أو بيانات يقدّمها المؤسس (L4).
- لا قوائم مشتراة، ولا كشط بيانات خاصة، ولا أتمتة LinkedIn.
- كل افتراض منخفض الدليل (L0/L1) يُصاغ كاحتمال: «غالبًا»، «قد»، «في هذا النوع من الشركات» — لا كحقيقة مؤكدة.
- ممنوع أي وعد بإيراد مضمون في أي حقل.
- `recommended_system` يجب أن يكون أحد المعرّفات الخمسة فقط: `revenue-operating-system`، `executive-command-os`، `follow-up-recovery-os`، `whatsapp-client-os`، `proposal-proof-os`.

## شرح الحقول

| الحقل | المعنى | إلزامي |
|------|--------|:------:|
| `company` | اسم الشركة كما يظهر علنًا | نعم |
| `website` | الموقع الرسمي إن وُجد | لا |
| `country` | الدولة (غالبًا السعودية) | لا |
| `city` | المدينة إن أمكن تحديدها من مصدر عام | لا |
| `sector` | القطاع (تدريب، عقار، عيادات، وكالة تسويق، استشارات...) | نعم |
| `public_contact_channels` | قنوات تواصل عامة منشورة (نموذج موقع، بريد عام، رقم معلن) | لا |
| `likely_decision_maker` | صاحب القرار المرجّح بالاسم أو الصفة (يُصاغ كاحتمال إن لم يُؤكَّد) | لا |
| `best_contact_role` | الدور الأنسب للتواصل، ضمن `ops.contact_roles` للنظام الموصى به | نعم |
| `signal` | إشارة عامة قابلة للملاحظة تبرّر التواصل (إعلان توظيف، حملة، توسّع) | لا |
| `likely_pain` | الألم المرجّح، مرتبط بإشارة عامة لا بافتراض جازم | نعم |
| `recommended_system` | معرّف أحد الأنظمة الخمسة | نعم |
| `why_this_system` | لماذا هذا النظام تحديدًا يناسب الإشارة والألم | نعم |
| `first_mission` | أول مهمة/مخرج صغير يقدّمه النظام خلال أيام | نعم |
| `proof_angle` | زاوية الإثبات: ما الذي سنكشفه أو نقيسه أولًا | لا |
| `email_subject` | عنوان بريد قصير (≤ 90 حرفًا) بلا مبالغة | نعم |
| `email_draft` | مسودة بريد تمهيدية مخصّصة (تبقى مسودة حتى موافقة المؤسس) | نعم |
| `call_opener` | جملة افتتاح بشرية للمكالمة | لا |
| `call_questions` | أسئلة اكتشاف قصيرة | لا |
| `expected_objections` | اعتراضات متوقعة من هذا النوع من الشركات | لا |
| `mini_proposal_angle` | زاوية العرض المصغّر إن ظهر اهتمام | لا |
| `next_action` | الإجراء التالي الواضح | لا |
| `risk_level` | مستوى المخاطرة: `low` / `medium` / `high` | نعم |
| `evidence_level` | مستوى الدليل: `L0`/`L1`/`L2`/`L3`/`L4` | نعم |

## مستويات الدليل

| المستوى | المصدر |
|--------|--------|
| L0 | تخمين قطاعي |
| L1 | موقع الشركة |
| L2 | صفحة/وظيفة/خبر محدد |
| L3 | عدة مصادر عامة متوافقة |
| L4 | بيانات مقدّمة من الشركة |
