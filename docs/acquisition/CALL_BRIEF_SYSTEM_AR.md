# نظام موجز المكالمة (Call Brief System)

> الغرض: تزويد المتصل البشري بموجز جاهز قبل مكالمة المتابعة التي تلي بريدًا معتمدًا — لا اتصال آلي إطلاقًا.

المخطط المرجعي: `schemas/call_brief.schema.json`.

## مبادئ

- المكالمات بشرية فقط؛ لا نظام اتصال آلي ولا رسائل صوتية مؤتمتة.
- الموجز يُبنى من حزمة استخبارات الشركة وبطاقة الاحتياج بعد إرسال البريد المعتمد.
- هدف المكالمة فهم الوضع وعرض تشخيص قصير، لا الضغط أو إغلاق صفقة بالقوة.
- ممنوع أي وعد بإيراد مضمون. الألم يُطرح كاحتمال إن كان دليله L0/L1.

## شرح الحقول

| الحقل | المعنى | إلزامي |
|------|--------|:------:|
| `company` | اسم الشركة | نعم |
| `contact_role` | دور جهة الاتصال (ضمن أدوار النظام الموصى به) | نعم |
| `recommended_system` | معرّف أحد الأنظمة الخمسة | نعم |
| `likely_pain` | الألم المرجّح (صياغة احتمالية) | نعم |
| `email_sent_summary` | ملخّص البريد المُرسَل مسبقًا ليبني المتصل عليه | لا |
| `call_objective` | هدف المكالمة بجملة واحدة | نعم |
| `opening_line` | جملة الافتتاح البشرية | نعم |
| `discovery_questions` | أسئلة اكتشاف (عنصر واحد على الأقل) | نعم |
| `expected_objection` | الاعتراض الأرجح | لا |
| `best_response` | أفضل رد آمن على ذلك الاعتراض | لا |
| `next_step` | الخطوة التالية المتفق عليها | نعم |
| `evidence_level` | `L0`–`L4` | لا |

## مثال مُعبّأ

```json
{
  "company": "SkillUp Arabia",
  "contact_role": "Founder / CEO",
  "recommended_system": "follow-up-recovery-os",
  "likely_pain": "غالبًا استفسارات دورات تدخل عبر واتساب والنموذج وتُترك بعضها دون متابعة في وقتها",
  "email_sent_summary": "بريد تمهيدي عن استعادة الاستفسارات المتروكة عبر قائمة متابعة ورسائل جاهزة خلال 7 أيام",
  "call_objective": "فهم كيف تُتابَع الاستفسارات حاليًا، والاتفاق على تشخيص سريع إن ظهر اهتمام",
  "opening_line": "السلام عليكم، معك [الاسم] من Dealix، بعثت لكم بريدًا قبل قليل عن متابعة تسجيلات الدورات — عندكم دقيقتان؟",
  "discovery_questions": [
    "كيف تصلكم تسجيلات الدورات حاليًا (واتساب، نموذج، اتصال)؟",
    "من يتابع الاستفسار غير المكتمل، وبأي إيقاع؟",
    "هل عندكم قائمة متابعة موحدة أم متفرقة بين الأشخاص؟"
  ],
  "expected_objection": "عندنا أحد يتابع الاستفسارات",
  "best_response": "ممتاز أن لديكم من يتابع. ما نضيفه هو قائمة موحدة ورسائل جاهزة حسب الحالة، حتى لا تعتمد المتابعة على اجتهاد فردي. نبدأ بتشخيص قصير يكشف أين تتوقف الاستفسارات فعلًا.",
  "next_step": "حجز تشخيص سريع 15 دقيقة، أو إرسال عرض مصغّر بعد موافقة المؤسس",
  "evidence_level": "L2"
}
```

## قائمة تحقق للمتصل قبل الاتصال

- [ ] قرأت ملخّص البريد المُرسَل (`email_sent_summary`).
- [ ] الدور صحيح وضمن أدوار النظام الموصى به.
- [ ] جاهز بجملة الافتتاح وأسئلة الاكتشاف.
- [ ] أعرف الرد الآمن على الاعتراض المتوقع.
- [ ] لن أقدّم أي وعد مضمون بالإيراد.
- [ ] سأسجّل نتيجة المكالمة والخطوة التالية بعد انتهائها.
