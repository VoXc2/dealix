# نظام Call Brief

لكل حساب من الأوائل، يُنتج النظام **مذكّرة مكالمة** جاهزة تجعل المؤسس مستعدًا في
30 ثانية — دون اتصال آلي.

## المخرجات

- `data/acquisition/call_briefs.jsonl`
- `data/acquisition/objection_responses.jsonl` (مكتبة اعتراضات)

## بنية المذكّرة

| الحقل | المعنى |
| --- | --- |
| `buyer_role` | الدور الشرائي المستهدف |
| `primary_need`, `core_system` | الاحتياج والنظام |
| `opening_ar` | جملة افتتاح قصيرة (إذن قبل المتابعة) |
| `questions` | ≥ 2 أسئلة تشخيصية |
| `value_points` | نقاط قيمة بلا وعود مضمونة |
| `objections_ref` | مرجع مكتبة الاعتراضات |
| `next_step` | الخطوة التالية (إرسال Mini Proposal بعد الاعتماد) |

## مبادئ

- المكالمة تشخيص، لا ضغط.
- لا اتصال آلي ولا واتساب بارد — المؤسس يقرر التوقيت.
- كل وعد قابل للإثبات بمخرجات ومعايير قبول.

## التسليم اللاحق

`Email → Call → Mini Proposal → Delivery`، وكل انتقال يحتاج اعتماد المؤسس.
