# سياسة اكتشاف القنوات العامة (Contact Discovery)

القاعدة الذهبية:

```
لا نخترع أي اسم أو رقم أو بريد. نستخدم فقط معلومات عامة أو يوفّرها المؤسس.
```

## ما المسموح

- قنوات **عامة** منشورة من الشركة نفسها: صفحة «اتصل بنا»، نموذج تواصل على
  الموقع، رقم/بريد معلن صراحة على الموقع الرسمي، حساب رسمي عام.
- بيانات يوفّرها **المؤسس** بنفسه (مصدر `founder_provided`).

## ما الممنوع

- تخمين بريد شخصي (مثل نمط الاسم@النطاق) — ممنوع.
- شراء قوائم أو كشط بيانات شخصية — ممنوع.
- أرقام جوال شخصية غير معلنة — ممنوع.
- واتساب بارد لأشخاص — ممنوع.

## مستويات الثقة

| المستوى | المعنى |
| --- | --- |
| `high` | قناة عامة رسمية مؤكدة من موقع الشركة |
| `medium` | قناة عامة محتملة تحتاج تأكيدًا |
| `low` | إشارة عامة فقط (مثل وجود الموقع) — لا تُستخدم للإرسال بثقة |

## شكل البيانات

- `data/contacts/contact_discovery.jsonl` — لكل شركة: `channels[]` + `invented:false`.
- كل قناة: `{type, value, source: public|founder_provided, confidence}`.
- `type` ∈ website / contact_form / email / phone / linkedin / whatsapp_business.

## بيانات العرض (Demo)

حزم العرض في هذا الريبو **اصطناعية** (`demo:true`)، ولا تحتوي إلا قناة موقع
عامة بثقة منخفضة، دون أي بريد أو رقم. التشغيل الحقيقي يستبدلها بقنوات عامة
حقيقية أو بيانات المؤسس فقط.

## البوابة

`check_security_privacy_gates.py` يرفض أي سجل `invented != false`، و
`check_account_pack_contract.py` يرفض أي قناة تحوي بريدًا مُختلقًا.
