# Contact Confidence Levels — مستويات ثقة التواصل

> لكل قناة تواصل **مستوى ثقة** من C0 إلى C4. النظام لا يخترع رقمًا ولا يملأ فراغًا.

---

## 1. السلّم (C0 → C4)

| المستوى | المعنى | مثال |
|---------|--------|------|
| **C0** | غير موجود | لا قناة تواصل عامة إطلاقًا |
| **C1** | قناة عامة غير مؤكدة | حساب اجتماعي فقط، أو موقع بلا صفحة تواصل |
| **C2** | صفحة تواصل رسمية | صفحة "اتصل بنا" / نموذج تواصل |
| **C3** | إيميل/هاتف منشور | عنوان عام مذكور صراحةً في الموقع |
| **C4** | تأكيد مباشر | بيانات قدّمها العميل أو أكّدها |

---

## 2. أمثلة عملية

### قناة بثقة C3 (إيميل عام منشور)
```yaml
email_if_public: info@example.sa
contact_confidence: C3
source: official_contact_page
```

### لا توجد قناة مباشرة (الحالة الشائعة في العيّنة)
```yaml
phone_if_public: null
email_if_public: null
best_contact_route: contact_form_or_general_email
contact_confidence: C1   # أو C2 إن وُجدت صفحة تواصل
```

> النظام **لا** يخترع رقمًا ولا يملأ فراغات. غياب القناة يُسجَّل بصدق.

---

## 3. أثر الثقة على الترتيب والطوابير

- `contact_confidence` جزء من **Contact availability** (20 نقطة) في Account Score.
- C0/C1 → غالبًا يخرج الحساب من Top 100 ويُدرج في `MISSING_CONTACTS_REVIEW`.
- C2+ → مؤهل للإرسال بعد الاعتماد البشري.

| الثقة | الجاهزية للإرسال |
|------|------------------|
| C4 | عالية جدًا |
| C3 | عالية |
| C2 | متوسطة (عبر صفحة/نموذج) |
| C1 | منخفضة — يحتاج بحثًا |
| C0 | غير مؤهل |

---

## 4. توزيع الثقة في العيّنة

| الثقة | عدد الحسابات | الحسابات |
|------|:-----------:|----------|

## 5. التحقق الآلي

`scripts/account-factory-check.mjs`:
- يتحقق أن `contact_confidence` ∈ {C0..C4}.
- يفرض أن سجلات C0/C1 **لا تحمل** إيميلًا/هاتفًا مباشرًا (إثبات عدم الاختراع).
