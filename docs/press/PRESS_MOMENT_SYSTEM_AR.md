# نظام اللحظة الصحفية — Press Moment System (AR)

كيف نميّز **لحظة صحفية حقيقية** تستحق طرحاً، مقابل حدث عادي لا يستحقّه. اللحظة
مشروطة دائماً بإنجاز إثبات حقيقي مستوفٍ للمعايير — وإلا فلا لحظة ولا طرح.

## القاعدة الحاكمة
- لا لحظة صحفية بلا إنجاز قابل للنشر (`docs/press/PROOF_MILESTONES_AR.md`).
- الندرة سياسة: لحظة حقيقية أحياناً خير من طروحات متكرّرة بلا قيمة.
- لكل لحظة: ≤ 3 طروحات مستهدَفة، بلا رسائل جماعية (`PRESS_OUTREACH_OS_AR.md`).

## اختبار «هل هذه لحظة صحفية؟»
| السؤال | إذا «لا» |
|--------|----------|
| هل وراءها إنجاز حقيقي موثّق ومستوفٍ للمعايير؟ | ليست لحظة → لا طرح |
| هل تهمّ فئة جهة واقعياً (لا نحن فقط)؟ | محتوى لا خبر |
| هل كل رقم فيها حقيقي بمصدر و`evidence_level`؟ | يُحذف الرقم أو تُؤجَّل |
| هل النطاق واضح (ما يُقال وما لا يُقال)؟ | يُضبَط قبل الطرح |
| هل اعتمدها المؤسّس؟ | لا طرح حتى الاعتماد |

## أنواع اللحظات المشروعة (قوالب)
- لحظة منتج: قدرة حقيقية جديدة في السلّم `DLX-L0..L6`.
- لحظة تعلّم موثّق: نمط مؤكَّد عبر تشخيصات/تسليمات حقيقية بموافقة.
- لحظة شراكة: توقيع نموذج شراكة موثّق ضمن سياسة الهامش (≥ 15%).
- لحظة محطّة للشركة: محطّة واقعية موثّقة، بلا «الأول/الأكبر» إلا بدليل.

## ضوابط التوقيت والصدق
- لا اختلاق لحظة لملء فراغ زمني؛ غياب الإنجاز = صمت صحفي مقبول.
- لا تضخيم حدث صغير ليبدو محطّة كبرى.
- لا تكرار نفس الإنجاز كعدّة لحظات منفصلة.

## ممنوعات صارمة
- لا تغطية مزعومة، لا جوائز مُختلَقة، لا شهادات مزيّفة، لا أسماء عملاء، لا أرقام بلا مصدر.
- لا عبارات محظورة (نضمن/نضاعف الإيرادات/نتائج مضمونة/بدون مخاطرة/10x/
  guaranteed revenue/no risk). الأفعال المسموحة فقط: نساعد/نقلّل/نرتّب/نحلّل/نوضّح/نلاحظ.
- لا PII (الأدوار فقط)؛ لا `Re:`/`Fwd:` مزيّفة.
