# اعتماد الطرح الصحفي — Media Pitch Approval (AR)

سير عمل اعتماد كل طرح صحفي. القاعدة: **المؤسّس يعتمد كل طرح** قبل أي إرسال، ولا
إرسال جماعي، ولا تجاوز للحدّ الأقصى (3 طروحات لكل لحظة).

## الإعدادات الافتراضية
- `dry_run=true`، `approval_required=true`، `send_enabled=false` افتراضياً.
- لا يتحوّل أي طرح من مسودّة إلى مُرسَل إلا بموافقة المؤسّس الصريحة على ذلك الطرح.
- مستوى مخاطرة التواصل الخارجي ≥ `high`.

## بوّابات الاعتماد (كلّها إلزامية)
| البوّابة | الشرط للمرور | بدونه |
|---------|--------------|-------|
| إنجاز الإثبات | إنجاز حقيقي مستوفٍ للمعايير (`PROOF_MILESTONES_AR.md`) | لا طرح |
| اللحظة الصحفية | اجتياز اختبار اللحظة (`PRESS_MOMENT_SYSTEM_AR.md`) | لا طرح |
| الحدّ الأقصى | ≤ 3 طروحات مستهدَفة، بلا تعميم | يُرفض الزائد |
| الفئة لا الاسم | جهات بالفئة، بلا ادّعاء علاقة (`MEDIA_TARGETS_AR.md`) | يُصحَّح |
| صدق الأرقام | كل رقم حقيقي بمصدر و`evidence_level` | يُحذف الرقم |
| اللغة | لا عبارات محظورة، أفعال مسموحة فقط | يُعاد للمراجعة |
| موافقة المؤسّس | اعتماد صريح لكل طرح على حدة | لا إرسال |

## مسار الحالة
`draft → needs_revision? → approved → sent` (لكل طرح، بلا قفز فوق الاعتماد).
كل انتقال يُسجَّل في `reports/press/PRESS_PIPELINE.md`، والفرص في `PRESS_OPPORTUNITY_QUEUE.md`.

## قائمة تحقّق قبل الاعتماد
- [ ] يوجد إنجاز إثبات حقيقي وموثّق ومستوفٍ للمعايير.
- [ ] عدد الطروحات ≤ 3 ومستهدَفة بفئة مناسبة.
- [ ] لا تغطية/جوائز/شهادات مزعومة، لا أسماء عملاء، لا أرقام بلا مصدر.
- [ ] كل رقم يحمل `evidence_level` ومصدراً حقيقياً.
- [ ] لا عبارات محظورة؛ الأفعال المسموحة فقط.
- [ ] لا PII (الأدوار فقط)؛ لا `Re:`/`Fwd:` مزيّفة.
- [ ] اعتمد المؤسّس هذا الطرح صراحةً.

## ممنوعات صارمة
- لا إرسال جماعي، لا قوائم تعميم، لا تجاوز للحدّ الأقصى.
- لا عبارات محظورة (نضمن/نضاعف الإيرادات/نتائج مضمونة/بدون مخاطرة/10x/
  guaranteed revenue/no risk). الأفعال المسموحة فقط: نساعد/نقلّل/نرتّب/نحلّل/نوضّح/نلاحظ.

## الربط

- بنك السرد: `docs/press/FOUNDER_NARRATIVE_BANK_AR.md`؛ قصّة المؤسّس: `docs/press/FOUNDER_STORY_AR.md`.

## مبادئ الصوت
مختصر، واضح، واثق، سعودي، مؤسّسي، عملي. اعتماد لكل طرح، صدق بلا تلفيق.
