# معمارية الموقع (عربي أولًا)

الموقع تطبيق **Vite + React + React Router** (SPA)، عربي RTL، بثيم Dealix
الأخضر. القاعدة الحاكمة:

```
العميل لا يرى التعقيد.
العميل يرى: 5 أنظمة جوهرية + حلول حسب قطاعه + تشخيص سريع.
الأربعون نظامًا الداخلية والخمسون سبرنت لا تُعرض كقائمة عامة.
```

## المسارات العامة

| المسار | الوصف |
| --- | --- |
| `/` | الصفحة الرئيسية (موجودة: `src/pages/LandingPage.tsx`) |
| `/systems` | الأنظمة الخمسة الجوهرية |
| `/systems/:slug` | صفحة نظام واحد (5 أنظمة) |
| `/solutions` | الحلول حسب القطاع |
| `/solutions/:sector` | صفحة قطاع واحد (20 قطاعًا) |
| `/pricing` | الأسعار المبدئية للسبرنت |
| `/diagnostic` | تشخيص سريع يحوّل الزائر إلى احتياج |

> ملاحظة معمارية: المشروع ليس Next.js. لذلك نُولّد صفحات `/systems/:slug` و
> `/solutions/:sector` ديناميكيًا من بيانات عامة بدل إنشاء عشرات الملفات
> المتكررة. مصدر البيانات: `src/marketing/catalog.ts` (مُولّد من
> `scripts/dealix/generate_site_catalog.py`).

## مصدر المحتوى العام

`src/marketing/catalog.ts` يحتوي فقط:

- الأنظمة الخمسة (اسم + وعد + مخرجات).
- العشرون قطاعًا مع أسماء الاحتياجات والنظام الجوهري المرتبط — **بدون** ذكر
  الأنظمة الأربعين الداخلية ولا منطق التوجيه.

## التحقق

`scripts/checks/check_site_routes.py` يتأكد من:

- تغطية المسارات العامة (5 أنظمة + 20 قطاعًا + الصفحات الأساسية).
- أن الكتالوج العام لا يسرّب أي نظام داخلي.
- أن `src/App.tsx` يربط مسارات `/systems` و`/solutions` و`/pricing` فعليًا.
