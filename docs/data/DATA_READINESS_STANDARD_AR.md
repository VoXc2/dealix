# معيار جاهزية البيانات — الدستور · المعايير التأسيسية

**الطبقة:** الدستور · المعايير التأسيسية
**المالك:** قائد البيانات
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [DATA_READINESS_STANDARD.md](./DATA_READINESS_STANDARD.md)

## السياق
لا يبدأ أي تطبيق ذكاء اصطناعي في Dealix قبل معرفة جاهزية البيانات. هذا
المعيار بوابة تحمي العميل من بناء سير عمل ذكي فوق بيانات غير صالحة.
يُستشهد به في `docs/DEALIX_OPERATING_CONSTITUTION.md`، وفي
`docs/BEAST_LEVEL_ARCHITECTURE.md` لأثر المعمارية، وفي
`docs/AI_STACK_DECISIONS.md` لانعكاسات النموذج وخط البيانات، وفي
`docs/services/data_readiness_assessment/offer.md` للخدمة المنتَجة.
يُفعَّل المعيار عبر طبقة حوكمة وقت التشغيل؛ الدرجة المنخفضة تُنتج إما
خدمة تنظيف أو قرار إيقاف.

## الغرض
جعل كل مجموعة بيانات تدخل سير عمل Dealix مُقيَّمة ومُصنَّفة وقابلة
للتدقيق قبل تشغيل أي ذكاء اصطناعي عليها.

## مكوّنات الدرجة
درجة جاهزية البيانات (DRS) ناتج مرجّح لسبعة مكونات. مجموع الأوزان 100.

| المكوّن | الوزن |
|---|---:|
| تغطية المصدر | 20 |
| الاكتمال | 15 |
| التناسق | 15 |
| الحداثة | 10 |
| إزالة التكرار | 10 |
| التعامل مع البيانات الشخصية | 15 |
| الربط بالأعمال | 15 |

كل مكوّن يُقيَّم من 0 إلى 100. الدرجة الإجمالية تساوي مجموع الأوزان
المرجَّحة مقسومًا على 100. الصيغ في `docs/data/DATA_QUALITY_SCORE.md`.

## مستويات الجاهزية
| المدى | التسمية | القرار |
|---|---|---|
| 85-100 | جاهزة | يمكن المضي في سير العمل الذكي |
| 70-84 | قابلة للاستخدام مع المراجعة | المضي مع تسجيل ملاحظات المخاطر |
| 50-69 | تحتاج تنظيفًا | تنفيذ خدمة تنظيف بيانات أولًا |
| < 50 | غير جاهزة | لا تطبيق ذكاء اصطناعي؛ معالجة فقط |

لا تُرفَع مجموعة بيانات لمستوى أعلى دون إعادة تقييم.

## الحقول الإلزامية لبيانات حسابات الأعمال
الحقول التالية إلزامية لأي مجموعة بيانات حسابات أعمال تستخدم في عمل
الإيرادات أو العملاء أو العمليات.

- `company_name`
- `sector`
- `city`
- `source_type`
- `allowed_use`
- `relationship_status`
- `notes`

## الحقول الاختيارية
- `website`
- `contact_role`
- `email`
- `phone`
- `estimated_size`
- `last_interaction_date`

## التعامل مع البيانات الشخصية
إذا كانت `email` أو `phone` أو اسم شخصي أو معرّفات خاصة أخرى موجودة،
يجب توفر الشروط الأربعة قبل أن تتجاوز الدرجة 70.

1. `source_type` معروف وموثّق.
2. `allowed_use` مُعرَّف ومُسجَّل.
3. `relationship_status` مُعرَّف لكل سجل.
4. لا فعل خارجي بلا موافقة صريحة وفق مصفوفة الموافقات
   (`docs/governance/APPROVAL_MATRIX.md`).

التزامات نظام حماية البيانات الشخصية (PDPL) في
`docs/governance/PDPL_DATA_RULES.md`، والاتفاقية الملزمة في
`docs/DPA_DEALIX_FULL.md`.

## مخطط المخرج
تنتج خدمة جاهزية البيانات السجل التالي بصيغة JSON لكل مجموعة بيانات،
ويُسجَّل في سجل تشغيل الذكاء الاصطناعي ويظهر في حزمة الإثبات.

```json
{
  "dataset_id": "DS-001",
  "records_total": 500,
  "records_valid": 420,
  "records_invalid": 80,
  "duplicates_found": 46,
  "pii_fields_detected": ["email", "phone"],
  "source_coverage_score": 90,
  "completeness_score": 78,
  "consistency_score": 82,
  "freshness_score": 65,
  "pii_risk_score": 40,
  "overall_data_readiness": 76,
  "decision": "usable_with_review",
  "required_actions": [
    "review phone number lawful basis",
    "fill missing sector for 55 records"
  ]
}
```

## قواعد القرار
- إذا `overall_data_readiness >= 85` → `proceed`.
- إذا `70 <= overall_data_readiness < 85` → `proceed_with_review`
  مع تسمية المخاطر.
- إذا `50 <= overall_data_readiness < 70` → `cleanup_required` وتشغيل
  خدمة تنظيف البيانات.
- إذا `overall_data_readiness < 50` → `not_ready`؛ لا عمل ذكاء
  اصطناعي.

مجموعة فيها بيانات شخصية وبدون `allowed_use` تُخفَّض تلقائيًا إلى
`not_ready` بصرف النظر عن الرقم.

## الواجهات
| المدخلات | المخرجات | المالك | الوتيرة |
|---|---|---|---|
| بيانات العميل الخام | سجل جاهزية مُقيَّم | قائد البيانات | لكل مجموعة |
| تعريفات المخطط | تقرير تحقق | قائد البيانات | لكل مجموعة |

| سجل القرار | نطاق تنظيف أو إشارة مضي | قائد التسليم | لكل مجموعة |

## المقاييس
- **التزام جاهزية البيانات قبل الذكاء** — نسبة تشغيلات الذكاء التي
  لديها سجل DRS أحدث من 30 يومًا. المستهدف: 100%.
- **تحويل التنظيف** — نسبة المجموعات `cleanup_required` التي أنتجت
  ارتباطًا تنظيفًا مدفوعًا. المستهدف: ≥ 50%.
- **عدد تجاوزات البيانات الشخصية** — صفر.
