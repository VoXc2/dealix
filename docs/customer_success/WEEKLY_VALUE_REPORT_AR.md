# Weekly Value Report — تقرير القيمة الأسبوعي
**Dealix — Agent #3**

> **الغرض:** تقرير أسبوعي لكل عميل، يُسلّم في نفس اليوم/الوقت، يوثّق القيمة.

---

## 1. The Report Mission

**Weekly Value Report = "ما الذي تسلّمته هذا الأسبوع، ولماذا يهم."**

ليس:
- ❌ Status report (للمهام فقط)
- ❌ Time report (للساعات)
- ❌ Invoice (للمال)

**بل:**
- ✅ Value delivered
- ✅ Issues addressed
- ✅ Next steps
- ✅ Approval requests

---

## 2. The Format

### 2.1 Header
```
Weekly Value Report — [Client Name]
Week: [Date Range]
Author: [CS Agent]
Approved: [Founder]
Version: 1
```

### 2.2 Sections

**1. What was done (ما تم)**
- 3-5 bullet points
- Specific (not vague)
- Tied to success metric

**2. What improved (ما تحسّن)**
- Metric (with is_estimate)
- Comparison to last week
- Trend

**3. What is blocked (ما هو معلق)**
- Issues
- Why blocked
- What's needed

**4. What needs approval (ما يحتاج موافقة)**
- Decisions pending
- Recommendations
- Sign-off needed

**5. Next best action (أفضل خطوة قادمة)**
- This week
- Top 3 priorities

**6. Renewal/Expansion signal (إشارة تجديد/توسعة)**
- Health score update
- Any signals
- Next renewal milestone

---

## 3. Length

- 1-2 pages max
- Bullet points
- Data highlights
- Action items

---

## 4. Frequency

- **Weekly** (every Monday for prior week)
- Or per package agreement
- Never miss a week (consistency)

---

## 5. Approval

- CS Agent drafts
- Founder approves
- Sent to client
- Archive in CRM

---

## 6. The 5 Sections — Detailed

### 6.1 What was done

```
✅ [Specific deliverable 1]
✅ [Specific deliverable 2]
✅ [Specific deliverable 3]
✅ [Issue resolved]
```

**Example:**
- ✅ Follow-up workflow live for 3 channels
- ✅ 5 draft templates created
- ✅ 1 dashboard configured
- ✅ Slack channel created
- ✅ Resolved: WhatsApp template issue

### 6.2 What improved

```
📈 [Metric]: from [X] to [Y] (is_estimate)
📈 [Metric]: from [X] to [Y] (is_estimate)
```

**Example:**
- 📈 Response time: from 24h to 4h (is_estimate)
- 📈 Follow-up completion: 30% → 70% (is_estimate)

### 6.3 What is blocked

```
⚠️ [Issue]: [Why blocked]: [What's needed]
```

**Example:**
- ⚠️ HubSpot integration: waiting for API key from client: need IT contact

### 6.4 What needs approval

```
🔐 [Decision]: [Recommendation]: [Sign-off needed]
```

**Example:**
- 🔐 Add 2 more templates: recommend yes: founder approval needed

### 6.5 Next best action

```
Next week:
1. [Priority 1]
2. [Priority 2]
3. [Priority 3]
```

**Example:**
1. Reschedule integration call
2. Add 2 more templates
3. First 30-day report prep

### 6.6 Renewal/Expansion signal

```
Health: [score] [color]
Renewal: [status]
Expansion: [status]
```

**Example:**
- Health: 75 (Yellow)
- Renewal: 60 days out
- Expansion: 1 workflow requested (good signal)

- Versioned
- Searchable

---

## 10. Common Mistakes

### 10.1 ❌ Don't Do
- ❌ Vague bullets
- ❌ Status only (no value)
- ❌ No data
- ❌ No next steps
- ❌ No approval requests
- ❌ Too long
- ❌ No visual

### 10.2 ✅ Do
- ✅ Specific
- ✅ Value-focused
- ✅ Data + is_estimate
- ✅ Clear next steps
- ✅ Approval requests
- ✅ Concise
- ✅ Visual when useful
