# قائمة الجاهزية للإطلاق (Ready-to-Launch Checklist)

## المجالات المطلوبة

- [x] الموقع (5 أنظمة + حلول قطاعية + تشخيص)
- [x] الأنظمة الجوهرية الخمسة
- [x] كتالوج أنظمة الأعمال (40 نظام داخلي)
- [x] ذكاء الاحتياج (25 احتياج / 20 قطاع / 50 سبرنت)
- [x] حزم الحسابات (400/يوم — عقد المخرجات)
- [x] اكتشاف القنوات العامة (بلا اختلاق)
- [x] مصنع البريد (مسودات للاعتماد)
- [x] Call Briefs
- [x] العروض المصغّرة
- [x] أتمتة التسليم + تقارير القيمة
- [x] التمويل والمؤشرات (Cash Priority)
- [x] الأمان والخصوصية (untrusted data + كبح)
- [x] أمر المؤسس اليومي
- [x] CI/CD + بوابات الجودة

## درجة الإطلاق

| النطاق | الحالة |
| ---: | --- |
| 90–100 | جاهز للإطلاق (Launch Ready) |
| 75–89 | جاهز لإطلاق ناعم (Soft Launch) |
| 60–74 | تشغيل داخلي تجريبي (Internal Dry Run) |
| < 60 | غير جاهز |

## كيف تتأكد؟

```bash
python dealix.py seed          # يولّد كل شيء
python dealix.py launch-score  # يحسب الدرجة ويكتب البطاقة
```

ثم راجع `reports/operating_factory/READY_TO_LAUNCH_SCORECARD.md`.

## تعريف «وصلت»

كل ملفات الـ manifest موجودة، كل schema تتحقق، كل البيانات صالحة، كل البوابات
تمر، الموقع يبني، وكل فحوص GitHub Actions خضراء، والدرجة ≥ 90.
