# Risk-Adjusted Pricing

Higher **risk** or **friction** → higher **price** or **narrower** scope—not the same price for harder work.

## Add premium when

| Factor | Typical premium / action |
|--------|----------------------------|
| High data sensitivity | +20–40% or separate DPA scope |
| Complex stakeholder / approval chains | +15–30% |
| Aggressive timeline | +20–50% or extend timeline |
| Integration / API work | **Separate** scoped phase |
| Enterprise compliance load | **Enterprise** pricing band |
| No clean data | Start with **diagnostic** or data cleanup sprint |

## Rule

Do not absorb incremental risk at the **same** price as a clean sprint. Log pricing rationale in [`../ledgers/DECISION_LEDGER.md`](../ledgers/DECISION_LEDGER.md) when non-standard.

See [`PRICING_DECISION.md`](PRICING_DECISION.md), [`PRICE_INCREASE_RULES.md`](PRICE_INCREASE_RULES.md).
