# Dealix Internal KPI System

Use [`../DASHBOARD.md`](../../DASHBOARD.md) (repo root) for **weekly numbers**; this file defines **what to measure**.

## Revenue

- Project revenue (cash / billed)
- MRR (retainers)
- Gross margin by service ([`SERVICE_ECONOMICS.md`](SERVICE_ECONOMICS.md))
- Proposal → close rate

## Delivery

- On-time delivery %
- Average **QA score**
- Rework count / hours
- Proof packs delivered / projects

## Product

- Manual steps automated (count / severity)
- Active modules touching delivery
- Feature adoption (when productized)

## Governance

- PII or policy incidents (target: 0 critical)
- Approval coverage (% external sends with approver)
- Blocked unsafe actions (count + category)

## Growth

- Sprint → retainer conversion
- Referrals sourced
- Anonymized case studies / proof assets published

## Loop

Feed winners into [`../market/AI_OPS_MARKET_INTELLIGENCE.md`](../market/AI_OPS_MARKET_INTELLIGENCE.md) and [`PUBLIC_AUTHORITY_ENGINE.md`](../growth/PUBLIC_AUTHORITY_ENGINE.md).
