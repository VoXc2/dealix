# Hiring Triggers

Hire only when a **system** exists for the role to follow—otherwise you import chaos.

## First delivery analyst / associate

When:

- **3+** concurrent projects repeatedly
- Tasks **repeat** (same checklist shape)
- **Templates + workbench** are in daily use
- **Margin** supports payroll

## First product engineer

When:

- **3+** repeated manual workflows with evidence ([`FEATURE_BACKLOG.md`](../product/FEATURE_BACKLOG.md) scores **80+**)
- Internal tools save **measurable** hours
- Governance patterns are stable enough to code

## First customer success

When:

- **3+** retainers
- Monthly reporting / QBR rhythm is real
- Founder bottleneck on **client follow-up**

## First governance reviewer (part-time acceptable)

When:

- Sensitive data projects increase
- Enterprise interest appears
- Approvals / redaction **bottleneck** QA

## Rule

**Document → simulate → hire.** See [`SCALE_DECISION.md`](SCALE_DECISION.md).
