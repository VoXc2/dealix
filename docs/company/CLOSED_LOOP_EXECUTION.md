# Closed-Loop Execution

Every Dealix engagement must move through **one closed loop**—from first request to expansion—so each client compounds into system capability ([`COMPOUNDING_SYSTEM.md`](COMPOUNDING_SYSTEM.md)).

**Context:** Scaling AI often fails when models are dropped onto **legacy operating environments** without integration and talent ([Gartner on AI scaling roadblocks in supply chain](https://www.gartner.com/en/newsroom/press-releases/2026-04-29-gartner-survey-finds-technology-integration-and-talent-perceived-as-key-roadblocks-to-scaling-ai-in-supply-chain)). Dealix sells **process + governance + proof**, not “AI on chaos.”

## Loop

1. **Request** — classified intake ([`../operations/REQUEST_INTAKE_SYSTEM.md`](../operations/REQUEST_INTAKE_SYSTEM.md))
2. **Qualification** — score before price ([`../sales/QUALIFICATION_ENGINE.md`](../sales/QUALIFICATION_ENGINE.md))
3. **Scope** — written boundary ([`../delivery/SCOPE_ENGINE.md`](../delivery/SCOPE_ENGINE.md))
4. **Governance check** — allow / approve / block ([`../governance/GOVERNANCE_DECISION.md`](../governance/GOVERNANCE_DECISION.md))
5. **Delivery** — execution against checklist
6. **QA** — scored gate ([`../delivery/DELIVERY_DECISION.md`](../delivery/DELIVERY_DECISION.md))
7. **Proof pack** — evidence bundle ([`../templates/proof_pack_2.md`](../templates/proof_pack_2.md))
8. **Client review** — structured meeting ([`../delivery/CLIENT_REVIEW_MEETING.md`](../delivery/CLIENT_REVIEW_MEETING.md))
9. **Learning review** — `POST_PROJECT_REVIEW` + playbooks
10. **Productization** — templates, backlog, features ([`../product/PRODUCTIZATION_DECISION.md`](../product/PRODUCTIZATION_DECISION.md))
11. **Expansion** — retainer / next sprint ([`../growth/EXPANSION_OFFER_SYSTEM.md`](../growth/EXPANSION_OFFER_SYSTEM.md))

## Rule

No project is **internally complete** until:

- Client output is **delivered** (per scope)
- **QA** is passed (threshold per policy)
- **Governance** is **logged** (project-level + client-level as needed)
- **Proof pack** is delivered
- **Post-project learning** is captured ([`LEARNING_LOOP.md`](../templates/LEARNING_LOOP.md))
- **Next-step offer** is made (explicit options)
