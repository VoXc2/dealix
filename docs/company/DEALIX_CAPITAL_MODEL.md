# Dealix Capital Model

> **Rule**: Dealix does not measure itself only by revenue. It measures
> itself by the capital that accumulates after every customer.
> 
> Every project must increase **at least two** of the five capitals below.
> A project that just generates cash without compounding any capital is
> selling time, not building a company.

## The 5 capitals

### 1. Service Capital
Reusable offers, scopes, intake forms, delivery checklists, QA standards,
SOWs. Lives in `docs/services/*` + `docs/company/SERVICE_REGISTRY.md` +
`templates/sow/`.

### 2. Product Capital
Internal tools, modules, APIs, dashboards, automations that reduce
delivery effort. Lives in `auto_client_acquisition/*` + `dealix/*` +
`docs/product/CAPABILITY_MATRIX.md`.

### 3. Knowledge Capital
Playbooks, sector insights, objection counters, data patterns,
governance risks, templates. Lives in `docs/playbooks/*` +
`docs/sales/objection_handling.md` + `docs/assets/templates/`.

### 4. Trust Capital
Proof Packs, anonymized case studies, governance records, audit history,
testimonials, safety reputation. Lives in `docs/assets/proof_packs/` +
`docs/assets/case_studies/` + `docs/governance/RISK_REGISTER.md`.

### 5. Market Capital
Audience, partners, referrals, benchmarks, authority content. Lives in
`docs/growth/*` + `docs/assets/case_studies/` + content channels.

## Project closure question

Before marking a project closed (per `COMPOUNDING_SYSTEM.md`):

```
Did this project increase…
  [ ] Service Capital? (new template, refined scope, better checklist)
  [ ] Product Capital? (new feature shipped, automation saved time)
  [ ] Knowledge Capital? (playbook updated, new objection counter, new insight)
  [ ] Trust Capital? (Proof Pack written, anonymized case study drafted)
  [ ] Market Capital? (referral surfaced, content published, partner intro)
```

Must check ≥ 2. Ideally ≥ 3. Less than 2 → project is operationally
incomplete; founder fixes before next.
