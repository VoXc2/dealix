# Dealix Decision Rules (CEO)

This is the **compact constitution**. The full decision map lives in [`DECISION_OPERATING_SYSTEM.md`](DECISION_OPERATING_SYSTEM.md). Every material call must still produce **evidence · score · risk · owner · next action**.

## Rule 1 — Do not sell what is not ready

Offer ≥ 85 · Delivery ≥ 85 · Governance ≥ 90 (معيار يدوي + نجاح التحقق) · Demo ≥ 85 · Sales ≥ 85 — تفاصيل [`SELLABILITY_POLICY.md`](SELLABILITY_POLICY.md).

## Rule 2 — Beta must be labeled

Score 70–84 → **pilot/beta** فقط بتوقعات مكتوبة وسعر مناسب.

## Rule 3 — No unsafe automation

Scraping بلا إذن · cold WhatsApp · LinkedIn automation واسعة · fake proof · ادعاءات مبيعات مضمونة · PII في السجلات · أي إجراء خارجي بلا موافقة — [`FORBIDDEN_ACTIONS.md`](../governance/FORBIDDEN_ACTIONS.md).

## Rule 4 — No client report without next action

كل تقرير للعميل: ماذا وجدنا · ماذا يعني · ماذا نفعل بعد · proof pack · عرض upsell مقترح.

## Rule 5 — Build after repetition

Feature فقط إذا: تكرّر **3 مرات** **أو** دفع عميل مقابلها **وتقلل زمن التسليم** **أو** ترفع الجودة/الحوكمة — سجل في [`FEATURE_CANDIDATE_LOG.md`](FEATURE_CANDIDATE_LOG.md).

## Rule 6 — Services become products

كل خطوة تتكرر → template · checklist · script · API · dashboard · أتمتة آمنة — انظر [`CAPABILITY_MATRIX.md`](../product/CAPABILITY_MATRIX.md).

## Intake (No chaos)

طلب جديد → تصنيف خدمة → فحص readiness → risk → scope → ثم موافقة بيع. لا «نجرب مجاناً» بدون scope وسعر ونتيجة متفق عليها.
