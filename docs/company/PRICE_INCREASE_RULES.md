# Price Increase Rules

Complements [`PRICING_DECISION.md`](PRICING_DECISION.md).

## When to raise price

Raise when **several** are true:

- Closed **3+** deals at current price **without** meaningful pushback
- **Delivery** got faster / more repeatable (you capture value)
- **Proof packs** and case patterns are strong
- **Demand** exceeds capacity for a sustained window
- Client **sees ROI** (even qualitative—documented)
- Service is **officially** repeatable ([`SELLABILITY_DECISION.md`](SELLABILITY_DECISION.md))

## Typical bands (tune to market)

```text
Entry / diagnostic:     +10–20%
Core sprint:           +20–30%
Retainer (monthly):    +15–25%
Enterprise / custom:   priced per risk + scope
```

## How to communicate

- Anchor on **value + risk reduction + speed**, not inflation
- Grandfather existing contracts until renewal unless scope changes ([`../delivery/CHANGE_REQUEST_SYSTEM.md`](../delivery/CHANGE_REQUEST_SYSTEM.md))
