# AI FinOps

## Purpose

Control AI costs without hurting quality.

## Track

- cost per service  
- cost per client  
- cost per AI run  
- cost per report  
- cost per proof pack  
- cost per workflow  

## Optimize

- cache repeated outputs  
- route simple tasks to cheaper models  
- use stronger models for high-risk outputs  
- reduce unnecessary context  
- batch processing when possible  

## Rules

- each service has an AI budget  
- cost overrun requires review  
- enterprise clients get AI cost reporting
