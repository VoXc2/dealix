# سجل المخاطر — المخاطر القائمة والضوابط

**الطبقة:** L7 · محرك التنفيذ
**المالك:** المؤسس / الرئيس التنفيذي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [RISK_REGISTER.md](./RISK_REGISTER.md)

## السياق
هذا هو سجل المخاطر القائم لديليكس بوصفها شركة عمليات ذكاء اصطناعي
يقودها مؤسس واحد في الشرق الأوسط وشمال أفريقيا. يُسمّي كل بند نمط فشل
متكرراً، وأثره على العمل، والضابط الذي يحتويه. السجل هو المُكمّل
التشغيلي لدليل الحوادث في `docs/V6_OBSERVABILITY_AND_INCIDENT_RUNBOOK.md`
والدستور في `docs/DEALIX_OPERATING_CONSTITUTION.md`.

## المخاطر القائمة

| المخاطرة | الأثر | الضابط |
|---|---|---|
| الإفراط في البناء | استنزاف الوقت ورأس المال بلا إيراد. | البناء بعد 3 تكرارات لنفس الطلب من العملاء. |
| ضعف المبيعات | لا إيراد؛ ضغط على المدرج. | التركيز على ICP واحد؛ الحدود الدنيا الأسبوعية لحلقة البيع. |
| تمدّد النطاق | هامش منخفض؛ إرهاق المؤسس. | SOW صارم + طلب تغيير مكتوب. |
| ضعف البيانات | مخرج سيّئ؛ يُلام الذكاء الاصطناعي. | بوّابة جاهزية البيانات قبل تنفيذ الذكاء. |
| مشكلة امتثال | ضرر للثقة؛ تعرّض قانوني. | Governance OS؛ بوّابة PDPL. |
| عربية ضعيفة | ضعف ثقة المشتري؛ خسارة مصداقية إقليمية. | دليل جودة العربية؛ مراجعة من ناطق أصلي على كل أصل عربي. |
| لا إثبات | لا توسعة؛ لا وقود لدراسات الحالة. | حزمة إثبات إلزامية لكل تسليم. |
| اختناق المؤسس | لا توسعة؛ تراجع الجودة. | `delivery_os` + مُحفّزات التوظيف (`docs/HIRING_CSM_FIRST.md`). |
| تكلفة الذكاء | خسارة هامش؛ ضغط تسعير. | بوّابة LLM + حارس تكلفة؛ استراتيجية التوجيه. |

## مستويات المخاطر

| المستوى | التعريف | إيقاع المراجعة |
|---|---|---|
| قائمة | في هذا الملف؛ متكررة عبر الشركة. | ربعي |
| ارتباط | خاصة بـ SOW واحد. | لكل سبرنت |
| حادثة | تحققت؛ تُتابَع في دليل الحوادث. | عند وقوعها |

## دورة حياة المخاطرة

```
تحديد ─► تصنيف ─► إضافة ضابط ─► رصد ─► مراجعة
                                       │
                                       ▼
                                    حادثة (إن تحققت)
```

لا تُغلَق المخاطرة بمجرد وجود الضابط، بل عند تشغيل الضابط لربع كامل دون
خرق. حتى ذلك الحين تبقى *مفتوحة بتخفيف*.

## ملكية الضوابط

كل ضابط أعلاه مربوط بأصل مُسمّى:

| المخاطرة | أصل الضابط |
|---|---|
| الإفراط في البناء | بوّابة Capability backlog (المرحلة 4 من الخارطة). |
| ضعف المبيعات | `docs/sales/ONE_PAGER.md`، الحلقات الأسبوعية. |
| تمدّد النطاق | قالب SOW؛ نموذج طلب تغيير. |
| ضعف البيانات | `data_os.data_quality_score`. |
| الامتثال | `governance_os.policy_check`. |
| جودة العربية | قائمة جودة العربية. |
| لا إثبات | `reporting_os.proof_pack`. |
| اختناق المؤسس | دليل `delivery_os`. |
| تكلفة الذكاء | `docs/COST_OPTIMIZATION.md`، بوّابة LLM. |

## إضافة مخاطرة جديدة

يحتاج الإدخال الجديد إلى:
- اسم بلغة بسيطة.
- جملة واحدة عن الأثر.
- ضابط مُسمّى أو مسودة ضابط مع مالك.
- مقياس رصد.

المخاطر بلا ضابط أو مقياس رصد ليست مسموحة هنا، بل تعيش في دليل الحوادث
بنوداً مفتوحة حتى تنضج.

## الواجهات
| المدخلات | المخرجات | الملّاك | الإيقاع |
|---|---|---|---|

## المقاييس
- المخاطر المفتوحة مع تخفيف — العدد.
- تحويل مخاطر إلى حوادث — نسبة المخاطر التي تحوّلت إلى حوادث.
- تغطية الضوابط — نسبة المخاطر بضابط مُسمّى.
- اكتمال المراجعة الربعية — صح/خطأ.
