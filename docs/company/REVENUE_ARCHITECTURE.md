# Revenue Architecture

Where money **should** come from over time.

## Streams

1. **Diagnostics** — low-risk entry, trust-building  
2. **Sprints** — main **cash** engine  
3. **Pilots** — operational proof in a timebox  
4. **Retainers** — **recurring** revenue  
5. **Enterprise** — high-value custom (gated)  
6. **Academy** — training / certification (later)  
7. **Platform** — workspace + usage modules (later)

## Financial intent (directional)

```text
Cash now      = Sprints
Stability     = Retainers
Scale         = Platform (only when repeatable)
Authority     = Academy (after evidence mass)
Large deals   = Enterprise (after readiness)
```
