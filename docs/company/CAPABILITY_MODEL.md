# Dealix Capability Model

بدل «نبيع خدمات»، قل **«نبني قدرات تشغيلية»**.

Dealix builds **seven core capabilities** for clients.

## 1. Revenue Capability

The ability to identify, prioritize, and convert revenue opportunities.

## 2. Customer Capability

The ability to understand, classify, respond to, and learn from customers.

## 3. Operations Capability

The ability to automate repeatable processes with human approval and audit.

## 4. Knowledge Capability

The ability to retrieve trusted company knowledge with sources.

## 5. Data Capability

The ability to prepare, clean, score, and use data for decisions and AI.

## 6. Governance Capability

The ability to use AI safely with policies, approvals, logs, and controls.

## 7. Reporting Capability

The ability to generate executive reports and value dashboards from operational data.

---

## Capability Maturity

| Level | Name | Meaning |
|---:|---|---|
| 0 | Absent | no process, no owner, no data |
| 1 | Manual | work happens manually and inconsistently |
| 2 | Structured | process and owner exist |
| 3 | AI-Assisted | AI drafts, scores, classifies, or reports |
| 4 | Governed Workflow | approvals, QA, audit, and metrics exist |
| 5 | Optimized OS | recurring cadence, dashboard, and improvement loop exist |

## Proposal language (required)

Every proposal states:

```text
Current capability level:
Target capability level:
Service to move from current to target:
```

This positions Dealix as **strategic**, not task labor.
