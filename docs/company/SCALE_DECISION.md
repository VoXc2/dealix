# Scale Decision (Hire, Delegate, Partner)

Scale **process**, not chaos.

## Scale only when

- [ ] Service is **repeatable** (same shape of delivery)
- [ ] **Checklists** and templates exist (`docs/templates/`, `clients/_TEMPLATE/`)
- [ ] **QA** is stable and predictable
- [ ] **Another person can deliver** using docs + simulation (see `docs/training/`)
- [ ] **Margins** support payroll or partner share
- [ ] Demand is **recurring**, not one-off luck

## Rule

**Do not hire to fix undocumented delivery.** Document first → simulate → then add capacity.

## First hire (typical triggers)

Scale toward a hire/partner when:

1. **3+** paid projects in a repeatable shape
2. Same **service** repeated with manageable variance
3. Checklists + QA **exist and are used**
4. **Margin** supports the role
5. Founder time is **trapped in delivery** vs sales/product
