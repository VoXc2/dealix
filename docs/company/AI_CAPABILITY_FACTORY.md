# Dealix AI Capability Factory

> Dealix is not a project shop. It is a **factory** that takes any
> business problem and turns it into an AI-enabled operating capability
> with workflows, agents, governance, QA, proof, and a continuous-
> improvement cadence.

## The 11-step factory flow

```
1. Intake               → Request Ledger entry, classification
2. Capability Assessment → score 7 capabilities, pick the gap
3. Data Readiness        → quality + sources + PII + lawful basis
4. Workflow Design       → trigger / input / AI step / governance / output
5. Governance Design     → which Runtime Governance checks apply
6. AI Build              → schema-bound agent outputs, LLM Gateway routed
7. QA + Evals            → 5-gate QA + 100-point score + eval registry
8. Delivery              → handoff with executive report + Proof Pack
9. Proof Pack            → before/after evidence in Value Ledger
10. Capability Roadmap   → maturity advance plan (Level X → X+2)
11. Retainer / Expansion → ongoing operating cadence
```

## Hard rule

Every Dealix engagement is **named by the capability it builds**, not by
the deliverable it ships.

| Service (deliverable) | Capability it builds |
|-----------------------|----------------------|
| Lead Intelligence Sprint | Revenue Capability |
| AI Quick Win Sprint | Operations Capability |
| Company Brain Sprint | Knowledge Capability |
| AI Support Desk Sprint | Customer Capability |
| AI Governance Program | Governance Capability |
| Data Readiness Assessment | Data Capability |
| Executive Reporting Automation | Reporting Capability |

## Sales reframe (use everywhere)

Don't say: *"We sell a Company Brain Sprint."*
Say: *"We build your Knowledge Capability — from scattered documents
(Level 1) to a cited internal assistant (Level 3) in 21 days, with the
roadmap to Level 5."*

## Per-capability blueprint

Each of the 7 capabilities has a canonical blueprint in
`docs/capabilities/<capability>.md` covering: business purpose, typical
problems, required inputs, AI functions, governance controls, KPIs, and
which Dealix services build / advance it.

(See `docs/capabilities/revenue_capability.md` for the seed blueprint.)
