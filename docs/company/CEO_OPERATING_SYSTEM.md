# CEO Operating System — مجلس التشغيل والأسئلة القيادية

**الفكرة:** صمّم الشركة وكأن هناك **CEO Operating Council** — حتى لو المؤسس وحده اليوم؛ كل «مقعد» يطرح سؤالًا مختلفًا فيرفع جودة القرار.

**الطبقة الكاملة:** [`../group/DEALIX_OPERATING_GOVERNANCE_LAYER.md`](../group/DEALIX_OPERATING_GOVERNANCE_LAYER.md) · **البوابات التشغيلية:** [`SERVICE_READINESS_GATE.md`](SERVICE_READINESS_GATE.md)

---

## 1. CEO Council — من يسأل ماذا؟

| المقعد | السؤال |
|--------|--------|
| **CEO / Strategy** | هل هذا **يبني الشركة** (ليس المشروع فقط)؟ |
| **Product** | هل يتكرر ويستحق التحويل إلى software؟ |
| **Delivery** | هل نستطيع تسليمه بجودة ثابتة؟ |
| **Growth** | هل يساعد البيع أو الثقة أو الشراكات؟ |
| **Governance** | هل فيه خطر بيانات أو سمعة أو امتثال؟ |
| **Finance** | هل الهامش صحيّ؟ |
| **Capital / Knowledge** | ما **الأصل** الذي نبنيه بعد هذا المشروع؟ |

---

## 2. أسئلة يومية (بدون جدول «أيام»)

1. أهم إجراء إيراد اليوم؟  
2. أي تسليم **في خطر**؟  
3. أي **خطر حوكمة** ظهر؟  
4. أي **proof** يمكن إنتاجه اليوم؟  
5. أي **خطوة يدوية** تكرّرت مؤخرًا؟  

---

## 3. أسئلة أسبوعية

1. ماذا بِعنا؟  
2. ماذا سلّمنا؟  
3. ماذا أثبتنا (proof)؟  
4. ماذا منتجنا (productization)؟  
5. ماذا تعلّمنا؟  
6. ماذا **نتوقف** عنه؟  
7. ماذا **يجب** أن يصبح retainer؟  

---

## 4. أسئلة شهرية (Holding)

1. أي **وحدة** الأقوى؟  
2. أي **عرض** أفضل هامشًا؟  
3. أي **قدرة** الأعلى طلبًا؟  
4. أي **playbook** يصبح venture؟  
5. أي **قناة شركاء** تعمل؟  
6. أي **وحدة برمجية** تقلّل وقت التسليم؟  

---

## 5. مؤشرات «وصلنا مستوى قابضة»

- Core OS عبر كل الخدمات · proof + capital لكل مشروع · **2+** retainers · **3+** playbooks  
- Productization Ledger حيّ · KPIs لكل وحدة · شريك يبيع لك  
- Client Workspace مطلوب · Academy/Standard قابلة للتسويق  

---

## 6. Endgame flywheel

```text
Services feed Cloud — Proof feeds Sales — Academy trains Partners
Partners feed Services — Labs create Ventures — Ventures use Core OS
Standards create Category
```
