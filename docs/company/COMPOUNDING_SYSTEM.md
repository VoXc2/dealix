# Dealix Compounding System

Every project must make Dealix stronger.

## The six internal assets (after each project)

1. **Client Outcome** — مخرجات العميل.
2. **Proof Asset** — دليل الأثر.
3. **Template** — شيء يعاد استخدامه.
4. **Playbook Update** — تعلم جديد عن قطاع أو حالة استخدام.
5. **Product Feature Candidate** — خطوة متكررة تتحول إلى feature.
6. **Sales Asset** — شيء يساعدك تبيع العميل التالي.

## After each project, capture

1. What did we deliver?
2. What proof did we create?
3. What template can be reused?
4. What playbook should be updated?
5. What manual step should become a feature?
6. What sales asset can be created?

## Rule

**A project is not complete until internal learning is captured.**  
If there is no Proof Pack, no Post-Project Review, no Feature Candidate note, and no Playbook Update, the project is **internally incomplete** even if the client paid.

## Where to capture

| Output | Primary location |
|--------|------------------|
| Post-project review | `projects/<client>/<project>/POST_PROJECT_REVIEW.md` (from template) |
| Proof pack (client) | `clients/<client>/06_proof_pack.md` + deliverables |
| Anonymized proof for sales | `docs/assets/proof_packs/anonymized_<sector>_<service>_proof_pack.md` |
| Templates & playbooks | `docs/assets/templates/`, `docs/playbooks/` |
| Feature candidates | [`FEATURE_CANDIDATE_LOG.md`](FEATURE_CANDIDATE_LOG.md) |
