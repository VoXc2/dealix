# Capability Packages — Commercial Repackaging

> The customer-facing presentation. Same Dealix services as
> `docs/company/SERVICE_REGISTRY.md`, repackaged by **capability built**
> rather than by deliverable shipped. This is how executives buy.

## Why packages instead of individual services

Executives don't buy "a Lead Intelligence Sprint". They buy **a Revenue
Capability upgrade**. Packaging by capability:

- Anchors the conversation in business outcomes.
- Makes the upsell path obvious (Sprint → Pilot → Retainer = capability levels 3 → 4 → 5).
- Justifies higher prices (capability, not deliverable).
- Reduces "compare us to a cheaper agency" objections.

## The 7 capability packages

### 1. Revenue Capability Package

**For**: Companies with messy leads / unused CRM / unclear pipeline.
**Capability moved**: Level 1 → Level 5 over time.
**Components**:
- Revenue Diagnostic (SAR 3,500–7,500) → Level 1 → measured baseline.
- **Lead Intelligence Sprint (SAR 9,500)** → Level 1 → Level 3.
- Pipeline Setup (SAR 8,000–25,000) → Level 2 → Level 3.
- Pilot Conversion Sprint (SAR 22,000–45,000) → Level 3 → Level 4.
- Monthly RevOps OS (SAR 15,000–60,000 / mo) → Level 4 → Level 5.

### 2. Customer Capability Package (Phase 2)

**For**: Companies with overloaded WhatsApp / email / inconsistent replies.
**Components**:
- AI Support Desk Sprint *(when Sellable)*.
- Inbox Operations Setup.
- Feedback Intelligence.
- Monthly Support AI.

### 3. Operations Capability Package

**For**: Companies losing hours to manual recurring work.
**Components**:
- **AI Quick Win Sprint (SAR 12,000)** → Level 0–1 → Level 3.
- Workflow Automation Sprint (Phase 2) → Level 3 → Level 4.
- Executive Reporting Automation (Phase 2).
- SOP Automation (Phase 2).
- Monthly AI Ops.

### 4. Knowledge Capability Package

**For**: Companies with rich documents and "where is that PDF" pain.
**Components**:
- **Company Brain Sprint (SAR 20,000)** → Level 1 → Level 3.
- Sales Knowledge Assistant.
- Policy Assistant.
- Proposal Assistant.
- Monthly Company Brain Management.

### 5. Data Capability Package

**For**: Companies that "want AI" but data isn't ready.
**Components**:
- AI Ops Diagnostic (SAR 7,500–25,000).
- Data Readiness Assessment (planned).
- Data Cleanup & Unification.
- Data OS Retainer (Phase 3+).

### 6. Governance Capability Package (Phase 3)

**For**: Companies with shadow-AI usage / PDPL exposure / compliance fear.
**Components**:
- AI Readiness Review.
- AI Usage Policy.
- PDPL-Aware Data Review.
- AI Governance Program.

### 7. Reporting Capability Package (Phase 2)

**For**: Executives drowning in manual weekly reports.
**Components**:
- Executive Reporting Automation (setup + monthly).
- Monthly AI Ops Retainer (reporting-focused variant).

## Cross-cut: Diagnostic gateway

For prospects who don't know which capability to build first, the
**AI Ops Diagnostic** (SAR 7,500–25,000) runs the Capability Assessment
across all 7 capabilities and recommends the first Sprint.

## Sales motion (binding)

When pitching:

1. Open with capability framing ("Which capability is most painful right now?").
2. Run the Diagnostic (paid) if the prospect can't answer.
3. Quote the first Sprint in the relevant package, NOT the SOW deliverable.
4. Show the capability ladder (Level 1 → 5) so the prospect sees the path.
5. End with the next package step (Pilot or Retainer) framed as "Level X → Level X+1".
