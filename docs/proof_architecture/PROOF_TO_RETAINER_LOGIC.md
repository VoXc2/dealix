# Proof-to-Retainer Logic

Continue / Expand / Pause + Retainer Gate.

## Three motions

- **Continue** — same capability, recurring.
- **Expand** — adjacent capability.
- **Pause** — value not yet sufficient.

## Retainer Gate

All of:
- proof_score ≥ 80
- client_health ≥ 70
- workflow recurring
- owner exists
- monthly value clear
- governance risk controlled

## Retainer mapping

| Proof type | Retainer |
| --- | --- |
| Revenue | Monthly RevOps OS |
| Knowledge | Monthly Company Brain |
| Risk | Monthly Governance |
| Time / Quality | Monthly AI Ops |

## Typed surface

`operating_manual_os.proof_to_retainer.evaluate_retainer_gate()`. Layer 10 also adds `proof_architecture_os.proof_to_retainer` as a thin alias for the value-architecture caller path.
