# Dealix Enterprise Proof & Value Architecture

> The firm that cannot prove AI value will keep selling promises. Dealix must sell **Proof before Features**.

Tenth strategic layer. Consolidated doctrine. Typed surfaces live in `auto_client_acquisition/proof_architecture_os/`.

## 1. Proof has five levels

1. **Activity Proof** — records imported, AI runs completed, drafts generated.
2. **Output Proof** — clean account list, ranked opportunities, executive report.
3. **Quality Proof** — QA score, Arabic quality, citation coverage, schema pass rate.
4. **Business Value Proof** — time saved, pipeline value, risk reduced, decision speed.
5. **Operating Capability Proof** — workflow owner, monthly cadence, approval path, dashboard, ongoing proof metrics, retainer path.

Level 5 is what opens the retainer.

## 3. Proof Score — 8 weights, total 100

| Component | Weight |
| --- | ---: |

Tiers: 85+ case candidate, 70–84 sales support, 55–69 internal learning, <55 weak proof.

Hard rules:
- No Proof Pack <85 used as a public case.
- No Proof Pack <70 used in sales.
- No retainer when proof score <80.

## 4. Value Metrics by Offer

- **Revenue Intelligence Sprint:** records imported, duplicates removed, data quality score, accounts scored, top opportunities identified, drafts generated, unsafe outreach blocked, pipeline value estimated.
- **Company Brain Sprint:** sources registered, documents indexed, questions tested, citation coverage, insufficient evidence rate, knowledge gaps found, answer QA score.
- **AI Governance Review:** use cases inventoried, risk rules created, approval paths defined, unsafe actions blocked, policy gaps found, audit coverage designed.
- **AI Quick Win Sprint:** manual steps mapped, workflow created, approval path defined, hours saved estimate, QA pass rate, handoff completion.
- **Executive Reporting Automation:** manual reports reduced, sources connected, report freshness, decision recommendations generated, QA score, time-to-report reduction.

## 5. Proof-to-Retainer

Continue / Expand / Pause. Retainer gate: proof score ≥ 80, client health ≥ 70, recurring workflow, named owner, clear monthly value, governance risk controlled.

## 7. ROI Discipline — three levels

- **Estimated** — assumption-based forecast.
- **Observed** — measured during the engagement.
- **Verified** — confirmed by the client or measured after operating cadence.

Rule: never mix Estimated with Verified; never use Estimated as marketing; only Verified powers public cases.

## 8. Case-Safe Proof Summary

Anonymized version of the Proof Pack. Sector + problem pattern + work performed + anonymized metrics + lessons + general insight. No confidential data. No client name without consent.

## 10. Proof + Governance Loop

Every Proof Pack records **what was blocked**:

```
cold WhatsApp blocked
unknown-source PII blocked
guaranteed claim removed
source-less answer rejected
external action queued for approval
```

Risk prevented = business value.
