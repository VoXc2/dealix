# Case-Safe Proof Summary

The anonymized version of a Proof Pack.

## Required fields

- sector
- problem pattern
- work performed
- anonymized metrics
- lessons
- general insight

## Anonymization

- No client-identifying data unless consent_to_publish is true.
- No PII.
- No confidential metrics.

## Example

> A Saudi B2B services company uploaded a messy account file. Dealix identified duplicate records, scored priority accounts, generated draft-only outreach, and blocked unsafe external actions.

## Typed surface

`proof_architecture_os.case_safe_summary.CaseSafeProofSummary` rejects client-name disclosure without consent.
