## Core belief (EN)

**Dealix is built as an AI Operations Holding Company** — legally one entity early on, **architecturally** a group: shared Core OS + business units + ledgers + academy/partners/ventures.

---

## Operating loop (EN)

```text
Service → Proof → Retainer → Product → Playbook → Business Unit → Venture
```

---

## Core OS (EN)

All units share:

- Data OS  
- Governance OS  
- LLM Gateway  
- Proof Ledger  
- Capital Ledger  
- Quality OS  
- Client Workspace  

---

## Rule — project completion (EN)

**No project is complete unless it creates:**

1. **Cash** (or defensible path to cash in defined sprint scope)  
2. **Proof** (pack + value signal)  
3. **Capital asset** (trust, knowledge, product, service, or market)  
4. **Next offer** (expansion / retainer / upsell path)  
5. **Productization signal** (what repeated, what to module next)

---

## الجملة القيادية (AR)

**Dealix لا تنجح لأنها تستخدم AI. تنجح لأنها تبني نظامًا يجعل كل استخدام للـ AI محكومًا، قابلًا للقياس، قابلًا للبيع، وقابلًا للتحول إلى أصل مركّب داخل شركة قابضة.**

**الصيغة المختصرة**

```text
Cash now. Proof always. Governance by default. Productize repetition.
Compound every client. Scale through units.
```

→ Product Modules → Business Units → Academy / Partners → Ventures → Holding Company
