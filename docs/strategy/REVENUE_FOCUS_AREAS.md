# Dealix Revenue Focus Areas

This file prioritizes where Dealix should focus commercial and product effort.

## Highest-value focus

| Focus area | Why it matters | Evidence to track |
|---|---|---|
| Demo request reliability | Converts website interest into pipeline. | Request success, routing success, response SLA. |
| Quote and inquiry follow-up | Common pain in Saudi B2B services. | Response time, qualified meetings, objections. |
| Agency operating dashboard | Agencies need repeatable client reporting. | Reporting time saved, handoff quality, client clarity. |
| SaaS revenue control plane | SaaS teams need pipeline and proof visibility. | Demo rate, lead quality, proof-pack usage. |
| Proof packs | Buyers need evidence, not generic AI claims. | Source evidence, owner, customer-safe summary. |

## Best starting sectors

1. B2B services.
2. Logistics and operations services.
3. Agencies and growth firms.
4. SaaS and IT service providers.
5. Property service operators.

## 30-day operating plan

- Pick one sector.
- Define one measurable operating problem.
- Prepare one diagnostic offer.
- Run 3 to 5 diagnostics.
- Turn the strongest proof into one case asset.
- Update the no-overclaim register before publishing any claim.

## Stop rules

Do not scale commercial activity until:

- Production smoke is passing.
- Demo request routing is verified.
- Public claims have evidence.
- Rollback and incident paths are known.
