# Proof Pack Example (Trust Page)

A **proof pack** ties delivery to evidence and value:

- **Scope** — what was in / out of the sprint.  
- **Inputs** — data sources and assumptions (redacted as needed).  
- **Outputs** — artifacts produced (reports, workflows, assistant behavior).  
- **QA / eval** — scores or checks run (governance, claims, grounding).  
- **Value** — link to [`VALUE_LEDGER.md`](../../ledgers/VALUE_LEDGER.md) entry (metric, baseline, result, next value).  

This is illustrative; real packs follow the client’s confidentiality rules.
