# Dealix Client Journey

Stages from first touch to referral and expansion.

```
Awareness → Diagnostic → Sprint → Pilot → Retainer → Expansion → Enterprise → Referral
```

## Stage 1: Awareness

**Goal:** the client understands what Dealix is and when it fits.  
**Assets:** content, demo, sample report (`docs/assets/demos/`, `docs/assets/reports/`).

## Stage 2: Diagnostic

**Goal:** identify a concrete business use case and readiness.  
**Assets:** readiness / diagnostic assessment (see service-specific intakes).

## Stage 3: Sprint

**Goal:** deliver first measurable output in a bounded timebox.  
**Assets:** proof pack, delivery checklist, QA sign-off.

## Stage 4: Pilot

**Goal:** operate the workflow for ~30 days with clear cadence.  
**Assets:** operating report, weekly or biweekly review rhythm.

## Stage 5: Retainer

**Goal:** recurring operation, optimization, and governance.  
**Assets:** monthly dashboard / operating scorecard, decision log.

## Stage 6: Expansion / Enterprise

**Goal:** multi-workflow "AI OS" with integrations and SLAs where needed.  
**Assets:** governance pack, integration map, escalation paths.

## Stage 7: Referral

**Goal:** turn success into new pipeline (with consent and anonymization).  
**Assets:** anonymized case study and proof pack in `docs/assets/case_studies/` and `docs/assets/proof_packs/`.

See also: `docs/sales/SALES_PIPELINE.md`, `docs/company/COMPOUNDING_SYSTEM.md`.
