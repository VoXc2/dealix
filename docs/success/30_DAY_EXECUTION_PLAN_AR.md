## 2. الأسبوع 1 — Foundation

```txt
- إغلاق الموقع (رسالة بسيطة + 5 أنظمة + صفحات قطاعات + تسعير).
- الأنظمة الخمسة معرّفة بوضوح.
- Delivery Packs للأنظمة (تبدأ بالأولوية الأعلى).
- Launch Scorecard جاهز.
```

**مخرج الأسبوع:** أساس قابل للعرض + أول Delivery Pack جاهز للبيع.

---

## 3. الأسبوع 2 — Acquisition Engine

```txt
- Account Packs.
- Contact Discovery (سياسة، لا تخمين جهات اتصال).
- Email Drafts (مسودات فقط، بلا إرسال).
- Call Briefs.
- Mini Proposals.
- Top 100 Queue.
```

**مخرج الأسبوع:** قناة جلب جاهزة تُنتج فرصاً مرتّبة بانتظار موافقة المؤسس.

---

## 4. الأسبوع 3 — Controlled Soft Launch

```txt
- 50–100 Account Packs/day.
- 10–25 sends يدوياً (بعد موافقة).
- 10–20 calls.
- Mini Proposals.
- قياس الردود.
```

**مخرج الأسبوع:** أول إشارات سوق حقيقية + بيانات ردود قابلة للتحليل.

> **حدود صارمة:** كل إرسال بموافقة المؤسس. لا اتصال آلي. لا واتساب بارد. لا قوائم مشتراة.

---

## 5. الأسبوع 4 — Learning + Scale Decision

```txt
- تحليل القطاعات.
- أفضل Sprint.
- أفضل price point.
- أول صفقات.
- تحسين التسليم.
- قرار رفع الإنتاج.
```

**مخرج الأسبوع:** قرار مبني على بيانات: هل ننتقل من Launch Mode إلى Scale Mode؟

---

## 6. بوابتا الانتقال إلى التوسّع

لا يُرفع الإنتاج قبل تحقّق البوابتين:

### Launch Score (هل أطلقنا فعلاً؟)

```txt
[ ] الموقع والأنظمة الخمسة منشورة.
[ ] Delivery Pack واحد على الأقل مكتمل وقابل للبيع.
[ ] بوابات الموافقة فعّالة (إرسال/عرض/تسعير/تسليم).
[ ] أول ردود سوق مُسجّلة.
```

### Scale Score (هل نوسّع؟)

```txt
[ ] أول صفقة مُسلَّمة بـ Weekly Value Report.
[ ] هامش الـ Sprint > 60%.
[ ] سعة تسليم كافية للحجم الأعلى.
[ ] حلقة تعلّم أسبوعية تعمل.
```
