# AI Ops Diagnostic

**SKU / folder:** `ai_ops_diagnostic` — use as the **entry gate** when the client says: “We want AI but don’t know where to start.”

## Promise

Identify the best **AI use cases**, **data readiness** gaps, and **operational risks** before implementation.

## Deliverables

- **Capability assessment** ([`CAPABILITY_ASSESSMENT.md`](CAPABILITY_ASSESSMENT.md)) — seven capabilities × maturity
- **AI / ops readiness** narrative (axes + evidence)
- **Data readiness** review (sources, gaps, PII); deep dive → [`../data_readiness_assessment/README.md`](../data_readiness_assessment/README.md)
- **Top 5** use cases (prioritized)
- **Risk map** (governance, claims, channels)
- **30-day** roadmap + **recommended sprint** ([`SERVICE_REGISTRY.md`](../../company/SERVICE_REGISTRY.md))
- Client templates: [`../../clients/_TEMPLATE/CAPABILITY_ROADMAP.md`](../../clients/_TEMPLATE/CAPABILITY_ROADMAP.md), [`../../clients/_TEMPLATE/AI_OPERATING_MODEL.md`](../../clients/_TEMPLATE/AI_OPERATING_MODEL.md)

## When to use

- Unclear problem / many ideas
- Messy or unknown data estate
- Need one **defensible** next step vs. generic “AI strategy”
