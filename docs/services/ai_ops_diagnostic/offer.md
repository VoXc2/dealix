# Offer — AI Ops Diagnostic

## Promise

Identify the **highest-value, lowest-risk** AI **operating** opportunities in the company—grounded in capability, data, process, and governance ([`CAPABILITY_ASSESSMENT.md`](CAPABILITY_ASSESSMENT.md)).

## Deliverables

- **Capability assessment** (seven capabilities × levels)  
- **Data readiness** review (high level; deep dive → Data Readiness Assessment if needed)  
- **Process readiness** review (owners, inputs, outputs)  
- **Governance risk** map  
- **Top 5** use cases (prioritized)  
- **Recommended first sprint** + **30-day** roadmap  
- Alignment with [`../company/CAPABILITY_MATURITY_MODEL.md`](../company/CAPABILITY_MATURITY_MODEL.md) for sales language (“L1 → L3”)  

## Diagnostic score dimensions (prioritization)

Weight mentally across:

```text
Business value
Data readiness
Process clarity
Owner readiness
Governance risk
Implementation complexity
Time-to-value
```

## Decision rule

```text
High value + low risk = first sprint
High value + high risk = readiness work first
Low value + high complexity = do not start
```

## What the client gets (summary)

- Structured assessment of opportunity **vs** reality  
- Prioritized shortlist with **risk** and **governance** notes  
- Clear **next sprint** from catalog (or explicit “not ready”)  

## Outcomes (KPI)

- One **primary** success definition for the next phase  
- Clear **go/no-go** on at least one sprint  

## Exclusions (defaults)

- No production implementation in this engagement unless separately scoped  
- No cold outbound automation  
- No scraping without explicit legal/client-supplied data path  
- No guaranteed business results
