# Delivery Checklist

Phased over ~45 days. Each phase ends with a documented artifact.

## Phase 1 — Discovery

- [ ] stakeholder map and interviews complete
- [ ] AI use inventory across departments
- [ ] current tools, vendors, and data flows recorded
- [ ] readiness review criteria agreed and signed

## Phase 2 — Policy

- [ ] company AI policy drafted
- [ ] allowed / prohibited use defined
- [ ] employee AI usage rules drafted
- [ ] policy reviewed with client owners

## Phase 3 — Risk Matrix

- [ ] AI risk matrix populated by use case
- [ ] likelihood / impact scoring agreed
- [ ] mitigations assigned to owners

## Phase 4 — Approval Classes

- [ ] approval classes defined (e.g. auto / review / restricted)
- [ ] data-handling rules mapped to each class
- [ ] sign-off authority recorded per class

## Phase 5 — Audit & Logging

- [ ] audit logging design documented
- [ ] human-approval workflows specified
- [ ] retention and access rules agreed

## Phase 6 — PDPL Readiness

- [ ] PDPL gap assessment complete
- [ ] remediation actions prioritized
- [ ] readiness statement drafted (not legal sign-off)

## Phase 7 — Vendor / Model Risk

- [ ] vendor and model risk review complete
- [ ] data residency and processing terms checked

## Phase 8 — Dashboard

- [ ] executive governance dashboard specified
- [ ] key indicators and owners defined

## Phase 9 — Readiness Review

- [ ] internal QA complete ([`qa_checklist.md`](qa_checklist.md))
- [ ] readiness review held against agreed criteria
- [ ] handoff package delivered ([`handoff.md`](handoff.md))
