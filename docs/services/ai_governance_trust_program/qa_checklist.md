# QA Checklist

Run before the readiness review.

## Content

- [ ] AI policy covers allowed, prohibited, and approval-required use
- [ ] risk matrix has owner and mitigation for every entry
- [ ] approval classes map to data-handling rules with no gaps
- [ ] audit logging and human-approval workflows fully specified
- [ ] PDPL readiness presented as gap assessment, not certification

## Compliance

- [ ] no scraping, cold WhatsApp, live send, or live charge described
- [ ] no fake or unverified proof used in any artifact
- [ ] no PII (real names, emails, phones, national IDs) in samples
- [ ] no legal advice language — readiness program only

## Copy

- [ ] forbidden terms absent (e.g. "guaranteed" / "نضمن")
- [ ] commitment language used ("we commit" / "نلتزم")
- [ ] customer-facing files end with the bilingual disclosure line
- [ ] AR and EN sections balanced where bilingual

## Sign-off

- [ ] readiness review criteria match what was agreed in discovery
- [ ] client owners named for every governance area
