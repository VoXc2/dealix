# Delivery Checklist

## Setup (onboarding)

- [ ] Engagement scope and pricing confirmed in writing
- [ ] Executive sponsor and approver named
- [ ] Time-saving commitment agreed and recorded as baseline
- [ ] Data sources connected (revenue, customer-health, operational-risk)
- [ ] Security review completed for each connected source
- [ ] Hard gates confirmed: no_live_send, no_live_charge, no_cold_whatsapp, no_linkedin_auto, no_scraping, no_blast
- [ ] Signal thresholds and opportunity radar calibrated with leadership
- [ ] Approval queue configured with named approvers
- [ ] Proof Ledger access provisioned
- [ ] First weekly executive brief format reviewed and signed off

## Weekly cadence

- [ ] Refresh revenue, customer-health, and operational-risk signals
- [ ] Update opportunity radar
- [ ] Draft decision memos for items needing an executive call
- [ ] Run QA checklist before the brief reaches the executive
- [ ] Deliver weekly executive brief
- [ ] Log approvals and decisions to the Proof Ledger

## Monthly cadence

- [ ] Compile monthly Proof Pack (see [`proof_pack_template.md`](proof_pack_template.md))
- [ ] Review time-saving commitment vs baseline
- [ ] Recalibrate signal thresholds if false positives are rising
- [ ] Confirm no hard gate has been crossed

## Quarterly cadence

- [ ] Quarterly growth strategy review with leadership
- [ ] Reassess opportunity radar priorities
- [ ] Update success metric and retainer scope if needed
- [ ] Confirm readiness gates still hold (see [`readiness_scorecard.md`](readiness_scorecard.md))
