# Tool Usage Rules

## Approved tools

Maintain a single table: tool name, vendor, data residency notes, allowed data classes, retention notes.

## Account hygiene

- no shared personal accounts for company work where avoidable  
- SSO/preferred enterprise plans when available  

## Logging

Where feasible: record prompts/outputs policy (what is logged vs not) per tool class.

## Model use

- default to least-sensitive model tier for low-risk tasks  
- escalate model tier for high-risk tasks with extra QA  

## Exit

Offboarding checklist: revoke access, export required records, delete per retention.
