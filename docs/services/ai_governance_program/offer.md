# AI Governance Program — Offer / عرض

**Pillar:** Govern AI / حوكمة الذكاء الاصطناعي · **Modules:** Governance + Knowledge + Reporting + Compliance

## Promise / الوعد
Stand up a PDPL-aware AI governance program for the customer's organization in 1–3 months: inventory every AI tool and data flow, run a Saudi-specific risk assessment, install an approval matrix, write the AI usage policy, and produce audit-ready evidence. Outputs are review-ready for SAMA / NCA / ZATCA / SDAIA inspections.

## Commercials / التجاريات
- **Price / السعر:** SAR 35,000–150,000 (excl. VAT).
  - SAR 35,000 — SME (1 BU, < 50 employees, <= 20 tools).
  - SAR 75,000 — mid-market (3 BUs, <= 75 tools).
  - SAR 150,000 — enterprise (multi-BU, > 75 tools, multi-jurisdiction, board reporting).
- **Duration / المدة:** 1–3 months (SME = 1 month; mid-market = 2 months; enterprise = 3 months).
- **Payment / الدفع:** 40% kickoff / 30% mid-program / 30% on delivery.
- **SOW template:** `templates/sow/ai_governance_program.md` (to be authored).

## Deliverables / المخرجات
1. AI tool inventory (every model, every vendor, every internal agent) — see `ai_tool_inventory.md`.
2. Data inventory + ROPA-aligned mapping — see `data_inventory.md`.
3. Saudi-specific risk assessment (PDPL + SAMA + NCA where applicable) — see `risk_assessment.md`.
4. AI usage policy + acceptable-use addendum (bilingual) — see `policy_template.md`.
5. Approval matrix tuned to the customer's actions and roles — see `approval_matrix.md`.
6. Audit-trail blueprint + monthly evidence pack template — see `audit_requirements.md`.
7. Bilingual training (3 hours, recorded) for staff + 1-hour board briefing.
8. Stage-6 (Deliver) handoff with evidence pack and 60/30/15-min agenda — see `handoff.md`.

## Best For / مناسب لـ
- Regulated industries (BFSI, healthcare, gov-adjacent) preparing for inspection.
- Mid-market and enterprise customers introducing GenAI org-wide.
- Boards demanding a documented governance posture before scaling AI.

## Compliance / الامتثال
- **PDPL Art. 13/14** referenced in every customer-facing flow that touches personal data.
- **Approval matrix** authored on top of `dealix/trust/approval_matrix.py` — every high-stakes action requires the matched approver role.
- Outputs cite `docs/policy/revenue_os_policy_rules.md` for enforcement points.
- Enterprise risk register cross-referenced: `docs/legal/enterprise_risk_register.md`.
