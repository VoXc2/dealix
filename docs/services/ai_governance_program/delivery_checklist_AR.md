# قائمة تسليم — برنامج حوكمة الذكاء الاصطناعي

**الطبقة:** كتالوج الخدمات · العدّة التشغيلية
**المالك:** قائد التسليم — الحوكمة
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [delivery_checklist.md](./delivery_checklist.md)

## السياق
سيناريو التشغيل مرحلة-بمرحلة لبرنامج حوكمة AI مدته 4–12 أسبوعًا. قابل للتكرار والتدقيق وفق `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md` و`docs/governance/RUNTIME_GOVERNANCE.md` و`docs/quality/QUALITY_STANDARD_V1.md`. كل خانة تُربَط بحدث في حزمة الإثبات.

## الأدوار
- **قائد التسليم (DL)** — محاسَب واحد.
- **مراجِع الحوكمة (GR)** — مالك المحتوى الأساسي.
- **اختصاصي الامتثال (CS-comp)** — PDPL، ZATCA، طبقات قطاعية.
- **قائد القدرة (CL-cap)** — التوقيع + الطبقات القطاعية (مثلًا: العيادات).
- **راعي العميل (SP)** — القبول النهائي.
- **DPO العميل** — نظير العميل الأساسي.
- **مستشار العميل** — مُشرَك لأي جانب تنظيمي.
- **مراجِع الجودة (QA)** — مستقل.

## الخطة مرحلة-بمرحلة

### T-10 إلى T-1 — قبل الإطلاق
- [ ] إرسال حزمة الترحيب. (DL)
- [ ] إنشاء لوحة البرنامج الداخلية. (DL)
- [ ] إعادة تحقّق العلاوة؛ تطبيق علاوة القطاع. (DL + ضابط الهامش)
- [ ] تهيئة حزمة الإثبات بـ `program_initialized`. (DL)
- [ ] التحقق من اتصال مستشار-التسجيل. (DL)

### المرحلة 1 — الجرد + خريطة المخاطر (الأسابيع 1–3)
- [ ] ورشة إطلاق (90 دقيقة) مع SP وDPO ومراقب المستشار. (DL + GR)
- [ ] استلام قائمة أدوات AI. (العميل + GR)
- [ ] المرور بكل أداة مع مالكها المُسمَّى. (GR)
- [ ] تشغيل ورشة تدفّق البيانات إن لزم (ساعتان). (GR)
- [ ] تقييم الأساس القانوني لكل مجموعة بيانات. (CS-comp + DPO العميل)
- [ ] صياغة سجل المخاطر الأوّلي: أعلى 30 خطر مُرتَّبًا. (GR)
- [ ] مراجعة الجرد وسجل المخاطر مع SP. (DL)
- [ ] أحداث إثبات: `ai_uses_inventoried = N`, `risks_logged = N`. (GR)

### المرحلة 2 — السياسة + مصفوفة الموافقات (الأسابيع 3–5)
- [ ] تحليل السياسات القائمة؛ تحديد الثغرات. (CS-comp)
- [ ] صياغة مصفوفة الموافقات: أنماط حالات استخدام AI × أدوار المُعتمِدين. (GR + CS-comp)
- [ ] تحديث حتى 5 سياسات (استخدام AI، معالجة البيانات، الموردين، الحوادث، الموافقات). (CS-comp)
- [ ] مستشار العميل يراجع تحديثات السياسة. (المستشار + DL)
- [ ] اختبار مصفوفة الموافقات مقابل الجرد: كل استخدام AI له مُعتمِد واضح. (GR)
- [ ] أحداث إثبات: `approvals_documented = N`, `policy_updates = M`. (GR)

### المرحلة 3 — التدقيق + سجل المخاطر + مصفوفة الضوابط (الأسابيع 5–8)
- [ ] تصميم عملية التدقيق: تدقيق ذاتي ربعي + قائمة سنوية بنمط طرف ثالث. (CS-comp + GR)
- [ ] إنهاء سجل المخاطر: مالك، معالجة، إيقاع مراجعة، خطر متبقٍ. (GR + DPO العميل)
- [ ] بناء مصفوفة الضوابط، مُتوائمة مع `docs/enterprise/CONTROLS_MATRIX.md`. (GR + CS-comp)
- [ ] تسجيل نضج الضابط لكل صف. (GR)
- [ ] تحديث/إنشاء Runbook استجابة الحوادث. (CS-comp + DPO العميل)
- [ ] تدقيق تجريبي على 3 استخدامات AI مُختارة عشوائيًا. (GR + QA)
- [ ] أحداث إثبات: `controls_implemented = N`, `incidents_addressed = N`. (GR + CS-comp)

### المرحلة 4 — التدريب + التسليم (الأسابيع 8–12)
- [ ] بناء حزمة التدريب: تنفيذي (45 دقيقة)، مالك (90 دقيقة)، باني (3 ساعات)، مستخدم (30 دقيقة). (DL + GR)
- [ ] جلسة تدريب-المدرّبين مع العميل (ساعتان). (GR)
- [ ] تسليم الطبقات القطاعية إن انطبق (عيادات، مالية، حكومة). (CL-cap)
- [ ] صياغة تقرير الإثبات النهائي + الملخص التنفيذي. (DL)
- [ ] تمرير الجودة النهائي. (QA)
- [ ] ورشة تسليم (3 ساعات) مع SP وDPO ومراقب المستشار. (DL + GR)
- [ ] SP يوقّع مذكرة التسليم؛ DPO يوقّع كذلك. (SP + DPO)
- [ ] أحداث إثبات: `program_delivered`, `proof_pack_signed_off`. (DL)
- [ ] تشغيل حركة البيع الإضافي. (DL + CSM)

## الضوابط الشاملة
- البيانات الحساسة تُعالَج فقط في الخزنة؛ لا تُنسَخ أبدًا.
- مكالمة توجيه أسبوعية مع SP وDPO وDL وGR.
- مراقب المستشار مدعوّ لكل الورش الكبرى.
- لعملاء الرعاية الصحية / الحكومة: CL-cap يوقّع إضافة إلى QA.

## الواجهات
| المدخلات | المخرجات | المالكون | الإيقاع |
|---|---|---|---|
| SOW موقَّع + الاستلام | لوحة البرنامج | DL | T-10 |
| توجيه أسبوعي | تحديثات البرنامج | DL | أسبوعيًا |
| نتائج QA + GR + CL-cap | تذاكر إصلاح أو توقيع | QA + GR + CL-cap + DL | نهاية كل مرحلة |
| التسليم + التقرير النهائي | التسليم | DL + SP + DPO | الأسبوع الأخير |

## المقاييس
- **التسليم في الوقت** — الهدف ≥ 85%.
- **اكتمال الجرد** — الهدف ≥ 95%.
- **تغطية الأساس القانوني** — الهدف = 100%.
- **تغطية مصفوفة الموافقات** — الهدف = 100%.
