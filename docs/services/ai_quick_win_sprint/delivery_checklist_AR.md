# قائمة تسليم — سبرنت المكسب السريع للذكاء الاصطناعي

**الطبقة:** كتالوج الخدمات · العدّة التشغيلية
**المالك:** قائد التسليم — العمليات
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [delivery_checklist.md](./delivery_checklist.md)

## السياق
سيناريو التشغيل اليومي لسبرنت AI Quick Win مدة 7 أيام. يجب أن يكون كل سبرنت قابلًا للتكرار والتدقيق وفق `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md` و`docs/quality/QUALITY_STANDARD_V1.md`. كل خانة تُربَط بحدث حزمة إثبات يُستهلَك في `docs/templates/PROOF_PACK_TEMPLATE.md`.

## الأدوار
- **قائد التسليم (DL)** — محاسَب واحد.
- **المصمّم (DS)** — تصميم وبناء سير العمل.
- **مالك سير العمل (WO)** — نقطة الاتصال الوحيدة لدى العميل، توفّر ساعة يوميًا.
- **مراجِع الجودة (QA)** — مراجِع مستقل.
- **الراعي (SP)** — القبول النهائي.

## الخطة يومًا بيوم

### T-2 إلى T-1 — قبل الإطلاق
- [ ] إرسال بريد الترحيب مع المجلد المُشارَك وملخص الاستلام وجدول اليوم 1. (DL)
- [ ] إنشاء لوحة السبرنت الداخلية في Notion. (DL)
- [ ] إعادة تحقّق العلاوة مقابل الاستلام. (DL + ضابط الهامش)
- [ ] تهيئة حزمة الإثبات بحدث `sprint_initialized`. (DL)

### اليوم 1 — رسم خريطة الحالة الراهنة
- [ ] مكالمة إطلاق (30 دقيقة) مع SP وWO. (DL)
- [ ] قفل اختيار سير العمل. (DL + SP)
- [ ] WO يأخذ DS عبر العملية الحالية. (WO + DS)
- [ ] DS يُنتج مسودة workflow_map.md. (DS)
- [ ] حدث إثبات: `workflow_mapped`. (DS)

### اليوم 2 — إنهاء الخريطة
- [ ] WO يُراجع ويصحّح خريطة سير العمل. (WO)
- [ ] بدء قياس الأساس اليدوي (دورة واحدة من العملية الحالية مُؤقَّتة). (WO + DS)
- [ ] قفل الأساس بالساعات/دورة. (WO + DS)
- [ ] حدث إثبات: `baseline_measured`. (DS)

### اليوم 3 — تصميم خطوة AI
- [ ] DS يُصمّم حدود خطوة AI. (DS)
- [ ] تحديد موقع بوّابة الموافقة. (DS)
- [ ] اختيار الأدوات من `docs/AI_STACK_DECISIONS.md`. (DS)
- [ ] فحص الخطر: لا auto-send / auto-pay / auto-contract. (QA + DS)
- [ ] WO + SP يوقّعون التصميم. (WO + SP)
- [ ] حدث إثبات: `ai_step_designed`. (DS)

### اليوم 4 — بناء المسوّدة
- [ ] DS يبني تنفيذًا بحد أدنى صالح. (DS)
- [ ] DL يراجع المخرج الوسيط في منتصف اليوم. (DL)
- [ ] تجهيز مدخلات الاختبار من نماذج العميل. (DS)
- [ ] حدث إثبات: `draft_built`. (DS)

### اليوم 5 — مراجعة المالك
- [ ] WO يمشي عبر المسوّدة مباشرة مع DS. (WO + DS)
- [ ] WO يُنتج قائمة تعديلات. (WO)
- [ ] DS يُطبّق التعديلات نفس اليوم. (DS)
- [ ] حدث إثبات: `owner_reviewed`. (WO)

### اليوم 6 — دورة الاختبار
- [ ] WO يشغّل دورة كاملة عبر التدفّق الجديد. (WO)
- [ ] قياس الوقت لكل خطوة. (WO + DS)
- [ ] حساب تقدير الوقت المُوفَّر. (DS)
- [ ] QA يشغّل قائمة الجودة. (QA)
- [ ] أي بوّابة فشلت → إصلاح نفس اليوم. (DL)
- [ ] حدث إثبات: `test_cycle_complete`, `manual_steps_reduced = N`. (WO + DS)

### اليوم 7 — SOP + الإثبات
- [ ] صياغة SOP بالعربية + الإنجليزية. (DS)
- [ ] إنهاء حزمة الإثبات. (DL + QA)
- [ ] صياغة المخرج التنفيذي. (DL)
- [ ] تمرير الجودة النهائي. (QA)
- [ ] مكالمة تسليم (45 دقيقة) مع SP وWO. (DL)
- [ ] SP يوقّع مذكرة التسليم. (SP)
- [ ] حدث إثبات: `sprint_delivered`, `hours_saved = X`. (DL)
- [ ] تشغيل حركة البيع الإضافي (وفق `upsell.md`). (DL + CSM)

## الضوابط الشاملة
- يُرصَد توفّر WO يوميًا؛ غياب يومَيْن فأكثر يُحفّز توقّفًا.
- يُتتبَّع كل تغيير في المُسلَّمات في لوحة السبرنت.
- لا تُسلَّم مُسلَّمة دون توقيع QA.
- تُحدَّث حزمة الإثبات باستمرار.

## المقاييس
- **التسليم في الوقت** — الهدف ≥ 95%.
- **نجاح الجودة المرة الأولى** — الهدف ≥ 75%.
- **التزام القائمة** — الهدف = 100%.
- **توفّر المالك** — `% days where WO was available ≥ 1h`. الهدف ≥ 90%.
