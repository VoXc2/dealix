# قائمة الجودة — سبرنت مكتب الدعم بالذكاء الاصطناعي

**الطبقة:** كتالوج الخدمات · العدّة التشغيلية
**المالك:** مراجِع الجودة (مستقل)
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [qa_checklist.md](./qa_checklist.md)

## السياق
بوّابات الجودة الواجب اجتيازها بحلول اليوم 14. يجب ألا يكون المراجِع قد أنتج المُسلَّمات. يُنفِّذ البروتوكول `docs/quality/QUALITY_STANDARD_V1.md` ومصفوفة الإنسان-في-الحلقة في `docs/governance/HUMAN_IN_THE_LOOP_MATRIX.md`. يرتبط بالخطة الاستراتيجية في `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md`.

## كيفية الاستخدام
1. افتح سجل السبرنت في Notion.
2. امشِ عبر كل بوّابة: نجاح / فشل / لا ينطبق + مؤشّر دليل.
3. الإخفاقات → تذاكر إصلاح نفس اليوم.
4. تنجح الجولة عندما تكون كل البوّابات Pass أو N/A.

## البوّابة 1 — الفئات تغطّي ≥ 90% من العيّنة
- [ ] عيّنة الاحتجاز مُصنَّفة بتغطية ≥ 90%.
- [ ] لسلّة "أخرى / غير مُصنَّف" قاعدة خطوة تالية قابلة للتنفيذ.
- [ ] كل فئة لها تعريف واضح + 3 رسائل نموذجية.

## البوّابة 2 — كل ردّ مُقترَح بعلامة `draft_only`
- [ ] كل ردّ في المكتبة له `draft_only` في الميتاداتا.
- [ ] لا ردّ يحتوي تعليمات "أرسل الآن" / "ردّ تلقائي".
- [ ] لا ردّ ينتحل وكيلًا بشريًا ("أنا جون من الدعم").

## البوّابة 3 — المواضيع الحساسة مُصعَّدة لإنسان
- [ ] كل نوع حالة حساسة يُوجَّه لإنسان مُسمَّى.
- [ ] لا حالة حساسة لها ردّ تلقائي، أبدًا.
- [ ] علامة مائية مُطبَّقة على الردود الحساسة.
- [ ] حالات اختبار لكل نوع حساس نجحت.
- [ ] لعملاء العيادات: الحالات الطبية صلة تُوجَّه لطبيب.

## البوّابة 4 — لا إرسال تلقائي في MVP
- [ ] لا مسار شيفرة يُحفّز رسالة صادرة دون موافقة بشرية.
- [ ] التكامل مع واتساب / البريد للقراءة فقط أو للمسوّدة فقط.
- [ ] لا مُجدوِل خلفي يستطيع إرسال ردود.

## البوّابة 5 — لا PII في السجلات
- [ ] السجلات تستخدم معرّفات، لا قيمًا خامًا للعملاء.
- [ ] النماذج الحساسة لا تغادر الخزنة المُشفَّرة.
- [ ] التصديرات المُجهَّلة للتسويق تجتاز فحص PII.

## البوّابة 6 — النبرة العربية ناجحة
- [ ] السجل السعودي/الخليجي مُطبَّق على الردود العربية.
- [ ] فحص النبرة مقابل نماذج العميل.
- [ ] لا آثار ترجمة حرفية من الإنجليزية للعربية.
- [ ] علامات اللطف ملائمة للسياق (رسمي مقابل ودّي).

## البوّابة 7 — فحص الحوكمة مُسجَّل
- [ ] وضع علامات الحساسية موجود لكل فئة.
- [ ] إقرار الأساس القانوني موجود.
- [ ] احتفاظ PDPL مُطبَّق.
- [ ] الموقف العابر للحدود يطابق `CROSS_BORDER_TRANSFER_ADDENDUM`.
- [ ] لعملاء العيادات: قائمة `clinics_playbook` موقَّعة من GR.

## البوّابة 8 — متتبّع SLA يعمل
- [ ] كل الفئات لها أهداف SLA.
- [ ] المتتبّع يلتقط أزمنة أول ردّ وحل.
- [ ] مدخلات اختبار تدفّقت بشكل صحيح.
- [ ] عرض اللوحة يُظهِر الخروقات بالأحمر.

## البوّابة 9 — حزمة الإثبات كاملة
- [ ] كل الأحداث المطلوبة موجودة.
- [ ] كل حدث له طابع زمني وفاعل وقيمة.
- [ ] حزمة الإثبات موقَّعة من DL + QA + GR + SL.

## البوّابة 10 — التسليم والبيع الإضافي
- [ ] مذكرة التسليم موقَّعة.
- [ ] توصية البيع الإضافي مُسوَّدة.
- [ ] مراجعة اليوم 30 مجدوَلة (اختياري).

## التصعيد
- 3+ إخفاقات → صعِّد لمالك القدرة.
- إخفاق البوّابة 3 أو 4 → تُعلَّق المهمّة؛ إعادة تصميم الحالة الحساسة.
- إخفاق البوّابة 7 → تُعلَّق المهمّة؛ مراجعة قانونية.
- أي إخفاق بوّابة عيادات → صعِّد لمراجِع clinics_playbook.

## المقاييس
- **نجاح المرة الأولى** — الهدف ≥ 70%.
- **إخفاقات البوّابات لكل سبرنت** — الهدف ≤ 2.
- **حوادث PII** — الهدف = 0.
- **حوادث إرسال تلقائي** — الهدف = 0 (يجب أن يبقى 0 للأبد).
