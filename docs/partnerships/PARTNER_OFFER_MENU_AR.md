# قائمة العروض للشركاء — Partner Offer Menu

## مقدمة

يحدد هذا المستند العروض المتاحة للشركاء مع تفاصيل كل عرض، الشروط، والأسعار.

---

## نظرة عامة على العروض

| العرض | الوصف | الجمهور المستهدف | هامش الشريك |
|-------|-------|-----------------|-------------|
| Referral Standard | إحالة عميل مقابل عمولة ثابتة | جميع الشركاء | 10-15% |
| Referral Premium | إحالة + دعم في البيع | الشركاء النشطون | 15-20% |
| Revenue Share | نسبة من الإيرادات المتكررة | مطبقو CRM، الاستشاريون | 20-25% |
| White-label | تنفيذ بالنيابة | وكالات الويب، الأتمتة | 30-40% |
| Joint Delivery | تنفيذ مشترك | مطبقو CRM، المستشارون | متغير |
| Reseller | إعادة بيع | الموزعون | 25-35% |
| Strategic Alliance | شراكة طويلة الأمد | الشركات الكبرى | حسب الاتفاق |

---

## 1. Referral Standard

### الوصف
يتابع الشريك العميل المحتمل ويقدمه لـ Dealix. عند إغلاق الصفقة، يحصل الشريك على عمولة.

### التفاصيل
| البند | الشرح |
|-------|-------|
| **دور الشريك** | تحديد وتمرير العملاء المحتملين |
| **دور Dealix** | البيع والإغلاق والتنفيذ |
| **ملكية العميل** | Dealix |
| **الهامش** | 10-15% من قيمة الصفقة |
| **توقيت الدفع** | عند إغلاق الصفقة |
| **مسؤولية التسليم** | Dealix |
| **مسؤولية الدعم** | Dealix |
| **ملكية التجديد** | Dealix |
| **المخاطر** | منخفضة |
| **مستوى الموافقة** | مدير الشراكات |

### الشروط
- العميل يجب أن يكون جديداً على Dealix
- العميل لا يكون في_pipeline Dealix خلال 90 يوم السابقة
- الإحالة موثقة عبر نموذج رسمي
- الدفع خلال 30 يوم من استلام الفاتورة

### حالات الاستخدام
- وكالات تسويق تحيل عملاء يحتاجون CRM
- مكاتب محاسبة تحيل عملاء جدد
- مدربو مبيعات يحيلون خريجي برامجهم

## 3. Revenue Share

### الوصف
يتقاسم الشريك في الإيرادات المتكررة للعميل الذي أحاله.

### التفاصيل
| البند | الشرح |
|-------|-------|
| **دور الشريك** | تحديد وتمرير + دعم في الاحتفاظ |
| **دور Dealix** | البيع والتنفيذ والإدارة |
| **ملكية العميل** | Dealix |
| **الهامش** | 20-25% من MRR |
| **توقيت الدفع** | شهرياً، بعد 60 يوم من التحصيل |
| **مسؤولية التسليم** | Dealix |
| **مسؤولية الدعم** | Dealix |
| **ملكية التجديد** | Dealix |
| **المخاطر** | متوسطة |
| **مستوى الموافقة** | المؤسسون (⚠️) |

### الشروط
- مدة العقد: سنتان كحد أدنى
- نسبة Revenue Share تنخفض بعد السنة الأولى (20% السنة الأولى، 15% السنة الثانية)
- يتم الدفع فقط للعملاء الذين يدفعون
- مدة الاحتفاظ: الشريك يستحق العمولة طالما العميل نشط

### حالات الاستخدام
- مطبقو CRM الذين يحافظون على علاقة مستمرة
- مستشارون يستثمرون في نجاح عملائهم

---

## 4. White-label

### الوصف
ينفذ الشريك بيع وتنفيذ Dealix باسمه الخاص.

### التفاصيل
| البند | الشرح |
|-------|-------|
| **دور الشريك** | البيع والتنفيذ والدعم |
| **دور Dealix** | توفير المنتج والترخيص |
| **ملكية العميل** | Dealix (أو مشترك) |
| **الهامش** | 30-40% من قيمة العقد |
| **توقيت الدفع** | شهرياً حسب العقد |
| **مسؤولية التسليم** | الشريك |
| **مسؤولية الدعم** | الشريك (مع دعم Level 2 من Dealix) |
| **ملكية التجديد** | Dealix |
| **المخاطر** | متوسطة-عالية |
| **مستوى الموافقة** | المؤسسين (⚠️) |

### الشروط
- الشريك يجب أن يجتاز تدريب المنتج
- Dealix يوفر branding toolkit
- اتفاقيات استخدام العلامة التجارية
- مراجعة دورية للجودة

### حالات الاستخدام
- وكالات ويب كبيرة تقدم CRM كخدمة
- مستشارو أتمتة يقدمون حلاً متكاملاً

## 5. Joint Delivery

### الوصف
تنفيذ مشترك بين Dealix والشريك.

### التفاصيل
| البند | الشرح |
|-------|-------|
| **دور الشريك** | جزء من التنفيذ (مثل التدريب أو التخصيص) |
| **دور Dealix** | البيع والتنفيذ الأساسي |
| **ملكية العميل** | Dealix |
| **الهامش** | متغير (حسب المساهمة) |
| **توقيت الدفع** | عند إغلاق المشروع |
| **مسؤولية التسليم** | مشترك |
| **مسؤولية الدعم** | Dealix |
| **ملكية التجديد** | Dealix |
| **المخاطر** | متوسطة |
| **مستوى الموافقة** | مدير الشراكات + المدير المالي |

### الشروط
- نطاق العمل محدد في اتفاقية المشروع
- سعر ثابت أو تكلفة + هامش
- تحديد واضح للمسؤوليات

### حالات الاستخدام
- مشاريع CRM مع مكون تدريب
- عمليات migrate مع خدمات نقل البيانات

### التفاصيل

| **توقيت الدفع** | مقدم أو شهري |
| **مسؤولية التسليم** | الشريك |
| **مسؤولية الدعم** | الشريك (مع دعم Level 2 من Dealix) |
| **ملكية التجديد** | الشريك |
| **المخاطر** | عالية |
| **مستوى الموافقة** | المؤسسين (⚠️) |

### الشروط
- اتفاقيات توزيع رسمية
- تدريب منتج إلزامي
- أهداف مبيعات دنيا (TBD)
- ضمانات للدفع

### حالات الاستخدام
- موزعو برمجيات في أسواق محددة
- Value-added resellers (VARs)

## 7. Strategic Alliance

### التفاصيل
| البند | الشرح |
|-------|-------|
| **دور الشريك** | متعدد (تسويق، مبيعات، تطوير) |
| **دور Dealix** | متعدد |
| **ملكية العميل** | حسب الاتفاق |
| **الهامش** | حسب الاتفاق |
| **توقيت الدفع** | حسب الاتفاق |
| **مسؤولية التسليم** | مشترك |
| **مسؤولية الدعم** | مشترك |
| **ملكية التجديد** | حسب الاتفاق |
| **المخاطر** | عالية |
| **مستوى الموافقة** | المؤسسين (⚠️) |

### الشروط
- اتفاقية شراكة استراتيجية
- أهداف مشتركة محددة
- مراجعات دورية (ربع سنوية)
- فريق dedicated من كل طرف

### حالات الاستخدام
- شراكات مع منصات كبيرة
- تكاملات تقنية عميقة
-共同 go-to-market

## ⚠️ قواعد صارمة

### لا نهائي

| القاعدة | السبب |
|---------|-------|
| ❌ لا اتفاقيات Revenue Share نهائية بدون موافقة المؤسسين | تأثير مالي مباشر وطويل الأمد |
| ❌ لا اتفاقيات قانونية كملاحظة نهائية | يجب المراجعة القانونية الرسمية |

### يجب

| القاعدة | السبب |
|---------|-------|
| ✅ توثيق كل اتفاق شفهي | حماية لجميع الأطراف |
| ✅ استخدام النماذج المعتمدة | الاتساق والسرعة |
| ✅ المراجعة القانونية لأي اتفاقية جديدة | حماية قانونية |

## اختيار النموذج المناسب

### شجرة القرار

```
هل لديك فريق تقني/تنفيذي؟
├── نعم → هل تريد بيع تحت علامتك؟
│   ├── نعم → White-label أو Reseller
│   └── لا → Joint Delivery أو Strategic
└── لا → هل تريد عمولة واحدة أم متكررة؟
    ├── واحدة → Referral Standard/Premium
    └── متكررة → Revenue Share
```

### جدول المقارنة السريعة

| المعيار | Referral | Revenue Share | White-label | Reseller | Strategic |
|---------|----------|---------------|-------------|----------|-----------|
| سهولة البدء | عالية | متوسط | منخفض | منخفض | منخفض |
| العائد المحتمل | محدود | جيد | كبير | كبير | كبير |
| الجهد المطلوب | منخفض | منخفض | عالي | عالي | عالي |
| التحكم | كامل | كامل | محدود | محدود | مشترك |
| المخاطرة | منخفضة | متوسطة | متوسطة | عالية | عالية |
| السرعة | سريع | سريع | بطيء | بطيء | بطيء جداً |
