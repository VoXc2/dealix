# مصفوفة الموافقات — الدستور · المعايير التأسيسية

**الطبقة:** الدستور · المعايير التأسيسية
**المالك:** قائد الحوكمة
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [APPROVAL_MATRIX.md](./APPROVAL_MATRIX.md)

## السياق
تُحوّل مصفوفة الموافقات المبدأ التشغيلي «لا فعل خارجي بلا موافقة» إلى
قاعدة وقت تشغيل صلبة. تُصنّف كل فعل تقوم به Dealix أو قوى الذكاء
لديها إلى إحدى خمس فئات، وتُحدد من يعتمد وما الدليل المطلوب قبل
التنفيذ. تُستشهد المصفوفة في `docs/DEALIX_OPERATING_CONSTITUTION.md`،
وفي `docs/governance/HUMAN_IN_THE_LOOP_MATRIX.md` (الشقيق L3 لنمط
المتدخل البشري)، وفي `docs/governance/RUNTIME_GOVERNANCE.md` الذي
يُنفّذها وقت التشغيل.

## المصفوفة
| فئة الفعل | المعتمِد | الدليل المطلوب |
|---|---|---|
| A — استبصار داخلي | لا أحد (مع تسجيل) | حدث تدقيق |
| B — مخرج للعميل | مراجع الجودة | درجة جودة ≥ 85 |
| C — تغيير داخلي | المالك | خطة تراجع |
| D — تواصل خارجي | المالك + قائد التسليم | قائمة مستلمين، موافقة، فحص ادعاءات |
| E — فعل خارجي مستقل | محظور | غير منطبق |

## تعريف الفئات
- **A — استبصار داخلي**: مخرج ذكاء يستهلكه فريق Dealix فقط للتحليل.
  مُسجَّل في سجل تشغيل الذكاء؛ الحدث هو الدليل.
- **B — مخرج للعميل**: أي مادة تُسلَّم للعميل (مسودات، تقارير، حزم
  إثبات). يلزم مراجعة جودة وفق
  `docs/quality/QUALITY_STANDARD_V1.md` بدرجة ≥ 85.
- **C — تغيير داخلي**: تعديلات تهيئة أو مطالبة أو مخطط أو سير عمل
  داخل Dealix أو داخل بيئة عميل. يلزم موافقة المالك وخطة تراجع.
- **D — تواصل خارجي**: أي رسالة خارجة أو بريد أو استدعاء API أو نشر في
  نظام طرف ثالث. يلزم اعتماد المالك وقائد التسليم. الدليل: قائمة
  المستلمين، وحالة الموافقة/العلاقة لكل مستلم، وفحص الادعاءات وفق
  `docs/governance/FORBIDDEN_ACTIONS.md`.
- **E — فعل خارجي مستقل**: أي فعل خارجي من وكيل ذكاء بلا معتمِد بشري.
  محظور بشكل دائم.

## قواعد التوجيه
- يُعلن كل سير عمل فئة الفعل الذي يُنتجه.
- عند تطبق عدة فئات، تحكم الفئة الأعلى.
- مخرج فئة B موجَّه لطرف ثالث يُرقَّى تلقائيًا إلى D.
- فعل D يحوي ادعاءً ممنوعًا يُنزَّل إلى BLOCK ويُوجَّه لقائد الحوكمة.

## مخطط الدليل
كل موافقة تُسجَّل كحدث تدقيق بهذا الشكل:

```json
{
  "approval_id": "APV-001",
  "action_class": "D",
  "approver_ids": ["delivery_lead_01", "owner_07"],
  "evidence": {
    "recipient_list": "RCP-091",
    "consent_record": "CSN-014",
    "claims_check": "passed"
  },
  "ai_run_id": "AIR-2031",
  "decision": "APPROVED",
  "expiry": "2026-05-20T00:00:00Z"
}
```

تنتهي الموافقات. موافقات فئة D تنتهي بعد سبعة أيام من الإصدار ما لم
تُمدَّد صراحة.

## الواجهات
| المدخلات | المخرجات | المالك | الوتيرة |
|---|---|---|---|
| مخرج تشغيل ذكاء | طلب موافقة | قائد منصة الذكاء | لكل مخرج |
| طلب موافقة | سجل موافقة | المعتمِد | لكل طلب |
| سجل موافقة | إطلاق الفعل أو حظره | قائد الحوكمة | لكل طلب |
| مؤقّت انتهاء | سحب الموافقة | قائد الحوكمة | يومي |

## المقاييس
- **تغطية الموافقات** — نسبة أفعال B-D ذات موافقة مُسجَّلة. المستهدف:
  100%.
- **زمن الموافقة** — ساعات بين الطلب والقرار. المستهدف: ≤ 24 لفئة D.
- **سوء استخدام موافقة منتهية** — صفر.
