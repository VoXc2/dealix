## القسم العربي

### المدخلات (الأنواع فقط — بلا أسرار)

- ~380 صف حساب (مثال توضيحي).
- 3 أنواع مصادر (إحالة، نموذج موقع، فعالية).

### المسارات المُراجَعة

| المسار | الخطوات | نقطة التسرّب التقديرية |
|---|---|---|
| الجذب | استلام ← إدخال يدوي ← تصنيف | مصدر غير مُسجَّل في ~22% من الصفوف. |
| المتابعة | تذكير ← تواصل ← ملاحظة | لا سجل يثبت أن المتابعة تمت أو نتيجتها. |
| الإغلاق | عرض ← موافقة ← فاتورة | لا نقطة اعتماد موثَّقة قبل إرسال العرض. |

### ملخص المعالجة

- حُدِّدت الصفوف المكررة المحتملة ووُضعت للمراجعة (مثال: 41 صفاً).
- وُسِّم الغموض في المصدر بدل تخمينه.
- رُسم كل مسار كخطوات قابلة للقياس مع نقاط القرار.

### المخرجات

- خريطة مرئية لثلاثة مسارات إيرادية.
- قائمة نقاط التسرّب المُرتَّبة حسب الأثر التقديري.
- توصية بنقاط قياس بسيطة لكل مسار.

### كيف سيستخدمها العميل

يستخدم الفريق الخريطة لتحديد أين يُضاف سجل أدلة أولاً، وأين توضع نقطة الاعتماد. القرار للعميل.

### ملاحظات الحوكمة

- كل مسودة تواصل خارجي تمر باعتماد بشري قبل الإرسال — لا إرسال ذاتي.
- موافقة (PDPL) والموافقة على البيانات تُراعى حيث تنطبق.
- لا تُنشر أي مخرجات حقيقية بدون موافقة موقَّعة.

## English Section

### Inputs (types only — no secrets)

- ~380 account rows (illustrative example).
- 3 source types (referral, website form, event).
- One service description document set.

### Workflows reviewed

| Workflow | Steps | Estimated leakage point |
|---|---|---|
| Acquire | Intake → manual entry → tagging | Source unrecorded on ~22% of rows. |
| Follow-up | Reminder → contact → note | No log proving follow-up happened or its outcome. |
| Close | Offer → approval → invoice | No documented approval point before the offer is sent. |

### Processing summary

- Likely duplicate rows identified and queued for review (example: 41 rows).
- Source ambiguity flagged rather than guessed.
- Each workflow mapped as measurable steps with decision points.

### Outputs

- A visual map of three revenue workflows.
- A leakage-point list ordered by estimated impact.
- A recommendation of simple measurement points per workflow.

### How the client will use it

The team uses the map to decide where to add an evidence trail first, and where to place the approval point. The decision is the client's.

### Governance notes

- Every external outreach draft passes human approval before sending — no autonomous sends.
- PDPL and data consent are respected where applicable.
- No real outputs are published without signed consent.
