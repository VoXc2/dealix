# Asset Graduation System

Not every deliverable is equal. Assets **climb stages** before they become IP or product.

## Stages

1. **Raw output** — created for one client (project-specific).
2. **Reusable template** — cleaned, generalized, de-identified.
3. **Standard asset** — used across **3+** projects with little friction.
4. **Productized asset** — supported by **tool**, script, or UI.
5. **Market asset** — used in **sales**, **content**, case studies, or **benchmark** reports.

## Example path

```text
Client report
→ report template
→ executive report standard
→ reporting_os / automation module (when built)
→ sample / anonymized report on website or deck
```

This is the path from **delivery labor** to **IP**. Track graduation in [`CAPITAL_LEDGER.md`](../ledgers/CAPITAL_LEDGER.md) and [`../company/IP_REGISTRY.md`](../company/IP_REGISTRY.md).

## Rule

Do not call something “standard” until **stage 3** evidence exists (three clean reuses).
