# Dealix Asset Library

**Rule:** any strong artifact from delivery should land here (or be linked here) so the next project starts from a higher baseline.

## Layout

```
docs/assets/
  reports/           # sample or anonymized report structures
  proof_packs/       # templates + anonymized proof for sales
  sales_assets/      # objections, pitches, before/after
  templates/         # reusable doc templates
  playbooks/         # (mirror or deep link to docs/playbooks/)
  demos/             # demo scripts / narrative
  case_studies/      # anonymized case studies
  benchmarks/        # market / ops intelligence
```

## Naming

- Reports: `*_sample_report.md` or sector-specific samples
- Proof packs: `proof_pack_template.md`, `anonymized_<sector>_<service>_proof_pack.md`

See `docs/company/COMPOUNDING_SYSTEM.md`, `docs/company/DEALIX_CAPITAL_MODEL.md`, `docs/playbooks/README.md`, and [`ASSET_GRADUATION_SYSTEM.md`](ASSET_GRADUATION_SYSTEM.md).

## Graduation

Assets progress **Raw → Template → Standard → Product → Market** — see [`ASSET_GRADUATION_SYSTEM.md`](ASSET_GRADUATION_SYSTEM.md).
