# Enterprise Readiness (Command View)

Restates the five-level ladder under the Command Model. Sell what you can operate. The level numbers are stable across docs.

## 1. Level 1 — Trust Pack

- Documented data handling
- AI policy in writing
- No unsafe automation in any offer
- Proof Pack format defined and used

## 2. Level 2 — Auditability

- Structured audit events
- AI Run Ledger active
- Approval records persisted
- Source Passports for every dataset in flight

## 3. Level 3 — Control

- Policy engine deployed
- Role-based approvals
- Risk Index per agent
- Incident response runbook

## 4. Level 4 — Enterprise Platform

- RBAC
- SSO
- Configurable data retention
- Audit exports
- SLA on uptime and approval latency
- Defined support process

## 5. Level 5 — Enterprise AI OS

- AI Control Tower
- Multi-workflow governance
- Business-unit reporting
- Executive value dashboard

## 6. Command Model usage

The Command Center displays the firm’s **current operating level**, the next level it is preparing to operate, and the gaps holding back the transition.

The Intelligence OS recommends `BUILD` decisions only when they close a gap to the next operable level — not because a feature looks impressive.

## 7. Anti-pattern

Selling Level N capabilities when the firm operates at Level N−2. Buyers walk away when they discover the gap. The doctrine prevents this by gating enterprise offers behind level evidence.

## 8. The principle

> Operate Level N before you sell Level N. Every level has a real artifact behind it.
