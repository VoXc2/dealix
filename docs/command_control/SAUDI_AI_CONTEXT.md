# Saudi AI Context

Saudi Arabia is building an enormous national AI stack — infrastructure, models, data centers, cloud, and Arabic-first multimodal models. Dealix sits **on top** of that stack and converts it into capability inside companies.

## 1. The stack

```
HUMAIN / infrastructure / models / cloud
Dealix / operations / workflows / governance / proof
```

## 2. The public-facing sentences

- **EN:** Saudi Arabia is building AI infrastructure. Dealix helps companies operationalize AI safely on top of it.
- **AR:** السعودية تبني بنية الذكاء الاصطناعي. Dealix تساعد الشركات على تشغيل AI داخل أعمالها بأمان وقياس.

## 3. Distribution moments

National events such as LEAP and DeepFest are used as **distribution moments** for the Capability Diagnostic and the Saudi B2B AI Readiness Benchmark — not as hype demos. The right outcome from an event is a number of paid diagnostics booked, not impressions counted.

## 4. Posture toward national initiatives

- Complementary, not competitive.
- Dealix consumes national compute, models, and platforms via the Core OS.
- Dealix encodes residency, identity, and data rules in the governance runtime.
- Dealix does not claim to be the national stack.

## 5. Operating discipline

- Engagements default to in-region residency.
- Engagements default to Arabic governance disclosures alongside English.
- Engagements default to a Source Passport per dataset.
- Partner relationships prioritize regulated sectors first.

## 6. Failure modes

- Marketing posture that competes with national initiatives.
- Treating Arabic as a translation surface instead of a strategic language.
- Sponsoring events without measuring Capability Diagnostic bookings.
- Inferring approval from event proximity rather than from documented engagements.

## 7. Why this anchors the firm

A Saudi-AI-native posture is not optional. It is the strongest moat available to a Riyadh-headquartered AI operating company, and the only one that scales naturally across MENA.
