# Control Metrics

A small, ordered set of metrics that drive Dealix decisions. Vanity metrics do not appear here.

## 1. North Star

> **Proof-backed AI operating capabilities created.**

That is, the count of operating capabilities Dealix has installed inside client organizations with a corresponding Proof Pack.

This North Star prevents counting customers without counting capability, and prevents counting projects without counting proof.

## 2. Supporting metrics

- MRR
- Project revenue
- Gross margin
- Proof packs delivered
- Proof strength score (composite of metric quality and evidence level)
- Capital assets created
- Manual steps productized
- Governance incidents prevented
- Retainer conversion
- Business unit maturity

## 3. Why not “number of customers”

A customer count can rise alongside delivery chaos. A proof-backed capability count rises only when the firm has actually built something operable, governed, and provable.

## 4. Operating discipline

- Each supporting metric has an owner.
- Each metric has a definition fixed in a single place — the Decision Log.
- Changes to definitions require a recorded decision; no silent redefinition.
- Metrics flagged by the Intelligence OS as drifting are investigated.

## 5. Reporting cadence

- **Weekly** — delivery and governance metrics.
- **Monthly** — financial and capital metrics.
- **Quarterly** — BU maturity, North Star roll-up.

## 6. Failure modes

- Adding metrics because they are easy to collect.
- Letting marketing redefine metrics without going through the Decision Log.
- Reporting hero numbers without baselines.
- Optimizing for a supporting metric in a way that decreases the North Star.

## 7. The principle

> What gets measured wrong, gets done wrong. What gets measured well, compounds.
