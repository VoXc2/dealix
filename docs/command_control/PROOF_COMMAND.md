# Proof Command

Every engagement must produce **proof**. Proof Command is the operating layer that turns proof from an artifact into a decision.

## 1. Proof Pack contents

- Problem
- Inputs
- Work Completed
- Metrics
- Before/After
- AI Outputs
- Governance Events
- Business Value
- Risks
- Recommended Next Step

## 2. Proof types

| Type | Example evidence |
| --- | --- |
| Revenue Proof | Pipeline value, opportunities created |
| Time Proof | Hours saved per workflow per week |
| Quality Proof | Error reduction, rework reduction |
| Risk Proof | Unsafe actions blocked, approvals logged |
| Knowledge Proof | Citation coverage, insufficient-evidence rate |

## 3. Proof decision rule

```
Proof score >= 85 → case candidate
70–84             → sales support
< 70              → internal learning only
```

The proof score combines:

- Metric quality (baseline + delta)
- Evidence level (L0–L5)
- Reproducibility
- Governance event richness

## 4. Hard rules

- No proof, no claim.
- No proof, no case study.
- No proof, no retainer push.

## 5. Operating discipline

- Reporting OS owns Proof Pack generation.
- Capital OS sweeps reusable assets from each pack.
- Intelligence OS reads proof scores into the decision engine.
- Governance OS contributes the audit and approval evidence.

## 6. Cross-cutting checks

- Public claims at evidence level ≥ L4 require explicit consent.
- Aggregated benchmarks anonymize and require an aggregation rule.
- A Proof Pack referenced by a marketing claim must be retrievable by ID.

## 7. Failure modes

- Verbal proof claimed without a Proof Pack ID.
- Reuse without rechecking consent.
- Metrics selected after delivery to look favorable.
- Single-period numbers without baselines.
- Hero charts without governance context.

## 8. The principle

> Proof is the unit. Every claim, every offer, and every retainer is denominated in it.
