# نموذج المحرّكَين — نموذج التركيب القابض

**الطبقة:** Holding · Compound Holding Model
**المالك:** الرئيس التنفيذي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [DUAL_ENGINE_MODEL.md](./DUAL_ENGINE_MODEL.md)

## السياق
تعمل دياليكس على **محرّكَين في آن واحد**: محرّك نقد يدفع التكاليف الآن، ومحرّك تركيب يبني قاعدة أصول قابلة للدفاع للمستقبل. تفشل أغلب الشركات الناشئة لأنها تُشغّل محرّكاً واحداً فقط؛ الوكالات الخالصة تُشغّل محرّك النقد دون تركيب، وشركات المنتج الخالصة تُشغّل محرّك التركيب وتنفد قبل الجذب. هذا الملف هو القاعدة التشغيلية التي تُلزم كل مشروع دياليكسي بخدمة **المحرّكَين معاً**. ومرتكزه `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md` و`docs/BUSINESS_MODEL.md`.

## المحرّك 1 — محرّك النقد

محرّك النقد هو **عمل الخدمات والاحتفاظات** في دياليكس. يموّل الرواتب والاستضافة والنمو.

| المكوّن | الوصف |
|---|---|
| التشخيص | عرض قصير (1–2 أسبوع). يكشف الفرص. |
| السبرنت | تكليف مدفوع 2–6 أسابيع. يُسلّم مساراً محوكَماً. |
| التجربة | تجربة منتجة 1–3 أشهر. تُثبت العائد. |
| الاحتفاظ | عقد شهري متكرّر لقدرة تشغيلية. |

مقاييس محرّك النقد: الحجوزات، النقد المحصّل، الهامش الإجمالي، تحويل الاحتفاظ، أيام الفوترة. يُقاس بحيث **يكون كل مشروع متعادلاً أو إيجابي المساهمة** — انظر `docs/UNIT_ECONOMICS_AND_MARGIN.md`.

## المحرّك 2 — محرّك التركيب

محرّك التركيب هو **عمل الأصول والملكية الفكرية** في دياليكس. يُسهم فيه كل مشروع دون أن يُفوتر مباشرة على العميل.

| المكوّن | ما يُراكِم |
|---|---|
| Core OS | قدرات منصّة قابلة لإعادة الاستخدام |
| الكتيبات | كتيبات تسليم لكل قطاع وقدرة |
| Proof Ledger | نتائج موثّقة لكل مشروع |
| Capital Ledger | برومبتات وبيانات وتقييمات ووكلاء |
| الأكاديمية | تدريب وشهادات ومنهجية |
| الشركاء | روافع توزيع عبر الوكالات والمكاملين |
| المنصّة | تدفقات منتجة بنموذج SaaS |

مقاييس محرّك التركيب: الأصول لكل مشروع، معدّل إعادة الاستخدام، عدد التدفقات المنتجة، إيراد من الشركاء، خرّيجو الأكاديمية.

## القاعدة غير القابلة للتفاوض

> **كل مشروع في دياليكس يجب أن يخدم كلا المحرّكَين.**
>
> - مشروع يخدم النقد فقط = **عمل وكالة** — ممنوع على المقياس لأنه لا يُراكم.
> - مشروع يخدم التركيب فقط = **فراغ منتجي** — ممنوع لأنه لا يكسب قوته.

تُفرض القاعدة عبر [`SUCCESS_ASSURANCE_SYSTEM`](./SUCCESS_ASSURANCE_SYSTEM.md) (التجاري + رأس المال خضراء) و[`VALUE_FLYWHEEL`](./VALUE_FLYWHEEL.md) (كل مشروع يدور عبر المحرّكَين).

## قائمة قبول المشروع
قبل قبول المشروع للتسليم:

- [ ] محرّك النقد: مُحدّد النطاق ومُسعّر، هامش ≥ 60%.
- [ ] محرّك التركيب: أصل رأسمالي قابل لإعادة الاستخدام واحد على الأقل مخطّط.
- [ ] محرّك التركيب: حدث إثبات واحد على الأقل مخطّط.
- [ ] الحوكمة: السياسات المعنيّة مؤكّدة.
- [ ] صلاحية التنميط: درجة 1–5 على معيار التنميط.

ما يفشل في محرّك يُرفض أو يُعاد تشكيله أو يُحوّل إلى تجربة Labs.

## ربط الوحدات

| الوحدة | وزن النقد | وزن التركيب | الأصل التركيبي الأساسي |
|---|---|---|---|
| Revenue | عالٍ | عالٍ | وكلاء تسجيل الحسابات، قوالب حِزَم الصياغة |
| Operations | عالٍ | متوسّط | مكتبة المسارات، مولّد SOP |
| Brain | متوسّط | عالٍ | سجل المصادر، خرائط معرفة قطاعية |
| Support | عالٍ | متوسّط | نماذج اقتراح ردود، قواعد التصعيد |
| Governance | متوسّط | عالٍ جداً | سجل السياسات، سجل المخاطر، سجل التدقيق |
| Data | متوسّط | عالٍ | نماذج جودة البيانات، كاشفات PII |

## الواجهات
| المدخلات | المخرجات | المالك | الإيقاع |
|---|---|---|---|
| قبول مشروع جديد | قرار اجتياز / فشل | مدير الوحدة + رئيس Core | لكل مشروع |
| إغلاق المشروع | نقد محصّل + أصول مسجَّلة | مدير الوحدة | لكل مشروع |
| Capital Ledger | طلبات إعادة الاستخدام | رئيس Core | لكل مشروع |
| مراجعة التنميط | مرشّحات احتفاظ / منتج | مجلس المنتج | كل أسبوعَين |

## المقاييس
- **معدّل اجتياز المحرّكَين** — % المشاريع المقبولة المجتازة للاثنَين.
- **حصّة مشاريع النقد فقط** — % المشاريع بـ 0 أصول (الهدف → 0%).
- **مساهمة التركيب لكل دولار** — الأصول لكل 1000$ إيراد.
- **معدّل إعادة الاستخدام** — % المشاريع التي تعيد استخدام ≥ 1 أصل.
- **انسياب الاحتفاظ** — % التجارب المحوَّلة لاحتفاظ خلال 60 يوماً.
