## خريطة الربح والخسارة

| الكيان | نوع P&L | مصدر الإيراد | مصدر التكلفة |
|---|---|---|---|
| Dealix Core (Platform) | منصة | فوترة داخلية + إيراد API مستقبلاً | البحث والتطوير، البنية، الحوكمة |
| Dealix Services (Delivery) | خدمة | رسوم المشاريع والاحتفاظات | كادر التسليم، CSM، الجودة |
| الوحدات الست | فرعية | سُلّم الخدمات + الاحتفاظات + المنتجات | تسليم مخصّص، Core OS مخصّص، إدارة الوحدة |
| الأكاديمية | مركّبة | رسوم الدورات والشهادات | المنهج، المدرّبون |
| الشركاء | مركّبة | حصص الشركاء، رسوم البرنامج | كادر عمليات الشركاء |
| المختبرات | مركز تكلفة ثم P&L | تجارب عمودية | الباحثون |
| الاستثمارات | حصص / بناء مشترك | عوائد الحصص، رسوم البناء | العناية الواجبة، القانون |
| المالية / العمليات | مركز تكلفة | لا يوجد | المصاريف العامة |

## نموذج فوترة الخدمات المشتركة

تُموّل الخدمات المشتركة من إيرادات الوحدات عبر فوترة داخلية شفّافة:

1. **المبيعات** — % من ARR الصافي المغلَق لكل وحدة (8% افتراضي).
2. **التسليم / CSM** — معدّل يومي محمّل بالكامل لكل يوم تسليم.
3. **المنتج / الجودة** — رسم منصة شهري ثابت لكل وحدة (يغطّي استخدام Core OS وبوّابات الجودة).
4. **المالية / القانون / الامتثال** — % من إيراد الوحدة (4% مدمج افتراضي).
5. **العلامة / المحتوى** — رسم شهري ثابت لكل وحدة؛ الحملات الكبرى تُفوتر منفصلة.

المدير العام للوحدة مسؤول عن البقاء داخل ظرف الهامش الضمني. المرجع الحي في `docs/FINANCE_DASHBOARD.md`.

## حقوق القرار

| القرار | المالك | يُستشار | يُبلَّغ |
|---|---|---|---|
| إطلاق وحدة جديدة | الرئيس التنفيذي | الرئيس التشغيلي ومديرو الوحدات | المجلس |
| الالتزام بميزة في النظام الأساسي | رئيس Dealix Core | مديرو الوحدات، رئيس الخدمات | كل الوحدات |
| تغيير سلّم الخدمات داخل وحدة | المدير العام للوحدة | رئيس الخدمات، العلامة | الرئيس التنفيذي |
| توظيف فوق المستوى L5 | الرئيس التنفيذي | الرئيس التشغيلي، المدير المعنيّ | المجلس |
| تغيير السعر > 15% | الرئيس التنفيذي | المدير العام، المدير المالي | المجلس |
| طبقة شركاء جديدة | رئيس النمو | المدير المالي، القانون | الرئيس التنفيذي |
| سياسة إعادة استخدام الأصول الرأسمالية | رئيس Dealix Core | القانون، مديرو الوحدات | الرئيس التنفيذي |

## تتابع التوظيف
سياسة "CSM أولاً" في `docs/HIRING_CSM_FIRST.md` هي الموقف الافتراضي للمجموعة: التوظيف الأول لأي وحدة هو CSM، ثم تسليم، ثم مبيعات، ثم مدير عام، ثم متخصّصون. هذا يُبقي الإثبات والاحتفاظ مرتفعَين قبل توسيع الأنبوب.

## المقاييس
- **هامش مساهمة الوحدة** — إيراد الوحدة ناقص التكلفة المباشرة + المخصّصة.
- **نسبة تكلفة الخدمات المشتركة** — تكلفة الخدمات المشتركة ÷ إيراد المجموعة.
- **نطاق الإشراف** — المرؤوسون المباشرون لكل مدير (الهدف 5–8).
- **زمن الإشغال** — الأيام من الطلب إلى التوقيع.
- **عمق خلَف المدير العام** — عدد الخلَفاء المسمَّون لكل مقعد.
