# برج التحكم الاستراتيجي — نموذج التركيب القابض

**الطبقة:** Holding · Compound Holding Model
**المالك:** الرئيس التنفيذي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [STRATEGIC_CONTROL_TOWER.md](./STRATEGIC_CONTROL_TOWER.md)

## السياق
برج التحكم الاستراتيجي هو **قُمرة التشغيل الأسبوعية** لمجموعة دياليكس: شاشة واحدة، عرضان (عرض المجلس وعرض الوحدة)، إيقاع أسبوعي واحد، ومفردات قرار ثابتة (بيع / بناء / إيقاف / رأس مال / توظيف). يربط `docs/V14_FOUNDER_DAILY_OPS.md` و`docs/ops/DAILY_OPERATING_LOOP.md` بسطح قرار واحد متكرّر. وفيه تتحوّل إشارات [`HOLDING_SCORECARD.md`](./HOLDING_SCORECARD.md) إلى إجراءات، وتُفرض سلسلة [`SUCCESS_ASSURANCE_SYSTEM.md`](./SUCCESS_ASSURANCE_SYSTEM.md).

## العرضان

### عرض المجلس
للرئيس التنفيذي والمجلس. يُظهر مستوى الكيان القابض فقط.

| الكتلة | المحتوى |
|---|---|
| لوحة القياس | صفوف اللوحة العشرة بأحمر/أخضر |
| النقد | الرصيد، الأشهر المتبقّية، الحرق |
| الأنبوب | ARR الأنبوب حسب المرحلة والمصدر |
| المخاطر | أعلى 5 مخاطر مفتوحة |
| القرارات | القرارات الجارية، مالكها، تاريخ استحقاقها |
| أهداف الربع | حالة 3–5 OKRs ربعية |

### عرض الوحدة
لمديري الوحدات ورئيس الخدمات. يُظهر صحّة الوحدة.

| الكتلة | المحتوى |
|---|---|
| P&L الوحدة | الإيراد، هامش المساهمة، عدد الموظفين |
| أنبوب الوحدة | قمع المراحل، نسبة الفوز، زمن الإغلاق |
| التسليم | درجة الجودة، الالتزام بالموعد، الاسترداد |
| الإثبات | حِزم الإثبات في آخر 30 يوماً |
| التنميط | الخطوات الجديدة المُمنتَجة في Core OS |
| رأس المال | أصول رأسمالية في آخر 30 يوماً |
| الاحتفاظ | Sprint→Retainer للوحدة |

## مفردات القرار الثابتة

كل مراجعة أسبوعية تنتهي بقرارات صريحة من هذه المجموعة:

| الفعل | متى | مثال |
|---|---|---|
| **بيع** | الطلب يفوق الطاقة والهامش قوي | "افتح Dealix Brain لتجربتين جديدتين." |
| **بناء** | تكرار العمل يبرّر التنميط | "نمّط مراجعة بيانات PDPL داخل Governance Workspace." |
| **إيقاف** | خدمة أو تجربة لا تستحق التكلفة | "أوقف تجربة الدعم في القطاع المالي." |
| **رأس مال** | جمع رأس مال أو رفع سعر | "ارفع سعر سبرنت الإيراد 15%." |
| **توظيف** | الطاقة هي القيد الملزم | "وظّف CSMs اثنَين لوحدة Brain." |

ما خرج عن هذه المفردات فهو خارج نطاق البرج — مكانه مراجعات المشاريع.

## جدول الأعمال الأسبوعي (90 دقيقة)

| دقيقة | الكتلة | المالك |
|---|---|---|
| 0–10 | قراءة لوحة القياس (أخضر/أحمر) | المدير المالي |
| 10–25 | تحديث 90 ثانية لكل وحدة | مديرو الوحدات |
| 25–40 | القرارات المفتوحة: الحالة | مالكو القرارات |
| 40–60 | قرارات جديدة: نقاش وإغلاق | الرئيس التنفيذي |
| 60–75 | مراجعة المخاطر | DPO / GC |
| 75–85 | تتبّع أهداف الربع | الرئيس التنفيذي |
| 85–90 | تأكيد التوثيق والاستعداد للأسبوع | رئيس الديوان |

## سجل القرارات
كل قرار يصبح صفّاً في `docs/EXECUTIVE_DECISION_PACK.md` بـ: المعرّف، الفعل، النطاق، المالك، الاستحقاق، مقياس النجاح، الحالة. الحالة تتنقّل: `proposed → approved → in-flight → done | reversed`.

## مضادّات النمط
- مناقشة تذاكر تشغيلية — مكانها وقفة الوحدة لا البرج.
- إصدار قرارات خارج المفردات الخمس — يُكرّس الغموض.
- لا بيانات من المدير المالي — كل أحمر مدعوم برقم لا بإحساس.
- قرارات بلا مالك أو استحقاق — رفض تلقائي من رئيس الديوان.

## الواجهات
| المدخلات | المخرجات | المالك | الإيقاع |
|---|---|---|---|
| لوحة القياس | الأحمر والأخضر | المدير المالي | أسبوعي |
| لوحات الوحدات | عرض الوحدة | مديرو الوحدات | أسبوعي |
| سجل المخاطر | أعلى 5 | DPO / GC | أسبوعي |
| CRM الأنبوب | أنبوب المجموعة | رئيس المبيعات | أسبوعي |
| Decision Pack | سجل القرارات | الرئيس التنفيذي + رئيس الديوان | أسبوعي |

## المقاييس
- **قرارات في الأسبوع** — عدد قرارات بيع/بناء/إيقاف/رأس مال/توظيف.
- **زمن دورة القرار** — الوسيط من المُقترح إلى المعتمد.
- **معدّل الرجوع** — % القرارات المعكوسة خلال 60 يوماً.
- **زمن الأحمر إلى القرار** — أيام من الإشارة إلى أول قرار مرتبط.
- **بلوغ أهداف الربع** — % OKRs المُسلَّمة.
