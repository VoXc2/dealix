# Unit Charter + Venture Factory

## Unit Charter (قالب مؤسسي)

كل وحدة لها نفس الهيكل:

| حقل | وصف |
|-----|-----|
| Name | مثال: Dealix Revenue |
| Problem | فرص مبيعات مبعثرة ومتابعة ضعيفة |
| Primary Customer | مثال: شركات خدمات B2B |
| Primary Offer | Lead Intelligence Sprint |
| Recurring Offer | Monthly RevOps OS |
| Product Module | Revenue OS |
| Proof Type | Revenue Proof Pack |
| KPIs | accounts scored · pipeline · qualified opps · retainer conversion |
| Capital Assets | scoring model · templates · playbook · benchmarks |
| Risk Profile | (قطاع، بيانات، قنوات) |
| Owner | (مسؤول الوحدة) |

**مثال مكتوب:** [`units/revenue/CHARTER.md`](units/revenue/CHARTER.md)

---

## Venture Factory — مسار التخرج

```text
Service Line → Business Unit → Product Module → Retainer Line
→ Venture Candidate → Venture
```

### Venture Gate

لا venture إلا إذا: **5+** عملاء مدفوعين · **2+** retainers · تسليم متكرر · module واضح · owner · playbook **~80+** · هامش صحي · مكتبة proof · **إشارة شريك/قناة** (عند التطبيق).

### أمثلة مرشحة

Dealix Clinics OS · Real Estate OS · Logistics OS · Education OS · Revenue OS · Governance Cloud — **كلها تعتمد على Core OS** ولا تُبنى من الصفر.

### القواعد

```text
No venture without proof.
No venture without repeatability.
No venture without Core OS dependency.
```
