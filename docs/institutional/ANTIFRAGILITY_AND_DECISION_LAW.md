# Anti-Fragility + Decision Law

## نظام Anti-Fragility

**كل خطر يجب أن يقوّي المؤسسة عند المعالجة.**

```text
Risk found → Rule created → Test added → Checklist updated
→ Playbook improved → Trust page / محتوى حوكمة
```

**مثال:** طلب عميل LinkedIn automation → قاعدة `no_linkedin_automation` · استثناء عروض · رد اعتراض مبيعات · مقال حوكمة.

---

## Dealix Decision Law — قانون القرار

**أي قرار يمرّ بستة أسئلة:**

1. **Does it sell?**
2. **Does it deliver?**
3. **Does it prove?**
4. **Does it govern?**
5. **Does it compound?**
6. **Does it scale?**

**القاعدة:** إن لم يحقق **3 على الأقل** — لا تفعل (أو أعد الصياغة).

### مثال — Lead Intelligence Sprint

sell ✓ · deliver ✓ · prove ✓ · govern ✓ · compound ✓ · scale ✓ → **افعل.**

### مثال — Custom chatbot بلا مصادر

معظم المحاور ضعيفة → **ارفض** أو أعد صياغته كـ Company Brain بحوكمة مصادر.
