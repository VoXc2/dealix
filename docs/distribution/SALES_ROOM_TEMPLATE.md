# Dealix — Sales Room Template (Per-Opportunity Record)
<!-- PHASE 3 | Owner: Founder | Date: 2026-05-17 -->
<!-- Arabic primary — العربية أولاً -->

> **الغرض:** لكل فرصة بيع سجلٌّ واحد مُوحَّد — "غرفة بيع" — يجمع كل ما يخص الصفقة في مكان واحد. لا ذاكرة في الرأس، لا متابعة مبعثرة.
> **Purpose:** one unified record — a "sales room" — per opportunity. No memory in the head, no scattered follow-up.

هذه الوثيقة قالب. تُنسخ مرة لكل فرصة وتُحدَّث حتى الإغلاق أو الإقفال. تُغذّي حقولها وحدة `auto_client_acquisition/sales_os/` (التأهيل، توليد العرض، شجرة القرار، درجة مخاطر العميل).

---

## القواعد الثلاث — The Three Rules

هذه القواعد غير قابلة للتفاوض. أي غرفة بيع تخالفها تُعتبر غير صالحة.

1. **لا فرصة بدون إجراء تالٍ.** No opportunity without a next action.
2. **لا إجراء تالٍ بدون تاريخ.** No next action without a date.
3. **لا مرحلة بدون دليل.** No stage without evidence — كل تقدّم في المرحلة يحتاج مصدراً حقيقياً (رسالة، مكالمة موثقة، فاتورة، رد مكتوب).

> أي رقم أو نتيجة تُذكر داخل الغرفة تُوسَم "تقديري" ما لم يكن لها مصدر مُتحقَّق. لا أدلة مُلفَّقة (`no_fake_proof`).

## القالب — Sales Room Record

انسخ الكتلة التالية لكل فرصة جديدة.

### 1. الهوية — Identity
- **Company / الشركة:**
- **Buyer / صانع القرار:** (الاسم الوظيفي والدور — لا تُخزَّن بيانات تواصل حسّاسة هنا دون موافقة)
- **Segment / القطاع:** (fintech / عقار / استشارات / تجزئة B2B / أخرى)
- **Opportunity ID:** `SR-YYYY-NNN`
- **Opened / تاريخ الفتح:**

### 2. السياق — Context
- **Pain / المشكلة المحورية:** (بكلمات العميل نفسه إن أمكن)
- **Source / مصدر الفرصة:** (تشخيص مجاني / إحالة / شريك / محتوى — مصدر مشروع فقط؛ لا scraping، لا واتساب بارد)
- **Current state / الوضع الحالي:** كيف يدير العميل المتابعة بعد الـ lead اليوم

### 3. التقدّم — Progress
- **Stage / المرحلة:** `discovery → qualified → proof_sent → demo → proposal → invoiced → won/lost`
- **Proof sent / الإثبات المُرسَل:** (نوع المخرَج + تاريخه + رابط الملف — مثال: عيّنة Sprint، Proof Pack v1)
- **Demo date / تاريخ العرض:**
- **Objections / الاعتراضات:** (سجِّل كل اعتراض كما ورد + الرد المُقدَّم)

### 4. العرض — The Offer
- **Scope / النطاق المقترح:** (الدرجة على سلّم العروض)
- **Recommended offer / العرض الموصى به:** اختر من السجل الرسمي فقط —
  - التشخيص المجاني — 0 SAR
  - سبرنت إثبات الإيرادات — 499 SAR (دفعة واحدة)
  - حزمة من البيانات إلى الإيراد — 1,500 SAR (دفعة واحدة)
  - عمليات النمو الشهرية — 2,999 SAR/شهر
  - إضافة دعم العمليات — 1,500 SAR/شهر
  - غرفة قيادة الإدارة — 7,500 SAR/شهر
  - منصة شركاء الوكالات — مخصص + rev-share 15–30%
  - للنطاقات والشروط الكاملة انظر [`../OFFER_LADDER_AND_PRICING.md`](../OFFER_LADDER_AND_PRICING.md).
- **Invoice / الفاتورة:** (الحالة: غير صادرة / صادرة / مدفوعة + رقم الفاتورة + التاريخ)
- **Pricing transparency:** لا تسعير مخفي (`no_hidden_pricing`) — السعر مكتوب صراحةً قبل أي التزام.

### 5. القرار — Decision
- **Decision criteria / معايير القرار:** ما الذي يحتاجه المشتري ليقول "نعم"؟ (دليل قيمة، موافقة شريك، ميزانية، توقيت)
- **Decision date target / تاريخ القرار المتوقع:**

### 6. الإجراء التالي — Next Action (إلزامي)
- **Next action / الإجراء:** (جملة واحدة واضحة وقابلة للتنفيذ)
- **Owner / المسؤول:**
- **Due date / التاريخ:** (إلزامي — بلا تاريخ تُعتبر الغرفة غير مكتملة)

### 7. الأدلة — Evidence Log
سجل زمني لكل دليل يدعم المرحلة الحالية. كل سطر: التاريخ + النوع + الرابط/المرجع.
- `YYYY-MM-DD` — نوع الدليل — مرجع
- ...

## دورة الحياة — Lifecycle

| المرحلة | شرط الدخول (دليل مطلوب) | الإجراء التالي النموذجي |
|---|---|---|
| discovery | فرصة مسجَّلة بمصدر مشروع | حجز تشخيص مجاني |
| qualified | استمارة تأهيل مكتملة عبر `sales_os` | إرسال إثبات/عيّنة |
| proof_sent | مخرَج حقيقي مُرسَل وموثَّق | مكالمة عرض |
| demo | مكالمة موثقة + اعتراضات مسجَّلة | إرسال عرض |
| proposal | عرض مُرسَل من `sales_os/proposal_renderer` | تأكيد معايير القرار |
| invoiced | فاتورة صادرة وموثقة | تأكيد الدفع |
| won / lost | دفع مؤكَّد أو سبب إقفال مكتوب | تسليم → [`ONBOARDING_TO_VALUE.md`](ONBOARDING_TO_VALUE.md) أو أرشفة درس |

> أي إرسال خارجي للعميل (رسالة، عرض) يتطلب موافقة بشرية صريحة قبل الإرسال. لا إرسال آلي مباشر.

## مراجعة الأنبوب — Pipeline Hygiene

مراجعة أسبوعية بسيطة:
1. كل غرفة لها إجراء تالٍ بتاريخ؟ إن لا — أصلِح أو أقفل.
2. أي إجراء تجاوز تاريخه؟ يُعاد جدولته أو يُغلق.
3. أي مرحلة بلا دليل؟ تُرجَع للمرحلة السابقة.

تسلسل المتابعة بعد الاتصال الأول موصوف في [`../sales-kit/dealix_followup_cadence.md`](../sales-kit/dealix_followup_cadence.md) — لا تكرّره هنا، اربط حقل "الإجراء التالي" بخطوته المناسبة في تلك السلسلة.
