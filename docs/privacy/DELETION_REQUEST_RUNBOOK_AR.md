# دليل تنفيذ طلب الحذف (Deletion Request Runbook)

طلب الحذف = **إجراء حسّاس بشري** (`requires_human_handoff` + `privacy_request`).

## الخطوات
1. **استلام:** يصل الطلب (بريد/واتساب/رد). يُصنَّف `privacy_request` تلقائياً.
2. **تسليم بشري:** يتوقف أي رد آلي؛ يُرفع للمؤسس فوراً.
3. **كبح فوري:** أضف الجهة لقائمة الكبح بسبب `privacy_request` (دائم).
4. **تحقق الهوية:** تأكد أن الطالب صاحب البيانات أو مفوّض.
5. **تحديد النطاق:** ما البيانات المحفوظة؟ (محتمل/ردود/عميل).
6. **تنفيذ الحذف/الإخفاء:** خلال ≤ 30 يوماً؛ مع إبقاء **سجل الكبح** و**سجل الحوكمة**.
7. **تأكيد:** أبلغ الطالب بالإنجاز.
8. **توثيق:** سجّل في `ai_action_ledger.jsonl` (نوع: deletion_request، الحالة: resolved).

## ملاحظات
- لا حذف لسجل الكبح (يمنع إعادة التواصل).
- إن تضمّن الطلب بُعداً قانونياً → `LEGAL_HANDOFF_TRIGGERS_AR.md`.
- خرق محتمل أثناء التنفيذ → بروتوكول 72 ساعة في `SAUDI_PDPL_OUTBOUND_POLICY_AR.md`.
