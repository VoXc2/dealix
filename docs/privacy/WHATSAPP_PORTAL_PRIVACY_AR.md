## 1. الأساس القانوني للمعالجة (PDPL المادة 5)

| القناة | الأساس القانوني | الشرط |
|--------|----------------|-------|
| واتساب — رسائل أولى | موافقة صريحة أو رد إيجابي أو طلب حجز | `consent_basis` يجب أن يكون في: `{explicit_optin, positive_reply, booking, form_submission, existing_client}` |
| واتساب — عملاء حاليون | `existing_client` | يجب توثيق `consent_timestamp` و`consent_source` |
| البوابة الآمنة | عقد الخدمة + موافقة صريحة | إقرار العميل عند التسجيل |
| تحليل بيانات العميل | عقد الخدمة (لغرض محدد) | الغرض: تحليل إيرادات B2B فقط |

### محظورات قنوات التواصل
- **واتساب البارد (`cold`) محظور تمامًا** — أي رسالة بلا `consent_basis` صالح هي انتهاك فوري.
- **طلب الأسرار/المفاتيح عبر واتساب محظور** — الأسرار تنتقل عبر البوابة الآمنة حصرًا.
- **`consent_basis ∈ {none, cold, scraped, unknown}` = رفض فوري** في الفحص الآلي.

## 2. إخفاء الهوية وتقليل البيانات

### مبدأ التدخل المحدود
- تُطلَب بيانات الحد الأدنى اللازم لتحقيق الهدف التحليلي فقط.
- بيانات العملاء (عملاء عميل Dealix) تُخفى قبل المعالجة بالذكاء الاصطناعي.
- لا تُدخَل بيانات PII في أدوات الذكاء الاصطناعي العامة.

### آلية الإخفاء
| نوع البيانات | المعالجة |
|-------------|---------|
| أرقام الجوال | `+9665XXXXXXXX` — يُخفى آخر 8 أرقام |
| أسماء الأشخاص | `[عميل-1]`, `[عميل-2]` — بمعرّف مجهول |
| البريد الإلكتروني | `user@REDACTED.com` أو حذف كلي |
| بيانات الدفع | `portal://payment-data/REDACTED` |
| مفاتيح API / أسرار | `portal://secrets/[ref-id]` — مرجع فقط لا قيمة |

## 4. الاحتفاظ بالبيانات والحذف

### جدول الاحتفاظ الافتراضي

| نوع البيانات | فترة الاحتفاظ | الإجراء بعد الانتهاء |
|-------------|--------------|---------------------|
| بيانات CRM الخام للعميل | 90 يومًا من نهاية العقد | حذف آمن + تأكيد للعميل |
| سجلات المحادثات (واتساب) | 90 يومًا | حذف تلقائي |
| Proof Packs (مُخفاة) | بموافقة العميل (يمكن إبقاؤها كدراسة حالة) | الاحتفاظ فقط بالمعطيات المجهولة الهوية |
| سجلات الإجراءات (`ai_action_ledger`) | 12 شهرًا | للتدقيق والامتثال |
| وثائق الحوكمة | غير محدد | تُراجَع سنويًا |

### حق الحذف
- العميل يستطيع طلب حذف بياناته في أي وقت.
- يُنفَّذ الحذف خلال 72 ساعة من الطلب.
- يُؤكَّد الحذف للعميل كتابةً.

## 5. الموافقة والانسحاب

### الموافقة الأولية
- تُوثَّق عند أول تفاعل (`consent_timestamp`, `consent_source`, `consent_basis`).
- صياغة واضحة ومحددة الغرض — لا موافقات عامة مبهمة.

### آلية الانسحاب (Opt-Out)
- متاح في أي وقت وبأي طريقة (رسالة واتساب، طلب مباشر).
- يُوقَف **فورًا** كل تواصل واتساب عند `opt_out`.
- يُسجَّل الانسحاب في `sessions.jsonl` ولا يُعاد التواصل بعده بدون موافقة جديدة.
- لا يُشترط تقديم سبب للانسحاب.

### جدول قيم consent_basis المسموح بها

| القيمة | المعنى | الحالة |
|--------|--------|--------|
| `explicit_optin` | موافقة صريحة موثَّقة | ✅ مسموح |
| `positive_reply` | رد إيجابي من العميل | ✅ مسموح |
| `booking` | طلب حجز أو موعد | ✅ مسموح |
| `form_submission` | تعبئة نموذج | ✅ مسموح |
| `existing_client` | عميل حالي ذو علاقة تعاقدية | ✅ مسموح |
| `none` / `cold` / `scraped` / `unknown` | بلا أساس قانوني | ❌ محظور — رفض فوري |

## 6. حقوق صاحب البيانات (PDPL الفصل الثالث)

| الحق | الآلية | الجدول الزمني |
|------|--------|---------------|
| الحق في المعرفة | إفصاح عند التعاقد + نموذج الاستقبال | عند بدء العلاقة |
| الحق في الوصول | طلب عبر البوابة الآمنة | خلال 15 يوم عمل |
| الحق في التصحيح | طلب عبر البوابة الآمنة | خلال 10 أيام عمل |
| الحق في الحذف | طلب مباشر | خلال 72 ساعة |
| الحق في الاعتراض | رسالة واتساب أو البوابة | فوري |
| الحق في نقل البيانات | بصيغة مقروءة آليًا | خلال 15 يوم عمل |

## 7. إشعار الخرق الأمني

| الخطوة | الإجراء | الجدول الزمني |
|--------|---------|---------------|
| الاكتشاف | رصد السجلات + تحليل الأثر | فوري |
| التقييم | تحديد نطاق الخرق والبيانات المتأثرة | خلال 24 ساعة |
| إشعار SDAIA | إبلاغ الهيئة بالخرق | **خلال 72 ساعة** من الاكتشاف |
| إشعار الأشخاص المتأثرين | إبلاغ أصحاب البيانات | حسب توجيه SDAIA |
| المعالجة | إصلاح السبب الجذري + منع التكرار | فوري |

## 8. الذكاء الاصطناعي والامتثال (SDAIA AI Ethics)

- الإنسان في الحلقة دائمًا — لا قرار آلي يؤثر على العميل بدون مراجعة بشرية.
- الشفافية: يُعلَم العملاء باستخدام الذكاء الاصطناعي في التحليل.
- لا معالجة آلية تؤدي إلى قرارات ذات أثر قانوني أو مالي بدون موافقة المؤسس.
- مراجعة دورية لمخرجات الذكاء الاصطناعي للكشف عن الانحياز.
