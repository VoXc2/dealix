# تقليل بيانات العملاء المحتملين (Prospect Data Minimization)

المبدأ: **اجمع الحد الأدنى فقط** مما يلزم للتأهيل والتواصل المشروع.

## الحقول المسموح جمعها (B2B عام)
`company, segment, website, decision_maker_role, business_pain, public_signal,
status, next_action, score`.

> ملاحظة: نفضّل **الدور الوظيفي** (مثل "Founder/CEO") على الاسم الشخصي عند الإمكان.
> سكربت `scripts/governance_check.py` يرفع تنبيهاً عند وجود أسماء شخصية في `prospects.csv`.

## الحقول الممنوعة افتراضياً
- بيانات شخصية حساسة (دينية/صحية/مالية شخصية).
- أرقام هوية وطنية، تفاصيل بطاقات، كلمات مرور، أي أسرار.
- بيانات مكشوطة من مصادر تنتهك شروطها، أو قوائم مشتراة.

## قواعد التشغيل
1. لا تُرسل بيانات شخصية إلى نماذج الذكاء الاصطناعي دون ضرورة، وبعد التنقيح.
2. أخفِ/نقّح المعرّفات قبل التحليل حيثما أمكن.
3. سجّل كل وصول للبيانات في `ai_action_ledger.jsonl`.
4. عند الشك → عامل الحقل كبيانات حساسة واستبعده.
