# Dealix Privacy — التعامل مع بيانات العملاء

> *آخر تحديث: 2026-06-03*
> يكمّل ولا يكرّر: `company_os/governance/data_handling_checklist.md` و `pdpl_checklist.md`.

بمجرد توقيع العميل، تنتقل بياناته إلى مسار خاص محكوم بحوكمة PDPL/SDAIA.

---

## دورة حياة بيانات العميل

```txt
استلام آمن  →  تجهيل قبل تحليل AI  →  معالجة محلية  →  مخرجات مُراجَعة
→  احتفاظ محدود (افتراضي 90 يومًا)  →  حذف/تجهيل + تأكيد
```

---

## قواعد ثابتة

```txt
✅ data minimization — اطلب فقط المدخلات اللازمة لكل Delivery Pack
✅ purpose limitation — تحليل النظام المتعاقد عليه فقط
✅ anonymize before AI — تجهيل الأسماء وPII قبل أي تحليل AI
❌ لا لصق بيانات عميل في أدوات AI عامة
❌ لا خلط بيانات العميل ببيانات العملاء المحتملين
```

المدخلات المطلوبة لكل نظام معرّفة في `QUALITY_GATES_AR.md` §5 — ولا نطلب أكثر منها.

---

## الفصل والتخزين

```txt
Client workspace منفصل لكل عميل.
تخزين مشفّر، وصول محدود (المؤسس).
لا نقل عبر إيميل شخصي أو مشاركة ملفات عامة.
```

(تفاصيل الاستلام/التحليل/ما بعد التعاقد في `data_handling_checklist.md`.)

---

## حقوق صاحب البيانات (PDPL)

| الحق | التنفيذ |
|------|---------|
| المعرفة | نموذج الاستلام يوضّح كل المعالجة |
| الوصول | يمكن للعميل طلب بياناته |
| التصحيح | يمكن طلب التصحيح |
| الحذف | حذف خلال 90 يومًا + عند الطلب |

---

## الربط بالتسليم

- Acceptance Gate: كل مخرج يُراجَع من المؤسس قبل تسليمه (لا تسليم تلقائي).
- كل معالجة AI لبيانات عميل تُسجَّل في `ai_action_ledger.jsonl`.

راجع أيضًا: `SECRET_HANDLING_POLICY_AR.md`،
`docs/security/EXTERNAL_CONTENT_UNTRUSTED_DATA_POLICY_AR.md`.
