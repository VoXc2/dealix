# Dealix Sovereign Command System — نظام القيادة السيادي

**الفكرة:** ربط **الاستراتيجية** بالقرارات اليومية، والقرارات **بالموارد**، والموارد **بـ Proof**، والـ Proof **بالتوسع**. فوق Core OS، لا بد من **Command System** يحدد *ماذا يحدث* و*لماذا* و*متى* نرفع، نوقف، نوسع، أو نفصل وحدة.

---

## 1. لماذا الآن؟

سوق AI يتحرك بسرعة، وكثير من المشاريع تبقى pilots أو تفشل بسبب **استراتيجية غير واضحة**، **ROI**، و**تصميم عمل** ضعيف. تغطية [IT Pro — AI adoption and enterprise FOMO](https://www.itpro.com/business/business-strategy/ai-adoption-projects-keep-failing-but-enterprise-fomo-means-investment-is-still-rising) تشير إلى ضغط استثمار ومخاطر تنفيذ؛ وبحث [Orgvue — firms pressing ahead with AI investment](https://www.orgvue.com/news/firms-pressing-ahead-with-investment-in-ai-despite-being-unclear-on-business-impact/) يوضح فجوة بين الإنفاق ووضوح الأثر. **Dealix تبيع «نظام قيادة AI»** لا مجرد أدوات.

في السياق السعودي، استخدام GenAI واسع مع **ثقة حذرة** وتركيز على **التدريب والخصوصية والاستخدام المسؤول** — انظر [Generative AI in Saudi Arabia: A National Survey (arXiv:2601.18234)](https://arxiv.org/abs/2601.18234). موقع Dealix: **عربي/سعودي + حوكمة + تشغيل + تدريب + Proof**.

## 2. الشكل الأعلى (تسع طبقات)

| # | الطبقة | السؤال المركزي |
|---|--------|----------------|
| 1 | **Strategy Layer** | الاتجاه، الـ wedge، الـ moat، ماذا **لا** نفعل |
| 2 | **Capital Allocation Layer** | أين الوقت والمال والطاقة |
| 3 | **Operating Intelligence Layer** | ماذا تقول events / ledgers / metrics |
| 4 | **Governance Command Layer** | ماذا يضر الثقة أو العميل أو Dealix |
| 5 | **Productization Command Layer** | ماذا تكرر ويصبح منتجًا |
| 6 | **Business Unit Command Layer** | أي service line أصبحت وحدة أعمال |
| 7 | **Venture Command Layer** | أي وحدة تستحق الانفصال |
| 8 | **Market Command Layer** | كيف نملك السوق واللغة |
| 9 | **Standards Command Layer** | أي منهجية يتبنى السوق |

**حوكمة وقت التشغيل:** سلوك الوكلاء **path-dependent**؛ السياسات تحتاج تقييمًا هوية الـagent، مسار التنفيذ، الفعل المقترح، وحالة المؤسسة — انظر [Runtime governance / agent identity (arXiv:2603.16586)](https://arxiv.org/abs/2603.16586).

## 3. استراتيجية مختصرة (أمثلة قرارات)

- التركيز أولًا على **Revenue Intelligence** (كمثال اتجاه).  
- **لا** scraping · **لا** cold WhatsApp (انظر سياسات الثقة في الريبو).  
- **خدمات مُنتَجة** قبل SaaS self-serve واسع.  
- **Proof** عملة الشركة في القرار والتوسع.

## 6. المنتج القابل للتكرار

`Manual → Template → Script → Internal Tool → Client Feature → SaaS Module`  

**لا feature** إلا إن تحقق: تكرار 3+ · مرتبط إيراد · يوفر وقت · يقلل خطر **أو** يحسن هامش · قابل للاختبار · قابل لإعادة الاستخدام.

## 7. Venture وBU

عند **venture**: عملاء مدفوعون، retainers، تسليم متكرر، product module واضح، playbook ≥ ~80، مالك، مكتبة proof، هامش صحي — التفاصيل في [`VENTURE_SIGNALS.md`](../control_tower/VENTURE_SIGNALS.md) و[`BUSINESS_UNIT_MATURITY_SCORE.md`](BUSINESS_UNIT_MATURITY_SCORE.md).
