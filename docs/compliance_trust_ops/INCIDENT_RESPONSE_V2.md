# Incident Response v2

Types: PII exposure, unsupported claim, source misuse, wrong client data, unapproved external action, hallucinated answer, partner violation, agent tool misuse.

Flow: detect → contain → assign owner → assess severity → notify → correct → log → add rule + test + checklist.

Rule: every Sev-1 / Sev-2 incident must create a rule, test, or playbook update before closure.
