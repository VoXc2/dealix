# وكيل الإيراد — منظومة تحقيق القيمة

**الطبقة:** L3 · منظومة تحقيق القيمة
**المالك:** رئيس الذكاء الاصطناعي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [revenue_agent.md](./revenue_agent.md)

## السياق
وكيل الإيراد هو القلب التحليلي لخدمات Lead Intelligence وRevOps. يقيّم
الحسابات ويوصي بالإجراءات داخل سير عمل محكوم، حتى يركّز البائع البشري
على التحركات الأعلى قيمة. ينسجم هذا الوكيل مع عقد قوى العمل في
`docs/product/AI_WORKFORCE_OPERATING_MODEL.md` ومنطق التوجيه في
`docs/AI_MODEL_ROUTING_STRATEGY.md`.

## بطاقة الوكيل

- **الدور:** يقيّم الحسابات ويوصي بإجراءات الإيراد.
- **المدخلات المسموحة:** بيانات بإذن العميل، ICP، عرض الخدمة، دليل
  القطاع.
- **المخرجات المسموحة:** درجة الحساب، شرح الدرجة، توصية شريحة، الإجراء
  التالي.
- **الممنوع:** إرسال الرسائل، الكشط، الادعاءات المضمونة، البيانات
  الشخصية غير الموثّقة.
- **الفحوصات المطلوبة:**
  - وجود مصدر بيانات معتمد؛
  - وسم البيانات الشخصية عند وجودها؛
  - الدرجة قابلة للتفسير (ذكر أهم ثلاث ميزات)؛
  - اجتياز فحص الامتثال.
- **مخطط الإخراج:** `AccountScore { account_id, score, reasons, risks,
  recommended_action }`.
- **الاعتماد:** مراجعة بشرية قبل التسليم للعميل.

## نموذج التقييم (موجز)

- المدخلات: بيانات الشركة، أحداث الإشارة، ملاءمة ICP، تاريخ التفاعل.
- المخرَج: درجة 0–100 مع أهم ثلاثة أسباب.
- المخاطر: قِدم البيانات، عدم ملاءمة القطاع، التعرّض لقطاع حسّاس.
- الإجراء الموصى: `enrich` أو `engage` أو `nurture` أو `disqualify`.

## الأنماط الخاطئة

- التقييم على بيانات غير معتمدة.
- إنتاج درجات بلا تفسير.
- اقتراح إجراء تمنعه مصفوفة الإنسان في الحلقة.
- لغة "نتيجة مضمونة".

| Score explanations | Reviewer summary | CSM | Per engagement |
| Action recommendations | Workflow queue items | Workflow Agent | Per task |

## المقاييس
- Score Calibration — الارتباط بين الدرجة والتحويل اللاحق.
- Explanation Coverage — نسبة الدرجات ذات أهم 3 أسباب.
- Action Acceptance — نسبة الإجراءات التي يقبلها المراجع.
- Forbidden Attempt Rate — محاولات محظورة لكل ألف تشغيل.
