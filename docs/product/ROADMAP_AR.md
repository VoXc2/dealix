# Dealix — خارطة الطريق (Roadmap)

## المبدأ الأساسي

**Roadmap يحكمه Feedback، لا الخيال.**

كل قرار في الـ Roadmap مبني على:
- إشارات من العملاء
- Pain points في التسليم
- فرص الإيراد
- قيود الموارد

## Release Gates

كل milestone يحتاج:

| Requirement | Description |
|-------------|-------------|
| **Owner** | شخص مسؤول |
| **Metric** | مقياس للنجاح |
| **Tests** | اختبارات CI |
| **Runbook** | دليل التشغيل |
| **Rollback** | خطة التراجع |
| **Customer Evidence** | دليل من عميل |

## Decision Criteria

### P0 (Now)
- Launch blocker
- Revenue critical
- Time-sensitive

### P1 (Next)
- Significant revenue impact
- High customer demand
- Strategic value

### P2 (Later)
- Scale enabler
- Efficiency improvement
- Nice to have

### P3+ (Future)
- Nice to have
- Can wait
- Nice to have

---

## Governance

### Review Cadence
- **Weekly**: Progress check + blocker identification
- **Monthly**: Roadmap review + reprioritization
- **Quarterly**: Strategic alignment + OKR check

### Change Process
1. Submit change request with justification
2. Product review + impact assessment
3. Founder approval for P0 changes
4. Update roadmap + communicate
