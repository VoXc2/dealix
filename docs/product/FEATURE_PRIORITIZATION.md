# Feature Prioritization (from delivery, not imagination)

Prioritize product work from **repeated delivery pain** and **compound value**.

## Feature Prioritization Score (max 100)

| معيار | وزن |
|--------|-----|
| تكرر في مشاريع متعددة | 25 |
| يوفر وقت فريق التسليم | 20 |
| يقلل مخاطر (جودة / حوكمة / بيانات) | 20 |
| يزيد جودة المخرجات | 15 |
| يساعد المبيعات أو الإقفال | 10 |
| قابل للبناء بسرعة مع بنية Dealix الحالية | 10 |

## Decision bands

| Score | Action |
|-------|--------|
| **80+** | Build now |
| **60–79** | Backlog (scheduled) |
| **أقل من 60** | Ignore or park |

- `docs/company/FEATURE_CANDIDATE_LOG.md` — candidates from projects
- `POST_PROJECT_REVIEW.md` — "Product Feature Candidates" and "Manual Steps Repeated"
- `docs/company/WEEKLY_OPERATING_REVIEW.md` — recurring themes
