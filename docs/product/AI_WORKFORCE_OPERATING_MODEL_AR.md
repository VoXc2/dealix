# نموذج تشغيل قوى العمل بالذكاء الاصطناعي — منظومة تحقيق القيمة

**الطبقة:** L3 · منظومة تحقيق القيمة
**المالك:** رئيس الذكاء الاصطناعي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [AI_WORKFORCE_OPERATING_MODEL.md](./AI_WORKFORCE_OPERATING_MODEL.md)

## السياق
تستخدم ديلكس وكلاء الذكاء الاصطناعي لتسليم القيمة على نطاق واسع. ولحماية
الجودة والثقة يجب تشغيل هؤلاء الوكلاء بوصفهم عاملين محكومين — لا موظفين
ذاتيي القرار. يحدّد هذا الملف تصنيف الوكلاء، والعقد الإلزامي الذي يلتزم
به كل وكيل، وكيف تنسجم قوى العمل مع تدفق التنفيذ في
`docs/DEALIX_V3_AUTONOMOUS_REVENUE_OS.md` وتوجيه النماذج في
`docs/AI_MODEL_ROUTING_STRATEGY.md`. ويُلزم تنفيذ هذه القواعد بدستور
`docs/DEALIX_OPERATING_CONSTITUTION.md` وقرارات الحزمة في
`docs/AI_STACK_DECISIONS.md`.

## المبدأ

> وكلاء الذكاء الاصطناعي في ديلكس **ليسوا موظفين ذاتيي القرار**؛ بل عاملون
> محكومون داخل سير عمل مضبوط.

تُمنح الاستقلالية بحسب فئة الإجراء لا بحسب الوكيل. القراءة، الصياغة،
التصنيف، التوصية — مسموح بها عموماً. الإرسال، النشر، التصرّف على أنظمة
خارجية — مقيّد ومُعتمَد ومُسجَّل.

## أنواع الوكلاء

1. **وكلاء التحليل** — تحليل البيانات والتصنيف والتلخيص والتقييم.
2. **وكلاء الصياغة** — مسودات وتقارير وسكربتات وتوصيات.
3. **وكلاء الاسترجاع** — البحث في المعرفة والإجابة بمصادر.
4. **وكلاء سير العمل** — تحريك المهام عبر سير عمل محدّد.
5. **وكلاء الحارس** — فحص السلامة والادعاءات والبيانات الشخصية والسياسة.
6. **وكلاء التقارير** — إنتاج التقارير وحزم الإثبات.

كل نسخة وكيل (بطاقة) تعيش في `docs/product/agent_cards/`. لا يُسمح بوكيل
في الإنتاج بلا بطاقة.

## العقد الإلزامي

كل وكيل — أياً كان نوعه — يصرّح بـ:

- **الدور** — جملة واحدة للغرض.
- **الأدوات المسموحة** — قائمة بيضاء صريحة.
- **الإجراءات الممنوعة** — قائمة سوداء صريحة.
- **مخطط الإدخال** — هيكل مدخلات مُتحقَّق.
- **مخطط الإخراج** — هيكل مخرجات مُتحقَّق.
- **متطلب الجودة** — تقييم أو مسطرة قبل التسليم.
- **متطلب الاعتماد** — موضعه في مصفوفة الإنسان في الحلقة.
- **سجل التدقيق** — كل تشغيل ونسخة موجِّه وكلفة ومخرَج يُسجَّل.

## دورة الحياة

```
design → card → eval → pilot → promote → monitor → retire
```

1. **تصميم** — رسم الدور والمخطط وفئة المخاطر.
2. **بطاقة** — كتابة بطاقة الوكيل.
3. **تقييم** — تشغيل مجموعة التقييمات من `docs/EVALS_RUNBOOK.md`.
4. **تجربة** — استخدام محدود تحت مراجعة مكثّفة.
5. **اعتماد** — السماح الرسمي داخل سير العمل.
6. **مراقبة** — برج التحكم يراقب السلوك والكلفة.
7. **تقاعد** — التخلّي عند الاستبدال أو الاهتراء.

## تركيبة قوى العمل (هدف MVP)

| Agent | Type | Risk class |
|---|---|---|
| Data Quality Agent | Analyst | Medium |
| Revenue Agent | Analyst / Workflow | Medium |
| Outreach Agent | Drafting | High |
| Knowledge Agent | Retrieval | Medium |
| Reporting Agent | Reporting | Medium |
| Compliance Guard Agent | Guardrail | Critical |

## الواجهات
| المدخلات | المخرجات | المالكون | الإيقاع |
|---|---|---|---|
| Engagement workflow | Routed agent runs | Workflow Agent | Per task |
| Agent run log | Cost + QA telemetry | Control Tower | Continuous |
| Eval suite | Agent promotion decision | Head of AI | Per agent change |
| Guardrail decisions | Action allow/block | Compliance Guard Agent | Per action |

## المقاييس
- Agent QA Pass Rate — نسبة التشغيلات الناجحة في التقييم.
- Cost per Run — متوسط الكلفة لكل استدعاء بحسب الشريحة.
- Forbidden Action Attempts — محاولات محظورة أوقفها الحارس.
- Approval Latency — وسيط زمن انتظار اعتماد الإنسان.
