# Platform Path

Gradual **product surface**—avoid jumping to SaaS before repeatability.

## Stages

1. **Internal tools** — Dealix operators only  
2. **Client-visible reports** — PDF / portal read-only outputs  
3. **Client workspace** — uploads, approvals, audit trail  
4. **Self-serve modules** — guardrailed workflows (eligibility scored)  
5. **SaaS** — subscription workspace + usage-based modules  

## Rule — do not move stage until

- Same **workflow shape** repeats with ledgers proof  
- Client **understands value** (renewal / expansion signal)  
- **Support** load is modeled (no heroics)  
- **Governance** is enforceable in-product ([`API_GOVERNANCE.md`](API_GOVERNANCE.md), [`DATA_GOVERNANCE_BY_DESIGN.md`](DATA_GOVERNANCE_BY_DESIGN.md))

**Revenue lens:** [`REVENUE_ARCHITECTURE.md`](../company/REVENUE_ARCHITECTURE.md).  
**Service → product:** [`SERVICE_TO_PRODUCT_PATH.md`](SERVICE_TO_PRODUCT_PATH.md).

## Maximum development path (ordered)

```text
1. Productized services
2. Internal operating system + ledgers
3. Client workspace
4. Retainers
5. Academy
6. Enterprise (gated)
7. SaaS modules
8. MENA expansion
9. Category: “Dealix Standard” as shorthand for governed AI ops
```

Do **not** leap from (1) to (7) without capital compounding ([`DEALIX_CAPITAL_MODEL.md`](../company/DEALIX_CAPITAL_MODEL.md)).
