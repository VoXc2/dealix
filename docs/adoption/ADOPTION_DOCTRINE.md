# Dealix Enterprise Adoption & Change Management

> The real win is not the demo. It is when employees and managers **use** the system, **trust** it, enter it into monthly cadence, and **ask** for natural expansion.

Fifteenth strategic layer. Typed surfaces live in `auto_client_acquisition/adoption_os/`.

## 1. Adoption model

```
Awareness → Trust → First Use → Repeated Use →
Operating Cadence → Managerial Reliance → Retainer → Platform Pull
```

## 3. Change Management — 5 layers

1. Executive Alignment ("We are improving a business capability.").
2. Workflow Ownership (no owner = no adoption).
3. User Enablement.
4. Governance Comfort (draft-only, requires approval, source-backed, audit-logged, human-reviewed).
5. Value Reinforcement (monthly).

## 4. Human-amplified AI

`AI prepares. Humans decide. System logs. Proof validates.`

## 5. Five client-side roles

Executive Sponsor, Workflow Owner, Operator, Governance Reviewer, Dealix Success Owner.

## 6. Enablement Kit — 10 items

Overview, Role Guide, Workflow Guide, Approval Guide, **What Dealix Will Not Do**, Proof Pack Sample, Monthly Cadence Template, Governance FAQ, Arabic Executive Template, Escalation Path.

## 7. Onboarding flow

`Kickoff → Capability Diagnostic → Source Passport Setup → Data Readiness → Workflow Owner → Governance Boundary → First Sprint → Proof Pack → Adoption Review → Retainer Decision`

## 8. Adoption Review

After each sprint: did the client use the outputs? Who used them? Approvals completed? Repeat usage? Value visible? Workflow recurring? What blocks adoption? What training is needed?

Outcomes: Adopted / Partially Adopted / Blocked / Not Adopted.

## 9. Friction Log → Productization

Repeated friction becomes a product candidate (Approval Center, Proof Timeline, Source Passport Panel, Monthly Value Report Generator, Next Action Engine).

Rule: products are built from adoption friction, not imagination.

## 10. Retainer logic

Ready when: workflow owner exists, outputs used, approvals work, proof score ≥ 80, client asks for continuation, monthly value exists, governance risk controlled.

## 11. Adoption metrics

Usage / Behavior / Value / Health. Strongest signal: client asks for a continuous dashboard.

## 12. Saudi Adoption Layer

Arabic executive summaries, clear privacy explanation, human approval emphasis, WhatsApp boundary clarity, bilingual reports, role clarity, training-focused rollout.

## 13. Training products

Client side: AI Ops for Executives, Revenue Intelligence Operator, AI Governance Basics, Company Brain User. Internal Champions: Dealix Champion, Workflow Owner, Governance Reviewer, Operator.

## 14. Adoption Risk Register

R1 sponsor disengages, R2 owner unclear, R3 data not trusted, R4 users fear outputs, R5 slow approvals, R6 value invisible, R7 governance seen as friction, R8 unsafe automation requested. Each has a control.

## 15. The closing sentence

> Dealix wins when the success metric stops being "did we deliver?" and starts being "did the client adopt the capability, trust it, use it, see value monthly, and ask for continuation?".
