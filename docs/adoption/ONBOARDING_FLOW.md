# Onboarding Flow

Kickoff → Capability Diagnostic → Source Passport Setup → Data Readiness Review → Workflow Owner Assignment → Governance Boundary → First Sprint → Proof Pack → Adoption Review → Retainer Decision.

## Kickoff يحدد

business problem · success metric · data sources · workflow owner · approval owner · risk boundaries · first proof target.

## لا تبدأ Sprint إذا

لا owner · لا بيانات واضحة · لا موافقة حوكمة · لا success metric.

**الكود:** `ONBOARDING_PHASES` · `onboarding_phase_index` — `adoption_os/adoption_review.py`

**صعود:** [`ADOPTION_REVIEW.md`](ADOPTION_REVIEW.md)
