> **هذا هو الصوت التشغيلي لـ Dealix في كل ما هو عربي.** أي agent أو
> قالب أو رسالة WhatsApp أو proposal أو report يستخدم الصوت بدون إذن
> لتعديله. الاختلاف عن هذه القواعد يُسجَّل في
> `reports/localization/voice_drift_log.md`.

**التاريخ:** 2026-06-03

## 1. شخصية Dealix (Personality)

| البُعد | الوصف |
| --- | --- |
| **واثق** | يعرف ما يفعل، لا يحتاج مبالغة. |
| **مختصر** | يحترم وقت القارئ. جمل قصيرة. |
| **محترف** | لغة عمل، لا لغة تسويق. |
| **سعودي** | السياق السعودي حاضر دائماً (الرياض، توقيت السعودية، SAR، PDPL). |
| **محترم** | "حضرتك" أو "أنتم"، لا "أنت" في السياقات الرسمية. |
| **عملي** | يعطي خياراً واضحاً وخطوة تالية، لا وعوداً. |
| **صادق** | ما يقوله يفعله. الأرقام حقيقية أو موصوفة بوضوح. |

## 2. ما نحب (We Like)

- جمل قصيرة (< 20 كلمة في WhatsApp، < 35 في email).
- أرقام حقيقية مع مصدر.
- CTA واحد واضح في كل رسالة.
- لغة فصحى مهنية بدون تعقيد.
- ربط النتيجة بالسبب ("لأن" بدلاً من "لذلك").
- خياران واضحان عندما يكون القرار بيد القارئ.

## 3. ما نتجنبه (We Avoid)

| نتجنبه | البديل |
| --- | --- |
| "نتائج مضمونة" | "نتائج موثّقة" + رابط الـ proof |
| "أفضل شركة في..." | "متخصصون في..." + sector |
| "مجاناً!" | "تجربة مجانية لـ 14 يوم" |
| "عرض حصري ينتهي اليوم!" | "العرض متاح حتى 30 يونيو" |
| "حل سحري" | "نظام تشغيل" / "تشغيل آلي" |
| "ثورة" / "تحويل جذري" | "تحسين" / "تنظيم" / "تشغيل منضبط" |
| "ضمان" (بدون سياق) | "ضمان استرداد وفق السياسة" |
| "مضمون" | "موثّق" / "مع إثبات" |
| مبالغات أرقام ("3x أسرع!") | "تقليل 40% في زمن المتابعة" + baseline |
| "تواصل معنا الآن!" | "احجز 15 دقيقة: [Calendly URL]" |

## 4. الضمائر (Pronouns)

| السياق | الضمير |
| --- | --- |
| WhatsApp / رسالة قصيرة | "حضرتك" (احترام) |
| Email رسمي | "أنتم" (جمع محترم) |
| عرض (proposal) | "حضرتكم" / "الشركة" |
| تقرير weekly | "فريقكم" / "أنتم" |
| UI label | "حسابي" / "مشاريعي" (ملكية) |
| Error message | "حدث خطأ" (passive professional) |

## 5. لحظات الفرح والصراحة

- **لحظة نجاح:** "تم. النتيجة وصلت. [الرابط]"
- **لحظة اعتذار:** "نعتذر عن التأخير. السبب كان [...]. الحل: [...]"
- **لحظة رفض:** "لا أستطيع تقديم ذلك الآن. البديل: [...]"
- **لحظة انتظار:** "يحتاج وقتاً أطول. متوقع 5 أيام عمل."

## 6. التوقيع (Closing)

- Email رسمي: "مع التقدير، فريق Dealix"
- WhatsApp: "— Dealix"
- Proposal: "في انتظار ردّكم، Sami — Dealix"

## 7. الأرقام

- **دائماً** مع baseline أو مصدر.
- "3x" ⇒ "3x مقارنة بحالة البداية" + رابط proof event.
- "50%" ⇒ "نصف" أو "50%" — اختر.
- "1.5M" ⇒ `1.5 مليون ر.س` أو `SAR 1.5M` — اختر.

## 8. CTA (Call to Action)

كل رسالة لها CTA واحد فقط. الأنواع المعتمدة:

- **Booking:** "احجز 15 دقيقة: [Calendly URL]"
- **Approval:** "اعتمد المسودة: [الرابط]"
- **Review:** "راجع الحزمة: [الرابط] — تعليقك خلال 48 ساعة"
- **Payment:** "ادفع وفعّل: [رابط Moyasar]"
- **Reply:** "أجب على هذا السؤال: [السؤال]"

ممنوع: CTA مبهم ("تواصل معنا"، "اضغط هنا").

## 9. القنوات والنبرة

| القناة | الطول | النبرة |
| --- | --- | --- |
| WhatsApp | < 60 كلمة | ودية، مباشرة |
| Email رسمي | 100–300 كلمة | مهنية، منظمة |
| Proposal | 800–2000 كلمة | رسمية، مفصّلة |
| Report weekly | 300–800 كلمة | تنفيذية، أرقام |
| SMS | < 160 حرف | قصير، رابط |
| Push | < 80 حرف | معلومة + CTA |

## 10. الاختبار (Voice Test)

قبل نشر أي artifact، اسأل:

1. هل هذه الجملة محترمة؟
2. هل هي مختصرة (< target channel length)؟
3. هل فيها رقم غير موثّق؟
4. هل فيها ادعاء ممنوع في §3؟
5. هل CTA واضح؟
6. هل يمكن اختصارها بنسبة 30% بدون فقدان المعنى؟

إذا نعم على 1، 2، 5، 6 ⇒ منشور. إذا لا ⇒ راجع.
