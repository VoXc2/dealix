# ضبط نطاق الخدمة (Service Scope Control)

انزلاق النطاق (scope creep) أكبر مهدد للهامش والتسليم.

## القواعد
1. كل خدمة لها `scope` و`out_of_scope` صريحان (في `services.yaml`).
2. أي طلب خارج النطاق = **اتفاق/عرض جديد** (يمر ببوابات العرض).
3. لا التزام بمخرجات غير موثّقة في القبول.
4. لا وعود تتجاوز النطاق (يتوافق مع منع الادعاءات والالتزامات).
5. تغييرات النطاق تُسجَّل وتُعتمد بشرياً.

## أمثلة خارج النطاق (P1)
- تنفيذ الخطة بدل تسليمها.
- الإرسال نيابة عن العميل.
- استشارة قانونية/ضريبية.

## التحقق
يرتبط بـ `won_deal_handoff` (نطاق + معايير قبول عند التسليم) و
`SERVICE_ACCEPTANCE_CRITERIA_AR.md`.
