# معايير قبول الخدمة (Service Acceptance Criteria)

معايير واضحة تحدد "تم التسليم" لكل خدمة — تحمي الطرفين وتمكّن التجديد.

## P1 Sprint
- [ ] خريطة تسرّب الإيراد مُسلّمة (استجابة/متابعة/عرض/اعتراضات).
- [ ] Proof Pack مُنقّحة بالأدلة.
- [ ] خطة 30 يوم بأولويات.
- [ ] جلسة عرض (readout) مكتملة.

## P2 Retainer (شهري)
- [ ] مراجعة تشغيلية شهرية مُسلّمة.
- [ ] playbooks المتابعة/الرد معتمدة بشرياً.
- [ ] لوحة القمع/الوحدة الاقتصادية محدّثة.

## قواعد
- لا "تم" بدون استيفاء كل المعايير.
- التجديد (P2) يتطلب قيمة مُسلّمة موثّقة (`renewal_allowed`).
- أي توسعة نطاق = اتفاق جديد (`SERVICE_SCOPE_CONTROL_AR.md`).
