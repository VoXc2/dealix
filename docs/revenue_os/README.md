## ما هذا المجلد / What this folder is

هذا توثيق **نظام تشغيل الإيراد (Revenue OS)** لـ Dealix — نموذج التشغيل
التجاري كاملاً مقسّماً إلى 7 طبقات، مع طبقة أتمتة، مصفوفة موافقات، خطة
30/60/90، ومكتبة رسائل. هذه **وثائق** وليست كوداً؛ تنفيذ الكود متسلسل في
`ENGINEERING_ROADMAP.md`.

This folder documents the Dealix **Revenue OS** — the full commercial
operating model split into 7 layers, plus an automation layer, an approval
matrix, a 30/60/90-day plan, and a messaging library. These are **documents,
not code**; the code is sequenced in `ENGINEERING_ROADMAP.md`.

---

## دائرة مصنع الإيراد / The Revenue Factory loop

الحملة تنتهي. المصنع يتكرر. كل عميل واعتراض وproof pack يصبح أصلاً جديداً.

A campaign ends. A factory repeats. Every customer, objection, and proof pack
becomes a new asset.

```
Audience → Signal → Trust → Proof → Qualification → Meeting → Scope
   → Payment → Delivery → Proof Pack → Upsell → Referral → Content → Audience
```

## حدود لا تُكسر / Hard boundaries

هذه الوثائق تحترم الـ 11 non-negotiables. تحديداً:

- التواصل الخارجي (LinkedIn / email) **مسودة + موافقة المؤسس فقط** — لا إرسال
  ذاتي حي (`no_live_send`).
- كشف إشارات الشراء **يدوي / من مصادر بموافقة فقط** — لا scraping (`no_scraping`).
- لا لغة ضمان نتائج في أي قالب (`no_fake_proof`).
- لا تسجيل إيراد قبل `invoice_paid`؛ لا case study بدون موافقة العميل.

External outreach is **draft + founder approval only**. Buying-signal detection
is **manual / consented-source only — no scraping**. No guaranteed-outcome
language. No revenue before `invoice_paid`; no case study without client
approval.

> `docs/OFFER_LADDER_AND_PRICING.md` (تشخيص مجاني، sprint بـ 499 ريال).
> تسعير التشخيص المدفوع المقترح موجود **فقط** في
> `PRICING_PROPOSAL_diagnostic_tiers.md` وليس قراراً نافذاً.
