# Dealix Enterprise Rollout & Operating Adoption Playbook

> Dealix is not installed as a tool. Dealix is **adopted as a way of operating**: start with a diagnostic, prove value, enter cadence, then expand across departments.

Sixteenth strategic layer. Typed surfaces live in `auto_client_acquisition/enterprise_rollout_os/`.

## 1. Rollout model

```
Land → Prove → Adopt → Operate → Expand → Standardize → Institutionalize
```

## 2. Enterprise entry message

> Your AI risk is not tool access. Your risk is unmanaged AI execution: unclear data, unapproved outputs, unlogged agents, no ROI proof, and no operating cadence.

Entry offers: AI Governance & Readiness Review, or Governed AI Workflow Pilot.

## 3. Department-by-department rollout

- **Sales / Revenue** → Revenue Intelligence Sprint.
- **Operations** → AI Quick Win Sprint.
- **Knowledge / HR** → Company Brain Sprint.
- **Governance / Compliance / IT** → AI Governance Review.
- **Executive Office** → Executive Value Dashboard.

## 4. Seven adoption gates

1. **Sponsor Gate** — executive sponsor + business owner + problem.
2. **Data Gate** — Source Passport + PII status + allowed use.
3. **Workflow Gate** — workflow owner + approval owner + success metric.
4. **Governance Gate** — external actions + draft-only boundaries + blocked actions.
5. **Proof Gate** — Proof Pack complete + score ≥ 70 + limitations.
6. **Adoption Gate** — outputs reviewed + approvals completed + workflow used twice.
7. **Retainer Gate** — proof score ≥ 80 + adoption ≥ 70 + monthly value + owner committed.

## 5. Rollout Roles

Executive Sponsor, Business Owner, Data Owner, Governance Reviewer, Operators, Dealix Success Owner.

## 6. Rollout Kit — 12 items

Executive Brief, Capability Diagnostic, Data Request Template, Source Passport Form, Governance Boundary Document, Workflow Owner Guide, Approval Matrix, Proof Pack Template, Monthly Value Report Template, Adoption Review Template, Incident Response Guide, **What Dealix Refuses to Build**.

## 7. Trust narrative

> Dealix does not automate chaos. Dealix governs AI before scaling it.

## 8. Saudi rollout layer

Arabic-first, privacy-aware, human-approval-centered, training-oriented, executive-friendly.

## 9. Agent rollout rules

MVP autonomy 0–3. Forbidden: autonomous external action, unapproved sending, scraping, LinkedIn automation, cold WhatsApp, data movement without audit.

## 10. Platform pull signals

Client asks for dashboard, multiple stakeholders need access, approval workflow repeats, monthly reports repeat, audit logs requested, executive asks for live view. Only then build Client Workspace, Approval Center, Proof Timeline, Governance Log, Value Dashboard.

## 11. Enterprise expansion path

`Department Sprint → Department Retainer → Cross-Department Governance → Client Workspace → Executive Value Dashboard → AI Control Plane → Enterprise Program`

## 12. Risk register

R1–R10 with controls: Sponsor Gate, Source Passport, Governance Boundary, Adoption Review, Approval Matrix, Proof Score, No Unsafe Automation Policy, Security Brief, Expectation Setting, Agent Card.

## 13. Rollout metrics

Adoption / Trust / Value / Expansion.

## 14. The closing sentence

> Dealix wins inside enterprises when AI is not an added tool, but an operating capability with an owner, documented data, governance, approvals, Proof, and a monthly cadence that opens expansion.
