# Dealix — الخريطة العليا من الصفر إلى شركة قابضة

**النسخة:** مرجع استراتيجي وتشغيلي علوي (ليس دليل تنفيذ API).  
**الجمهور:** المؤسس، القيادة، الشريك الاستراتيجي، التصميم المؤسسي للمنتج.

---

## الفكرة الأم

Dealix **ليست**:

- AI tool  
- automation agency  
- lead generation system  
- chatbot builder  
- consulting service  

Dealix **يجب أن تكون**:

**Saudi Governed AI Operations Company**  
شركة سعودية **لتشغيل** الذكاء الاصطناعي **المحكوم** داخل الشركات.

### المعنى التشغيلي

نأخذ فوضى الشركات:

- بيانات مبعثرة  
- عمليات يدوية  
- متابعة ضعيفة  
- معرفة متفرقة  
- مخاطر AI  
- غياب proof  

ونحوّلها إلى:

- قدرات تشغيلية  
- محكومة  
- مقاسة  
- قابلة للتدقيق  
- قابلة للتوسع

### غير قابل للتفاوض

- No cold WhatsApp automation.

- No fake proof.

## 2. طبقة الفئة السوقية (Category)

Dealix تمتلك فئة اسمها:

**Governed AI Operations** — تشغيل الذكاء الاصطناعي المحكوم.

### التعريف

تحويل AI من أدوات وتجارب عشوائية إلى **workflows داخل الشركة** فيها: بيانات واضحة، موافقة بشرية، حوكمة، سجل تدقيق، Proof، وتشغيل شهري.

### الرسالة

المشكلة ليست أن الشركات لا تستخدم AI.  
المشكلة أنها تستخدم AI **بدون نظام تشغيل**.

## 6. طبقة التشخيص (Diagnostic)

قبل أي مشروع كبير: **Capability Diagnostic**.

### يقيس

Revenue، Data، Workflow، Knowledge، Governance، Reporting، AI Readiness.

### المستويات (0–5)

0 = Absent → 1 Manual → 2 Structured → 3 AI-Assisted → 4 Governed Workflow → 5 Optimized OS.

### النتيجة

## 7. طبقة Data OS

بدون بيانات واضحة لا يوجد AI.

### المكونات

Source Passport، Import Preview، Data Quality Score، PII Detection، Deduplication، Normalization، Allowed Use Check، Retention Policy.

### Source Passport (أسئلة إلزامية)

من المالك؟ من أين جاء؟ هل فيه PII؟ ما الاستخدام المسموح؟ هل يسمح AI؟ هل يسمح استخدام خارجي؟ متى يحذف؟

### القواعد

- No Source Passport = no AI use.  
- Unknown source = no outreach.  
- PII + external use = approval required.  

---

## 8. طبقة Governance OS

أهم طبقة دفاعية. كل مخرج يأخذ قرارًا:

ALLOW، ALLOW_WITH_REVIEW، DRAFT_ONLY، REQUIRE_APPROVAL، REDACT، BLOCK، ESCALATE.

### تفحص

source status، PII status، allowed use، channel risk، claim risk، agent autonomy، approval requirement.

### مثال

أرقام جوال + طلب إرسال واتساب → **BLOCK** أو **DRAFT_ONLY** — reason: no cold WhatsApp automation.

---

## 9. طبقة LLM Gateway

لا يوجد AI call مباشر خارج البوابة.

### كل AI يمر عبر

Prompt Registry، Model Router، Redaction، Schema Validator، Cost Guard، Eval Hook، AI Run Ledger.

### القواعد

- No AI call outside gateway.  
- No prompt without version.  
- No output without schema.  
- No client output without governance.  

---

## 10. طبقة Agent OS

إذا تُستخدم agents فلتكن محكومة.

### كل agent

Identity، Owner، Purpose، Allowed tools، Forbidden tools، Autonomy level، Auditability card، Kill switch.

### مستويات الاستقلالية (0–6)

0 Read → 1 Analyze → 2 Draft/Recommend → 3 Queue for approval → 4 Internal execution after approval → 5 External restricted → 6 Autonomous external **forbidden**.

### قاعدة MVP

Dealix MVP يسمح **0–3 فقط**.

### ممنوع الآن

agent يرسل إيميل؛ واتساب؛ scraping؛ تعديل CRM خارجي بدون موافقة؛ بيانات بين عملاء.

---

## 11. طبقة Secure Runtime

تشغيل agents بأمان.

### Runtime loop

Observe → Classify → Check policy → Decide → Execute/Block/Escalate → Log evidence → Update state.

### حالات agent

SAFE، WATCH، RESTRICTED، ESCALATED، PAUSED، KILLED.

### Kill Switch

Soft kill، Tool kill، Client kill، Agent kill، Fleet kill.

أي agent يطلب أداة خطرة أو إجراء خارجي بدون موافقة → **تقييد أو إيقاف**.

### Evidence Graph

Source → AI Run → Governance Decision → Review → Approval → Output → Proof → Value → Board Decision.

### الهدف

## 13. طبقة Proof OS

Dealix لا تبيع كلامًا — تبيع **proof**.

### Proof Pack (محتوى)

Executive Summary، Problem، Inputs، Source Passports، Work Completed، Outputs، Quality Scores، Governance Decisions، Blocked Risks، Value Metrics، Limitations، Recommended Next Step، Capital Assets Created.

### Proof Score

- 85–100 = case candidate  
- 70–84 = sales support  
- 55–69 = internal learning  
- <55 = weak proof  

### القانون

No proof, no claim. No proof, no case study. No proof, no retainer push.

---

## 14. طبقة Value OS

**Estimated Value** — تقدير بافتراضات.  
**Observed Value** — ملاحظ داخل المشروع (مثال: 46 duplicate accounts).  
**Verified Value** — أكدها العميل أو قيست بعد التشغيل.

### القانون

لا تستخدم Estimated كدعاية. استخدم **Verified** فقط كـ case خارجي.

## 40. الخلاصة

### منطق البناء

- لا تبدأ بمنصة — ابدأ بقدرة.  
- لا تبدأ بوكلاء أحرار — ابدأ بـ **draft-only governed agents**.  
- لا تبدأ بسوق كامل — ابدأ بقطاع واضح.  
- لا تبدأ بوعود — ابدأ بـ **Proof**.  
- لا تبدأ بتوسع — ابدأ بـ **Retainer**.  
- لا تبدأ بـSaaS — ابدأ بتكرار **مدفوع**.  

### الجملة النهائية

Dealix تفوز عندما تصبح **النظام** الذي يحوّل فوضى الشركات إلى قدرات AI محكومة: بيانات واضحة، workflows مملوكة، AI مساعد، مراجعة بشرية، حوكمة وقت التشغيل، Evidence، Proof، قيمة شهرية، وأصول تتراكم مع كل مشروع — حتى تصبح الشركة قابلة للتوسع إلى **وحدات أعمال ومعايير وأكاديمية ومشاريع مشتقة وشركة قابضة**.
