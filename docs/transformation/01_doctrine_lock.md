# 01 — Doctrine Lock

## Objective

Freeze and align company doctrine across strategy, product, trust, and execution so every team operates on one non-contradictory operating model.

## Doctrine anchors

- `docs/strategic/DEALIX_MASTER_OPERATING_MODEL_AR.md`
- `docs/company/DEALIX_MASTER_OPERATING_SYSTEM.md`
- `docs/reports/enterprise_control_plane_hardening_report.md`
- `readiness/enterprise_readiness_model.md`
- `readiness/mission_critical_ready.md`

## Non-negotiable constraints

1. No external high-risk action without approval.
2. No measured value claim without source evidence.
3. No cross-tenant operational access.
4. No production autonomy without rollback path.
5. No proof-level overclaiming beyond available evidence level.

## Decisions frozen

- Golden chain is the default execution path for all commercial workflows.
- Offer architecture remains five doors with productized service ladders.
- Approval-first is mandatory for external and irreversible actions.
- Control-plane verification scripts are release blockers.
- Trust and value artifacts are required for enterprise procurement narratives.

## Governance cadence for doctrine changes

- Proposed via RFC in `docs/transformation/rfcs/`.
- Requires review from Product, Trust, Delivery, Revenue, and Platform owners.
- Requires explicit impact mapping to `todo_registry.yaml` and `kpi_registry.yaml`.
- Requires green verification checks before merge.
