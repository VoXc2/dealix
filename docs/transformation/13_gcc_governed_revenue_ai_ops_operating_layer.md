## Governance doctrine (non-negotiable)

- No external action without approval.
- No value claim without source reference.
- No security claim without evidence.
- No case study without client permission.
- No autonomous external messaging agents.
- Every agent action must be auditable.

## Operating loops

- **Demand loop:** signal -> content -> proof request -> lead -> meeting -> learning
- **Sales loop:** lead -> score -> draft -> approval -> conversation -> scope -> invoice -> payment
- **Delivery loop:** payment -> onboarding -> diagnostic -> proof pack -> value confirmation -> upsell
- **Product loop:** manual repetition -> template -> automation -> module
- **Governance loop:** proposed action -> risk -> approval -> action -> evidence -> policy update

## Agent strategy

Agents prepare, classify, draft, score, warn, summarize, route, and log.  
External execution remains approval-gated through orchestrator policy.

Orchestrator minimum responsibilities:

- receive event
- check policy
- choose agent
- enforce tool limits
- require source
- route approval
- log evidence
- update state

## 14/30/90 operating checkpoints

- **14 days:** first market proof signals (targets, conversations, meetings, scopes, first invoice)
- **30 days:** first paid diagnostic + delivered proof pack + objections library
- **90 days:** 3–5 paid diagnostics + 1 sprint + 1 retainer candidate + repeatable checklist

## Build discipline

Build only after paid signal.  
Automate only repeated workflows.  
Productize only with proof-backed repetition across clients.
