## Why these layers
Dealix runs as a stack of **13 operating layers**. Each layer answers
one question and produces one kind of artefact. Together they turn the
strategy in `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md` and the rules
in `docs/DEALIX_OPERATING_CONSTITUTION.md` into a repeatable system
that compounds: every diagnostic, sprint, retainer, and enterprise
engagement adds to the same capability factory, value ledger, and
holding company portfolio.

This index is the **front door**. Open it first; then jump to the
canonical doc for the layer you need. Each layer entry shows: one-line
purpose, file count, canonical umbrella doc, and the folder where the
rest of the layer lives. Five strategic anchors stitch all 13 layers
together:

- `docs/company/DEALIX_MASTER_OPERATING_BLUEPRINT.md` — five-layer umbrella
- `docs/company/DEALIX_CONSTITUTION.md` — the rules
- `docs/company/DEALIX_CEO_STRATEGY.md` — the CEO/CTO/CSO view
- `docs/holding/DEALIX_HOLDING_OS.md` — the compound holding model
- `docs/intelligence/INTELLIGENCE_LAYER.md` — the operating brain

## Cross-layer flow
The 13 layers are not parallel silos. They flow:

```
   Capability Diagnostic (L2 / ai_ops_diagnostic)
                       |
                       v
              Service Sprint (S)
                       |
                       v
            Proof Pack (M / templates)
                       |
                       v
        Capital Asset (L1 / ledgers + assets)
                       |
                       v
             Retainer (L6 / growth)
                       |
                       v
       Productization (L4 / capability factory)
                       |
                       v
      Business Unit (H / business_units)
                       |
                       v
           Venture (H / holding)
                       |
                       v
       Standard (H / standards) -----> feeds back into L2
                       |
                       v
        Holding Company (H / holding os)
```

Read **bottom-up** for strategy ("we are building a holding company
of operating standards"); read **top-down** for delivery ("today I
need to run a capability diagnostic well").

## How to keep this index honest
- Update **layer counts** when files are added or removed; do not let
  this index drift more than 2 weeks behind reality.
- When a new layer is added, append it to the table and add a new
  section; never silently re-number existing layers.
- The index is **scan-friendly first**, exhaustive second — if a layer
  section grows past 30 lines, push detail into its umbrella doc.
- Validate links monthly via `scripts/validate_doc_links.py`.

- **Link validity** — % of links that resolve (target 100%).

  from this index.
