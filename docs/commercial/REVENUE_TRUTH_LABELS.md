# نظام تصنيفات الحقيقة / Revenue Truth Labels

## 1. الغرض / Purpose

**العربية:** نظام تصنيفات الحقيقة يحمي دياليكس من المبالغة في النتائج. كل رقم أو ادعاء في أي مخرج يحمل تصنيفًا واحدًا فقط يوضّح مدى رسوخ الدليل خلفه. هذا ما يجعل دياليكس بمستوى المؤسسات: لا إثبات مزيّف، ولا أرقام بلا مصدر.

**English:** The truth-label system protects Dealix from overstating results. Every number or claim in any artifact carries exactly one label that states how solid the evidence behind it is. This is what makes Dealix enterprise-grade: no fake proof, no numbers without a source.

## 2. التصنيفات الستة / The Six Labels

| التصنيف / Label | المعنى / Meaning |
|---|---|
| Estimate / تقدير | رقم محسوب أو متوقع، لم يُقَس بعد. يحمل بادئة `~`. / A calculated or projected number, not yet measured. Carries the `~` prefix. |
| Observed / مُلاحَظ | قاسته دياليكس مباشرة من بيانات العميل. / Measured directly by Dealix from client data. |
| Client-confirmed / مؤكَّد من العميل | أكّده العميل صراحةً كتابةً. / Explicitly confirmed by the client in writing. |
| Payment-confirmed / مؤكَّد بالدفع | إيراد فعلي مرتبط بسجل دفع موثّق. / Actual revenue tied to a verified payment record. |
| Repeated workflow / سير عمل متكرر | نمط ظهر في 3 مشاريع أو أكثر بنفس الشكل. / A pattern seen in 3+ projects in the same form. |
| Retainer-ready / جاهز للاشتراك | سير عمل مستقر يبرّر عرضًا شهريًا. / A stable workflow that justifies a monthly offer. |

## 3. القاعدة الصارمة / The Hard Rule

**العربية:** كل رقم في أي مخرج موجَّه للعميل يحمل تصنيفًا واحدًا بالضبط — لا صفر ولا اثنين. التقديرات تحمل أيضًا بادئة `~` بالعربية والإنجليزية معًا، اتساقًا مع معيار Proof Pack v6. الرقم بلا تصنيف لا يخرج.

**English:** Every number in any customer-facing artifact carries exactly one label — not zero, not two. Estimates also carry the `~` prefix in both Arabic and English, consistent with the Proof Pack v6 Standard. A number without a label does not ship.

```text
ONE NUMBER  ->  ONE LABEL
  estimate   ->  "~12 leads (Estimate)"  /  "~12 عميلًا محتملًا (تقدير)"
  measured   ->  "37 leads (Observed)"
  confirmed  ->  "SAR 8,000 (Payment-confirmed)"
```

## 4. مثال محلول / Worked Example

سيناريو: تدقيق متابعة العملاء المحتملين لشركة خدمات.
Scenario: a lead follow-up audit for a services company.

| العنصر / Element | القيمة / Value | التصنيف / Label | السبب / Why |
|---|---|---|---|
| ثغرة متابعة العملاء / Lead follow-up gap | 41% من العملاء بلا متابعة / 41% of leads not followed up | Observed | قِيس مباشرة من سجل CRM العميل / Measured from the client CRM log |
| القيمة المحتملة / Potential value | ~SAR 60,000 سنويًا / ~SAR 60,000 per year | Estimate | إسقاط، لم يتحقق بعد / A projection, not yet realized |
| ألم العميل / Client pain | "نخسر صفقات بسبب بطء الرد" / "We lose deals to slow response" | Client-confirmed | قاله العميل كتابةً في مكالمة التسليم / Stated in writing by the client |
| الإيراد / Revenue | SAR 0 | Payment-confirmed | لا يُسجَّل إيراد إلا بسجل دفع / Revenue is logged only with a payment record |

**العربية:** الإيراد يحمل تصنيف Payment-confirmed فقط — لا يجوز أبدًا وصف قيمة محتملة بأنها إيراد.

**English:** Revenue carries the Payment-confirmed label only — a potential value may never be described as revenue.

## 5. جدول قرار التصنيف / Label-Decision Table

| التصنيف / Label | الدليل المطلوب / Evidence required | يظهر في / Appears in |
|---|---|---|
| Estimate | منهجية حساب أو افتراض موثّق / A documented calculation method or assumption | داخلي + قابل للمشاركة / Internal + customer-shareable |
| Observed | قياس مباشر من بيانات العميل / Direct measurement from client data | داخلي + قابل للمشاركة / Internal + customer-shareable |
| Client-confirmed | تأكيد كتابي من العميل / Written confirmation from the client | داخلي + قابل للمشاركة / Internal + customer-shareable |
| Payment-confirmed | سجل دفع موثّق / A verified payment record | داخلي فقط، ويُشارَك مجهّلًا / Internal only; shared only anonymized |
| Repeated workflow | 3+ مشاريع موثّقة بنفس النمط / 3+ documented projects, same pattern | داخلي فقط / Internal only |
| Retainer-ready | تقييم استقرار سير العمل / A workflow-stability assessment | داخلي فقط / Internal only |

## 6. لماذا يمنع هذا الإثبات المزيّف / Why This Prevents Fake Proof

**العربية:** التصنيف يجبر كل ادعاء على إعلان قوته. عندما يرى العميل "Observed" يعرف أن الرقم مقيس، وعندما يرى "~ Estimate" يعرف أنه إسقاط. هذا الفصل الواضح يبني ثقة دائمة ويحمي دياليكس قانونيًا وتجاريًا.

**English:** Labelling forces every claim to declare its strength. When a client sees "Observed" they know the number is measured; when they see "~ Estimate" they know it is a projection. This clear separation builds durable trust and protects Dealix legally and commercially.

> Estimated outcomes are not guaranteed outcomes / النتائج التقديرية ليست نتائج مضمونة.
