# معيار Dealix — SOAEN / The Dealix Standard — SOAEN
<!-- COMMERCIAL EMPIRE | Owner: Founder | Date: 2026-05-17 -->

> القانون / Canonical: see [PROOF_PACK_V6_STANDARD.md](../PROOF_PACK_V6_STANDARD.md)

معيار Dealix SOAEN قاعدة واحدة: أي سير عمل يستحق الأتمتة يجب أن يملك خمسة عناصر — مصدر، مالك، موافقة، دليل، خطوة تالية. غياب أي عنصر يعني أن سير العمل غير جاهز للأتمتة.
The Dealix SOAEN Standard is one rule: any workflow worth automating must have five elements — Source, Owner, Approval, Evidence, Next Action. A missing element means the workflow is not ready for automation.

## 1. العناصر الخمسة / The Five Elements

| الحرف / Letter | العنصر / Element | التعريف / Definition |
|---|---|---|
| S | المصدر / Source | من أين جاءت البيانات أو الحدث، وهل المصدر موثّق |
| O | المالك / Owner | شخص واحد مسؤول عن الخطوة التالية |
| A | الموافقة / Approval | موافقة بشرية إلزامية قبل أي إجراء خارجي |
| E | الدليل / Evidence | سجل يثبت ما حدث فعلًا، بوسم حقيقة |
| N | الخطوة التالية / Next Action | فعل محدد وقابل للتنفيذ، لا توصية مبهمة |

## 2. الصياغات البيعية / The Sales Framings

استخدم هذه الصياغات في الاكتشاف والعرض. كل صياغة تحوّل غياب عنصر إلى مشكلة يفهمها المشتري.
Use these framings in discovery and pitch. Each turns a missing element into a problem the buyer understands.

```text
Lead with no owner          = not a pipeline
عميل محتمل بلا مالك          = ليس خط أنابيب

Follow-up with no evidence  = not operations
متابعة بلا دليل             = ليست عمليات

AI action with no approval  = risk
إجراء ذكاء اصطناعي بلا موافقة = مخاطرة

Dashboard with no next action = a report only
لوحة بلا خطوة تالية         = مجرد تقرير
```

## 3. منهج Dealix الثماني / The Dealix 8-Step Method

المعيار SOAEN يصف ما يجب أن يملكه سير العمل. المنهج الثماني يصف كيف نوصله. هذا المنهج هو ما يميّز Dealix عن وكالة ذكاء اصطناعي أو مستشار CRM أو مزوّد روبوت محادثة.
SOAEN describes what a workflow must have. The 8-step method describes how we deliver it. This method is what makes Dealix different from an AI agency, a CRM consultant, or a chatbot provider.

```text
1. Detect    كشف     — رصد leads أو سير عمل
2. Diagnose  تشخيص   — تصنيف الحالة والمالك والفجوة
3. Decide    قرار    — ترتيب الأولوية
4. Draft     تجهيز   — صياغة draft_only
5. Approve   موافقة  — موافقة بشرية إلزامية
6. Deliver   تسليم   — تنفيذ المعتمد فقط
7. Prove     إثبات   — Proof Pack بوسوم حقيقة
8. Expand    توسيع   — ترقية النطاق
```

| المنافس / Competitor | ما يقدّمه / What they offer | ما تضيفه Dealix / What Dealix adds |
|---|---|---|
| وكالة ذكاء اصطناعي / AI agency | بناء نماذج وأتمتة | حد الموافقة + سجل الدليل |
| مستشار CRM / CRM consultant | تنفيذ النظام | التشخيص والإثبات قبل التنفيذ |
| مزوّد روبوت محادثة / chatbot provider | ردود آلية | المالك والخطوة التالية الموثّقة |

## 4. قائمة التحقق — جاهزية الأتمتة / Checklist — Automation Readiness

أي سير عمل ينقصه عنصر واحد من الخمسة غير جاهز للأتمتة. أجب بنعم أو لا على كل بند.
Any workflow missing one of the five elements is not ready for automation. Answer yes or no on each line.

```text
[ ] S — هل المصدر موثّق؟              / Is the source documented?
[ ] O — هل يوجد مالك واحد محدد؟        / Is there one named owner?
[ ] A — هل يوجد حد موافقة بشرية؟       / Is there a human approval gate?
[ ] E — هل يُسجَّل دليل بوسم حقيقة؟     / Is evidence logged with a truth label?
[ ] N — هل الخطوة التالية محددة؟       / Is the next action specific?
```

إذا كان أي بند "لا"، أصلِح العنصر أولًا. لا تُؤتمت سير عمل ناقص — الأتمتة تضاعف الفجوة.
If any line is "no", fix the element first. Do not automate an incomplete workflow — automation multiplies the gap.

## 5. الربط بالعقيدة / Doctrine Link

المعيار يفرض الموافقة البشرية على كل إجراء خارجي ووسم الحقيقة على كل دليل. لا يسمح بإثبات مزيّف ولا بادّعاء عائد مضمون: الدليل يصف ما حدث فقط.
The standard enforces human approval on every external action and a truth label on every piece of evidence. It allows no fake proof and no guaranteed-return claim: evidence describes only what happened.

> Estimated outcomes are not guaranteed outcomes / النتائج التقديرية ليست نتائج مضمونة.
