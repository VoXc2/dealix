# Scope Creep Policy — سياسة التمدد في النطاق
**Dealix — Agent #3**

> **الغرض:** كيف نمنع ونتعامل مع scope creep — أكبر قاتل للهامش.

---

## 1. What is Scope Creep?

**Scope creep = إضافة scope بدون agreement على price/timeline.**

أمثلة:
- "بس أضف هذا الworkflow"
- "سويها بنفس السعر"
- "ضيف هذه الـ feature"
- "نفّذ هذا التكامل الإضافي"

---

## 2. Why Scope Creep is Dangerous

### 2.1 Impact
- ❌ Margin erosion
- ❌ Delivery delay
- ❌ Team burnout
- ❌ Client disappointment
- ❌ Founder time stolen
- ❌ Reputation risk (if rushed)

### 2.2 Common Causes
- Vague initial scope
- No change order process
- "Yes" culture
- Weak founder discipline
- Client's unclear expectations
- Sales over-promised

---

## 3. Prevention (The Best Cure)

### 3.1 Clear Scope from Start
- ✅ Detailed proposal
- ✅ Explicit in/out list
- ✅ Acceptance criteria
- ✅ Founder-approved

### 3.2 Change Order Process (Always)
- ✅ Document change
- ✅ Estimate cost
- ✅ Price adjustment
- ✅ Founder approval
- ✅ Client signature
- ✅ Implement only after

### 3.3 Founder Discipline
- ✅ Don't say "yes" without thinking
- ✅ Always check scope
- ✅ Always estimate
- ✅ Always get approval

### 3.4 Client Education
- ✅ Set expectations in proposal
- ✅ Remind when asking
- ✅ Reinforce "extra = extra cost"

---

## 4. Detection

### 4.1 During Discovery
- Vague requirements
- "We'll add later"
- No specific deliverables
- Open-ended timeline

### 4.2 During Delivery
- Tasks not in original scope
- "Can you also..."
- "Just one more thing"
- "This is similar to what we have"

### 4.3 Red Flags
- "Small change" (×10)
- "Quick addition"
- "Other vendors include this"
- "We assumed this was included"

---

## 5. Response

### 5.1 The Script
```
"هذا خارج نطاق اتفاقنا الأصلي.
أقدر أحضّر change order:
- Additional scope: [X]
- Additional cost: [Y] SAR
- Additional time: [Z] days

هل تريد أن نبدأ؟"
```

### 5.2 Variations

**For "small" additions:**
```
"أقدّر اهتمامك. هذا change order صغير،
لكن نشتغل بنظام change order للحفاظ على الجودة.
أقدر أحضّر proposal خلال يوم."
```

**For mid-scope additions:**
```
"هذا إضافة mid-scope. أقدر أحضّر change order
خلال 3 أيام. founder يوافق قبل ما نبدأ."
```

**For large additions:**
```
"هذا إضافة كبيرة. أقدر أحضّر change order
كامل، أو new SOW، خلال أسبوع.
founder يوافق قبل ما نبدأ."
```

## 6. Change Order Process

### 6.1 Step 1: Identify
- Document the additional ask
- Be specific
- Reference original scope

### 6.2 Step 2: Estimate
- Delivery hours
- Founder time
- Tools
- Subcontractor (if any)
- Margin

### 6.3 Step 3: Price
- Within margin floor
- Reasonable
- Justified

### 6.4 Step 4: Approval
- L1 for small (< 5K)
- L2 for mid (5-15K)
- L3 for large (> 15K)
- L4+ for strategic

### 6.5 Step 5: Document
- Format: change order form
- Signatures
- Date

### 6.6 Step 6: Implement
- Only after approval
- Update plan
- Update timeline

## 8. The "Yes" Trap

### 8.1 Pressure to Say Yes
- Client pressure
- Founder time pressure
- Team pressure
- Deadline pressure

### 8.2 The Right Answer
- **Small (1-2h):** "Yes, this time, as goodwill. Next time = change order."
- **Medium (4-8h):** "I'll prepare a small change order, takes 1 day."
- **Large (> 8h):** "This needs full change order. I'll send within 1 week."

### 8.3 Never Do
- ❌ "Sure, no problem" (without checking)
- ❌ "We'll add it for free"
- ❌ "Just this once" (multiple times)
- ❌ "It's part of the deal" (when not)

---

## 9. The "Discount" Compromise

### 9.1 When to Offer
- Client truly can't pay more
- Long-term value
- Founder decision

### 9.2 How
- Reduce scope, not price
- "I can keep the price if we remove [X] and [Y]"
- Trade-offs, not discounts

## 10. The "Founder Discipline" List

### 10.1 Always Check
- Is this in scope?
- Is this a real change?
- What's the cost?
- What's the price?
- Who approves?

### 10.2 Always Document
- The change
- The cost
- The price
- The approval
- The implementation

### 10.3 Always Communicate
- Clearly
- Professionally
- Without blame
- With options

---

## 11. Common Scenarios

### 11.1 "Just one more workflow"
- Check: is it really just one?
- If yes: small change order
- If many: bigger change order

### 11.2 "Quick training for 2 more people"
- If 1-2h: goodwill
- If more: change order

### 11.3 "Add this integration"
- Always change order
- Estimate: 8-40h typically
- Price: 5-15K typically

### 11.4 "We need this for the project to work"
- Was it in original scope?
- If yes: no extra
- If no: change order
- If unclear: clarify

### 11.5 "You said you would"
- Reference original scope
- If unclear: founder decision
- If change: change order

---

## 12. Scope Reduction Strategy

When client can't pay more:
1. List all scope items
2. Mark critical vs nice-to-have
3. Remove nice-to-have first
4. Propose: "I'll keep price if we remove these"
5. Client chooses
