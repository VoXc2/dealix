# سير مراجعة المسوقين بالعمولة — Affiliate Review Workflow

نسخة المسوّق بالعمولة (affiliate copy) **تُراجَع قبل النشر**. لا نشر بلا مراجعة امتثال، ولا دفع عمولة بلا استيفاء بوابات محددة.

## النطاق / Scope

- يصف workflow مراجعة نسخة المسوّق بالعمولة من التقديم حتى استحقاق العمولة.
- الـ disclosure هنا بوابة تقنية وتشغيلية، لا نصيحة.
- كل نسخة مسوّق تخضع للـ 11 non-negotiable.

## المصدر القانوني / Canonical source

- سلم الخدمات والأسعار: [`../OFFER_LADDER_AND_PRICING.md`](../OFFER_LADDER_AND_PRICING.md).
- قواعد إحالة الشركاء: [`../commercial/PARTNER_ONBOARDING_KIT.md`](PARTNER_ONBOARDING_KIT.md) و[`../partners/PARTNER_ONBOARDING.md`](../partners/PARTNER_ONBOARDING.md).

## الـ workflow

```
affiliate_submits_copy → compliance_review → approved / edit_required / blocked
   → tracking link → lead submitted → qualified → invoice_paid → commission_eligible
```

| المرحلة | الوصف |
|---------|-------|
| **affiliate_submits_copy** | المسوّق يقدّم نص الإعلان/المنشور قبل أي نشر |
| **compliance_review** | مراجعة امتثال إلزامية مقابل القائمة أدناه |
| **approved** | النسخة مطابقة — يُسمح بالنشر |
| **edit_required** | تعديل محدد مطلوب قبل إعادة التقديم |
| **blocked** | النسخة تخالف non-negotiable — لا نشر، لا رابط تتبّع |
| **tracking link** | يُصدر رابط تتبّع فقط بعد الموافقة |
| **lead submitted** | وصول lead عبر الرابط |
| **qualified** | تأهيل الـ lead — مالك قرار + جدّية |
| **invoice_paid** | سداد فاتورة العميل فعلياً |
| **commission_eligible** | استحقاق العمولة بعد استيفاء كل البوابات |

## بوابات الدفع — العمولة لا تُدفع قبل

- **invoice_paid** — سداد الفاتورة فعلياً.
- لا إشكال في فترة الاسترداد (no refund-period issue).
- الـ lead ليس مكرّراً (not a duplicate).
- النسخة مطابقة للامتثال (copy compliant).
- الـ disclosure موجود وواضح (disclosure present).

أي بوابة غير مستوفاة = العمولة **غير مستحقة**.

## الـ disclosure كبوابة تقنية وتشغيلية

الـ disclosure ليس نصيحة أخلاقية بل **شرط نظامي للدفع**. إفصاحات المسوّقين كثيراً ما تكون مفقودة أو غير واضحة؛ لذلك يحجب النظام صرف العمولة تلقائياً ما لم يكن الـ disclosure موجوداً وواضحاً ومرئياً في النسخة المنشورة. لا disclosure واضح = `blocked` = لا رابط تتبّع = لا عمولة.

## قائمة مراجعة الامتثال — Compliance review checklist

| البند | المعيار | النتيجة عند الفشل |
|-------|---------|---------------------|
| Disclosure | إفصاح علاقة العمولة موجود وواضح ومرئي | `blocked` |
| لا scraping | لا ذكر أو تلميح لاستخراج بيانات | `blocked` |
| لا إرسال بارد | لا WhatsApp/LinkedIn automation ولا تواصل بارد | `blocked` |
| لا ضمان نتيجة | لا وعد ROI أو مبيعات أو conversion كحقيقة | `blocked` |
| لا proof مزيّف | لا عملاء وهميون ولا أرقام غير مُتحقَّقة | `blocked` |
| لا PII | لا أسماء/هواتف/إيميلات حقيقية في النسخة | `blocked` |
| دقة العرض | الأسعار والنطاق مطابقة لسلم الخدمات المعتمد | `edit_required` |
| اللهجة | تجارية واضحة بلا مبالغة تسويقية | `edit_required` |
| الروابط | روابط تتبّع رسمية فقط | `edit_required` |
| المصدر | أي ادعاء له مصدر — لا إجابة بلا مصدر | `edit_required` |

النسخة تمرّ فقط إذا اجتازت كل بنود `blocked`؛ بنود `edit_required` تُصحَّح ثم يُعاد التقديم.
