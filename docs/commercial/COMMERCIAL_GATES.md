# البوابات التجارية الخمس / The Five Commercial Gates

## 1. الغرض / Purpose

**العربية:** خمس بوابات تجارية متسلسلة. لا يمكن الانتقال إلى المرحلة التالية قبل اجتياز البوابة الحالية. هذا يمنع بناء منتجات قبل وجود طلب حقيقي، ويحمي دياليكس من الاستثمار في وهم.

**English:** Five sequential commercial gates. You cannot advance to the next stage until the current gate passes. This prevents building products before real demand exists and protects Dealix from investing in an illusion.

```text
GATE 1  Manual Market Penetration
   |
GATE 2  Paid Pilot
   |
GATE 3  Proof Pack
   |
GATE 4  Expansion
   |
GATE 5  Repeatability  -->  ONLY HERE: build a module or SaaS feature
```

## 2. البوابة 1 — اختراق السوق اليدوي / Gate 1 — Manual Market Penetration

| الحقل / Field | التفصيل / Detail |
|---|---|
| معايير الدخول / Entry criteria | عرض واضح وقائمة جمهور أولية / A clear offer and an initial audience list |
| معايير الخروج / Exit criteria | إيقاع يومي ثابت لمدة أسبوعين / A steady daily cadence sustained for two weeks |
| الأدلة المطلوبة / Evidence required | 10 تواصلات/يوم، 5 متابعات/يوم، محادثة شريك/يوم، منشور مؤسس/يوم، بطاقة أداء يومية / 10 touches/day, 5 follow-ups/day, 1 partner conversation/day, 1 founder post/day, a daily scorecard |
| تصنيف الحقيقة / Truth label | Observed |

كل تواصل يدوي وبموافقة بشرية. لا واتساب بارد، لا سحب بيانات. / Every touch is manual and human-approved. No cold WhatsApp, no scraping.

## 3. البوابة 2 — التجربة المدفوعة / Gate 2 — Paid Pilot

| الحقل / Field | التفصيل / Detail |
|---|---|
| معايير الدخول / Entry criteria | عميل مهتم من البوابة 1 / An interested lead from Gate 1 |
| معايير الخروج / Exit criteria | أول دفعة أو التزام مكتوب + نطاق متفق عليه / First payment OR written commitment + agreed scope |
| الأدلة المطلوبة / Evidence required | نطاق واضح، 10 عملاء محتملين أو سير عمل واحد، جدول تسليم / Clear scope, 10 leads or one workflow, a delivery timeline |
| تصنيف الحقيقة / Truth label | Payment-confirmed أو Client-confirmed / Payment-confirmed or Client-confirmed |

العرض المربوط: Revenue Proof Sprint 499 أو Data-to-Revenue Pack 1,500. / Wired offer: Revenue Proof Sprint 499 or Data-to-Revenue Pack 1,500.

## 4. البوابة 3 — حزمة الإثبات / Gate 3 — Proof Pack

| الحقل / Field | التفصيل / Detail |
|---|---|
| معايير الدخول / Entry criteria | تجربة مدفوعة قيد التنفيذ / A paid pilot in progress |
| معايير الخروج / Exit criteria | Proof Pack مُسلَّم خلال 48 ساعة للتجربة / Proof Pack delivered within 48h for a pilot |
| الأدلة المطلوبة / Evidence required | مصدر + مالك + موافقة + أثر أدلة + خطوة تالية / Source + owner + approval + evidence trail + next action |
| تصنيف الحقيقة / Truth label | Observed + Estimate (موسومة) / Observed + Estimate (labelled) |

يجب أن يجتاز Proof Pack بطاقة جودة التسليم بـ 8/10. / The Proof Pack must pass the Delivery QA scorecard at 8/10.

## 5. البوابة 4 — التوسّع / Gate 4 — Expansion

| الحقل / Field | التفصيل / Detail |
|---|---|
| معايير الدخول / Entry criteria | Proof Pack مقبول من العميل / A client-accepted Proof Pack |
| معايير الخروج / Exit criteria | قيمة مؤكَّدة + توصية Sprint + طلب إحالة + تعريف بشريك / Value confirmed + Sprint recommendation + referral ask + partner intro |
| الأدلة المطلوبة / Evidence required | تأكيد كتابي للقيمة، توصية موثّقة، إحالة أو تعريف مسجّل / Written value confirmation, a documented recommendation, a logged referral or intro |
| تصنيف الحقيقة / Truth label | Client-confirmed |

العرض المربوط: Growth Ops Monthly 2,999/mo أو Support OS Add-on 1,500/mo. / Wired offer: Growth Ops Monthly 2,999/mo or Support OS Add-on 1,500/mo.

## 6. البوابة 5 — القابلية للتكرار / Gate 5 — Repeatability

| الحقل / Field | التفصيل / Detail |
|---|---|
| معايير الدخول / Entry criteria | عملاء توسّع متعددون / Multiple expansion clients |
| معايير الخروج / Exit criteria | 3 سير عمل متكررة بنفس الألم والمشتري والشكل / 3 repeated workflows, same pain, same buyer, same proof format |
| الأدلة المطلوبة / Evidence required | 3 Proof Packs موثّقة بنمط مطابق / 3 documented Proof Packs with a matching pattern |
| تصنيف الحقيقة / Truth label | Repeated workflow |

## 7. القاعدة الصارمة / The Hard Rule

**العربية:** فقط بعد اجتياز البوابة 5 نبني وحدة برمجية أو ميزة SaaS. قبل ذلك، أي بناء منتج هو مخاطرة غير مبررة. التكرار المثبت — لا الحماس — هو إذن البناء.

**English:** Only after Gate 5 passes do we build a module or SaaS feature. Before that, any product build is an unjustified risk. Proven repetition — not enthusiasm — is the permit to build.

أي طبقة أكبر من العروض المربوطة: Roadmap — not wired to checkout / خارطة طريق — غير مربوط بالدفع.

> Estimated outcomes are not guaranteed outcomes / النتائج التقديرية ليست نتائج مضمونة.
