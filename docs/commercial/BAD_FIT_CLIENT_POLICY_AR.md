# سياسة العميل سيّئ الملاءمة — Dealix

المبدأ: العميل الباحث عن spam أو ضمانات أو scraping **ليس فرصة، بل خطر**. نتعامل معه
بـ**استبعاد + تسجيل**، لا بمحاولة إقناعه بتجاوز الحوكمة. أي تنازل عن السلامة لإرضائه = فشل.

> يستند إلى [`WALK_AWAY_RULES_AR.md`](./WALK_AWAY_RULES_AR.md) (WA-01..10) و
> [`DISQUALIFICATION_RULES_AR.md`](./DISQUALIFICATION_RULES_AR.md).

## علامات العميل سيّئ الملاءمة
- يطلب **إرسالاً جماعياً/رسائل باردة** بالنيابة (يخالف OBJ-008 وسياسة الإرسال).
- يطلب **ضمان مبيعات/نتائج مضمونة** أو لغة «بدون مخاطرة».
- يطلب **scraping** أو جمع بيانات مشبوه.
- **يرفض الموافقة** أو **أساسيات الخصوصية**.
- يريد **عملاً مخصّصاً ثقيلاً مجاناً** قبل أي اعتماد أو دفع.
- **لا قدرة على الدفع** أو **لا صاحب قرار** أو **لا فرص متكرّرة**.

## الإجراء: استبعاد + تسجيل (disqualify + log)
1. **استبعاد:** لا تأهيل، لا عرض، لا إرسال. نوقف المسار التجاري فوراً.
2. **تحديد السبب:** ربط بـ `WA-NN` المناسب من قائمة الانسحاب.
3. **تسجيل (log):** نوثّق في سجل الفرصة: المعرّف، السبب، التاريخ، القرار.
4. **تحديد المرحلة:**
   - `do_not_contact` لطلبات spam/scraping/رفض خصوصية صريح.
   - `lost` (مع سبب) لعدم الملاءمة التجارية (لا دفع/لا قرار/لا فرص متكرّرة).
   - `nurture` إن كان الرفض ظرفياً وقابلاً للتغيّر لاحقاً (مثل غياب صاحب قرار حالياً).

## صياغة التعامل (آمنة، بلا تجريح)
«نقدّر اهتمامكم. ما نقدّمه يعمل بموافقتكم ووفق سياسة واضحة للإرسال والخصوصية، ولا
يشمل الإرسال الجماعي أو ضمان النتائج. لذلك لا نستطيع المضي في هذا الطلب.» — واضح،
مهذّب، بلا عبارات محظورة وبلا وعود زائفة.

## ممنوعات صارمة
- تفعيل إرسال جماعي/بالنيابة لإرضاء العميل (`send_enabled=false` دائماً).
- قبول scraping أو تجاوز الخصوصية.
- استخدام عبارة محظورة (نضمن/نتائج مضمونة/بدون مخاطرة/10x…) للإبقاء على العميل.
- تقديم عمل مخصّص ثقيل بلا اعتماد ودفع.
- تسجيل أي PII للأفراد؛ نكتفي بالقطاع والدور وسبب الاستبعاد.

## «مثال توضيحي»
- «مثال توضيحي»: Nexus IT Solutions يطلب حملة باردة جماعية → استبعاد (WA-04) + log + `do_not_contact`.
- «مثال توضيحي»: عميل يطلب «ضمان مضاعفة المبيعات» → استبعاد (WA-05) + log + `lost`.
- «مثال توضيحي»: Growth Labs SA بلا قدرة دفع حالياً → استبعاد (WA-03) + log + `nurture`.

## قائمة تحقّق
- [ ] رُصدت علامة عدم ملاءمة واضحة.
- [ ] استبعاد فوري + ربط بـ `WA-NN`.
- [ ] تسجيل السبب والمرحلة (`lost`/`nurture`/`do_not_contact`).
- [ ] لا تنازل عن السلامة، لا عبارات محظورة، لا PII.
