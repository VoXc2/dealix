# Delivery checklist — Lead Intelligence Sprint (Dealix)

Internal use. Customer-facing deliverables use Arabic naming in the proposal; this checklist is bilingual for the delivery team.

## Phase A — Intake (Day 0–1)

- [ ] Signed SOW + DPA on file (`docs/commercial/templates/SCOPE_SPRINT_SAR.md` filled).
- [ ] Data received: CSV/XLSX + column mapping confirmed.
- [ ] **Forbidden requests scan** (no purchased lists, no scraping, no cold WhatsApp / LinkedIn automation).
- [ ] Tier1 / source tags assigned per batch policy (`source_registry`).

## Phase B — Processing (Day 1–7)

- [ ] Import preview row counts + null rates documented.
- [ ] Dedupe hints generated; duplicates reviewed with client sign-off on merge rules.
- [ ] Scoring model version noted in appendix (internal).
- [ ] Top 50 + Top 10 actions drafted in executive report template.
- [ ] Up to **20 outreach drafts** prepared — **draft-only**, Arabic/English as agreed.

## Phase C — Outputs (Day 7–10)

- [ ] Executive report delivered (Markdown/PDF as agreed).
- [ ] Mini CRM board stages agreed (lead → qualified → opportunity → next step).
- [ ] Compliance blockers section reviewed with founder.
- [ ] Handoff call 45–60 min; recording / notes stored per client policy.

## Phase D — Close

- [ ] Invoice / payment milestone checked (ops).
- [ ] Upsell path: Pilot Conversion Sprint or Monthly RevOps OS (`DEALIX_REVOPS_PACKAGES_AR.md`).
