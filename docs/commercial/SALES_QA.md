# ضمان جودة المبيعات — Sales QA

> **قاعدة ذهبية:** أي محادثة تحتوي ضماناً، أو ادعاء ROI غير مدعوم، أو إثباتاً
> زائفاً، أو خطوة تالية غير واضحة — تحتاج **تصحيحاً** قبل المتابعة.

## النطاق / Scope

هذا الملف يحدد كيف نراجع جودة المحادثات التجارية أسبوعياً. نراجع **5 محادثات
كل أسبوع** بعينة عشوائية، ونصحّح ما يخالف المعايير. الهدف اتساق ونزاهة البيع —
لا مراقبة شكلية.

## وتيرة المراجعة

- 5 محادثات أسبوعياً (مكالمات، اجتماعات، أو سلاسل رسائل مُوافَق عليها).
- اختيار عشوائي، لا انتقائي — لا تُختار «أفضل» المحادثات فقط.
- المراجعة يقودها المؤسس أو من ينوب عنه، وتُوثَّق في ورقة التقييم أدناه.

## قائمة فحص المحادثة — Checklist

| # | السؤال | المعيار |
|---|--------|---------|
| 1 | هل بدأنا بالألم (pain)؟ | الافتتاح يتناول مشكلة العميل، لا وصف المنتج |
| 2 | هل عرضنا Proof؟ | إثبات حقيقي أو نمط case-safe — لا ادعاء بلا مصدر |
| 3 | هل ضيّقنا النطاق (scope)؟ | عرض درجة واحدة محددة، لا «كل شيء ممكن» |
| 4 | هل طرحنا سؤال إغلاق؟ | دعوة واضحة لقرار أو خطوة |
| 5 | هل سجّلنا خطوة تالية؟ | إجراء تالٍ محدد بمالك وتاريخ |
| 6 | هل وعدنا بشيء خطير؟ | **لا** ضمانات، **لا** ROI كحقيقة، **لا** إثبات زائف |

## الوعود الخطيرة — ربطها بالمحظورات الـ11

«الوعد الخطير» في البند 6 ليس رأياً ذوقياً — هو مخالفة مباشرة للمحظورات
غير القابلة للمساومة:

| الوعد الخطير في المحادثة | المحظور المُنتهَك |
|--------------------------|--------------------|
| «نضمن لك نتيجة / صفقات» | No guaranteed-outcome claims |
| «ستحقق ROI بنسبة كذا» (كحقيقة) | No guaranteed ROI claims |
| عرض إثبات لا مصدر له أو مُجمَّل | No fake proof / No sourceless answers |
| «سنرسل نيابةً عنك» دون موافقة | No live send / Human approval for all external actions |
| «نخصم من بطاقتك الآن» دون تأكيد | No live charge |
| الترقية لدرجة أعلى دون إثبات الدرجة السابقة | No stage advance without verified evidence |
| إرسال مخرَج AI للعميل دون مراجعة | No client-facing AI output without QA |

أي ظهور لأحد هذه في محادثة → تصحيح فوري + تدوين في عمود الإجراء التصحيحي.

## ما يحدث بعد المراجعة

- وعد خطير مكتشف → تصحيح فوري + رسالة توضيح للعميل إن لزم + تحديث النص.
- نمط متكرر عبر محادثات → يُحدَّث سكربت البيع، لا فرد واحد فقط.
- محادثة نموذجية جيدة → تُحفظ كمرجع تدريب داخلي.
