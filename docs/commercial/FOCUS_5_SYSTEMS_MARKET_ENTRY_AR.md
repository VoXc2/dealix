# الأنظمة الخمسة الجوهرية — مدخل السوق

Dealix لا يبيع أدوات؛ يبيع **أنظمة تشغيل** بمخرجات ومعايير قبول واضحة. واجهة
السوق خمسة أنظمة فقط، وخلفها يعمل المحرّك الداخلي (40 نظام + 50 سبرنت + ذكاء
الاحتياج) لاختيار أفضل عرض لكل شركة.

## الأنظمة الخمسة

1. **نظام تشغيل الإيرادات** (`revenue-operating-system`)
   - الوعد: تنظيم رحلة العميل وكشف أين تضيع الإيرادات.
   - المخرجات: خريطة تسرب + عملية مبيعات + لوحة عملاء موحّدة.
2. **نظام القيادة التنفيذية** (`executive-command-os`)
   - الوعد: صورة يومية واضحة للمؤسس: ماذا يحدث، ما الأولوية، أين الخطر.
   - المخرجات: لوحة مؤشرات + أولويات يومية + مراجعة أسبوعية + توقع إيرادات.
3. **نظام استرجاع المتابعات** (`follow-up-recovery-os`)
   - الوعد: التقاط كل عميل لم تتم متابعته وإعادة تفعيل الصفقات المهجورة.
   - المخرجات: تسلسل متابعة + سرعة رد + استرجاع صفقات + تقرير استرجاع.
4. **نظام عملاء واتساب** (`whatsapp-client-os`)
   - الوعد: تحويل فوضى واتساب إلى قناة منظمة.
   - المخرجات: صندوق منظم + قوالب معتمدة + إدارة مواعيد + تقارير.
5. **نظام العروض والإثبات** (`proposal-proof-os`)
   - الوعد: عروض تُقنع وتُثبت القيمة برقم وقصة نجاح، لا بوعود.
   - المخرجات: عروض جاهزة + تسعير منضبط + حزمة إثبات + مكتبة اعتراضات.

## مبادئ التسعير

- كل نظام يبدأ بسبرنت قصير محدد المخرجات (Starter) لتقليل المخاطرة على العميل.
- التسعير المبدئي ثابت حسب التعقيد: منخفض 2,500 / متوسط 5,000 / مرتفع 9,000 ر.س.
- لا التزام طويل قبل إثبات القيمة.

## ممنوعات تجارية

- لا وعود مضمونة («نضمن/نضاعف»).
- لا نطاق مفتوح في العروض.
- لا إرسال أو اتصال آلي — كل تواصل باعتماد المؤسس.

## بيانات مرتبطة

- `data/business_os_catalog/systems.yaml` — الأنظمة الداخلية الأربعون.
- `data/business_os_catalog/system_pricing.yaml` — التسعير.
- `src/marketing/catalog.ts` — النسخة العامة (5 أنظمة فقط).
