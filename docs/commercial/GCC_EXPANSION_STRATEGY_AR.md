# GCC Expansion Strategy — استراتيجية التوسّع الخليجي

> المبدأ: **Saudi-first, GCC-ready.** نبدأ بالسعودية لأن هوية Dealix والسياق واللغة مبنية عليها، ثم نوسّع بمنطق GCC دون تبديل الهوية.
>
> **مبني على:** التموضع الحالي في `src/pages/LandingPage.tsx` ("للسعودية والخليج")، و`company_os/marketing/one_pagers/one_pager_arabic.md` ("Saudi AI Governance")، وحوكمة `company_os/governance/pdpl_checklist.md` (PDPL/SDAIA).

| المرحلة      | التموضع الخارجي                                            |

## 2. لماذا السعودية أولاً؟

1. الهوية والسياق واللغة مبنية على السوق السعودي.
2. الحوكمة جاهزة لـ PDPL/SDAIA (`governance/pdpl_checklist.md`).
3. قاعدة الـ prospects الحالية كلها سعودية (`revenue/prospects.csv`).
4. أسرع دورة شراء في القطاعات المستهدفة (وكالات تسويق، تدريب، B2B).

> **قاعدة category design:** لا نستهدف "السعودية" ككتلة واحدة، بل **قطاع × مدينة × قناة × Mission** (راجع `reports/outreach/GCC_TARGETING_REVIEW.md`).

## 4. اعتبارات لكل دولة

| الدولة    | اللغة الأساسية | حماية البيانات (إطار)            | ملاحظة دخول                                  |
| --------- | -------------- | -------------------------------- | -------------------------------------------- |
| السعودية  | العربية        | PDPL (SDAIA)                     | جاهز — قاعدة الانطلاق                         |
| الإمارات  | عربي + إنجليزي | PDPL اتحادي + مناطق حرة (DIFC/ADGM) | محتوى ثنائي اللغة؛ احترم أطر المناطق الحرة    |
| قطر       | عربي + إنجليزي | PDPPL                            | ميزانيات أعلى؛ بيع علائقي                     |
| الكويت    | العربية        | إطار ناشئ                        | شبكات علاقات قوية                            |
| البحرين   | عربي + إنجليزي | PDPL (بحريني)                    | بوابة شراكات إقليمية                          |
| عُمان     | العربية        | إطار ناشئ                        | نمو تدريجي عبر الإحالات                       |

> **التحقق القانوني قرار مؤسس** — لا يقدّم AI استشارة قانونية مباشرة (خط أحمر #6 في `governance/agent_permissions.md`). أي توسّع جديد يمرّ بفحص امتثال قبل الإطلاق.

## 5. ما الذي يُعاد استخدامه عبر الدول؟

| ثابت عبر GCC                          | يُحلّى محلياً لكل دولة/مدينة             |
| ------------------------------------- | --------------------------------------- |
| الأعمدة الأربعة (Radar/AI Team/Portal/Proof) | أمثلة القطاعات والمدن في الرسائل        |
| الـ 8 Missions                        | اللهجة/الأمثلة (سعودي ← خليجي ← إنجليزي) |
| الحوكمة وسير الموافقة                  | إطار حماية البيانات لكل دولة            |
| نموذج التسعير (anchors)               | عتبة التسعير حسب السوق                  |
| سياسة الإرسال (email-first)           | suppression list منفصلة لكل سوق/دومين   |

## 6. تسلسل التنفيذ (لا تفتح كل GCC دفعة واحدة)

```txt
1. أثبت السعودية: قطاع × مدينة × Mission واحدة بدليل (Proof Pack).
2. كرّر النمط في الإمارات (نفس القطاع، مدينة جديدة).
3. وسّع للقطر/الكويت بعد ثبات التسليم والامتثال.
4. البحرين/عُمان عبر الشراكات والإحالات.
```

**قاعدة:** لا ننتقل لمرحلة جديدة قبل وجود Proof Pack واحد على الأقل من المرحلة السابقة.
