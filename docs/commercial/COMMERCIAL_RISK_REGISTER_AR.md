# سجل المخاطر التجارية — Dealix

> يُراجَع دورياً عبر [`reports/commercial/COMMERCIAL_RISK_REVIEW.md`](../../reports/commercial/COMMERCIAL_RISK_REVIEW.md).
> مستويات الخطر: `low`/`medium`/`high` (التسعير ومعالجة البيانات والإرسال الخارجي ≥ `high`).

المبدأ: نوثّق المخاطر التجارية ونخفّفها استباقياً. أي مخاطرة تمسّ السلامة (عبارات
محظورة، اختلاق، PII، إرسال غير مُعتمد) تُعامل كأولوية قصوى وإيقاف فوري.

## آلية الإدارة
1. أي مخاطرة جديدة تُضاف بمعرّف `CR-NN` مع تخفيف ومالك.
2. مخاطر السلامة (CR-02..05، CR-14) = إيقاف فوري عند أي إشارة خرق.
3. تُراجَع المخاطر العالية كل أسبوع، والباقي شهرياً.
4. كل تخفيف مرتبط بمستند سياسة أو اختبار قابل للتحقّق.

## «مثال توضيحي»
- «مثال توضيحي»: CR-08 — لو طلب Nexus IT Solutions إرسالاً جماعياً، يُفعّل الاستبعاد ويُسجّل السبب.
- «مثال توضيحي»: CR-07 — لو طلب Growth Labs SA بنوداً خارج النطاق، تُحوَّل إلى change request بموافقة.
