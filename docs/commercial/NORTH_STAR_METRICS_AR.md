# مقاييس North Star — Dealix (شركة تشغيل محكوم للإيراد والذكاء الاصطناعي)

## نجم الشمال

**Governed Value Decisions Created** — عدد القرارات التشغيلية أو الإيرادية التي
اتُّخذت بـ **مصدر واضح، وموافقة واضحة، ودليل موثّق، وأثر قابل للقياس**.

هذا المقياس مبكّر-مناسب لأنه يجمع: الإيراد، والحوكمة، والذكاء الاصطناعي، والإثبات،
وقيمة العميل في رقم واحد — بدل مقياس سطحي مثل «عدد المستخدمين».

### أمثلة على «قرار قيمة محكوم»

- قرر العميل متابعة 7 حسابات عالية القيمة.
- قرر العميل إيقاف workflow خطر.
- قرر العميل عدم السماح برسالة AI خارجية بلا موافقة.
- قرر العميل تحويل workflow إلى Sprint.
- قرر العميل دفع diagnostic.
- قرر العميل الانتقال إلى retainer.

## قاعدة العدّ

يُحتسب القرار ضمن North Star فقط عند توفّر **الشروط الأربعة**:

| الشرط | الدليل المطلوب |
|-------|----------------|
| مصدر واضح | `source_ref` غير فارغ |
| موافقة واضحة | `approval_id` مسجّل عبر Approval Center |
| دليل موثّق | حدث في سجل الأدلة (Evidence Ledger) أو Proof Pack |
| أثر قابل للقياس | قيمة في `value_os` value ledger أو انتقال CEL مسجّل |

**الربط مع value ledger:** القرار الذي يصل إلى أثر مالي يُسجَّل في `value_os`،
ولا يُعترف بإيراد إلا بعد `invoice_paid` (`CEL7_confirmed`) — انظر
[آلة الحالة CEL](COMMERCIAL_EVIDENCE_STATE_MACHINE.md).

## جدول مقاييس مساندة (تشغيل داخلي، ليست وعوداً للعميل)

| المحور | المقياس | تعريف مختصر |
|--------|---------|-------------|
| Revenue | تحويل Sprint → Retainer | عملاء أكملوا Sprint ووقّعوا ريتينر |
| Revenue | هامش جروس الخدمات | تكلفة التسليم مقابل السعر — لا ادعاء إيراد بدون `invoice_paid` |
| Delivery | وقت التسليم | أيام من بداية الجلسة حتى تقرير معتمد |
| Delivery | درجة QA | بوابات QA مكتملة / مطلوبة |
| Product | تكرار الـ workflow | مسارات تكررت 3+ مرات (إشارة بوابة G7) |
| AI Quality | تغطية المصدر | نسبة الإجابات/المخرجات مع `source_id` |
| Governance | حوادث PII | عدد التسريبات المؤكدة — الهدف 0 |
| Governance | موافقات مسجّلة | نسبة الأفعال الخارجية بـ `approval_id` |
| Commercial | بوابات مجتازة | G1–G7 «تم / لم يتم» |

## ربط التنفيذ

- العدّ والملخص: `GET /api/v1/board-decision-os/overview`.
- أحداث الأدلة وانتقالات CEL: `POST /api/v1/evidence/events`.
- value ledger: وحدات `auto_client_acquisition/value_os/`.

الأرقام أهداف داخلية قابلة للمعايرة، وليست وعوداً للعميل.
