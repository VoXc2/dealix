# Daily First Customer Loop

> 4 short windows. Same time every day. Discipline > heroics.

## 9:00 AM — Standup

Open `docs/FIRST_CUSTOMER_LIVE_BOARD.md` and answer these 6 questions:

1. Who did I message yesterday? (count)
2. Who replied?
3. How many Mini Diagnostics are due today?
4. How many pilots are offered or ready to be offered?
5. How many Proof Packs are due in the next 48 hours?
6. What was blocked (compliance / safety) yesterday?

If any answer is "I don't know" → that's a board hygiene issue.
Update the board first, then continue.

## 12:00 PM — Follow-up window (≤ 30 min)

- Reply to interested prospects (do not push the uninterested).
- Collect any missing intake fields for Mini Diagnostics in flight.
- Send any Mini Diagnostic where all 10 intake answers arrived.

## 4:00 PM — Closing window (≤ 45 min)

- Convert interest → pilot offer per `docs/PILOT_499_OFFER.md`.
- Send manual payment fallback instructions per `docs/MANUAL_PAYMENT_FALLBACK.md`.
- Mark `paid` only after proof of payment (bank screenshot / STC Pay reference).
- Schedule day-1 of delivery for any newly-paid pilot.

## 6:00 PM — Scorecard

Append a line to `docs/DAILY_SCORECARD.md` (create on first run):

```
Date: 2026-MM-DD
Messages sent:                 N (cap 10/day in week 1)
Replies:                       N
Diagnostics requested:         N
Diagnostics sent:              N (with 24h-from-intake compliance)
Pilot offers:                  N
Payments / commitments:        N
Proof Packs delivered:         N
Risks blocked (compliance):    N
Best message variant:          AR / EN — short note why
Best segment so far:           e.g. "B2B agencies in Riyadh"
Tomorrow's focus:              one sentence
```

## Weekly review (Friday 5:00 PM)

Write `docs/WEEK_<n>_REVIEW.md` answering:

1. How many warm DMs sent this week? Reply rate?
2. How many Mini Diagnostics delivered? On-time rate?
3. How many pilots paid?
4. How many Proof Packs delivered?
5. How many risks blocked?
6. Best segment / best message — what to double down on?
7. Worst bottleneck — what to remove?
8. Next experiment (just one) for next week.

This becomes input for the eventual Executive Growth OS upsell — the
weekly cadence IS the product.

## Hard rules

| Rule | Reason |
| --- | --- |
| Cap 10 new DMs / day in week 1 | volume comes after first paid pilot, not before |
| 24h SLA on Mini Diagnostics | trust = on-time delivery |
| 7-day SLA on Proof Packs | same |
| One outbound channel per prospect | LinkedIn manual is the default; switch only if they reply elsewhere |
| Stop after 3 follow-ups | force `nurture`, never push |
| Friday review every week | even if outcome is "no progress" |
| No live external sends | all approval-first / draft-only |

## What "successful daily loop" looks like

Week 1:
- 10 warm DMs sent (manual)
- 2-3 replies
- 1-2 Mini Diagnostics delivered
- 0-1 paid pilots

Week 2:
- 10 more warm DMs (different segments)
- continued follow-ups from week 1
- 1-2 Mini Diagnostics + 1 pilot in delivery

Week 3:
- First Proof Pack delivered
- First Executive Growth OS upgrade conversation

This is the safe pace. Going faster usually means cutting corners on
intake, which kills proof quality, which kills the upsell.
