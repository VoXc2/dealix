## Release theme

**Commercial finalization & honest revenue.** The platform was
already feature-complete at v3.7.0; v3.8.x closes the
revenue-funnel breaks the commercial-readiness audit surfaced and
verifies every shipped claim end-to-end via a local smoke deploy.

## What's in this release

### Revenue flow unblocked (T13a)

`POST /api/v1/payment-ops/invoice-intent` now returns a real
Moyasar hosted-checkout URL when `MOYASAR_SECRET_KEY` is set and
`method == "moyasar_test"`. The landing checkout page redirects to
it. Without keys, the historic manual-evidence flow still works.
Constitution's `no_live_charge` / `no_fake_revenue` gates preserved.

### ZATCA-shaped invoice PDF + receipt email (T13b)

`dealix/billing/invoice_pdf.py` renders a bilingual invoice with a
TLV-encoded QR-code payload. `api/routers/billing_invoices.py`
serves it at `/api/v1/billing/invoices/{id}?t=<sig>` with signed-
link auth so the email link works before login. The Moyasar webhook
fires `_send_receipt_email_best_effort()` on every `paid` event.

### Honest comparison pages (T13c)

Six comparison pages (`landing/comparisons/{hubspot,salesforce,gong,
salesloft,outreach,apollo}.html`) re-shipped with conservative
language for Tabby/Tamara/TAP ("pending merchant onboarding"),
Hijri dual-calendar ("Roadmap Q3 2026"), WhatsApp Cloud
("Cadence engine live; Meta template approval required"), and
Saudi data residency ("Available on Enterprise contract; default
EU/US"). Each page now carries a "Last validated 2026-05-12"
footer. `/api/v1/billing/gcc/health` splits public-available from
operator-full.

### Vertical→handler coverage enforced (T13d)

`tests/unit/test_vertical_handler_coverage.py` +
`scripts/dev/check_vertical_truth.py` make claimed-but-unimplemented
agents impossible to ship.

### Sales kit (T13e)

`dealix/marketing/brochure_pdf.py` renders a single-page bilingual
sales brochure per vertical. `/api/v1/marketing/brochure/{vertical}.pdf`
serves it. `docs/sales/` ships `cold_outreach_{ar,en}.md` (3-touch
outbound sequence) + `objection_handling.md` (12 questions with
honest answers).

### Operational artefacts (T13f, T14)

- `docs/ops/vendor_cost_sheet.md` — every env var → vendor URL + cost.
- `docs/ops/go_live_checklist.md` — 30-item checklist.
- `docs/ops/go_live_one_pager.md` — single-screen CEO runbook.
- `docs/ops/key_paste_order.md` — paste-order .env template.
- `docs/ops/troubleshooting.md` — top 10 issues × diagnosis × fix.
- `docs/sales/first_customer_in_7_days.md` — the narrowest GTM playbook.
- `scripts/ops/post_deploy_smoke.sh` — green/red dashboard after every deploy.

### Smoke-deploy bug fixes (v3.8.1 patch)

A local end-to-end FastAPI smoke caught **5 production-grade bugs**
the unit tests didn't catch:

1. **Manifest path off by one parent** — `/api/v1/skills` returned 0.
2. **KSA-phone PII regex never matched** — `\b\+966` doesn't fire.
3. **Agent-builder silently normalised mixed-case ids** — typos passed.
4. **Feature-flag env fallback unreachable** — PostHog `False` shadowed env.
5. **Vertical lead-form widget read `f.name`; configs use `f.id`**.

All five fixed with regression tests. 101/101 unit tests pass.

## Breaking changes

None. All v3.7.0 endpoints + env vars still work. The behavioural
changes:

- `/api/v1/billing/gcc/health` now returns `{available, configured_count, *_configured}` (back-compat booleans preserved).
- Agent-builder rejects mixed-case ids that previously normalised silently — fix any customer manifests using `MIX-Case` ids by lowercasing them.
