# سياسة تجربة المستخدم في واتساب (UX Policy)
## رسائل قصيرة، خيارات واضحة، نبرة B2B سعودية

> **الغرض:** يحدد هذا المستند معايير تجربة المستخدم المُلزِمة لكل رسائل واتساب في Dealix — طول الرسالة، أسلوب الخيارات، نبرة الخطاب، وما هو ممنوع من ناحية UX.

---

## 1. مبادئ UX الأساسية

| المبدأ | التعريف |
|---|---|
| **قصير ومباشر** | رسالة واحدة = فكرة واحدة. لا فقرات طويلة |
| **خيارات لا أسئلة مفتوحة** | بدلًا من "ماذا تريد؟" → قائمة مرقّمة واضحة |
| **نبرة B2B سعودية** | محترمة، مباشرة، بدون تكلّف، عربية فصحى مبسّطة |
| **لا بوت عام** | Dealix ليس مساعدًا عامًا — كل رسالة لها هدف محدد |
| **"ما أعرف" دائمًا خيار** | لا يُترك العميل عالقًا بدون مخرج |

---

## 2. معايير طول الرسائل

| نوع المحتوى | الحد المسموح | إذا تجاوز الحد |
|---|---|---|
| رسالة ترحيب / سؤال | ≤ 300 حرف | اختصر أو قسّم |
| ملخص توصية (بطاقة) | ≤ 600 حرف | انقل التفاصيل للبوابة |
| رسالة إخطار (بوابة/تقرير) | ≤ 200 حرف | رابط البوابة للتفاصيل |
| عنوان البطاقة | ≤ 80 حرف | إلزامي (محدّد في السكيمة) |

للتفاصيل الكاملة: `docs/whatsapp/WHATSAPP_MESSAGE_LENGTH_POLICY_AR.md`.

---

## 3. هيكل الرسالة المثالية

```
[سطر تمهيدي — فكرة واحدة]
[معلومة داعمة — جملة أو جملتان]
─────────────────────────────
[1] الخيار الأول
[2] الخيار الثاني
[3] ما أعرف — اقترح علي   ← دائمًا موجود في بطاقات التوصية
```

**لا bullets غير ضرورية. لا جمل اعتراضية طويلة. لا emoji مبالغ فيه.**

---

## 4. النبرة المطلوبة

### خصائص النبرة

| سمة | نعم | لا |
|---|---|---|
| العربية أولًا | "أهلًا، نساعدك في…" | "Hello, we help you with…" |
| مباشر ومحترم | "تحب نبدأ الفحص؟" | "هل تودّ البدء بإجراء الفحص؟" |
| دافئ لكن مهني | "حلو 👍" عند التأكيد | "ممتاز جدًا! رائع!" (مبالغة) |
| لا ضغط | "بدون أي ضغط" | "فرصة لا تفوتك!" |
| يعترف بعدم اليقين | "ما عندنا بيانات كافية بعد" | ادّعاء اليقين الزائف |
| لا ضمان ROI | "نستهدف تحسين X بناءً على Y" | "ضامنين لك X% زيادة" |

### مثال مقارنة

**غير صحيح:**
> "بناءً على خبرتنا الواسعة وآلاف العملاء، نضمن لك تحسين إيراداتك بنسبة 40% خلال شهر واحد فقط!! 🔥🔥"

**الصحيح:**
> "بناءً على ردودك، جاهزية إيرادك 52/100. نقترح البدء بتشخيص تسرب الإيراد (مستوى الدليل: client_reported). تحب التفاصيل؟"

---

## 5. قواعد الخيارات

1. **ترقيم واضح:** كل خيار يبدأ بـ[1], [2], [3]...
2. **خيارات محدودة:** 2-4 خيارات في الرسالة الواحدة (لا إرباك)
3. **"ما أعرف — اقترح علي" في بطاقات التوصية:** إلزامي دائمًا (انظر `templates.yaml`)
4. **"كلّمني بشري":** دائمًا قابل للوصول (ثابت عالمي)
5. **لا خيارات مخادعة:** الخيار يجب أن يصف بدقة ما يحدث بعده

---

## 6. Dealix ليس بوتًا عامًا

**ممنوع الرد على:**
- أسئلة غير مرتبطة بالإيراد/المتابعة/التشغيل
- طلبات تصحيح نصوص، ترجمات، كتابة إبداعية
- نصائح طبية، قانونية، مالية شخصية

**الرد الصحيح عند الخروج عن النطاق:**
```
"هذا الموضوع خارج تخصصنا في Dealix.
أنا هنا بس أساعدك في تحسين إيراد شركتك. 
تحب نكمل؟"
─────────────────────
[1] نعم نكمل
[2] كلّمني بشري
```

---

## 7. التعامل مع ردود غير متوقعة

| نوع الرد | الإجراء |
|---|---|
| رقم لا يطابق خيارًا | طلب التوضيح مرة واحدة، ثم dont_know_suggest |
| 3 ردود غير مفهومة | تسليم بشري تلقائي |
| سؤال خارج النطاق | رسالة "خارج نطاقنا" + خيارات |
| كلام مسيء | قطع المحادثة + تسليم بشري + سجل في الطابور |

---

## 8. Emoji — متى وكيف؟

| مقبول | ممنوع |
|---|---|
| 👋 في الترحيب | إفراط في كل رسالة |
| 👌 أو ✅ تأكيد إيجابي | emoji تحفيزية مبالغ فيها |
| 🎉 للمبروكات الحقيقية (صفقة مكتملة) | emoji في سياقات جدية (حجب دفع، انسحاب) |
| 👏 في رسائل التجديد | — |
