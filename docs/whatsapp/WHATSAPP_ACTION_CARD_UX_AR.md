# تجربة استخدام بطاقات الإجراء (Action Card UX)
## الخيارات المرقّمة، انتهاء الصلاحية، والموافقة

> **الغرض:** يصف هذا المستند تجربة التفاعل مع بطاقات الإجراء من منظور العميل — كيف يرى الخيارات، ما الذي يحدث عند انتهاء صلاحية البطاقة، وكيف تعمل آلية الموافقة.  
> مرجع السكيمة: `schemas/whatsapp_action_card.schema.json`.

---

## 1. كيف تبدو البطاقة للعميل؟

```
┌─────────────────────────────────────┐
│  الخطوة المقترحة: فحص الجاهزية     │
│  (عنوان ≤ 80 حرف)                  │
├─────────────────────────────────────┤
│  بناءً على إشارات ضعف المتابعة،    │
│  نقترح البدء بـفحص الجاهزية ثم     │
│  تشخيص تسرب الإيراد.               │
│  (الدليل: client_reported)          │
├─────────────────────────────────────┤
│  [1] أكمل الفحص                    │
│  [2] ما أعرف — اقترح علي  ⭐       │
└─────────────────────────────────────┘
```

⭐ خيار "ما أعرف — اقترح علي" يظهر دائمًا في بطاقات التوصية.

---

## 2. أنواع الخيارات في البطاقات

### أ. خيارات إجراء (Action Options)

تُقدم للعميل قرارات واضحة:

```
[1] ابدأ الفحص         → يُفعّل تدفق readiness_scan
[2] رتّبوا مكالمة      → human_handoff → book_call
[3] ابغى التفاصيل      → proposal_review (البوابة)
```

### ب. خيار "ما أعرف — اقترح علي" (إلزامي)

```
is_dont_know: true
label: "ما أعرف — اقترح علي"
value: "dont_know_suggest"
```

ينقل العميل لتدفق `dont_know_suggest` الذي يقترح الخطوة الأنسب بشكل ودي.

### ج. خيار التسليم البشري (دائمًا متاح)

```
label: "كلّمني بشري"
value: "human_handoff"
```

يُنشئ `HHO-xxxx` فورًا ويُغيّر حالة الجلسة لـ`handoff`.

## 3. ما الذي يحدث بعد اختيار العميل؟

```
العميل يرسل رقم الخيار (مثلًا: "1")
              │
              ▼
   النظام يُطابق الرقم مع value
              │
              ├─ قرار لا يحتاج موافقة → يُنفَّذ الخطوة التالية مباشرةً
              │
              └─ قرار يحتاج موافقة المؤسس (L4/L5)
                          │
                          ▼
                 البطاقة تنتقل لـ status: awaiting_approval
                 إشعار للمؤسس في طابور الموافقات
                          │
                          ▼
                 المؤسس يوافق → status: approved
                          │
                          ▼
                 الإرسال يتم يدويًا (send_enabled يُفعَّل يدويًا)
```

## 4. انتهاء صلاحية البطاقة (expires_at)

### لماذا تنتهي الصلاحية؟

- منع الموافقة على قرارات قديمة قد تغيّرت ظروفها
- حماية من ضغط البطاقات المتراكمة
- ضمان أن البيانات المستخدمة في التوصية حديثة

### مدد الصلاحية الشائعة

| نوع البطاقة | مدة الصلاحية المقترحة |
|---|---|
| `recommendation` | 7 أيام |
| `permission` | 5 أيام |
| `proposal` | 3 أيام |
| `proof_pack` | 7 أيام |
| `payment_handoff` | 3 أيام |
| `renewal` | 14 يومًا |
| `support_escalation` | 24 ساعة |

### ما الذي يحدث عند الانتهاء؟

```
expires_at يمر بدون موافقة
         │
         ▼
    status: expired
         │
         ▼
إشعار للعميل:
"انتهت صلاحية هذه البطاقة. تحب نُنشئ واحدة جديدة؟"
─────────────────────
[1] نعم، جدّد البطاقة
[2] لا، شكرًا (opt_out خفيف)
```

**البطاقة المنتهية لا تُستعاد** — تُنشأ بطاقة جديدة (`status: superseded` للقديمة).

## 5. تجربة الموافقة من منظور المؤسس

```
طابور الموافقات (approval_queue)
──────────────────────────────────
WAC-0002 | permission | TrainMe KSA
risk: medium | evidence: assumption
expires: 2026-06-08

[✅ اعتمد]  [❌ ارفض]  [📋 تفاصيل في البوابة]
```

بعد الموافقة:
```json
{
  "approved": true,
  "approved_by": "founder",
  "approved_at": "2026-06-03T...",
  "status": "approved"
}
```

## 6. البطاقات المتسلسلة (Card Chains)

بعض الإجراءات تُنتج بطاقة تالية تلقائيًا:

```
WAC-0001 (recommendation: readiness_scan) → approved
    │
    ▼
WAC-0002 (permission: read_only_crm)
    │
    ▼
[CAS-0002 يُنشأ بعد الفحص]
    │
    ▼
WAC-0003 أو WAC-proposal (service_recommendation)
```

كل بطاقة في السلسلة مستقلة — لها موافقتها ومدة صلاحيتها.

---

## 7. ما العميل يراه مقابل ما لا يراه

| يراه العميل في واتساب | لا يراه العميل |
|---|---|
| `title` + `summary` | `evidence_level` الكامل (يُذكر مختصرًا فقط) |
| الخيارات المرقّمة | `risk_level` الداخلي |
| "مستوى الدليل: {level}" | حقول `dry_run`/`send_enabled` |
| رابط البوابة (مُقنَّع) | `approved_by`/`approved_at` الداخلية |
| "صلاحية انتهت" | التفاصيل الفنية للسكيمة |
