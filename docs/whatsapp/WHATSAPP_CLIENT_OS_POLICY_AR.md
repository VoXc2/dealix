# سياسة نظام عملاء واتساب (WhatsApp Client OS Policy)

> الغرض: تحديد كيف يُستخدم واتساب داخل **نظام عملاء واتساب** (`whatsapp-client-os`): قناة **خدمة وتحويل بعد ظهور الاهتمام**، لا رسائل باردة. هذه السياسة تحكم التسليم والواجهة مع العميل.

المصدر المرجعي: `data/systems.json` (النظام `whatsapp-client-os`). راجع أيضًا `docs/site/WEBSITE_COPY_AND_IA_AR.md` للنسخة العلنية.

## المبدأ الأساسي

- واتساب يُستخدم كقناة **خدمة + تحويل** بعد أن يُبدي العميل اهتمامًا (استفسار، حجز، دعم، طلب).
- **ممنوع** أي رسائل واتساب باردة، أو رسائل جماعية، أو أتمتة spam، أو قوائم مشتراة.
- لا نرسل نيابة عن العميل رسائل واردة باردة؛ نبني المسارات والبطاقات والسياسة، والتشغيل بيد فريقه.
- لا أي وعد بنتيجة مضمونة في أي مخرج.

## ما الذي يقدّمه النظام (حزمة التسليم)

مطابقة لـ `delivery_pack_ar` في `systems.json`:
- خريطة مسارات واتساب (WhatsApp Flow Map)
- فحص الجاهزية (Readiness Scan)
- بطاقات الإجراء (Action Cards) — ما يحتاج ردًا الآن
- سياسة التحويل للإنسان (Human Handoff Policy)
- دليل الربط الآمن مع بوابة الخدمة
- المراجعة الأسبوعية لواتساب

## WhatsApp Business Platform / Cloud API

- **ليست شرطًا للبداية.** كثير من الشركات تبدأ برقم WhatsApp Business عادي ومسارات منظمة.
- منصة الأعمال و **Cloud API** مناسبة للتكاملات الأكبر (ربط ببوابة خدمة، أنظمة داخلية، حجم رسائل عالٍ مع قوالب معتمدة).
- **تنبيه مهم:** Cloud API لا توفّر **واجهة جاهزة (frontend)** بحد ذاتها — هي طبقة API فقط. أي واجهة تشغيل/دعم تحتاج بناءً منفصلًا أو أداة وسيطة. نوضح في الدليل **متى** يحتاجها العميل فعلًا، فلا يدفع تعقيدًا قبل أوانه.

## سياسة التحويل للإنسان (Human Handoff)

- كل مسار محادثة يحدّد بوضوح **متى يُحوّل العميل لإنسان** (طلب معقّد، اعتراض، سعر، شكوى، طلب خارج النطاق).
- بطاقة الإجراء تُبرز المحادثات التي تحتاج ردًا بشريًا الآن، مع أولوية وزمن استجابة مستهدف.
- الهدف: لا تُترك محادثة مهمة بلا مسار، ولا يُترك العميل في حلقة آلية عندما يحتاج إنسانًا.

## التحويل الآمن إلى بوابة الخدمة (Secure Portal Handoff)

- البيانات الحساسة (دفع، مستندات، تفاصيل عقد) لا تُدار داخل دردشة واتساب المفتوحة.
- عند الحاجة، يُحوّل العميل إلى **بوابة خدمة آمنة** عبر رابط مخصّص، ويبقى واتساب لطبقة التواصل والتحويل فقط.
- يوثّق دليل الربط الآمن: متى يُستخدم الرابط، وما البيانات التي تبقى داخل واتساب وما الذي ينتقل للبوابة.

## معايير القبول (Acceptance)

مطابقة لـ `acceptance_criteria_ar`:
- كل نوع محادثة له مسار وبطاقة إجراء.
- سياسة التحويل للإنسان واضحة ومطبّقة.
- أول مراجعة أسبوعية تم تسليمها.
- لا توجد محادثة مهمة بلا مسار.

## الممنوعات (تلخيص)

- لا رسائل واتساب باردة أو جماعية أو أتمتة spam.
- لا قوائم مشتراة ولا كشط بيانات خاصة.
- لا وعود مضمونة بالأرقام.
- لا إدارة بيانات حساسة داخل دردشة واتساب المفتوحة.
