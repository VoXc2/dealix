# نظام عملاء واتساب — الدليل التشغيلي الكامل
## WhatsApp Client OS — Full Operating Guide

---

## 1. الغرض

نظام عملاء واتساب هو **مساعد تدفق أعمال** وليس بوت محادثة عام.
يُستخدم واتساب فقط بعد موافقة صريحة أو علاقة عميل موجودة.

---

## 2. شروط الموافقة (ALL required)

واتساب يُستخدم فقط عندما يتوفر واحد على الأقل:

- [ ] رد بريد إلكتروني إيجابي (positive reply)
- [ ] إرسال نموذج (form submission)
- [ ] حجز اجتماع (booking confirmed)
- [ ] موافقة واتساب صريحة (explicit WhatsApp consent)
- [ ] علاقة عميل موجودة (existing client relationship)

**موافقة صريحة تعني:**
- العميل قدم رقم واتساب للتفاعل مع Dealix
- العميل وافق كتابياً على التواصل عبر واتساب
- العميل بدأ محادثة مع Dealix على واتساب

**لا يُستخدم واتساب أبداً عندما:**
- [ ] الاتصال من قائمة مشتراة
- [ ] الاتصال من بيانات مجمعة بدون إذن
- [ ] الاتصال بأرقام مرشدة بدون علاقة
- [ ] الاتصال بدون موافقة مسبقة

---

## 3. القواعد الأمنية

| القاعدة | الوصف |
|---|---|
| لا مفاتيح API في نص واتساب | حتى لو سأل العميل، لا نرسل مفاتيح API |
| لا أسرار في محادثات واتساب | الملفات الحساسة عبر البوابة الآمنة فقط |
| تحويل بشري متاح دائماً | "تحويل لمختص" متاح في كل رسالة |
| لا بوت محادثة عام | هذا ليس ChatGPT |

## 4. قالب الترحيب بعد الموافقة

```
مرحباً [الاسم]! 👋

أنا [المساعد] من Dealix.
أقدر أتحدث معك بالعربي أو الإنجليزي.

1️⃣ كيف نبدأ التشخيص

3️⃣ عرض دليل الأسعار

```

**أزرار:**
- [تشخيص] → يبدأ فحص الجاهزية
- [أدلة] → يعرض دليل الأسعار
- [مساعد] → "ما أعرف — اقترح علي"

---

## 5. "ما أعرف — اقترح علي"

هذا المفتاح الذهبي. في كل رسالة، نقدم خيار "ما أعرف — اقترح علي":

**أمثلة:**

"بخصوص سير العمل الحالي — ما أعرف:
- تقدر تبدأ بـ Quick Win Sprint (3-7 أيام)
- أو تشوف التشخيص الكامل أولاً

**ما أعرف — اقترح علي:**"

**أزرار:**
- [Quick Win] → عرض Quick Win Sprint
- [تشخيص] → بدء فحص الجاهزية
- [فهم أكثر] → شرح الخيارات

## 6. فحص الجاهزية (Readiness Scan)

### 6.1 الأبعاد التسعة

| البُعد | السؤال | النقاط |
|---|---|---:|
| تدفق العملاء | كيف يأتيك عملاء جدد؟ | 0-10 |
| نضج المتابعة | هل تتابع العملاء المحتملين بانتظام؟ | 0-10 |
| جاهزية البيانات | هل عندك بيانات عن العملاء المحتملين؟ | 0-10 |
| نضج التقارير | هل تقيس نتائج التسويق والمبيعات؟ | 0-10 |
| الإلحاح | هل المشكلة مؤجلة أم ملحة؟ | 0-10 |
| ملاءمة الميزانية | هل الميزانية متاحة للـ AI Operations؟ | 0-10 |
| الوصول لصانع القرار | هل你能 الوصول لصاحب القرار؟ | 0-10 |
| حساسية الامتثال | هل عندك متطلبات امتثال خاصة؟ | 0-10 |
| جاهزية الأتمتة | هل هناك عمليات متكررة قابلة للأتمتة؟ | 0-10 |

1. **بطاقة التوصية** — توصية منتج أو خدمة
2. **بطاقة الاعتماد** — طلب موافقة من المؤسس
3. **بطاقة الإذن** — طلب إذن من العميل
4. **بطاقة المقترح** — عرض مقترح تجاري
5. **بطاقة دليل الأسعار** — عرض دليل إثبات القيمة

8. **بطاقة تصعيد الدعم** — تحويل للدعم

## 9. قالب طلب الإذن

```
قبل نبدأ، محتاج موافقتك على نقطتين:

1️⃣ كيف نتواصل؟
   - واتساب للمتابعة السريعة
   - البريد للوثائق الرسمية

2️⃣ أي معلومات حساسة؟
   - نحتفظ بالبيانات فقط للعرض
   - لا نشارك بياناتك مع أطراف ثالثة

**ما أعرف — اقترح علي:**

أزرار:
- [موافق، نبدأ] → يسجل الموافقة → يبدأ الفحص
- [عندسؤال] → يحول لمتخصص
```
