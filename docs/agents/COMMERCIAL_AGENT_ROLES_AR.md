# أدوار الوكلاء التجاريين (Commercial Agent Roles)

المجموعة التجارية تغطي مرحلة ما بعد الرد الإيجابي حتى التجديد.

| الوكيل | المهمة | المستوى | يتطلب موافقة |
|--------|--------|---------|--------------|
| Client Assessment Agent | تقييم وتأهيل العميل من نموذج الاستلام | L1 | — |
| Action Card Agent | تحويل التقييم إلى بطاقات إجراء (نصح فقط) | L1 | — |
| Proposal Agent | صياغة عرض مرتبط بالكتالوج لفرصة مؤهَّلة (نطاق سعري فقط) | L3 | نعم |
| Proof Pack Agent | تجميع حزمة إثبات من أدلة موثّقة ومُنقّحة | L1 | — |
| Payment Handoff Agent | تجهيز تسليم الدفع — يتم فقط بموافقة بشرية | L3 | نعم (حرِج) |
| Delivery Handoff Agent | تسليم الصفقات المربوحة للتنفيذ مع النطاق ومعايير القبول | L2 | نعم |
| Customer Success Agent | دفع التبني وتحقيق القيمة بعد التسليم | L1 | — |
| Renewal Agent | تحديد التجديد فقط عند وجود قيمة مُسلّمة موثّقة | L1 | — |
| Finance Agent | اقتصاديات الوحدة و CAC/الاسترداد (نصح فقط) | L1 | — |

## بوابات تجارية صارمة (مفروضة في `core/safety/commercial.py`)

1. **العرض ↔ الكتالوج:** كل عرض يجب أن يرتبط بمنتج/خدمة موجودة في
   `data/productized_services/services.yaml`.
2. **فرصة مؤهَّلة:** لا عرض بدون تأهيل.
3. **السعر النهائي:** يتطلب موافقة بشرية — الذكاء الاصطناعي يقترح **نطاقاً** فقط.
4. **تسليم الدفع:** يتطلب موافقة + تأهيل.
5. **الصفقة المربوحة:** تتطلب handoff للتنفيذ ولنجاح العميل قبل بدء العمل.
6. **التجديد:** يتطلب قيمة مُسلّمة موثّقة + اكتمال التسليم.

## ممنوع

التسعير النهائي التلقائي، أي التزام قانوني، معالجة مدفوعات، إرسال رابط دفع،
تجاوز قائمة الكبح، أو وعود تتجاوز النطاق المتفق عليه.
