# Capability Blueprint: Data Capability

## Business Purpose

Prepare trustworthy data for decisions and AI workflows.

## Typical Problems

- duplicates and missing fields  
- unclear source of truth  
- no freshness or ownership  
- PII mixed into operational datasets  

## Required Inputs

- data source list  
- field dictionaries (even draft)  
- access and retention constraints  
- example exports (redacted as needed)  

## AI Functions

- profile datasets  
- suggest schema and mappings  
- flag quality and sensitivity issues  
- recommend integration order  
- generate readiness report  

## Governance Controls

- registered sources only  
- labeled sensitivity  
- minimization and retention alignment  
- no use of unregistered extracts in client-facing outputs  

## KPIs

- completeness / consistency scores  
- duplicate rate trend  
- time to onboard a new source  
- governance flags resolved  
- readiness score movement  

## Services

- Data Readiness Assessment  
- Data Cleanup / Mapping Sprint  
- Data OS (managed)
