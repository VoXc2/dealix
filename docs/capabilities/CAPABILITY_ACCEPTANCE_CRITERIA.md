# Capability Acceptance Criteria

A capability is considered built when:

## Level 2 — Structured

- owner exists  
- input/output documented  
- process mapped  
- KPI defined  

## Level 3 — AI-Assisted

- AI output exists  
- human review exists  
- QA checklist exists  
- proof pack produced  

## Level 4 — Governed Workflow

- governance checks embedded  
- audit log exists  
- approval flow exists  
- recurring report exists  

## Level 5 — Optimized OS

- recurring cadence  
- dashboard exists  
- improvement backlog exists  
- monthly proof updates  
- retainer operating model  

---

## Example

Revenue Capability Level 3 is not complete unless:

- scoring exists  
- top accounts identified  
- draft pack exists  
- QA completed  
- proof pack delivered  
- next actions defined
