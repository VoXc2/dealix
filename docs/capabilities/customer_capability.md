# Capability Blueprint — Customer Capability

> One of the 7 capabilities Dealix builds inside customers (per
> `docs/company/CAPABILITY_OPERATING_MODEL.md`). Mirrors the structure of
> `docs/capabilities/revenue_capability.md`. Concerns the customer's
> support inbox, SLA adherence, feedback intelligence, and the human
> agents who serve end-users in Arabic and English.

## Business purpose
Help the customer understand and respond to their end-customers with
quality and speed — without giving up the human touch. Cut response time,
deflect repetitive questions to FAQ-backed drafts, and surface what
customers are actually saying so the business can act.

## Typical problems
- Inbox overload across email + WhatsApp + web chat + Instagram DMs.
- Inconsistent replies between agents; tone drifts; bilingual gaps (AR/EN).
- No SLA tracking; "first response time" is a guess.
- FAQ knowledge trapped in one senior agent's head.
- Feedback (complaints, NPS, reviews) collected but never analyzed.
- New agent ramp-up takes weeks; tribal knowledge lost on every churn.
- Manual escalation routing — wrong tickets reach the wrong team.

## Required inputs from customer
- Inbox access or export (email / WhatsApp Business / web chat history).
- Top 30–50 FAQs with current "correct" answers (or co-create on Day 1).
- Escalation rules per category (who handles refunds, complaints, legal).
- Tone-of-voice guide (or use Dealix bilingual default).
- SLA targets per channel (first response, resolution).
- Lawful basis for processing customer messages (PDPL Art. 5).

## AI functions that build this capability
- Intent classifier (6–10 categories tuned to the customer's volume).
- Bilingual reply-suggestion engine (AR + EN drafts; agent dispatches).
- FAQ indexing with citation to the source document (RAG-backed).
- Escalation routing recommendation (per intent and SLA risk).
- Sentiment + complaint extraction from feedback streams.
- Weekly support report: volume, response time, top intents, deflection
  rate, sentiment delta — generated by ReportingAgent.
- PII redaction on inbound messages before any draft is composed.

## Governance controls (binding)
- **Suggested-replies-only, no autosend in MVP.** Every draft requires an
  agent click to dispatch. Autosend is deliberately deferred; it returns
  only with a contracted upgrade and a renegotiated approval matrix
  (per `docs/services/ai_support_desk_sprint/offer.md`).
- PDPL Art. 13 footer present in any externally-shared customer message.
- PII detector runs on inbound + drafts before the agent sees them
  (per `docs/governance/RUNTIME_GOVERNANCE.md` Check 2).
- WhatsApp messages need explicit per-message approval; bulk
  WhatsApp is BLOCKED (per `docs/governance/HUMAN_IN_THE_LOOP_MATRIX.md`).
- No fake testimonials; no synthetic "social proof" reviews.
- Forbidden-claim filter on every draft ("نضمن / guarantee / 100%").

## KPIs (measured before/after)
- Median first-response time (minutes).
- Deflection rate (FAQ-resolved without agent edit, %).
- AR/EN coverage parity (% of inbound answered in source language).
- Agent edit distance (how much agents change AI drafts — proxy for quality).
- SLA adherence (% of tickets answered within target).
- Repeat-contact rate (was the answer good enough the first time?).
- CSAT / sentiment delta over the engagement window.

## Maturity ladder (per `docs/company/CAPABILITY_OPERATING_MODEL.md`)
- **Level 0** — no support function; founder answers DMs.
- **Level 1** — manual replies; one shared inbox; no SLA tracking.
- **Level 2** — channels documented; agents named; FAQ exists somewhere.
- **Level 3** — AI drafts replies + routes by intent + weekly report
  (AI Support Desk Sprint).
- **Level 4** — Drafts + approvals + audit log + feedback intelligence
  + monthly improvements (Monthly Support AI).
- **Level 5** — Customer OS with cross-channel inbox, SLA dashboard,
  knowledge feedback loop, partial self-service.

## Agents involved (per `docs/product/AI_AGENT_INVENTORY.md`)
- **SupportAgent** — bilingual draft composition, intent classification,
  FAQ retrieval. Autonomy level 2 (drafts only).
- **KnowledgeAgent** — pulls cited FAQ / policy / product answers from
  the indexed corpus. Refuses if no source qualifies.
- **ComplianceGuardAgent** — mandatory gate. Runs PII + forbidden-claim
  + PDPL-notice checks on every draft.
- **ReportingAgent** — weekly support report + monthly trend pack.

## Proof types produced
- **Time Proof** — median first-response time before/after; hours saved
  per agent per week.
- **Quality Proof** — deflection rate, repeat-contact rate, agent edit
  distance — evidence that drafts are landing.
- **Risk Proof** — PII redaction count, blocked-claim count, audit log
  of every dispatch decision (per
  `docs/governance/AUDIT_LOG_POLICY.md`).
- **Knowledge Proof** — FAQ coverage % and citation rate (handed off
  upstream to the Knowledge Capability if Company Brain is also active).
