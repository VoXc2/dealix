# Sovereign Operating Stack

The complete, layered stack that makes Dealix operationally sovereign. Each layer makes the next one stronger.

```
Constitution
→ Dealix Method
→ Core OS
→ Data Sovereignty
→ Runtime Governance
→ LLM Gateway
→ Agent Control Plane
→ Productized Services
→ Proof Economy
→ Capital Ledger
→ Intelligence OS
→ Command Center
→ Business Units
→ Standards
→ Academy / Partners
→ Ventures
→ Holding Company
```

## 1. Layer responsibilities

| Layer | Responsibility |
| --- | --- |
| Constitution | The doctrine and the prime directives |
| Dealix Method | The repeatable how |
| Core OS | The shared technical substrate |
| Data Sovereignty | Source Passport + access discipline |
| Runtime Governance | Decision vocabulary + audit |
| LLM Gateway | Model routing + cost + eval |
| Agent Control Plane | Identity + autonomy + containment |
| Productized Services | Fixed-scope, fixed-price, fixed-proof |
| Proof Economy | Proof Pack + score + tiers |
| Capital Ledger | Compounding assets |
| Intelligence OS | Decisions and capital allocation |
| Command Center | Real-time operation |
| Business Units | Operating lines |
| Standards | Method + QA + benchmarks |
| Academy / Partners | Distribution + authority |
| Ventures | Standalone product entities |
| Holding Company | Capital allocation across the stack |

## 2. Layering rules

- Lower layers may not depend on higher layers.
- New offers depend only on stable layers below them.
- Ventures inherit lower layers; they may never fork them.
- Standards capture patterns that emerge from repeated use of lower layers.

## 3. Why this stack is sovereign

- Each layer is owned by Dealix.
- Each layer has a control or a fallback for its external dependencies.
- Each layer has a proof artifact when it is operating well.
- Each layer can be replaced or refactored without breaking the layers above it, provided the contracts hold.

## 4. The single-line summary

> Build from the constitution upward. Refuse anything that asks you to build from the top down.
