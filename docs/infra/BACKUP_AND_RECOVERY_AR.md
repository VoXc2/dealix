# النسخ الاحتياطي والاسترجاع (Backup & Recovery)

> مسودة سياسة — التطبيق الفعلي TBD من المؤسس/المزوّد.

## النطاق
- قاعدة بيانات MySQL (الأساسي).
- ملفات S3 (إن استُخدمت لتخزين المخرجات).
- سجلات الحوكمة (`ai_action_ledger.jsonl`) وقائمة الكبح.

## الأهداف (مقترحة)
| المؤشر | الهدف (TBD) |
|--------|-------------|
| RPO (أقصى فقد بيانات) | ≤ 24 ساعة |
| RTO (زمن الاسترجاع) | ≤ 4 ساعات |
| تكرار النسخ | يومي + قبل كل هجرة |
| الاحتفاظ بالنسخ | 30 يوماً (متدرّج) |

## قواعد
- نسخ مشفّرة؛ تفضيل إقامة KSA.
- اختبار استرجاع دوري (restore drill) — لا نسخة بلا اختبار.
- قائمة الكبح وسجل الحوكمة ضمن النسخ دائماً.
- لا حذف بيانات إنتاج بدون نسخة مؤكدة + اعتماد.
