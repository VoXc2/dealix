# سياسة اعتماد النشر (Deployment Approval)

## القواعد الحمراء
- ❌ لا نشر إنتاج من فرع وكيل أو تلقائياً.
- ❌ لا إضعاف لبوابات النشر.
- ❌ لا تعديل أسرار إنتاج عبر CI/وكيل.
- ✅ نشر الإنتاج باعتماد بشري صريح فقط.

## التدفق
```
دمج إلى main (بعد بوابات CI) → بناء (vite + esbuild) → نشر staging (تلقائي)
   → فحوصات صحة + اختبارات دخان → اعتماد بشري → نشر إنتاج → مراقبة
```

## بوابات قبل الإنتاج
- كل بوابات CI خضراء (9 workflows).
- إصلاح بند مصادقة نقطة الموافقة (`api/governance-router.ts`).
- فحص صحة + هجرات قاعدة البيانات مُراجَعة.

## الوكلاء
الحد الأقصى L4 (staging)؛ `production_deploy` ممنوع للجميع.
