# تقييمات العميل/الإيراد/التسليم (Client · Revenue · Delivery Evals)

الحالات في `data/evals/client_revenue_delivery_cases.jsonl`، ويشغّلها
`tests/test_evals_runner.py`.

## ما الذي يُقيَّم

| المحرك | يتحقق من |
|--------|----------|
| `reply` | تصنيف الردود (إيجابي/غاضب/إلغاء/ارتداد/قانوني/شكوى/خصوصية) |
| `payment` | تسليم الدفع يتطلب موافقة + تأهيل |
| `renewal` | التجديد يتطلب قيمة مُسلّمة + اكتمال التسليم |
| `won_deal` | الصفقة المربوحة تتطلب handoff للتسليم ونجاح العميل |

## قواعد التوجيه الحرجة

- **الرد الإيجابي** → اكتشاف/واتساب/حجز موعد — **never** دفع مباشر.
- **غاضب/إلغاء/ارتداد** → كبح (suppression) تلقائي.
- **قانوني/شكوى/خصوصية** → تسليم بشري إلزامي.

## أمثلة

- `هذا يهمني، نقدر نحدد موعد؟` → `positive` (لا دفع).
- `هذا سبام، لا تزعجني` → `angry` (كبح).
- `احذف بياناتي` → `privacy_request` (تسليم بشري + كبح).
- دفع بدون موافقة بشرية → **block**.
- تجديد بدون قيمة مُسلّمة → **block**.
- صفقة مربوحة بدون handoff تسليم → **block**.
