## 2. الموافقة والرضا (Consent Framework)

### مستويات الموافقة

| المستوى | متى يُطلب | الشكل |
|---------|----------|-------|
| **L1 — تشغيلية** | عند استخدام Diagnostic | checkbox في النموذج |
| **L2 — تعاقدية** | قبل Sprint / Pilot | DPA موقّع |
| **L3 — تسويقية** | قبل case study / testimonial | موافقة مكتوبة منفصلة |
| **L4 — شراكة** | قبل مشاركة بيانات مع شريك | بند صريح في عقد الشراكة |

### ما يحدث بدون موافقة
```
بدون L1: الخدمة لا تبدأ
بدون L2: لا Sprint، لا billing
بدون L3: لا نشر، لا case study، لا شهادة
بدون L4: لا مشاركة مع شريك، بغض النظر عن النوع
```

## 3. آلية الإلغاء وحذف البيانات (Opt-out & Deletion)

### الطريقة الحالية (يدوية)
```
العميل يرسل: بريد إلكتروني أو واتساب لـ المؤسس
المطلوب: طلب صريح بـ "إلغاء المشاركة" أو "حذف بياناتي"
وقت الاستجابة: < 48 ساعة (إقرار) + < 7 أيام (تنفيذ)
التوثيق: سجل داخلي بتاريخ الطلب والتنفيذ
```

### الخطوات عند طلب الحذف
1. إقرار باستلام الطلب (< 48 ساعة)
2. التحقق من هوية الطالب
3. حذف البيانات الشخصية من Supabase
4. إبلاغ أي subprocessors مشاركين
5. إرسال تأكيد التنفيذ (< 7 أيام)
6. الاحتفاظ فقط بما يلزم للالتزامات القانونية

## 4. حدود بيانات العميل (Customer Data Boundary)

### ما يستطيع Dealix الوصول إليه
```
✓ البيانات المُدخلة في النماذج (diagnostic، onboarding)
✓ معلومات pipeline المشاركة بشكل صريح
✓ الرسائل في محادثات الدعم
```

### ما لا يستطيع Dealix الوصول إليه أبداً
```
✗ CRM الخاص بالعميل (بدون تكامل صريح + موافقة)
✗ قواعد بيانات العملاء الداخلية
✗ المراسلات الخاصة بخارج منصة Dealix
✗ بيانات موظفين العميل
✗ معلومات مالية (إلا ما أُعطي بشكل صريح)
```

### ما لا يُشارك مع أطراف ثالثة
```
✗ بيانات أي عميل مع منافسيه
✗ pipeline data مع شركاء Dealix بدون موافقة
✗ اسم العميل في أي تسويق بدون L3 consent
```

## 6. سياسة AI أولاً بموافقة (Approval-First AI Policy)

### المبدأ
```
النظام يحلل ويصيغ ويقترح → المؤسس/العميل يوافق → التنفيذ يدوي
```

### ما يفعله AI تلقائياً (بدون موافقة)
- تحليل البيانات المُدخلة
- توليد المسودات
- توليد التقارير الداخلية
- تصنيف الفرص

### ما يتطلب موافقة صريحة
- إرسال أي رسالة للخارج (`NO_LIVE_SEND`)
- نشر أي محتوى عام
- استخدام بيانات العميل في التسويق
- إصدار فاتورة / تحصيل مبلغ (`NO_LIVE_CHARGE`)
