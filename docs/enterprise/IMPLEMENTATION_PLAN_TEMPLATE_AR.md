## Phases

### Phase 0: Discovery & Signoff (Week 1)

**Activities:**
- [ ] Stakeholder mapping
- [ ] Current state assessment
- [ ] Success criteria agreement
- [ ] Scope signoff
- [ ] DPA / Contract signed

**Deliverables:**
- Signed SOW
- DPA executed
- Permission grants documented

**Exit criteria:**
- Contract signed
- Counterpart identified

### Phase 1: Setup & Configuration (Week 1–2)

**Activities:**
- [ ] Tenant provisioning
- [ ] User accounts + roles
- [ ] Integration credentials
- [ ] Brand voice capture
- [ ] Workflow config

**Deliverables:**
- Live sandbox
- Config documented
- Smoke tests passed

**Exit criteria:**
- Founder approves sandbox

### Phase 2: Pilot Workflows (Week 2–4)

**Activities:**
- [ ] First workflow(s) launched
- [ ] Drafts generated + approved
- [ ] Proof pack #1 produced
- [ ] Client training

**Deliverables:**
- ≥ 1 workflow in production
- 1+ proof pack delivered
- ≥ 1 client user trained

**Exit criteria:**
- Pilot metrics reviewed
- Client signs Phase 3

### Phase 3: Scale & Embed (Week 4–8)

**Activities:**
- [ ] Additional workflows
- [ ] Power user training
- [ ] Internal handover prep
- [ ] Renewal conversation

**Deliverables:**
- 3+ workflows in production
- ≥ 80% of intended users active
- Renewal proposal

**Exit criteria:**
- Acceptance signoff
- Renewal signed (or graceful close)

## Communication Plan

- **Daily:** async updates during setup
- **Weekly:** 30-min sync during pilot
- **Bi-weekly:** exec summary during scale
- **Monthly:** business review post-scale

## Risk Register (Per Implementation)

| Risk | Mitigation | Owner |
|------|------------|-------|
| Delayed approvals from client | Define approval SLA upfront | Client lead |
| Integration blocker | Vet integrations in Phase 1 | Dealix tech |
| Data quality issues | Onboarding assessment in Phase 0 | Both |
| Scope creep | Change control in SOW | Both |
| Champion turnover | Multi-stakeholder engagement | Dealix |

## Acceptance Criteria (Per Deliverable)

- Aligned to sold scope
- Clear, measurable
- Evidence-backed
- Client-readable (Arabic-first)
- No overclaim
- Has next action + owner + acceptance criteria

## Change Control

أي تغيير في scope:
1. Written request
2. Impact assessment (time, cost, risk)
3. Both parties sign
4. Updated SOW

## Termination

- 30-day notice either side
- Data return + deletion per DPA
- Knowledge transfer
