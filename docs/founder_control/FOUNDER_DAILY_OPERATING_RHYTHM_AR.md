# الإيقاع اليومي للمؤسس (Founder Daily Operating Rhythm)

إيقاع تشغيل يومي للمؤسس يقسّم اليوم إلى ثلاث محطات: الصباح، منتصف اليوم، نهاية اليوم. الهدف: خط اكتساب وتسليم منضبط، مع بقاء كل قرار حسّاس بيد المؤسس.

يكمّل هذا المستند `DAILY_SUPER_COMMAND_SYSTEM_AR.md`.

## الإيقاع اليومي

### الصباح
- توليد 400 مسودة (drafts) لليوم.
- إعداد حزم المعلومات (intelligence packs) للجهات المستهدفة.
- توليد قائمة أفضل 100 (Top 100).
- تجهيز موجزات المكالمات (call briefs) لمن يحتاج اتصالًا.
- توليد تقرير الأمر الأعلى اليومي (`reports/founder/DAILY_SUPER_COMMAND.md`).

### منتصف اليوم
- اعتماد رسائل البريد (approve emails) — لا شيء يُرسل قبل موافقتك.
- إسناد المكالمات (assign calls) للمسؤولين.
- تجهيز العروض المصغّرة (mini proposals) — كل عرض يحتاج موافقتك.
- تصنيف الردود الواردة (classify replies) وتحويلها للحالة المناسبة في المسار.

### نهاية اليوم
- تحديث مسارات التسليم (delivery pipelines) وحالاتها.
- إعداد/إرسال تقارير القيمة الأسبوعية (weekly value reports) المستحقة.
- مراجعة تقرير الأمر الأعلى اليومي (daily super command) وحسم «أهم قرار اليوم».
- متابعة المدخلات الناقصة لأي تسليم عالق في `intake_required`.

## القواعد الصلبة (Hard Rules)

| # | القاعدة |
|---|---|
| 1 | 400 مسودة يوميًا **مطلوبة**، لكن 400 إرسال يوميًا **غير مفعّل افتراضيًا**. |
| 2 | لا إرسال خارجي افتراضيًا. |
| 3 | كل بريد يبقى **مسودة** حتى موافقة المؤسس. |
| 4 | كل عرض مصغّر (mini proposal) يحتاج موافقة المؤسس. |
| 5 | لا اتصال آلي (no automated calling). |
| 6 | لا أتمتة واتساب أو LinkedIn باردة (no cold WhatsApp/LinkedIn automation). |
| 7 | لا قوائم مشتراة (no purchased lists). |

> هذه القواعد ضوابط ثابتة لا تُتجاوز. أي أتمتة في الاكتساب تتوقف عند حدّ «المسودة»، والإرسال قرار بشري صريح.

## كيف تترابط المحطات مع المسار

- **الصباح** يغذّي خط الاكتساب (مسودات، Top 100، موجزات) ويُنتج تقرير القيادة.
- **منتصف اليوم** يحوّل الجاهز إلى قرار: اعتماد البريد، إسناد المكالمات، اعتماد العروض المصغّرة (`mini_proposal_ready → proposal_sent` بموافقتك)، وتصنيف الردود.
- **نهاية اليوم** يحرّك التسليم: تحديث الحالات، تقارير القيمة الأسبوعية، ومتابعة الـ intake. تذكّر القاعدة الصلبة: لا `delivery_started` قبل اكتمال المدخلات الأساسية.

## قائمة فحص يومية سريعة

- ☐ صباحًا: 400 مسودة + intelligence packs + Top 100 + call briefs + تقرير الأمر الأعلى.
- ☐ منتصف اليوم: اعتماد البريد + إسناد المكالمات + تجهيز mini proposals + تصنيف الردود.
- ☐ نهاية اليوم: تحديث مسارات التسليم + تقارير القيمة الأسبوعية + حسم أهم قرار اليوم.
- ☐ تأكدت أن لا شيء أُرسل خارجيًا بلا موافقتي.
- ☐ راجعت العوائق وأي تسليم عالق على المدخلات.
