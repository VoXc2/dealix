# المراجعة الأسبوعية للمجلس — Weekly Board Review (عربي أولًا)

> إيقاع أسبوعي يرفع المؤسس من التنفيذ إلى القيادة: المقاييس، الصحة، المخاطر، الإيراد، والقرارات الكبرى.

---

## 1. جدول الأعمال الثابت
1. **الإيراد:** خط الأنابيب، العروض المعتمدة/المرفوضة، النقد (ربط `company_os/war_room/WEEKLY_CEO_BRIEF.md`).
2. **العملاء:** صحة العملاء (`reports/customer_success/CLIENT_HEALTH_REVIEW.md`)، الأخضر/الكهرماني/الأحمر.
3. **التسليم:** تقارير القيمة الأسبوعية، نقاط القبول، مخاطر التسليم.
4. **واتساب/البوابة:** تحويل ما بعد الرد، فجوات التجربة، مراجعة الصلاحيات.
5. **التجديد/التوسعة:** طابور التجديد/الترقية (مبني على قيمة).
6. **الحوكمة/الخصوصية/الأمن:** تنبيهات، انتهاكات (يجب أن تكون صفرًا)، مراجعة السجل.
7. **التعلّم:** تحديث المقاييس وحالات التقييم (`data/evals/*`).

## 2. المقاييس الأساسية (KPIs أسبوعية)
| المقياس | المصدر | الهدف |
|---|---|---|
| تحويل ما بعد الرد | `reports/whatsapp/POST_REPLY_CONVERSION_REVIEW.md` | ↑ |
| نسبة العروض المعتمدة | `reports/revenue_execution/PROPOSAL_QUEUE.md` | ↑ |
| زمن أول قيمة مُسلَّمة | `reports/delivery/*` | ↓ |
| صحة العملاء (أخضر%) | `reports/customer_success/*` | ↑ |
| فرص تجديد مؤهّلة | `reports/renewal/RENEWAL_QUEUE.md` | ↑ |
| انتهاكات أمن/خصوصية | فحص الأمن | = 0 |

## 3. القرارات الكبرى
تسعير استراتيجي، أولويات القطاعات، توظيف/طاقة التسليم، تفعيل `send_enabled` (قرار مؤسس صريح بعد اكتمال البوابات).

## 4. المخرج
`reports/founder/WEEKLY_BOARD_REVIEW.md` محدّثًا + قرارات مُدوّنة في `DECISION_LOG.md`.
