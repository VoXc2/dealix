# Commercial Gates — بوابات تجارية غير قابلة للمفاوضة

**الغرض:** تثبيت **حواجز** تمنع تآكل الثقة والتسعير والتسليم. تتماشى مع سياسة المؤسس في المستودع.

## بوابات الدفع والإثبات

- **لا revenue قبل invoice_paid** — لا اعتبار «مغلق» بدون تحصيل مؤكد.  
- **لا upsell كبير قبل Proof Pack** لنطاق متفق.  
- **لا L5/L6 دعائية** قبل اجتماع/scope حقيقي (استخدم مستويات الأدلة في الحوار الداخلي).

## بوابات الإرسال والامتثال

- **لا cold WhatsApp** — لا إرسال جماعي غير مرغوب.  
- **لا Gmail/تقويم خارجي** بدون موافقة صريحة.  
- **لا scraping إنتاجي** يخالف السياسة.

## بوابات البيانات

- **لا أرقام CRM أو إيراد مخترعة** — مصدر الحقيقة من استيراد KPI الحقيقي عند الربط، لا اختلاقاً في الأتمتة.

## بوابات التنفيذ والمنتج

- **لا custom build قبل دفع + نطاق + تكرار** مرئي.  
- **لا affiliate مفتوح** مبكراً — انظر [AFFILIATE_NETWORK.md](AFFILIATE_NETWORK.md).  
- **لا enterprise motion** قبل repeatability — انظر [ENTERPRISE_MOTION.md](ENTERPRISE_MOTION.md).

## حلقة Proof التجارية المقترنة

```text
~10 لمسات بشرية موافَق عليها يومياً
  → ~5 متابعات
  → ~1 محادثة شريك
  → ~1 ديمو
  → ~1 pilot صغير مدفوع
  → Proof Pack ≤ ~48h للـ pilots
  → insight مجهّل
  → referral / sprint / retainer
```

## أكبر الأخطاء (قائمة منع)

1. شرح المنتج بدل بيع أول Proof.  
2. بناء features قبل الدفع/التكرار.  
3. affiliate مفتوح مبكراً.  
4. enterprise قبل repeatability.  
5. WhatsApp بارد / أتمتة LinkedIn.  
6. بيع **ROI guarantee**.  
7. ترك العميل بعد Proof Pack.  
8. عدم تحويل التسليم إلى محتوى/إحالة.  
9. توظيف قبل تكرار workflow.  
10. SaaS قبل 3 retainers أو تكرار مثبت.
