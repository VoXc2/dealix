# Capital Model — نموذج رأس المال لكل مشروع

**الغرض:** كل مشروع يبني **أصولاً** لا نقوداً فقط. إن دخل مال ولم يترك أصلاً، فهو أقل قيمة من مشروع أصغر يولد **Proof + playbook**.

## أنواع رأس المال

| النوع | أمثلة |
|-------|--------|
| **Service Capital** | عروض، scopes، checklists، معايير QA |
| **Product Capital** | أدوات، وحدات، APIs، dashboards، أتمتة آمنة |
| **Knowledge Capital** | playbooks، اعتراضات، insights قطاعية، مخاطر حوكمة |
| **Trust Capital** | Proof packs، insights مجهّلة، شهادات، سجلات تدقيق |
| **Market Capital** | جمهور، شركاء، إحالات، benchmarks، محتوى سلطة |

## قاعدة القبول لكل مشروع

> **1 Trust Asset** + **1 Knowledge أو Product Asset** على الأقل.

### تعريف سريع

- **Trust Asset:** Proof Pack، أو insight معتمد، أو سياسة موافقة مُثبتة الاستخدام.  
- **Knowledge/Product Asset:** قالب، checklist، قاعدة بيانات اعتراضات، تحسين واجهة داخلية، إلخ.
