# Empire Folder — Dealix as an Operating Standard — مجلد الإمبراطورية

## How to use this folder — كيف تستخدم هذا المجلد

This folder defines Dealix as a **company-as-operating-standard**: not a tool that
gets sold, but a governed revenue and AI operations standard that gets adopted.
The top rule of the whole folder is one sentence:

> **Dealix must not be a tool that gets sold; it must be an operating standard
> that gets adopted.**

Read the files in group order. Category & Method explain *what Dealix teaches*.
Commercial explains *how it earns*. Proof & Trust explain *why it is believed*.
Distribution explains *how it spreads*. Scale explains *how it compounds*. Gates
explain *what is allowed to grow*. Every file is bilingual and ends with the same
disclosure. The operating doctrine chain runs through all 17:

`Signal → Source → Owner → Approval → Action → Evidence → Decision → Value → Asset`

Anything outside this chain is noise and is not built, sold, or measured.

### The final decision — القرار النهائي

The folder ends on one operational decision: **build the Commercial Proof Loop,
not features.** The loop is:

10 human-approved touches per day → 5 follow-ups → 1 partner conversation →
1 demo → 1 small paid pilot → Proof Pack within 48 hours → case-style insight →
referral, sprint, or retainer.

Features wait. The loop runs every working day. Every adopted standard starts
inside this loop.

## كيف تستخدم هذا المجلد

يعرّف هذا المجلد Dealix بوصفه **شركة-كمعيار-تشغيلي**: ليس أداة تُباع، بل معيار
حوكمة للإيرادات وعمليات الذكاء الاصطناعي يُتبنّى. القاعدة العليا للمجلد كله جملة
واحدة:

> **يجب ألّا تكون Dealix أداة تُباع؛ بل معيار تشغيلي يُتبنّى.**

اقرأ الملفات بترتيب المجموعات. «الفئة والمنهج» تشرح *ما الذي تعلّمه Dealix*.
«التجاري» يشرح *كيف تكسب*. «الإثبات والثقة» يشرح *لماذا تُصدَّق*. «التوزيع» يشرح
*كيف تنتشر*. «التوسع» يشرح *كيف تتراكم*. «البوابات» تشرح *ما الذي يُسمح له بالنمو*.
كل ملف ثنائي اللغة وينتهي بالإفصاح نفسه. وسلسلة الدوكترين التشغيلي تمرّ عبر
السبعة عشر ملفًا:

`إشارة ← مصدر ← مالك ← موافقة ← إجراء ← دليل ← قرار ← قيمة ← أصل`

أي شيء خارج هذه السلسلة ضجيج، ولا يُبنى ولا يُباع ولا يُقاس.

### القرار النهائي

ينتهي المجلد بقرار تشغيلي واحد: **ابنِ حلقة إثبات تجارية، لا ميزات.** الحلقة هي:

١٠ لمسات معتمَدة بشريًا يوميًا ← ٥ متابعات ← محادثة شريك واحدة ← عرض واحد ←
تجربة مدفوعة صغيرة واحدة ← Proof Pack خلال ٤٨ ساعة ← رؤية بأسلوب دراسة حالة ←
إحالة أو Sprint أو Retainer.

الميزات تنتظر. الحلقة تعمل كل يوم عمل. كل معيار يُتبنّى يبدأ داخل هذه الحلقة.
