# Productization Path — مسار المنتجة

**الغرض:** لا تقفز إلى **SaaS**. حوّل التكرار إلى منتج عبر طبقات واضحة.

## المراحل الستة

| Stage | الوصف |
|-------|--------|
| 1 | **Internal tools** — أدوات داخلية للمؤسس والتشغيل |
| 2 | **Client-visible reports** — تقارير/Proof يقرأها العميل |
| 3 | **Client workspace** — مساحة عمل للعميل |
| 4 | **Partner portal** — تمكين الشركاء |
| 5 | **Self-serve modules** — وحدات مختارة بخدمة ذاتية محكومة |
| 6 | **SaaS** — منتج واسع بعد تكرار الاقتصاد |

## قاعدة البناء

```text
workflow جديد → checklist
تكرر مرتين → template
تكرر 3 مرات → automation (ضمن حدود الموافقة)
تكرر مع عميلين → internal module
تكرر مع 3 عملاء + retainers → product feature
```
