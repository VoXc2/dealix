# Productization Path — مسار التحويل إلى منتج
<!-- Layer: empire | Owner: Founder | Date: 2026-05-17 -->

## English

Do not jump to SaaS. A product built before the workflow is proven is a guess
with a database. Dealix follows a gradient — each stage earns the next, and
nothing is built before real demand shows it is needed.

### The six stages

| Stage | What it is |
|---|---|
| 1 — Internal tools | Tools the founder uses to deliver faster |
| 2 — Client-visible reports | Outputs the customer sees, still founder-produced |
| 3 — Client workspace | A space the customer logs into |
| 4 — Partner portal | A workspace for partners and agencies |
| 5 — Self-serve modules | Pieces a customer can run without the founder |
| 6 — SaaS | A productized, self-serve platform |

Each stage is only entered when the stage below it is stable and repeatedly
used. Skipping a stage builds something nobody has asked for.

### The build rule

Building follows repetition, not enthusiasm:

| Trigger | Build |
|---|---|
| New workflow | Write a checklist |
| Repeated twice | Make a template |
| Repeated 3 times | Build an automation |
| Repeated with 2 customers | Build an internal module |
| Repeated with 3 customers + retainers | Build a product feature |

A workflow seen once is a note. Seen three times across customers with
retainers, it has earned a feature. This keeps engineering effort tied to
proven demand and protects the founder's time from speculative building.

## العربية

لا تقفز إلى SaaS. المنتج المبني قبل إثبات الـworkflow تخمين مع قاعدة بيانات.
تتّبع Dealix تدرّجاً — كل مرحلة تستحقّ التالية، ولا يُبنى شيء قبل أن يُظهر طلب
حقيقي أنه مطلوب.

### المراحل الستّ

| المرحلة | ما هي |
|---|---|
| 1 — أدوات داخلية | أدوات يستخدمها المؤسس للتسليم أسرع |
| 2 — تقارير مرئية للعميل | مخرجات يراها العميل، ما زال المؤسس يُنتجها |
| 3 — مساحة عمل للعميل | مساحة يدخل إليها العميل |
| 4 — بوابة شركاء | مساحة عمل للشركاء والوكالات |
| 5 — وحدات ذاتية الخدمة | أجزاء يُشغّلها العميل دون المؤسس |
| 6 — SaaS | منصّة ذاتية الخدمة مُحوَّلة إلى منتج |

لا يُدخَل في أي مرحلة إلا حين تكون المرحلة الأدنى منها مستقرّة ومُستخدَمة
بتكرار. تخطّي مرحلة يبني شيئاً لم يطلبه أحد.

### قاعدة البناء

البناء يتبع التكرار، لا الحماس:

| المُحفِّز | ابنِ |
|---|---|
| workflow جديد | اكتب قائمة تحقّق |
| تكرّر مرتين | اصنع قالباً |
| تكرّر 3 مرات | ابنِ أتمتة |
| تكرّر مع عميلين | ابنِ وحدة داخلية |
| تكرّر مع 3 عملاء + retainers | ابنِ ميزة منتج |

workflow يُرى مرة واحدة ملاحظة. يُرى ثلاث مرات عبر عملاء لديهم retainers، فقد
استحقّ ميزة. هذا يُبقي جهد الهندسة مربوطاً بطلب مُثبَت ويحمي وقت المؤسس من
البناء التخميني.
