# Trust Layer — طبقة الثقة كمنتج

**الغرض:** **عدم إخفاء القيود** — القيود تبني ثقة، خصوصاً مع GenAI وحوكمة الوكلاء.

## Dealix Trust Promise

- **No spam automation** — لا إزعاج مؤتمت كممر بيع.  
- **No scraping** إنتاجي يخالف السياسة.  
- **No cold WhatsApp** — إلا مع **opt-in** صريح.  
- **No fake proof** — لا أدلة مزيفة أو أرقام منقولة بدون مصدر.  
- **No guaranteed revenue claims** — لا ضمان عائد.  
- **No source-less knowledge answers** في سياقات تتطلب إثباتاً.  
- **Human approval** للأفعال الخارجية الحساسة.  
- **Audit logs** للأفعال الحساسة حيث ينطبق النظام.  
- **Proof packs** لكل مشروع مدفوع بنطاق متفق.

## الجملة البيعية

> Dealix **لا يؤتمت الفوضى**. Dealix **يحكم workflow قبل الأتمتة**.

## مواءمة التشغيل (Founder gates)

- لا إرسال حيّ خارجي بدون موافقة.  
- لا أتمتة فوترة عدوانية؛ إيراد عند **invoice_paid**.  
- كل شيء **inbound + human review + Proof Pack** كمسار افتراضي للتسليم الأول.
