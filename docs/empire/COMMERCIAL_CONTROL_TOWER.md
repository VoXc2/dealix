# Commercial Control Tower — برج التحكم التجاري

<!-- Layer: Empire | Owner: Founder | Date: 2026-05-17 -->
<!-- Arabic primary — العربية أولاً -->

> **قاعدة حاكمة:** ما لا يُراجَع بإيقاع ثابت لا يُدار. برج التحكم التجاري هو
> مجلس المراجعة الداخلي — يومي، أسبوعي، شهري — يحوّل النشاط إلى قرار.
> لا قرار بلا بيانات. بيانات ناقصة = `insufficient_data`، لا تخمين.

---

## الغرض — Purpose

برج التحكم التجاري ليس لوحة مؤشرات (dashboard) فارغة. هو **إيقاع مراجعة**
يجبر السؤال الصحيح في الوقت الصحيح. ثلاث وتيرات، لكل منها مخرَج قرار واحد:

| الوتيرة | السؤال المحوري | المخرَج |
|---------|----------------|---------|
| يومي | ماذا تحرّك اليوم؟ | أولوية الغد |
| أسبوعي | ماذا نضاعف وماذا نوقف؟ | قرار التركيز |
| شهري | إلى أين نتجه استراتيجياً؟ | الرهان التالي |

---

## المراجعة اليومية — Daily review

تُملأ نهاية كل يوم عمل. أرقام فقط، لا سرد. الحقول الفارغة = `insufficient_data`.

| الحقل | القياس |
|-------|--------|
| Messages sent | عدد الرسائل المُرسَلة (بعد موافقة) |
| Follow-ups | عدد المتابعات |
| Replies | عدد الردود الواردة |
| Demos | عروض/جلسات تشخيص |
| Scopes | نطاقات عمل أُرسلت |
| Invoices | فواتير صادرة |
| Paid / commitments | مدفوعات أو التزامات مؤكدة |
| Proof Packs delivered | عدد Proof Packs المُسلَّمة |
| Partner conversations | حوارات شراكة |
| Affiliate leads | leads واردة من المسوّقين بالعمولة |
| Blocked risks | مخاطر عالقة تحتاج قراراً |
| Best message | أفضل رسالة أداءً اليوم |
| Worst channel | أضعف قناة اليوم |
| Tomorrow's priority | أولوية واحدة للغد |

```
نهاية اليوم
  ├── سجّل الأرقام (لا سرد)
  ├── حدّد best message + worst channel
  ├── سجّل المخاطر العالقة
  └── اكتب أولوية واحدة للغد ← المخرَج الوحيد الإلزامي
```

## المراجعة الأسبوعية — Weekly review

تُعقد جلسة واحدة أسبوعياً. تحوّل بيانات سبعة أيام إلى قرار تركيز.

- **Best segment** — أي قطاع أعطى أفضل استجابة
- **Best message** — أي رسالة تكررت نتيجتها
- **Best offer** — أي عرض حرّك القرار
- **Best channel** — أي قناة جلبت أعلى جودة
- **Where the funnel stalled** — أين تعثّر القمع هذا الأسبوع
- **Which objection recurred** — أي اعتراض تكرر
- **Which partner brought quality** — أي شريك جلب leads جيدة
- **Which affiliate brought noise** — أي مسوّق جلب ضجيجاً لا جودة
- **What to double** — ما الذي نضاعفه الأسبوع القادم
- **What to stop** — ما الذي نوقفه
- **What NOT to build** — ما الذي لا نبنيه رغم الإغراء

> الحقل الأخير ضروري: قرار "ما لا نبنيه" يحمي التركيز بقدر قرار "ما نبنيه".

---

## المراجعة الشهرية — Monthly review

مراجعة استراتيجية واحدة شهرياً. ثمانية محاور، ينتهي كل منها بحكم واضح.

| المحور | السؤال |
|--------|--------|
| Revenue | ما الإيراد الفعلي المُحصَّل هذا الشهر؟ |
| Pipeline | ما حالة الفرص الحقيقية الموثّقة؟ |
| Delivery | هل سُلّمت الـ Proof Packs في وقتها وبجودة QA؟ |
| Support | ما حجم الدعم؟ ما أكثر مشكلة تكرّرت؟ |
| Partners | أي شريك نمى؟ أيٌّ ركد؟ |
| Governance | هل خُرق أي بند من اللاءات الثمانية؟ |
| Learning | ما الدرس الأكبر هذا الشهر؟ |
| Next strategic bet | ما الرهان الاستراتيجي التالي؟ |

```
المراجعة الشهرية
  ├── Revenue / Pipeline / Delivery / Support
  ├── Partners / Governance / Learning
  └── Next strategic bet ← رهان واحد، مكتوب، قابل للقياس
```

محور **Governance** غير قابل للحذف: إن خُرق أي بند من الدستور يُسجَّل صراحةً،
ويُفتح بنداً للتصحيح. لا تمرير صامت.

## قواعد الانضباط — Discipline rules

1. الحقل الفارغ لا يُملأ بالتخمين — يُكتب `insufficient_data`.
2. لا ادعاء نتيجة لم تُوثَّق. "تقديري" أو "نمط آمن للحالة" فقط.
3. كل مراجعة تنتج قراراً واحداً مكتوباً على الأقل.
4. المراجعة لا تُؤجَّل — الإيقاع أهم من اكتمال البيانات.
