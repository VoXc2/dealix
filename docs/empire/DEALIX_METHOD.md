# منهجية Dealix — The Dealix Method

> **AR:** أي شيء لا يدخل سلسلة المنهجية يُعتبر noise.
> **EN:** Anything that does not enter the method chain is treated as noise.

---

## السلسلة / The Doctrine Chain

كل منتج، خدمة، شريك، عميل، ووكيل AI يجب أن يمرّ بنفس السلسلة:

```text
Signal → Source → Owner → Approval → Action → Evidence → Decision → Value → Asset
```

| الحلقة / Link | المعنى / Meaning |
|---------------|------------------|
| Signal | إشارة سوق أو اهتمام / a market or interest signal |
| Source | من أين جاءت الإشارة / where the signal came from |
| Owner | من يملك المتابعة / who owns the follow-through |
| Approval | هل التصرف الخارجي معتمد / is the external action approved |
| Action | التنفيذ نفسه / the execution itself |
| Evidence | الدليل المُسجّل / the recorded evidence |
| Decision | القرار والخطوة التالية / the decision and next step |
| Value | القيمة الناتجة / the value produced |
| Asset | الأصل القابل لإعادة الاستخدام / the reusable asset |

---

## مثال تطبيقي / Worked Example

```text
Lead جاء من حملة
→ مصدره واضح (Source)
→ مالكه فريق المبيعات (Owner)
→ الرسالة تحتاج موافقة (Approval)
→ أُرسلت يدوياً بعد الاعتماد (Action)
→ الرد تسجّل (Evidence)
→ القرار: الخطوة التالية (Decision)
→ القيمة: scope أو اجتماع (Value)
→ الأصل: Proof Pack أو case insight (Asset)
```

هذا يجعل Dealix ليست «أتمتة» — يجعلها **نظام تشغيل إيراد محكوم**.

**EN:** This makes Dealix not "automation" but a *governed revenue operating system*.

## المعادلة الاستراتيجية / The Strategic Equation

المنهجية اليومية تتفرّع من المعادلة الأعلى للشركة (لا تقفز درجة):

```text
Signal → Diagnostic → Productized Sprint → Governed Delivery → Proof Pack
→ Retainer → Capital Asset → Product Module → Business Unit → Standard
→ Partner/Academy → Venture
```
