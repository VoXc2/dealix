# Dealix Method — The Operating Model in Layers / منهج Dealix

> Dealix is not "a product." It is built as layers that run together: a
> service-led company, an AI operations platform, a revenue proof system, a
> governance layer, a partner channel, and a Saudi-localized operating
> standard. This doc names the layers and the method flow that connects them.
>
> Dealix ليست "منتجاً". إنها مبنية كطبقات تعمل معاً: شركة قائمة على الخدمة،
> منصة تشغيل ذكاء اصطناعي، نظام إثبات إيراد، طبقة حوكمة، قناة شركاء، ومعيار
> تشغيل مُحلّى للسوق السعودي. هذه الوثيقة تسمّي الطبقات وتدفّق المنهج الذي يربطها.

**Date opened / تاريخ الفتح:** 2026-05-17
**Owner / المالك:** Founder
**Status / الحالة:** Strategic narrative — not a binding spec.

## 1. The identity / الهوية

Dealix is a **Governed Revenue & AI Operations Partner**. Not a tool vendor,
not an agency, not a consultancy alone — the combination that lets one
workflow be diagnosed, governed, and proven.

Dealix هي **شريك تشغيل محكوم للإيراد والذكاء الاصطناعي**. ليست بائع أدوات، ولا
وكالة، ولا استشارات وحدها — بل التركيبة التي تسمح بتشخيص سير عمل واحد، حوكمته،
وإثباته.

The difference is structural: a company that lives on chance closes deals when
luck arrives; a company that runs a system closes deals because the system
produces them. Dealix is the second kind.

الفرق بنيوي: الشركة التي تعيش على الصدفة تغلق الصفقات حين يحضر الحظ؛ الشركة
التي تشغّل نظاماً تغلق الصفقات لأن النظام يُنتجها. Dealix من النوع الثاني.

## 2. The method flow / تدفّق المنهج

Every engagement moves through six steps in the same order. Skipping a step is
not allowed — it is how proof breaks.

كل ارتباط يمرّ بست خطوات بالترتيب نفسه. تخطّي خطوة غير مسموح — هكذا ينكسر الإثبات.

| Step / الخطوة | What happens / ما الذي يحدث |
|---|---|
| 1. Diagnose / نشخّص | Name the workflow and where value leaks |
| 2. Prepare / نجهّز | Make the workflow ready: source, owner, data clarity |
| 3. Govern / نحكم | Apply approval and evidence rules before automation |
| 4. Prove / نثبت | Produce a Proof Pack with evidence, not a dashboard |
| 5. Deliver / نسلّم | Hand over drafts, status, and next actions |
| 6. Expand / نوسّع | Move to the next rung only after proof exists |

---

## 3. The seven layers / الطبقات السبع

Dealix runs as seven layers stacked on top of each other. A weak layer below
makes every layer above unreliable.

Dealix تعمل كسبع طبقات مرصوفة فوق بعضها. ضعف طبقة في الأسفل يجعل كل طبقة فوقها
غير موثوقة.

| # | Layer / الطبقة | Contains / يحتوي |
|---|---|---|
| 1 | Market / السوق | Category, education, content, benchmark |
| 2 | Demand / الطلب | Outbound, partners, workshops, referrals |
| 3 | Sales / المبيعات | Discovery, demo, proof, scope, invoice |
| 4 | Delivery / التسليم | Intake, analysis, drafts, Proof Pack, QA |
| 5 | Trust / الثقة | Approval, evidence, no fake claims, data clarity |
| 6 | Expansion / التوسع | Sprint, retainer, referral, case insight |
| 7 | Product / المنتج | Templates, automations, modules, workspace |

---

## 4. How the layers reinforce each other / كيف تعزّز الطبقات بعضها

The Market layer makes Demand cheaper. The Trust layer makes Sales shorter.
The Delivery layer feeds the Expansion layer with real case insight. The
Product layer turns repeated delivery into reusable assets — so the next
engagement starts further ahead.

طبقة السوق تجعل الطلب أرخص. طبقة الثقة تجعل المبيعات أقصر. طبقة التسليم تغذّي
طبقة التوسع برؤى حالات حقيقية. طبقة المنتج تحوّل التسليم المتكرر إلى أصول قابلة
لإعادة الاستخدام — فيبدأ الارتباط التالي من نقطة أبعد.

This is the compounding loop: each engagement should leave the system stronger
than it found it.

هذه حلقة التراكم: كل ارتباط يجب أن يترك النظام أقوى مما وجده.

---

## 5. What the method refuses / ما يرفضه المنهج

- No automation of a workflow that is not yet ready (see `SOAEN_STANDARD.md`).
- No pricing rung skipped to chase a bigger deal (see `OFFER_LADDER.md`).
- No outcome described as guaranteed — only diagnosed, governed, and proven.
- No external action without human approval.

— لا أتمتة لسير عمل غير جاهز بعد (انظر `SOAEN_STANDARD.md`).
— لا تخطّي درجة تسعير لملاحقة صفقة أكبر (انظر `OFFER_LADDER.md`).
— لا وصف نتيجة بأنها مضمونة — فقط مُشخَّصة ومحكومة ومُثبتة.
— لا فعل خارجي دون موافقة بشرية.

## Doctrine alignment / المواءمة مع الدستور

- The method is built around governance and proof, never around guaranteed
  revenue. / المنهج مبني على الحوكمة والإثبات، لا على ضمان الإيراد.
- The six-step flow enforces approval and evidence before any automation runs.
- Pricing is not set here; `../COMPANY_SERVICE_LADDER.md` governs. Only the
  499 SAR Sprint is a firm price; higher rungs are `recommended_draft`.
- No rung may be skipped. The flow is sequential by design.
