# عملية الموافقات للعميل

1. **مسودات فقط افتراضيًا:** أي نص مقترح للعملاء الخارجيين أو إرسال عبر قناة حساسة يبقى مسودة حتى موافقة بشرية من جهة العميل (وDealix حيث ينطبق).
2. **الادعاءات:** لا ادعاءات مبيعات مضمونة؛ أي رقم أو وعد يمر على Claim QA.
3. **البيانات الشخصية:** معالجة وفق الأساس المتفق؛ redaction قبل مشاركة نماذج مع أطراف ثالثة.
4. **التسجيل:** أحداث الموافقة/الرفض تُسجّل حيث يدعم النظام — انظر [`../governance/AUDIT_LOG_POLICY.md`](../governance/AUDIT_LOG_POLICY.md).

للمصفوفة التفصيلية: [`../governance/APPROVAL_MATRIX.md`](../governance/APPROVAL_MATRIX.md).
