# قوائم تسليم الأنظمة (System Delivery Checklists)

لكل نظام من أنظمة Dealix الخمسة قائمة تسليم مشتقّة من حزمة التسليم (`delivery_pack_ar`) في `data/systems.json`. كل عنصر في الحزمة يتحوّل إلى مهمة تسليم قابلة للتتبّع.

المرجع التقني: `schemas/delivery_task.schema.json` — كل مهمة لها `status` من بين: `todo` / `in_progress` / `blocked` / `done`، ويرتبط حقل `deliverable` بعنصر حزمة التسليم الذي تنتجه المهمة.

> تذكير: هذه المهام تُنشأ آليًا عند `won`، لكن لا تبدأ فعليًا قبل اكتمال المدخلات الأساسية (`delivery_started`).

## كيفية الاستخدام

- عند `won` تُنشأ هذه القوائم تلقائيًا داخل مجلد العميل كمهام `todo`.
- حدّث حالة كل مهمة أثناء العمل: `todo → in_progress → done`، أو `blocked` مع كتابة `blocker_reason`.
- المهمة لا تكتمل المسار إلى `accepted` إلا بعد اجتياز بوابة القبول الخاصة بالنظام (راجع `DELIVERY_ACCEPTANCE_GATES_AR.md`).
