# خارطة المنتج — Dealix

## حارس المنتج (السلسلة الذهبية)

كل مرحلة أدناه يجب أن تخدم **إشارة → Lead → Decision Passport → إجراء معتمد → تسليم → Proof** أو مقياساً تشغيلياً مباشراً (تحويل، تسليم، إثبات، احتفاظ). لا تُضاف ميزات «CRM عام» خارج مسار [DEALIX_MASTER_OPERATING_MODEL_AR.md](strategic/DEALIX_MASTER_OPERATING_MODEL_AR.md). موانع الهدر: كتالوج الإجراءات + `POST /revenue-os/anti-waste/check`؛ مستويات الإثبات L0–L5 و`ProofEvent` — راجع نفس المستند و [LAUNCH_READINESS_REPORT.md](LAUNCH_READINESS_REPORT.md).

## Phase 0 — Foundation (الحالي)

- PR v3 foundation، Personal Operator، Project memory migration، اختبارات ووثائق.

**نجاح:** API يعمل، اختبارات خضراء، قصة منتج موحدة.

**بوابة الخروج:** [LAUNCH_READINESS_REPORT.md](LAUNCH_READINESS_REPORT.md) + [LAUNCH_GATES.md](LAUNCH_GATES.md) (قبل أي إعلان خارجي).

## Phase 1 — Private Alpha

- سامي + فريق ضيق؛ فهرسة؛ موجز يومي؛ جاهزية إطلاق.

**مخاطر:** تشتت الميزات.

**خروج:** أسبوعان استخدام داخلي منتظم.

**بوابة الخروج:** [LAUNCH_READINESS_REPORT.md](LAUNCH_READINESS_REPORT.md) — تحديث قسم الجاهزية بعد أسبوعين.

## Phase 2 — Private Beta

- 10 مؤسسين؛ واتساب موافقات؛ Gmail/Calendar drafts؛ رادار؛ proof.

**مخاطر:** امتثال واتساب.

**خروج:** 3 قصص نجاح مكتوبة.

**بوابة الخروج:** [PHASE2_PRIVATE_BETA_CHECKLIST.md](PHASE2_PRIVATE_BETA_CHECKLIST.md) + [LAUNCH_GATES.md](LAUNCH_GATES.md) حيث ينطبق.

## Phase 3 — Paid Pilot

- 5 دافعين؛ onboarding؛ proof pack أسبوعي.

**خروج:** تجديد ≥60%.

**بوابة الخروج:** [LAUNCH_GATES.md](LAUNCH_GATES.md) + [POST_LAUNCH_SCORECARD.md](ops/POST_LAUNCH_SCORECARD.md) (مؤشرات التجديد والإثبات).

## Phase 4 — Public Launch

## Phase 5 — Platform

**بوابة الخروج:** [POST_LAUNCH_BACKLOG.md](ops/POST_LAUNCH_BACKLOG.md) (ship-when متفق عليها) + [LAUNCH_GATES.md](LAUNCH_GATES.md) لأي إعلان واسع جديد.
