# Board Dashboard

Twelve metrics. The board does not need everything.

## 1. The twelve

1. Revenue
2. MRR
3. Gross Margin
4. Proof Packs Delivered
5. Proof-to-Retainer Conversion
6. Governance Incidents
7. AI Run Audit Coverage
8. Capital Assets Created
9. Productization Candidates
10. Client Health
11. Business Unit Maturity
12. Market Power Signals

## 2. Why each matters

| Metric | Why |
| --- | --- |
| Revenue | proves the market pays |
| MRR | proves continuity |
| Gross Margin | proves scalability |
| Proof Packs | proves value is documented |
| Proof-to-Retainer | proves value converts to operation |
| Governance Incidents | protects trust |
| AI Run Audit Coverage | proves enterprise readiness |
| Capital Assets | prevents agency trap |
| Productization Candidates | proves services convert to software |
| Client Health | informs expansion or risk |
| BU Maturity | informs venture timing |
| Market Power | confirms category formation |

## 3. Typed surface

`board_ready_os.board_dashboard.BoardDashboardSnapshot` rejects unknown metric names.

## 4. The principle

> A board dashboard is a decision surface, not a metric pile.
