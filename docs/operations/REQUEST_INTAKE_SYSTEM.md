# Request Intake System

**Rule:** Nothing goes “straight to build.” Every request is **typed**, **scored for risk**, and **routed**.

## Request types

1. **Sales opportunity**
2. **Existing client request**
3. **Feature request** (product / internal)
4. **Governance request** (legal, PDPL, approvals)
5. **Support issue** (post-delivery)
6. **Partnership opportunity**
7. **Custom enterprise request**

## Intake template

```markdown
# Request Intake

## Request
What is being asked?

## Source
Who asked? (name / channel / role)

## Type
Sales / Client / Feature / Governance / Support / Partner / Enterprise

## Urgency
Low / Medium / High

## Revenue Potential
Low / Medium / High

## Risk
Low / Medium / High

## Existing Service Match
- Lead Intelligence
- AI Quick Win
- Company Brain
- Support Desk
- Governance Program
- AI Ops Diagnostic
- Custom / unclear

## Decision
- Route to service
- Offer diagnostic ([`../services/ai_ops_diagnostic/README.md`](../services/ai_ops_diagnostic/README.md))
- Reject
- Add to backlog
- Escalate (founder / legal)
```

Copy a blank from [`../../clients/_PROJECT_WORKBENCH/00_intake.md`](../../clients/_PROJECT_WORKBENCH/00_intake.md) when the request becomes a project.
