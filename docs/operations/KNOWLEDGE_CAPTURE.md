# Knowledge Capture

Every project must **feed** the company: playbooks, assets, product, pricing, risk.

## After each project, capture at least one of

- New **objection** → [`../sales/OBJECTION_HANDLING.md`](../sales/OBJECTION_HANDLING.md)
- New **sector insight** → [`../playbooks/`](../playbooks/README.md)
- New **template** → [`../assets/templates/`](../assets/templates/) or `docs/templates/`
- New **risk** or control → [`../governance/RISK_REGISTER.md`](../governance/RISK_REGISTER.md)
- **Feature candidate** → [`../company/FEATURE_CANDIDATE_LOG.md`](../company/FEATURE_CANDIDATE_LOG.md) + [`../product/FEATURE_BACKLOG.md`](../product/FEATURE_BACKLOG.md)
- **Capital compounding** → [`../ledgers/CAPITAL_LEDGER.md`](../ledgers/CAPITAL_LEDGER.md) + [`DEALIX_CAPITAL_MODEL.md`](../company/DEALIX_CAPITAL_MODEL.md)
- **Delivery improvement** → service checklist or [`SCOPE_ENGINE.md`](../delivery/SCOPE_ENGINE.md) notes

## Where it lives

| Kind | Primary location |
|------|------------------|
| Sector | `docs/playbooks/<sector>.md` |
| Reusable doc | `docs/assets/templates/` |
| Product idea | `docs/company/FEATURE_CANDIDATE_LOG.md`, `docs/product/FEATURE_BACKLOG.md` |
| Sales | `docs/sales/OBJECTION_HANDLING.md`, battlecards |
| Risk | `docs/governance/RISK_REGISTER.md` |
| Market | [`../market/AI_OPS_MARKET_INTELLIGENCE.md`](../market/AI_OPS_MARKET_INTELLIGENCE.md) |

Closure gate: [`POST_PROJECT_REVIEW`](../../projects/_TEMPLATE/POST_PROJECT_REVIEW.md), [`COMPOUNDING_SYSTEM.md`](../company/COMPOUNDING_SYSTEM.md), [`../ledgers/LEARNING_LEDGER.md`](../ledgers/LEARNING_LEDGER.md).
