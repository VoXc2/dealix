# حزمة الثقة والامتثال للبيع (Trust & Compliance Pack)

**الجمهور:** فرق مبيعات، نجاح عملاء، وأمن معلومات العميل.  
**تحذير إلزامي:** هذا المستند **ليس** استشارة قانونية. الامتثال لأنظمة المملكة يتطلب مراجعة من مختص مرخّص (DPO / محامٍ).

## 1. ما نبيعه في طبقة الثقة (C)

- **قابلية التدقيق:** أحداث structured عبر السجلات؛ مسارات لمس البيانات الشخصية تُسجّل على مستوى الـ API (انظر أدناه).
- **Decision Passport + Anti-Waste:** لا إجراء خارجي بلا جواز قرار؛ قيود توسعة ونشر حسب مستوى الأدلة.
- **موافقات بشرية:** مسار A1+ ومركز موافقات (حسب حالة المنتج في [no_overclaim.yaml](../../dealix/registers/no_overclaim.yaml)).

## 2. تدقيق الوصول (Audit logs) — بعد تقني

| العنصر | الوصف | المسار في الريبو |
|--------|--------|------------------|
| تسجيل طلبات HTTP | معرّف طلب، مدة، حالة | `RequestIDMiddleware` في [api/middleware/http_stack.py](../../api/middleware/http_stack.py) |
| مسارات بيانات شخصية (PDPL Art. 18 كمرجع تصميمي) | حدث `personal_data_access` لبادئات API محددة | `AuditLogMiddleware` في نفس الملف |
| إزالة بصمة الخادم | عدم الإبقاء على ترويسة Server | `SecurityHeadersMiddleware` في نفس الملف |

**للمشتري:** ربط مخرجات structlog بـ SIEM أو تخزين append-only يقع على عاتق التشغيل في بيئة العميل أو SaaS الخاص بك.

## 3. مستويات الأدلة (L0–L5)

- **API:** `GET /api/v1/decision-passport/evidence-levels`
- **كود مرجعي:** `auto_client_acquisition/proof_engine/evidence.py`
- **سياسة المنتج:** لا نشر عام إلا وفق L4+ وموافقة؛ التفاصيل في [DEALIX_MASTER_OPERATING_MODEL_AR.md](DEALIX_MASTER_OPERATING_MODEL_AR.md).

## 4. سياسات القنوات (ملخص للعرض التجاري)

- **واتساب:** لا إرسال بارد تلقائي؛ التزام بسياسات المنتج والقواعد في `.cursor/rules` ووثائق التشغيل.
- **LinkedIn / بريد:** مسودات وموافقة أولاً؛ لا أتمتة إرسال تعسفي.
- **Gmail / تقويم خارجي:** لا إجراء خارجي بدون موافقة صريحة.

## 5. خريطة PDPL — تقني مقابل تنظيمي (مختصر)

| الموضوع | جانب تقني (ما يدعمه التصميم) | جانب تنظيمي (مسؤولية العميل + قانون) |
|---------|-------------------------------|----------------------------------------|
| أساس قانوني للمعالجة | تقليل البيانات في السجلات؛ عدم تسجيل query params في مسار التدقيق | سياسات الخصوصية، العقود، سجلات الأغراض |
| حقوق الأفراد | واجهات/عمليات عند توفرها في المنتج؛ تصدير/حذف يتطلب تصميم سير عمل | إجراءات الاستجابة للطلبات، هوية طالب الحق |
| خرق البيانات | سجلات ومراقبة؛ جاهزية تشغيلية | إشعار الجهة المعنية وفق الأنظمة المعمول بها |
| التعاقد مع المعالجين | — | DPA مع البائعين الفرعيين |

## 6. الخطوة التالية للعميل Enterprise

- ورشة **TRUST** مع أمن المعلومات: مصدر الحقيقة للسجلات، الاحتفاظ، التشفير، الوصول.
- مراجعة قانونية لنماذج الموافقة ونشر قصص العملاء (L4).
