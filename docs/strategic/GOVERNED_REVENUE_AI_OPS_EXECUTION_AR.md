# Dealix — Governed Revenue & AI Operations (CEO/Board Blueprint)

## الحكم الاستراتيجي

المسار المعتمد:

**Dealix = Governed Revenue & AI Operations**

وليس شركة “AI عامة”.  
القيمة الأساسية: تحويل تجارب AI وعمليات الإيراد إلى تشغيل محكوم، قابل للقياس، ومربوط بالأدلة والموافقات.

## الرؤية النهائية

**Dealix تصبح طبقة التشغيل الخليجية للـ governed revenue وAI workflows.**

المخرجات التي يجب أن يضمنها النظام:

- Source clarity
- Approval boundaries
- Evidence trails
- Decision passports
- Proof of value
- Revenue actions
- Governance gates

## North Star

**Governed Value Decisions Created**

التعريف:
عدد القرارات الإيرادية/التشغيلية التي تستوفي:

1. مصدر واضح
2. موافقة واضحة
3. دليل موثق
4. أثر قابل للقياس

## العقيدة التشغيلية (Operating Doctrine)

كل workflow يمر بالسلسلة التالية:

**Signal → Source → Approval → Action → Evidence → Decision → Value → Asset**

أي نشاط لا يدخل هذه السلسلة يُعامل كـ drift حتى يثبت عكس ذلك.

## State Machine الحاكمة للأدلة والقيمة

- `prepared_not_sent` = L2
- `sent` = L4
- `replied_interested` = L4
- `meeting_booked` = L4
- `used_in_meeting` = L5
- `scope_requested` = L6
- `pilot_intro_requested` = L6
- `invoice_sent` = L7_candidate
- `invoice_paid` = L7_confirmed

القواعد غير القابلة للتفاوض:

- لا `sent` بدون `founder_confirmed=true`
- لا L5 بدون `used_in_meeting`
- لا L6 بدون `scope_requested` أو `pilot_intro_requested`
- لا L7 confirmed بدون `invoice_paid`
- لا إعلان revenue قبل `invoice_paid`

## معنى Full Ops في Dealix

Full Ops لا يساوي autonomous external execution.

Full Ops يعني:

- النظام يجهز
- النظام يقترح
- النظام يحذر
- النظام يسجل
- النظام يتحقق
- النظام يصنف
- النظام يولد drafts
- المؤسس يوافق على كل فعل خارجي

## نظام الحوكمة (4 طبقات)

1. AI Management System
2. Risk Management
3. Evidence & Records
4. Operational Approvals

ترجمة تشغيلية داخل Dealix:

- Govern = policies + owners + approval rules
- Map = workflow risks + data risks + AI use cases
- Measure = KPIs + quality + evidence + outcomes
- Manage = mitigations + logs + no-build decisions + reviews

## قواعد البناء (No-Build Discipline)

لا تُبنى ميزة إلا إذا تحقق شرط واحد على الأقل:

- عميل طلبها صراحة
- workflow تكرر 3 مرات
- تقلل خطرًا حقيقيًا
- تسرّع تسليمًا مدفوعًا
- تفتح retainer

غير ذلك: **No build**.
