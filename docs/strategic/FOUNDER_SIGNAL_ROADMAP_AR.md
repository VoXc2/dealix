# رودماب Founder Signal إلى Product Spine — خطة تنفيذ قيادية

**المرجع التشغيلي اليومي:** [FULL_MARKET_PROOF_RUN_AR.md](FULL_MARKET_PROOF_RUN_AR.md) (Full Market Proof Run + **Founder Signal Command**: classification → evidence → next action → build/no-build → verification).

**P0 اليومي:** شغّل أولًا قسم **[Founder Signal War Room](FULL_MARKET_PROOF_RUN_AR.md#founder-signal-war-room)** داخل RUN (لوحة الجدول + L4 + متابعة + تصنيف النتائج)؛ انتقل إلى **P1** عند أول **نتيجة خارجية موثّقة** في السجل (رد، صمت مسجّل، طلب PDF، اعتراض، إلخ) مع **verification PASS**.

**السياق الإستراتيجي الأوسع للموجات والكود:** [DEALIX_EXECUTION_WAVES_AR.md](DEALIX_EXECUTION_WAVES_AR.md).

**قاعدة التعامل مع هذا الملف:** الرودماب أدناه **لا تحلّ محل السجل**؛ أي انتقال بين مراحل يبقى مشروطًا بإدخالات في [`../../data/docs_asset_usage_log.json`](../../data/docs_asset_usage_log.json) و`founder_confirmed` المطابقة للحقيقة، ثم تشغيل السكربتات في بداية مرجع RUN.

---

## 1. لماذا هذا المسار؟

Dealix تبني **طبقة تشغيل محكومة** وليست «كثافة وثائق» فقط. الخريطة التالية تربط:

- **إشارة خارجية** واحدة في كل مرة،
- **قرارًا واحدًا في السجل**،
- **تحديث Evidence** في الريبو المولّذ،
- وبعدها فقط **SKU صغير أو عدم بناء**.

هذا يتوافق روحًا مع:

- إدارة السجلات كدورة حياة معلومات تجارية (مراجع في RUN: ISO 15489 / 23081)،
- وحوكمة AI القائمة على التحقق والوقت التشغيلي (مرجع RUN: arXiv governance stack).

## 3. مراحل الرودماب (عمود فقري — ليست تواريخ مطلقة)

كل مرحلة لها: **هدف خارجي** · **بوابة لا تُتجاوَز** · **ممنوعات** · **مخرج واحد واضح**.

### المرحلة P0 — Throughput أول signal (أسبوع الوعي)

**RUN:** [War Room](FULL_MARKET_PROOF_RUN_AR.md#founder-signal-war-room) + **§0–§2** (L4، متابعة، سجل).

**هدف خارجي:** تنفيذ **Follow-up Event** + إدخال `follow_up_sent` **واحد** (أو تصحيح L2 إن لزم) + **verification PASS**.

**بوابة المرحلة التالية:** لا تنتقل قبل **نتيجة واحدة** ظاهرة على الأقل: رد، صمت مسجّل، طلب PDF، أو اعتراض صريح — كلها تُصنَّف في Founder Signal.

**ممنوع:** أي PR خارج جدول الإشارات؛ أي L5 بدون اجتماع.

---

### المرحلة P1 — تحويل المحادثة (interested → وقت)

**RUN:** **§3–§5** (تصنيف الردود والمسارات السريعة).

**هدف خارجي:** `replied_interested` أو `meeting_requested` → **جدولة 30 دقيقة** فقط (لا حزم مرسلة دفعة واحدة).

**بوابة المرحلة التالية:** وجود `meeting_booked` في السجل أو حجز مؤكد متساوٍ مع الواقع.

**ممنوع:** «tell me more» يتحول إلى ملفات؛ الرد يبقى دعوة عملية واحدة.

---

### المرحلة P2 — أول L5 صادق

**RUN:** **§7** (اجتماع) + **§8** حتى L5 (`used_in_meeting`).

**هدف خارجي:** انعقاد اجتماع + تسجيل `used_in_meeting` للأصل **الذي وُجد فعلًا** في المحادثة + `commercial_next_action` واضح نحو intro أو شريحة.

**بوابة المرحلة التالية:** إما **pull** (انتقال لمرحلة L6) أو تعلّم إعادة صياغة (لا بناء).

**ممنوع:** تسجيل L5 لثلاثة أصول إن عُرض أصل واحد فقط؛ رفع درجات Revenue/Partner في الـ registry دون مصدر سلوك واضح قيد الاتفاق الداخلي.

---

### المرحلة P3 — مسار L6 قابل للتكرار (SKU ضبط التشخيص)

**RUN:** **§8** (L6) + **§10** (نطاق PR عند الإشارة).

**هدف خارجي:** `pilot_intro_requested` أو `scope_requested` حقيقي موثّق.

**مخرج منتجي مسموح ومحدود:** **قالب واحد** لتشخيص محكوم (نص/هيكل Markdown داخل الريبو أو أداة صغيرة)، **مرتبط بإشارة واحدة**؛ لا توسيع كتالوج خدمات قبل أول إغلاق حلقة عميل.

**ممنوع:** تحويل المرحلة هذه إلى «منتج كامل» قبل أول إتمام تسليم عميل مع Proof Pack في السجل.

---

### المرحلة P4 — إغلاق L7 الأول (مال أو commitment خطّي)

**RUN:** **§9** (L7 / فواتير والتزامات).

**هدف خارجي:** `invoice_sent` أو ما يعادل الـ commitment في سجلكم (`retainer_signed`, `sprint_accepted`, إلخ كما في RUN).

**بوابة المرحلة التالية:** دفع مؤكّد أو رفض واضح — كلاهما يُسجَّل ويُحدَّث الـ registry وفق [ASSET_USAGE_GOVERNANCE_AR.md](ASSET_USAGE_GOVERNANCE_AR.md).

**ممنوع:** تسويق حملة عامة أو رفع دعاءات Trust قبل شروطكم الداخلية للـ BU/trust gate.

---

### المرحلة P5 — بناء «أذرع» صغيرة فقط بحسب الطلب المتكرّر

**RUN:** **§10** (جدول إشارات البناء والحدود).

**هدف خارجي:** تكرّر نفس **`asks_for_*`** من طرفين مختلفين على الأقل ضمن إطار زمني معقول.

**SKU مسموحة:** وفق الجدول في RUN §10 بالضبط (PDF، one-pager English، pricing one-pager، Trust Pack Lite…).

**ممنوع:** «حزمة عربية كاملة + إنجليزي كامل + CRM»؛ كل زيادة = PR واحد وحده بحدود النطاق المكتوب.

---

### المرحلة P6 — Cockpit تشغيلي للمؤسس (بوابة وليس تجديدًا شاملاً)

**RUN:** **§11** (تشغيل المنتج بعد إشارة السوق) + [DEALIX_EXECUTION_WAVES_AR.md](DEALIX_EXECUTION_WAVES_AR.md).

**هدف:** ربط التجربة الأمامية / المشغل الشخصي بـ أعمدة الحقيقة (موافقات، Proof، ليدز) عندما **تثبت الأولوية تشغيليًا**.

**بوابة الدخول:** على الأقل **L5 واحد حقيقي** + عدم احتياجكم لطبقة منتج لتوليد ذلك الأسبوع.

**ممنوع:** إعادة هندسة الـ frontend بالكامل «لأنه حان وقت الـ cockpit».

---

### المرحلة P7 — توسعة رأس عمود واحد قبل توسعة جغرافية

**RUN:** [War Room](FULL_MARKET_PROOF_RUN_AR.md#founder-signal-war-room) + **§3–§10** حسب النتيجة (لا مسار موازٍ).

**هدف خارجي:** تكرار جزء واحد على الأقل من الحلقة (مثلاً شريحة ثانية ضمن المملكة أو قطاع مختلف) بنفس الانضباط في السجل.

**بوابة:** لا تطلق قطاعًا ثانيًا قبل أن يكون **`no_response`/تعلّم** قد وجّهكم إلى archetype مختلف وجرّبتموه واحدًا على الأقل.

## 4. «شبكة عدم الانحراف» لمقارنة المبادرات

| إذا ظهر هذا الاقتراح الداخلي… | قرار واحد أو قريبًا منه |
|------------------------------|---------------------------|
| «نطبّق منصّة مراقبة اشتراكات» | **متوقف** قبل signal متكررة من العملاء |
| «نوحّد UI كامل اليوم» | **متوقف** قبل P6 |
| «نترجم كل الريبو للإنجليزي» | **متوقف**؛ one-pager فقط عند الطلب الخارجي |
| «ندرّب نموذج على كل النصوص الداخلية» | **محظور خارج حدود الامتثال** قبل بوابات الأمان المتفق عليها |
| «نشغّل cold outreach آلياً» | **مرفوض وفق دستور Dealix** — مسودّة وموافقة فقط |

## 5. Definition of Done لكل خط نجاح خارجي مصغّر

| النجاح المرصود | تمّ المرحلة عندما… |
|----------------|---------------------|
| **Signal دخل** | entry في السجل + `verification` PASS |
| **L5** | `used_in_meeting` + قرار واحد للخطوة التالية |
| **L6** | pull مسجَّل بدون احتياج لمخترعات عميل |

## 6. المراجعة الدورية (ربط بالمجلس)

عند ظهور **نتيجة أول أسبوع** خارجية، شغّل **Mini Asset Council** كما في RUN، ثم اربط ذلك بقراءة موسّعة أكثر بحسب الحاجة: [MONTHLY_ASSET_COUNCIL_AR.md](MONTHLY_ASSET_COUNCIL_AR.md).

لا تخلط بين «اجتماع شهري» و«مجلس المصغّر المؤجل حتى بعد أول نتيجة» — كل واحد له غرض.

## 7. جملة واحدة لتوجيه المراجعة كل أسبوع

> إن لم يجد السجل خارجيًا جديدًا هذا الأسبوع، لم تكن مهمة هذا الأسبوع البناء؛ كانت مهمة **الاختبار خارجيًا أو اعتراض تعليمي واحد لا يمكن قوله قبل التجربة**.
