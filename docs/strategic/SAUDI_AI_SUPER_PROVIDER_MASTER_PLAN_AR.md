# Dealix — الخطة التنفيذية لتصبح أقوى مزود خدمات ذكاء اصطناعي للشركات في السعودية

هذا المستند ليس عرضًا نظريًا؛ هو خطة تشغيل + بيع + تسليم + حوكمة قابلة للتنفيذ مباشرة.

مرجع الكتالوج التنفيذي المرتبط بالخطة:
- `dealix/catalogs/saudi_ai_service_catalog_v2.json`

---

## 1) التمركز الاستراتيجي (Positioning)

**Dealix = Saudi AI Revenue Command Center**

القيمة التي تشتريها الشركة:
- رادار سوق سعودي
- عقل شركة (Company Brain)
- قرارات يومية عبر WhatsApp مع اعتماد
- تنفيذ تشغيلي منضبط
- إثبات قيمة (Proof) قابل للقياس
- توسع مدفوع بالأدلة

**قاعدة البيع**: لا نبيع “تقنية AI”؛ نبيع **نتائج أعمال قابلة للقياس** مع حوكمة واضحة.

## 2) محفظة الخدمات الكاملة (12 محرك × 3 مستويات = 36 خدمة)

كل محرك لديه 3 مستويات:
- `Foundation` (تأسيس سريع)
- `Managed` (تشغيل مُدار)
- `Command` (قيادة تنفيذية)

| المحرك | الهدف التجاري | مستويات الخدمة |
|---|---|---|
| Saudi Market Radar | اكتشاف فرص Why-Now | Foundation / Managed / Command |
| Lead Intelligence | تحويل البيانات إلى فرص أولوية | Foundation / Managed / Command |
| Company Brain | ذاكرة تشغيلية تقلل الفوضى | Foundation / Managed / Command |
| Decision Passport | قرار قبل أي تنفيذ | Foundation / Managed / Command |
| WhatsApp Decision Layer | اعتماد سريع وآمن | Foundation / Managed / Command |
| Action & Approval Engine | تنفيذ مضبوط بسياسات | Foundation / Managed / Command |
| Delivery OS | تحويل الخطة إلى مخرجات | Foundation / Managed / Command |
| Support & CS OS | دعم يحسّن الاحتفاظ | Foundation / Managed / Command |
| Payment & Revenue Truth | وضوح مالي وتحصيل | Foundation / Managed / Command |
| Proof & Expansion Engine | إثبات ثم توسع | Foundation / Managed / Command |
| Learning Flywheel | تحسين مستمر مبني على الواقع | Foundation / Managed / Command |
| Trust/Security/Compliance | ثقة جاهزة للبيع المؤسسي | Foundation / Managed / Command |

**المعيار الموحّد لكل خدمة** (إجباري):
1. ما المشكلة التجارية؟
2. ما القرار المطلوب؟
3. ما التنفيذ؟
4. ما الدليل؟
5. ما التوسع التالي؟

## 3) أفضل استراتيجية عروض وتسعير (Saudi Service Ladder)

### A) Offer 1 — Pilot Revenue Sprint (دخول سريع)
- الهدف: إغلاق أول قيمة خلال 7–14 يوم.
- يركز على: Market Radar + Lead Intelligence + Decision Passport + Proof baseline.
- مخرج البيع: Proof Event واضح + Next Action.

### B) Offer 2 — Managed Growth Ops (قيمة شهرية متكررة)
- الهدف: تحسين pipeline + سرعة القرار + جودة التنفيذ.
- يركز على: WhatsApp Decision Layer + Action Approval + Delivery OS + Support OS.
- مخرج البيع: تقارير أسبوعية + تحسين مؤشرات شهرية.

### C) Offer 3 — Executive Command Center (عرض عالي القيمة)
- الهدف: قيادة تنفيذية يومية + تحكم مخاطر + خطة توسع.
- يركز على: Revenue Truth + Proof & Expansion + Learning + Trust Layer.
- مخرج البيع: Executive Pack + Expansion Roadmap.

**مبدأ التسعير**:
- تربط الأسعار بالأثر التشغيلي والوضوح التنفيذي، وليس بعدد المزايا.
- كل عرض يملك KPI baseline + KPI target + review cadence.

## 4) أفضل استراتيجية Go-To-Market في السعودية

### قناة 1: Warm-intro Engine (أولوية قصوى)
- الشركاء: مستشارون، وكالات، مكاتب محاسبة، مزودو ERP/CRM.
- آلية التحويل: Diagnostic قصير + Proof sample + قرار واضح للعميل.

### قناة 2: Founder-led advisory selling
- بيع تنفيذي مباشر لمالكي الشركات ومديري النمو.
- قالب اللقاء: `pain -> decision -> proof -> next step`.

### قناة 3: Partner-led growth
- برنامج Agency Partner OS.
- عمولات واضحة + حوكمة نشر proof + متابعة قيمة مشتركة.

### قناة 4: Content with proof (بدون مبالغة)
- محتوى عربي قائم على learnings حقيقية.
- ممنوع أي ادعاء غير مدعوم أو “ضمان” غير مثبت.

## 5) أفضل استراتيجية تشغيل (Operating Cadence)

### Daily Founder Loop (60–90 دقيقة)
- مراجعة أفضل 5 فرص
- اعتماد/رفض Queue التنفيذ
- إزالة bottleneck واحد يوميًا
- توثيق proof event جديد إن وجد

### Weekly Leadership Loop
- تقرير 7 محاور: Radar, Pipeline, Delivery, Support, Payment, Proof, Risks
- قرار إيقاف/مضاعفة لكل channel أو offer
- تحديث Company Brain وLearning Report

### Monthly Board Loop
- قراءة Unit Economics
- تقييم Expansion Readiness
- قرار استثمار: منتج/قناة/قطاع

---

## 6) أفضل استراتيجية بناء الميزة التراكمية (Compounding Moat)

الميزة لا تأتي من النموذج، بل من هذا التراكم:
- كل تنفيذ يولد Proof
- كل Proof يحسن احتمالية البيع
- كل عميل يحدث Company Brain
- كل دورة تعلم تزيد دقة القرار التالي

هذا يحول Dealix من “أداة” إلى **Operating Layer يصعب استبداله**.

---

## 7) أفضل استراتيجية الثقة والحوكمة (غير قابلة للتفاوض)

يجب أن تبقى هذه القواعد ثابتة:
- لا Cold WhatsApp
- لا LinkedIn Automation
- لا Scraping
- لا إرسال خارجي حي بدون اعتماد
- لا تحصيل حي بدون اعتماد
- لا Proof مزيف ولا Revenue مزيف

وأي خدمة جديدة لا تمر إذا لم تلتزم بهذه القواعد.

## 8) خطة تنفيذ 180 يوم (Execution Roadmap)

### المرحلة 0 (أيام 1–14): تثبيت طبقة القرار
- تشغيل Decision Passport + Approval Queue + WhatsApp brief
- توحيد 36 خدمة داخل الكتالوج
- تفعيل verifier تلقائي للحوكمة

### المرحلة 1 (أيام 15–45): إيراد أولي + Proof
- تفعيل Pilot Revenue Sprint على أول مجموعة عملاء
- إنتاج Proof Packs مع موافقات واضحة
- تتبع مؤشرات: demo_booked, pilot_started, payment_confirmed

### المرحلة 2 (أيام 46–90): تشغيل مُدار قابل للتوسع
- تشغيل Managed Growth Ops لعملاء متكررين
- رفع كفاءة Delivery + Support + Payment Truth
- تفعيل Expansion Engine مبني على الأدلة

### المرحلة 3 (أيام 91–180): جاهزية Enterprise
- Trust Center تشغيلي + حزمة تدقيق
- Executive Command Center للعملاء الأعلى قيمة
- توسيع قنوات الشركاء القطاعية
