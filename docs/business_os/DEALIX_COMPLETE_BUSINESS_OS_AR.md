## ما هو Dealix Complete Business OS؟

ليس أداة واحدة، بل **منظومة أنظمة مترابطة** تأخذ الشركة السعودية B2B من *اهتمام السوق* إلى *إيراد مُحقَّق* إلى *قيمة مُسلَّمة* إلى *تجديد وتوسعة* — كل ذلك تحت سقف واحد: **موافقة المؤسس أولًا، تشغيل تجريبي افتراضيًا، مدفوع بالأدلة، واعٍ للخصوصية**.

```
اهتمام (GTM)  →  تنفيذ إيراد  →  واتساب/بوابة  →  عرض/إثبات/دفع  →  تسليم  →  نجاح  →  تجديد/توسعة
        (Agent #1)            (هذه الطبقات — Agent #2)
```

## المكوّنات (13 نظامًا)

1. **Market Production OS** — إنتاج الاهتمام (أساس Agent #1، لا يُكرَّر).
2. **Revenue Execution OS** — العرض/الإثبات/الدفع جاهز للموافقة.
3. **WhatsApp Client OS** — تجربة العميل بعد الموافقة عبر بطاقات إجراء.
4. **Secure Client Portal** — الأسرار والملفات والصلاحيات والمراجعات.
5. **Proposal / Proof / Payment OS** — عقود البيانات لكل ذلك.
6. **Client Delivery OS** — من صفقة مربوحة إلى قيمة مُسلَّمة.
7. **Renewal OS** — تجديد/توسعة مبنية على قيمة.
8. **Customer Success OS** — صحة العميل ونجاحه.
9. **Finance OS** — المال والفواتير (أساس قائم).
10. **Founder Control Room** — أمر واحد يومي ومراجعة أسبوعية.
11. **Agent Governance OS** — الصلاحيات والموافقات والسجل.
12. **Trust / Security / Privacy OS** — لا أسرار، PDPL، أقل امتياز.
13. **Metrics & Learning Loop** — القياس والتعلّم والتقييم.

## المبادئ الجوهرية

| المبدأ | المعنى العملي |
|---|---|
| موافقة المؤسس أولًا | لا إجراء خارجي بلا موافقة موثّقة (`approved_by`, `approved_at`). |
| تشغيل تجريبي افتراضيًا | `dry_run=true`, `send_enabled=false` حتى تكتمل البوابات. |
| مدفوع بالأدلة | كل توصية/عرض/تقرير يحمل `evidence_level`. |
| واعٍ للخصوصية | لا أسرار في أي مخرَج؛ الأسرار في البوابة فقط؛ PDPL. |
| واتساب بعد الموافقة | لا واتساب بارد إطلاقًا. |
| لا وعود | لا عائد مضمون، لا أدلة مزيّفة، لا وعد قانوني بلا تسليم بشري. |

## ما ليس Dealix
بوت واتساب بارد · بوت ذكاء عام · أداة سبام · مغلِق صفقات مستقل · نظام يطلب مفاتيح API في الدردشة · نظام يرسل روابط دفع بلا موافقة.
