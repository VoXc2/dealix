# Dealix — نموذج التشغيل
## Operating Model

---

## 1. نموذج الأعمال

Dealix هو نظام تشغيل إيرادات سعودي B2B مبني على ثلاثة أعمدة:

1. **إنتاج السوق** — إشارة ← مسودة ← موافقة ← إرسال
2. **تنفيذ الإيرادات** — رد ← تصنيف ← توجيه ← تحويل
3. **التسليم والاحتفاظ** — صفقة رابحة ← إلحاق ← تسليم ← تجديد

## 3. تدفق الإيرادات

```
السوق ← الإشارات ← البحث ← المسودات ← الامتثال
    ← الموافقة ← التسليم ← الردود ← التصنيف
    ← [إيجابي] ← واتساب ← التقييم ← التوصية
    ← [اهتمام] ← مقترح ← أدلة ← دفع
    ← [صفقة رابحة] ← تسليم ← تقرير ← تجديد
```

## 5. قواعد الامتثال الثابتة

| القاعدة | الوصف |
|---|---|
| dry_run = true | كل أنظمة الإرسال الافتراضية جافة |
| approval_required = true | كل الإجراءات الخارجية تتطلب موافقة |
| send_enabled = false | لا إرسال خارجي بدون موافقة صريحة |
| no_cold_whatsapp | واتساب فقط بعد موافقة أو علاقة موجودة |
| no_guaranteed_claims | لا ادعاءات مضمونة |
| no_fake_proof | لا أدلة مزيفة |
| no_pii_in_logs | لا بيانات شخصية في السجلات |
| evidence_level_required | كل توصية تحتاج مستوى دليل |

---

## 6. القواعد الستة الذهبية

1. **لا ضمانات** — "نضمن زيادة المبيعات" ممنوعة
2. **لا واتساب بارد** — فقط بعد موافقة صريحة
3. **لا ادعاءات مضمونة** — "10x revenue" ممنوعة
4. **لا أدلة مزيفة** — كل دليل يجب أن يكون موثقاً
5. **لا أسرار في المطالبات** — مفاتيح API لا تظهر في prompts
6. **لا إرسال بدون موافقة** — كل إرسال يحتاج اعتماد المؤسس
