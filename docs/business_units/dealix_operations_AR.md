# Dealix Operations — وحدة أعمال

**الطبقة:** Holding · Compound Holding Model
**المالك:** المدير العام لوحدة Operations
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [dealix_operations.md](./dealix_operations.md)

## السياق
Dealix Operations هي وحدة الأعمال التي تحوّل العمليات اليدوية كثيرة الأخطاء إلى مسارات محوكَمة قابلة للتكرار. هي الوحدة التي **تأخذ أكثر SOP داخلي مؤلم للعميل وتُسلّم نسخة معزَّزة بالذكاء الاصطناعي** قابلة للتدقيق ومتوافقة مع PDPL. تقع في طبقة الوحدات داخل [`docs/holding/DEALIX_HOLDING_OS.md`](../holding/DEALIX_HOLDING_OS.md)، ومقترنة بـ `docs/capabilities/operations_capability.md`، وتطبّق منهج `docs/standards/DEALIX_METHOD.md`.

## الوظيفة
تُشخّص Operations أين يقضي العميل ساعات متكرّرة، تُصمّم مساراً يجمع خطوات حتمية مع مساعدة ذكاء اصطناعي، تُسلّمه داخل Core OS، تربط الجودة والموافقات، وتحتفظ به شهرياً.

## الخدمات

| الخدمة | المدّة | المخرج |
|---|---|---|
| AI Quick Win | 1–2 أسبوع | تشخيص + مسار صغير حيّ |
| Workflow Automation | 2–6 أسابيع | 1–3 مسارات محوكَمة |
| Executive Reporting Automation | 2–4 أسابيع | تقارير مجدوَلة بمصادر وموافقات |
| Monthly AI Ops (احتفاظ) | مستمر | ضبط، مسارات جديدة، تقرير قيمة شهري |

## وحدات المنتج داخل Core OS

| الوحدة | الوظيفة |
|---|---|
| Workflow Builder | تركيب مسارات من خطوات معتمدة |
| Approval Flow | توجيه موافقات لكل مسار |
| SOP Generator | تحويل ملاحظات اجتماعات/مستندات إلى SOP |
| Ops Dashboard | الإنتاجية، نسبة الأخطاء، الوقت الموفَّر |

## المؤشرات

- **ساعات موفَّرة لكل مسار أسبوعياً.**
- **خطوات يدوية مستبدَلة.**
- **معدّل إكمال المسار** — %.
- **خفض الأخطاء** — معدّل العيوب بعد النشر.
- **تبنّي المسار** — # مستخدمون نشطون / مرخَّصون.

## اعتمادات Core OS

| الوحدة | كيف تستهلكها Operations |
|---|---|
| Data OS | سجل مصادر لكل مدخل |
| LLM Gateway | خطوات الذكاء عبر Gateway |
| Governance Runtime | موافقة + تدقيق لكل مسار |
| Proof Ledger | دليل الوقت الموفَّر |
| Capital Ledger | قوالب مسارات ومكتبات خطوات |
| AI Control Tower | كلفة لكل تشغيل، درجات تقييم |
