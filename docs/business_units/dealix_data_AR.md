# Dealix Data — وحدة أعمال

**الطبقة:** Holding · Compound Holding Model
**المالك:** المدير العام لوحدة Data
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [dealix_data.md](./dealix_data.md)

## السياق
Dealix Data هي وحدة الأعمال التي تُجهّز بيانات العميل **للقرارات والذكاء الاصطناعي** — لا لعروض "استراتيجية بيانات" نظرية، بل للمدخلات الملموسة التي تحتاجها الوحدات الأخرى (Revenue وOperations وBrain وSupport وGovernance). تقع في طبقة الوحدات داخل [`docs/holding/DEALIX_HOLDING_OS.md`](../holding/DEALIX_HOLDING_OS.md)، ومقترنة بـ `docs/capabilities/data_capability.md`، ومتّصلة بموقف البحيرة في `docs/ops/DATA_LAKE_PLAYBOOK.md`.

## الوظيفة
تُقيّم Data الجاهزية، تُنفّذ تنظيفاً (إزالة التكرار، إصلاح الحقول المفقودة، وسم PII)، تنشر درجة جودة، وتُشغّل سبرنت ذكاء بيانات يُسلّم مجموعة بيانات جاهزة للتحليل داخل Core OS.

## الخدمات

| الخدمة | المدّة | المخرج |
|---|---|---|
| Data Readiness Assessment | 1–2 أسبوع | بطاقة جاهزية + خطة معالجة |
| Data Cleanup | 2–4 أسابيع | إزالة تكرار، إصلاح حقول، وسم PII، أصل |
| Data Quality Dashboard | 1–2 أسبوع | درجة جودة حيّة لكل مصدر |
| Data Intelligence Sprint | 3–6 أسابيع | مجموعة جاهزة + أول لوحات |

## وحدات المنتج داخل Core OS

| الوحدة | الوظيفة |
|---|---|
| Data Quality Score | درجة 0–100 لكل مصدر |
| Deduplication | إزالة تكرار قابلة للضبط بسجل تدقيق |
| PII Detection | وسم تلقائي + ربط سياسة احتفاظ |
| Source Registry | جرد، مالكون، حداثة، أصل |

## المؤشرات

- **درجة جاهزية البيانات** — مركّبة 0–100.
- **التكرارات المُزالة.**
- **خفض الحقول المفقودة** — % من الأساس.
- **مخاطر PII المعالَجة.**
- **حداثة الأنبوب** — % مجموعات داخل SLA.
- **الاستهلاك التالي** — # وحدات تستهلك المجموعة المُنظَّفة.

## اعتمادات Core OS

| الوحدة | الاستهلاك |
|---|---|
| Data OS | المستهلك الأصلي |
| Governance Runtime | احتفاظ + DSR |
| LLM Gateway | عند استخدام الذكاء في التنظيف (تحليل الكيانات) |
| Proof Ledger | Data Proof Pack |
| Capital Ledger | قواعد تنظيف، مجموعات إزالة تكرار، مخطّطات قطاع |
| AI Control Tower | كلفة وجودة التنظيف بالذكاء |
