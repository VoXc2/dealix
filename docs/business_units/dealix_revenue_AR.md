# Dealix Revenue — وحدة أعمال

**الطبقة:** Holding · Compound Holding Model
**المالك:** المدير العام لوحدة Revenue
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [dealix_revenue.md](./dealix_revenue.md)

## السياق
Dealix Revenue هي وحدة الأعمال التي تحوّل بيانات العملاء والفرص إلى خط أنابيب مؤهّل وإيراد مُغلَق. هي **الوحدة الإسفينية** للقابض — التي تفتح المحادثات مع الحسابات الجديدة، ثم تُسلّم العمل الأوسع إلى وحدات أخرى. تُنفّذ الاستراتيجية في `docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md`، وتقع في طبقة الوحدات داخل [`docs/holding/DEALIX_HOLDING_OS.md`](../holding/DEALIX_HOLDING_OS.md)، ومقترنة بمواصفة القدرة في `docs/capabilities/revenue_capability.md`.

## الوظيفة
تأخذ Dealix Revenue قائمة مستهدفة وCRM، تُسجِّل الحسابات، تصوغ التواصل التالي الصحيح بمصادر، تُدير الموافقة المحوكَمة، وتُسلّم مجموعة حسابات وصياغات جاهزة لخط أنابيب فريق الإيراد لدى العميل — كل ذلك قابل للتدقيق ومتوافق مع PDPL.

## الخدمات

| الخدمة | المدّة | المخرج |
|---|---|---|
| تشخيص الإيراد | 1–2 أسبوع | قائمة محسوبة، أعلى 25 فرصة، تحليل فجوة الأنبوب |
| سبرنت ذكاء العملاء المحتملين | 2–4 أسابيع | نموذج تسجيل في الإنتاج، Draft Pack حيّ، تدريب |
| تحويل التجربة | 1–3 أشهر | أول صفقات مغلقة من الصياغات، Proof Pack |
| Monthly RevOps OS (احتفاظ) | مستمر | تسجيل مستمر، تحديث الصياغات، تقارير الأنبوب |

يتوافق مع `docs/OFFER_LADDER_AND_PRICING.md` و`docs/COMPANY_SERVICE_LADDER.md`.

## وحدات المنتج داخل Core OS

| الوحدة | الوظيفة |
|---|---|
| Revenue Workspace | واجهة المستأجر لفريق الإيراد |
| Account Scoring | نموذج تسجيل قابل للضبط بموافقة الحوكمة |
| Draft Pack | صياغات تواصل بمصادر |
| Pipeline Board | قمع المراحل، المالك، الإجراء التالي |
| Revenue Proof Pack | أدلّة العائد لكل فوج |

## المؤشرات

- **حسابات مسجَّلة** — عدد الحسابات المعالَجة.
- **حسابات مؤهَّلة** — أعلى من العتبة ببيانات صحيحة.
- **قيمة الأنبوب** — SAR/USD أُضيفت.
- **إجراءات متابعة منجَزة** — صياغات معتمدة ومرسلة.
- **تحويل Sprint→Pilot** — % خلال 60 يوماً.
- **تحويل Pilot→Retainer** — % إلى احتفاظ شهري.

## اعتمادات Core OS

| الوحدة | كيف تستهلكها Revenue |
|---|---|
| Data OS | سجل المصادر، وسم PII، درجة الجودة |
| LLM Gateway | كل صياغة عبر Gateway، سجل البرومبت، تقييم |
| Governance Runtime | موافقة على الصياغات؛ تدقيق لكل إرسال |
| Proof Ledger | Revenue Proof Pack لكل مشروع |
| Capital Ledger | ميزات تسجيل، برومبتات، قوالب قطاع |
| AI Control Tower | كلفة لكل صياغة، معدّل التقييم، الانحراف |
| Client Workspace | واجهة المستأجر، RBAC |

## المقاييس
- **MRR للوحدة** — احتفاظ شهري متكرّر.

- **اكتمال التدقيق** — % إجراءات لها قيد تدقيق.
