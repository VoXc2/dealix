# امتثال البريد البارد — Dealix (المرحلة 8)

قواعد امتثال الرسائل الباردة: هوية مُرسِل صادقة، وسيلة إلغاء اشتراك، لا تزييف لـ Re:/Fwd:، لا عناوين مضلّلة، اسم مُرسِل أمين، ومراعاة السياق السعودي. الهدف تواصل محترم وشفّاف ومتوافق مع PDPL — لا أي تحايل على المرشّحات. الإرسال موقوف افتراضيًا (`send_enabled=false`).

---

## 1. هوية المُرسِل

| العنصر | القاعدة |
|--------|---------|
| اسم المُرسِل (from-name) | اسم حقيقي للشخص/الجهة، لا تنكّر |
| عنوان المُرسِل (from-address) | عنوان حقيقي على نطاق Dealix |
| Reply-To | عنوان رد حقيقي يُراقَب |
| التعريف بالجهة | ذكر Dealix بوضوح |

---

## 2. العنوان والمحتوى

| الممارسة | الحكم |
|----------|-------|
| تزييف `Re:` أو `Fwd:` لرسالة لم تبدأ | ممنوع |
| عنوان مضلّل عن مضمون الرسالة | ممنوع |
| ادعاءات غير صحيحة أو مبالغ فيها | ممنوع |
| إخفاء أنها رسالة تواصل أولى | ممنوع |
| عنوان أمين يصف الغرض | مطلوب |

> يجب أن يطابق العنوان مضمون الرسالة بصدق؛ كل ادعاء يستند إلى أعلى مستوى دليل أمين (`evidence_level`).

---

## 3. إلغاء الاشتراك الإلزامي

- كل رسالة باردة تحمل وسيلة إلغاء اشتراك واضحة.
- يُحترَم الطلب خلال مهلة معلنة ويُحوَّل إلى الاستبعاد.
- التفصيل: `docs/outreach/UNSUBSCRIBE_POLICY_AR.md` و`data/outreach/suppression_list.jsonl`.

---

## 4. السياق السعودي (KSA)

| الجانب | القاعدة |
|--------|---------|
| الإطار النظامي | نظام حماية البيانات الشخصية (PDPL) — سدايا |
| الأساس | بيانات شركات عامة وجهات اتصال عمل بدور وظيفي |
| تقليل البيانات | جمع الحد الأدنى اللازم فقط |
| اللغة والنبرة | احترافية، مناسبة لسياق الأعمال السعودي |
| القوائم المشتراة | ممنوعة منعًا باتًا |

التوافق التفصيلي: `docs/privacy/SAUDI_PDPL_OUTBOUND_POLICY_AR.md` و`company_os/governance/pdpl_checklist.md`.

---

## 5. قائمة تحقق سريعة قبل أي خطة

| البند | الحالة |
|-------|--------|
| اسم وعنوان مُرسِل صادقان | ☐ |
| عنوان أمين غير مضلّل | ☐ |
| لا تزييف Re:/Fwd: | ☐ |
| وسيلة إلغاء اشتراك واضحة | ☐ |
| تطبيق قائمة الاستبعاد | ☐ |
| لا قوائم مشتراة | ☐ |

البوابة الكاملة: `docs/outreach/DELIVERABILITY_READINESS_CHECKLIST.md`.

---

## 6. القواعد غير القابلة للتفاوض

1. لا تزييف للعناوين ولا اسم/عنوان مُرسِل مضلّل.
2. لا تحايل على المرشّحات ولا تجاوز لإلغاء الاشتراك.
3. لا ادعاءات زائفة ولا قوائم مشتراة.
4. تبقى `send_enabled=false` حتى الحكم ≥ `LIMITED_SEND_READY` وموافقة المؤسس.
