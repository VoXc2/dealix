# نظام تدرّج الإرسال المتحكَّم به — Dealix (المرحلة 8)

التدرّج كنظام تشغيلي محكوم: بوابة الحكم، الحد اليومي، موافقة المؤسس لكل خطوة، ومُحفّزات الإيقاف التلقائي. هذا المستند يحوّل خطة `docs/outreach/SENDING_RAMP_PLAN_AR.md` إلى ضوابط قابلة للتطبيق. الإرسال موقوف افتراضيًا (`send_enabled=false`).

مخططات البيانات: `schemas/email_account.schema.json` · `schemas/sending_batch.schema.json`.

---

## 1. الافتراضات الحاكمة

| المتغيّر | القيمة الافتراضية |
|----------|-------------------|
| `dry_run` | `true` |
| `approval_required` | `true` |
| `send_enabled` | `false` |
| `daily_cap` | `0` حتى الموافقة |

> الإرسال الفعلي مسموح فقط عند الحكم ≥ `LIMITED_SEND_READY` **و** موافقة المؤسس.

---

## 2. بوابة الحكم (Verdict Gating)

| الحكم | السماح بالإرسال | الحد اليومي |
|-------|-----------------|-------------|
| `NOT_READY` | لا | 0 |
| `DRY_RUN_ONLY` (افتراضي) | لا — خطط ومسودات فقط | 0 إرسالة |
| `LIMITED_SEND_READY` | محدود بعد موافقة | حسب خطوة الأسبوع 0–1 |
| `RAMP_READY` | نعم ضمن الخطوة | حسب الجدول الأسبوعي |
| `PAUSE_REQUIRED` | لا — إيقاف فوري | 0 |

تعريفات الأحكام: `docs/gtm/MARKET_PRODUCTION_NAMING_CONVENTIONS.md`.

---

## 3. الحد اليومي لكل خطوة

| الأسبوع | الحد اليومي للإرسال | شرط التفعيل |
|---------|---------------------|-------------|
| 0 | 0–20 | تهيئة + موافقة |
| 1 | 25–50 | استقرار + موافقة |
| 2 | 50–100 | استقرار + موافقة |
| 3 | 100–150 | استقرار + موافقة |
| 4+ | 150–250 | مؤشرات صحية + موافقة |

> 250 **مسودة**/يوم مسموحة دائمًا؛ 250 **إرسالة**/يوم ممنوعة قبل اجتياز كل البوابات.

---

## 4. موافقة المؤسس لكل خطوة

1. تقييم الحكم عبر `docs/outreach/DELIVERABILITY_READINESS_CHECKLIST.md`.
2. التحقق من مؤشرات الصحة للخطوة الحالية.
3. موافقة صريحة من المؤسس (`approved_by`) قبل رفع `daily_cap`.
4. تفعيل الخطوة التالية فقط بعد التوثيق في `reports/outreach/SENDING_RAMP_READINESS.md`.

---

## 5. مُحفّزات الإيقاف التلقائي

| المؤشر | الحد | الإجراء التلقائي |
|--------|------|------------------|
| معدل الارتداد | ≥ 2% | `PAUSE_REQUIRED` · `send_enabled=false` · `daily_cap=0` |
| شكاوى السبام | ≥ 0.1% | `PAUSE_REQUIRED` · `send_enabled=false` · `daily_cap=0` |
| سمعة Postmaster | منخفضة | `PAUSE_REQUIRED` · إيقاف الخطوة |
| كسر المصادقة | SPF/DKIM/DMARC فشل | `NOT_READY` · إيقاف الإرسال |

إجراء الإيقاف الكامل: `docs/outreach/DOMAIN_HEALTH_RUNBOOK.md`.

---

## 6. القواعد غير القابلة للتفاوض

1. لا رفع للحد اليومي دون موافقة المؤسس لتلك الخطوة.
2. لا تجاوز لبوابة الحكم؛ `DRY_RUN_ONLY` = صفر إرسالات.
3. لا قوائم مشتراة ولا تحايل على المرشّحات.
4. تبقى `send_enabled=false` حتى الحكم ≥ `LIMITED_SEND_READY` وموافقة المؤسس.
