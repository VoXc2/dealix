# Delivery Lead SOP

## Responsibility

Own scope-to-delivery: timeline, client comms, artifact quality, and handoff to proof/QA.

## Must check before kickoff

- Scope file matches sold sprint (`00_scope.md` / SOW)
- Data request sent; governance baseline understood
- Success metric written in one sentence

## During delivery

- Update `DELIVERY_COMMAND.md` at least twice weekly
- Log material governance decisions in `governance_events.md`

## Closure

- Ensure `POST_PROJECT_REVIEW.md` is filled
- File anonymized proof in `docs/assets/proof_packs/` when applicable
- Propose playbook delta for the sector file in `docs/playbooks/`

## Authority

Can pause delivery if data access or approvals block safe work — escalate with a written reason.
