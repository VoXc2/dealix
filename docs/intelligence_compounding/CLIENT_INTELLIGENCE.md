# Client Intelligence — ذكاء العميل

## الأسئلة

- أي عميل يستحق التوسع؟ أي عميل في خطر؟  
- أي عميل يحتاج تمكينًا؟ أي عميل جاهز لـ retainer؟

## مقاييس مرجعية (تغطية)

مثال: Client Health، Adoption، Proof، Governance alignment، Expansion readiness، Data readiness، Stakeholder engagement — راجع `client_intelligence.py`.

## قواعد قرار تشغيلية (أمثلة)

- Proof عالي + Adoption عالي → **Retainer**  
- Proof عالي + Adoption ضعيف → **Enablement**  
- Proof ضعيف + Data ضعيفة → **Data Readiness**  
- مخاطر حوكمة عالية → **Governance Review**

## ربط تجاري

تتقاطع مع `client_os` و`proof_architecture_os` و`value_capture_os` — الوثيقة هنا تصف **ماذا نتعلم**؛ الكود في تلك الحزم يطبّق البوابات حيث وُجد.
