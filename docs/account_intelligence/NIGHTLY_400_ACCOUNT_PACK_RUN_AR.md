# Nightly 400 Account Pack Run — التشغيل الليلي لـ 400 حزمة

> كل ليلة: يُنتج النظام حتى **400 Account Pack** جاهزة للمراجعة صباحًا. الناتج **مسودات** — لا إرسال آلي.

---

## 1. التوزيع الليلي على الأنظمة (400)

| النظام | الحصة |
|--------|------:|
| Revenue Operating System | 100 |
| Follow-up Recovery OS | 90 |
| Executive Command OS | 70 |
| WhatsApp Client OS | 70 |
| Proposal & Proof OS | 70 |
| **الإجمالي** | **400** |

> التوزيع قابل للضبط حسب الأداء الأسبوعي (Learning Loop). يبدأ مائلاً نحو Revenue و Follow‑up لأنهما الأسرع في إثبات القيمة.

---

## 2. مراحل التشغيل (Run Stages)

| المرحلة | المدخل | المخرج | البوابة |
|---------|--------|--------|---------|
| 1. Seed | قطاعات/مدن مستهدفة | قائمة شركات مرشحة | لا تكرار، لا suppression list |
| 2. Intelligence | معلومات عامة | حقول الشركة | بيانات عامة فقط |
| 3. Contact Discovery | الموقع/السوشيال | قنوات + دور + ثقة | لا اختراع جهات اتصال |
| 4. Need + Fit | إشارات عامة | الألم + النظام | نظام واحد فقط |
| 5. Drafting | كل ما سبق | إيميل + Call Brief + زاوية عرض | لغة احتمالية لـ L0/L1، لا ادعاءات |
| 6. Scoring | الحزمة | Account Score + Cash Priority | المخطط صالح |
| 7. Gating | الدرجات | شريحة + قبول/رفض | شروط الاستبعاد |
| 8. Queues | الحزم المقبولة | Top 100 / Send 20 / Call 30 | جاهز للاعتماد البشري |

---

## 3. بوابات الجودة (Quality Gates)

تُرفض الحزمة أو تُعاد للبحث إذا:

```
- لا يوجد recommended_system
- لا يوجد evidence_level
- لا يوجد best_contact_route
- risk_level = high
- النص يحتوي ادعاءً مضمونًا (نضمن / 10x)
- L0/L1 بدون لغة احتمالية
- الشركة في suppression list
```

---

## 4. الطوابير الناتجة (Output Queues)

| الطابور | المعيار | الملف |
|---------|---------|-------|
| Top 100 | Account Score ≥ 75 ومستوفٍ للبوابات | `reports/account_intelligence/TOP_100_ACCOUNT_QUEUE.md` |
| Top 20 Send | الأعلى جاهزية للإرسال (قناة + دليل) | `reports/founder/DAILY_SUPER_COMMAND.md` |
| Top 30 Call | الأعلى ملاءمة للاتصال | `reports/founder/DAILY_SUPER_COMMAND.md` |
| Mini Proposals Ready | حزم باعتماد جاهزة لعرض | `reports/proposals/MINI_PROPOSAL_QUEUE.md` |
| Missing Contacts | حزم بقناة C0/C1 | `reports/contacts/MISSING_CONTACTS_REVIEW.md` |

---

## 5. تشغيل الفحص (Run the Check)

```bash
npm run factory:check
```

النتيجة الحالية على العيّنة (8 حزم تمثيلية بدل 400): راجع
`reports/account_intelligence/NIGHTLY_400_ACCOUNT_PACKS_REPORT.md`.

> **ملاحظة تشغيلية:** البيانات الحالية **عيّنة تمثيلية** (8 شركات توضيحية، أسماء غير حقيقية، بلا بيانات شخصية) لإثبات أن خط الإنتاج والعقود والتحقق تعمل من طرف إلى طرف. التوسع إلى 400 يتم بزيادة قائمة البذور مع الحفاظ على نفس البوابات.

## 6. حدود الواقعية (ماذا نضمن جمعه؟)

| غالبًا متاح | أحيانًا | لا يُضمن |
|-------------|---------|----------|
| اسم الشركة، الموقع، القطاع | اسم صاحب القرار | جوال شخصي مباشر |
| الخدمات الظاهرة، صفحة التواصل | رقم/إيميل عام منشور | إيميل شخصي مباشر |
| روابط اجتماعية عامة | إشارات توظيف/توسع | ميزانية الشركة |
| الدور الأنسب، الألم المحتمل | أخبار حديثة | قرار الشراء الحقيقي |

عند غياب جهة اتصال شخصية: نستهدف **الدور** ونوجّه عبر صفحة التواصل الرسمية. لا اختراع.
