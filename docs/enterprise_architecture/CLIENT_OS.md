# Client OS — تجربة العميل ومساحة العمل

## الهدف

مساحة عمل لكل عميل تعرض **الجاهزية، الحوكمة، المسودات، الإثبات، القيمة، الإجراءات التالية** — بدون إخفاء حالة الحوكمة.

## Client Workspace MVP (مرجعي)

ملف العميل، درجة القدرة، جاهزية البيانات، جدول ليدز/سير عمل، حالة الحوكمة، حزمة مسودات، حزمة إثبات، إجراءات تالية.

## لاحقًا

مركز موافقات، خط زمني للإثبات، تصديرات تدقيق، صلاحيات أدوار، لوحة تنفيذية.

## التنفيذ في الريبو

- `auto_client_acquisition/client_os/` — لوحات مثل `data_readiness_panel.py`, `governance_panel.py`, `proof_timeline.py`.
- الواجهة: `frontend/` — بوابة العميل/المشغل (مع قراءة عربية حيث طُبّق).
