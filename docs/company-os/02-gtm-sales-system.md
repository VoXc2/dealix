# GTM and Sales System

## Purpose
Build a repeatable revenue system for Dealix.

## ICP rule
Only work accounts that match the approved ICP unless founder explicitly approves an exception.

## Funnel stages

| Stage | Definition | Exit criteria | Owner |
|---|---|---|---|
| Target | Account matches ICP | Added to target list | Sales Ops |
| Researched | Account need and context known | Research note complete | Sales |
| Contacted | Approved message sent or intro requested | Follow-up date set | Sales |
| Meeting | Qualified meeting booked | Discovery completed | Sales |
| Diagnostic | Pain, urgency, owner, impact mapped | Business case draft | Founder/Sales |
| Proposal | Scope, price, timeline, success metric shared | Decision process known | Sales |
| Commit | Buyer confirms next step | Contract path started | Founder/Sales |
| Won | Agreement signed | Onboarding scheduled | Sales/CS |
| Lost | Not moving forward | Reason documented | Sales |

## Sales assets

- ICP card
- Account research note
- Discovery script
- Objection handling library
- Proof pack
- Pricing sheet
- Proposal template
- Mutual action plan
- Follow-up templates

## Discovery questions

1. What workflow is currently broken?
2. Why does it matter now?
3. What happens if nothing changes?
4. Who owns the result?
5. What system or process is used today?
6. What would a successful outcome look like?
7. What data or approval is required?
8. What could block the decision?
