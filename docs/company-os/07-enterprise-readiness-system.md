# Enterprise Readiness System

## Purpose
Prepare Dealix for serious B2B buyers who require trust, process, proof, security, and procurement readiness.

## Enterprise readiness map

| Area | Required artifact | Owner | Status |
|---|---|---|---|
| Company profile | Company one-pager | Founder | TBD |
| Product | Product overview and architecture note | Product | TBD |
| Security | Security overview and access policy | Engineering | TBD |
| Privacy/data | Data handling note | Ops/Security | TBD |
| Legal | MSA, SOW, NDA, DPA templates | Founder/Legal | TBD |
| GTM | Proof pack and case notes | Sales/Marketing | TBD |
| Support | SLA and escalation path | CS/Ops | TBD |
| Implementation | Onboarding and implementation plan | CS/Product | TBD |

## Buyer roles

| Role | Concern | Asset needed |
|---|---|---|
| Founder/CEO | Strategic value and trust | Executive narrative |

| Operations leader | Workflow reliability | Process map |
| Technical reviewer | Security and integrations | Technical note |
| Legal/procurement | Terms and data | Contract kit |

## Enterprise red flags

- Buyer has no clear owner.
- Buyer wants custom-only work.
- Legal/security requirements exceed current maturity.
- No measurable success criteria.
- No timeline or decision process.
- Deal distracts from core ICP.

## Output

Every enterprise opportunity should have:

- Deal review issue
- Account research note
- Mutual action plan
- Deal risk score
- Proof or evidence pack
- Implementation outline
