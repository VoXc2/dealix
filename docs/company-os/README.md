# Dealix Company OS

## Purpose
This folder contains the complete execution layer for Dealix as a company, not just a product.

Use it to operate the business across strategy, GTM, sales, customer success, data, AI, security, finance, risk, and executive decisions.

## Operating layers

1. Company Command Center
2. GTM and Sales System
3. Customer Success System
4. Data Governance System
5. Security and Incident System
6. Risk and Decision System
7. Enterprise Readiness System
8. KPI and Dashboard System
9. Weekly Business Review
10. Launch and Release Governance

## How to use

- Use `01-company-command-center.md` every week.
- Use `02-gtm-sales-system.md` to drive pipeline.
- Use `03-customer-success-system.md` after first customer contact or pilot.
- Use `04-data-governance-system.md` before building dashboards or AI flows.
- Use `05-security-incident-system.md` before production or customer-facing releases.
- Use `06-risk-decision-system.md` for founder-level decisions.
- Use `07-enterprise-readiness-system.md` before serious B2B sales.
- Use `08-kpi-dashboard-system.md` as the company metric source.
- Use `09-weekly-business-review.md` every week.
- Use `10-operating-calendar.md` to keep cadence.

## Rule
Every system must produce an artifact. If there is no artifact, the work is not operational.
