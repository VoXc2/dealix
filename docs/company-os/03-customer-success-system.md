# Customer Success System

## Purpose
Turn every customer into proof, retention, expansion, and product learning.

## Customer lifecycle

| Stage | Goal | Owner | Output |
|---|---|---|---|
| Handoff | Transfer sales context to CS | Sales/CS | Handoff note |
| Kickoff | Confirm success criteria | CS | Kickoff plan |
| Onboarding | Reach first value fast | CS/Product | 7-day plan |
| Adoption | Make workflow repeatable | CS | Usage review |
| Proof | Capture measurable outcome | CS/Marketing | Proof note |
| Renewal | Protect ongoing value | CS/Sales | Renewal plan |
| Expansion | Find next workflow/account | CS/Sales | Expansion note |

## Health score

| Signal | Green | Yellow | Red |
|---|---|---|---|
| Usage | Core workflow used weekly | Irregular use | No meaningful use |
| Outcome | Value visible | Value unclear | No value proof |
| Engagement | Active champion | Slow responses | No sponsor |
| Support | Issues resolved | Repeated issues | Critical unresolved issue |
| Renewal risk | Low | Medium | High |

## Save plan for red accounts

1. Assign owner.
2. Contact customer within 48 hours.
3. Identify blocker.
4. Offer focused success path.
5. Escalate product or founder if needed.
6. Document next action and review date.

## Proof capture

Every successful customer should produce:

- Before state
- After state
- Time saved
- Revenue or pipeline effect
- Risk reduced
- Workflow improved
- Customer quote if approved
- Evidence source

## Weekly CS review

- Green/yellow/red customers
- Onboarding blockers
- Product feedback
- Proof captured
- Expansion signals
- Renewal risks
- Escalations
