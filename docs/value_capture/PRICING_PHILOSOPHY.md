# Pricing Philosophy

Price for **operating capability transformation**, not hours.

## Reflect

- Problem value
- Data sensitivity
- Workflow complexity
- Governance level
- Expected proof
- Retainer potential
- Module reuse
- Client tier (SME / mid-market / enterprise)

## Do not price by

- Row count
- Message count
- Page count
- Hours of work

These are delivery measures, not value measures.

## Principle

> The price is the capability, not the labor.
