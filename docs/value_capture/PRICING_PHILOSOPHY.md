# Pricing Philosophy

لا تسعّر «ساعات» أو «صفوف» فقط — سعّر **تحويل قدرة تشغيلية**.

## يجب أن يعكس السعر

قيمة المشكلة · حساسية البيانات · تعقيد workflow · مستوى الحوكمة · قوة proof المتوقعة · إمكانية retainer · إعادة استخدام module · قطاع العميل (SME / mid-market / enterprise).

## لا يكفي وحده

عدد الصفوف · الرسائل · الصفحات · ساعات التنفيذ — مقاييس تشغيل لا مقاييس قيمة وحدها.
