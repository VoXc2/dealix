# Dealix Scale Gates

لا تنتقل من مرحلة لأخرى إلا إذا **البوابة** متحققة. تقييم آلي أولي: `auto_client_acquisition/scale_os/scale_gates.py`.

---

## Gate 1: Founder-led → Productized

- نفس الخدمة بيعت **2–3 مرات**  
- نفس **الألم** تكرر  
- نفس **المخرجات** تكررت  

---

## Gate 2: Productized → Tool-assisted

- خطوة يدوية تكررت **3 مرات**  
- تأخذ **أكثر من ساعتين**  
- تؤثر على **الجودة** أو **الهامش**  

---

## Gate 3: Tool-assisted → Team-assisted

- Checklists واضحة  
- QA واضح  
- الهامش يسمح  
- التسليم **لا يحتاج** معرفة المؤسس بالكامل  

---

## Gate 4: Team-assisted → Partner-assisted

- معيار تسليم **ثابت**  
- Trust Pack جاهز  
- **شهادة شريك** (certification) موجودة  
- **حقوق تدقيق** واضحة  

---

## Gate 5: Partner-assisted → Platform-assisted

- العميل يحتاج **workspace**  
- المخرجات **متكررة**  
- الموافقات / التقارير **متكررة**  

---

## Gate 6: Platform → Academy

- **10+** مشاريع  
- **3+** حالات proof-backed  
- Method **مستقر**  
- قوالب **ثابتة**  

---

## Gate 7: Academy → Venture

- وحدة لديها: عملاء · retainers · product module · owner · playbook  

---

## Scale Risk by Stage

| Stage | Risk | Control |
|-------|------|---------|
| Founder-led | founder bottleneck | document everything |
| Productized | fake standardization | QA gates |
| Tool-assisted | overbuilding | productization gate |
| Team-assisted | quality drop | training + QA |
| Partner-assisted | brand damage | certification + audit |
| Platform-assisted | unused software | client pull first |
| Academy | premature content | proof-backed curriculum |
| Ventures | fragmentation | Core OS dependency |
| Standards | empty thought leadership | real benchmarks |
