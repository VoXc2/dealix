# Dealix Operating Doctrine — دوران الاستراتيجية

1. **Measure before scaling** — لا توسّع خدمة قبل proof وmargin وQA.  
2. **Govern before automating** — لا أتمتة بلا مالك workflow ومخاطر وموافقات.  
3. **Prove before promising** — لا claim بلا proof.  
4. **Productize after repetition** — لا feature بلا تكرار مثبت.  
5. **Compound after delivery** — كل مشروع يترك **أصلًا**.  
6. **Spin out after maturity** — لا venture بلا عملاء وretainers وplaybook.
