# نتيجة تخصيص رأس المال — الذكاء · الدماغ التشغيلي

**الطبقة:** Intelligence · الدماغ التشغيلي
**المالك:** الرئيس التنفيذي + المدير المالي
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [CAPITAL_ALLOCATION_SCORE.md](./CAPITAL_ALLOCATION_SCORE.md)

## السياق
نتيجة تخصيص رأس المال هي الطريقة التي تُقرّر بها Dealix أين تستثمر الوقت
والمال المحدودَين بين الفرص — خطوط الخدمات والبناءات الداخلية وبرامج
الشركاء والتجارب. تُنتج رقماً وحيداً لكل فرصة يتطابق مع نطاق قرار،
بالإضافة إلى توزيع ثابت لإجمالي رأس المال على خمسة دلاء. بهذه الطريقة
تقول Dealix "لا" للمشتّتات الجاذبة و"نعم" للرهانات المُتراكِمة. راجع
`docs/V14_COMPREHENSIVE_STRATEGIC_PLAN.md` و
`docs/UNIT_ECONOMICS_AND_MARGIN.md` و
`docs/OFFER_LADDER_AND_PRICING.md` للاقتصاديات المُغذّية.

## صيغة نتيجة الأولوية
```
Priority Score =
  revenue_score * 0.25
+ repeatability_score * 0.20
+ proof_score * 0.20
+ productization_score * 0.15
+ strategic_moat_score * 0.10
+ risk_adjusted_score * 0.10
```

كل مكوّن على مقياس 0-100.

### تعريفات المكوّنات
- **revenue_score** — مساهمة الهامش الإجمالي المتوقّعة 12 شهراً، مُعيَّرة
  مقابل الخط الأعلى الحالي.
- **repeatability_score** — نسبة سير العمل المُقَوْلَب أو المُمَنتج
  (من Capital Ledger + Productization Ledger).
- **proof_score** — عدد وتنوّع أحداث الإثبات لكل مشروع (من Proof Ledger).
- **productization_score** — معدّل إعادة استخدام الوحدات + استخدام الأدوات
  الداخلية (من Productization Ledger).
- **strategic_moat_score** — القدرة الدفاعية — البيانات، الوصول التنظيمي،
  التوزيع، الموقع الفئوي للعلامة.
- **risk_adjusted_score** — `100 - delivery_risk - regulatory_risk -
  concentration_risk`، كل واحد على 0-30.

## نطاقات القرار
| النتيجة | النطاق | الفعل الافتراضي |
|---|---|---|
| 85-100 | استثمار / توسيع | تخصيص رأس مال، تعيين مالك، عرض موسّع |
| 70-84 | بناء بحذر | تمنتج، تجربة مع 2-3 عملاء |
| 55-69 | تجربة فقط | تجربة مدفوعة واحدة، بلا التزام خارطة طريق |
| 40-54 | إبقاء | الصيانة، بلا رأس مال جديد |
| < 40 | إيقاف | إيقاف الخدمة، كتابة الدروس في الذاكرة الاستراتيجية |

## المثال "أ" — ذكاء الإيراد (توسيع)
المدخلات (0-100):
- revenue_score = 90 (خط إيراد مُثبت لـ12 شهراً، هامش صحّي)
- repeatability_score = 80 (سير عمل مُقَوْلَب، وحدة تقييم حسابات)
- proof_score = 90 (إثباتات قيمة إيراد متعدّدة لكل عميل)
- productization_score = 75 (وحدة Revenue OS مستخدمة في 4 مشاريع)
- strategic_moat_score = 70 (بيانات + Playbook B2B، فئة MENA)
- risk_adjusted_score = 80 (مخاطر تسليم وتنظيمية منخفضة)

الحساب:
```
Priority Score =
   90 * 0.25  = 22.5
 + 80 * 0.20  = 16.0
 + 90 * 0.20  = 18.0
 + 75 * 0.15  = 11.25
 + 70 * 0.10  =  7.0
 + 80 * 0.10  =  8.0
 -----------------
            = 82.75 → 83
```
الحكم: **بناء بحذر (قمّة النطاق، على حافة التوسيع)** — يُرقّى إلى التوسيع
بمجرد إغلاق الاحتفاظين القادمين (مما يرفع revenue_score والـmoat).

## المثال "ب" — شات بوت مُخصّص لعميل واحد (إيقاف)
المدخلات:
- revenue_score = 30 (مرّة واحدة، هامش منخفض)
- repeatability_score = 10 (تدفّقات مخصّصة)
- proof_score = 15 (لا قيمة قابلة للقياس وراء تقليل التذاكر)
- productization_score = 5 (لا وحدة قابلة لإعادة الاستخدام)
- strategic_moat_score = 20 (لا قدرة دفاعية)
- risk_adjusted_score = 50 (مخاطر تسليم + مخاطر تركّز)

الحساب:
```
Priority Score =
   30 * 0.25  =  7.5
 + 10 * 0.20  =  2.0
 + 15 * 0.20  =  3.0
 +  5 * 0.15  =  0.75
 + 20 * 0.10  =  2.0
 + 50 * 0.10  =  5.0
 -----------------
            = 20.25 → 20
```
الحكم: **إيقاف** — تُكتب الدروس في `STRATEGIC_MEMORY.md` ويُرفض ما
يشبهها وفق `ANTI_AGENCY_RULES.md`.

## دلاء رأس المال
إجمالي رأس المال — وقت المؤسس + الفريق + النقد — مُوزَّع بحصص ثابتة على
خمسة دلاء. تُعاد الترجيح شهرياً في مراجعة تخصيص رأس المال.

| الدلو | الحصة المستهدفة | الغرض |
|---|---|---|
| Cash Engine | 40% | خطوط الإيراد الحالية، التجارب المدفوعة، الاحتفاظات |
| Core OS | 25% | المنصّة، الوكلاء، السجلات، الحوكمة |
| Proof + Capital Assets | 15% | حِزم الإثبات، دراسات الحالة، الأصول القابلة لإعادة الاستخدام |
| Growth + Partners | 10% | التسويق، برنامج الشركاء، تطوير الأعمال |
| Labs + Experiments | 10% | خطوط جديدة قيد التجربة أو دون 70 |

قواعد إعادة التوزيع:
- Cash Engine لا تنزل عن 30%.
- Labs + Experiments لا تتجاوز 15%.
- فرصة جديدة في نطاق التوسيع قد تنقل مؤقتاً حتى 5% من Labs إلى Cash
  Engine لدورة واحدة.

## إيقاع التقييم
- لكل فرصة عند الاستقبال.
- إعادة تقييم شهرية لكل فرصة نشطة في السجل.
- إعادة تقييم خلال 7 أيام من حدث مادي (إثبات، فوز احتفاظ، خسارة، حادثة
  حوكمة).

## الحوكمة
- التقييم يملكه رئيس الديوان باعتماد الرئيس التنفيذي + المدير المالي.
- تُحفظ النتائج في Venture Signal Ledger مع الأوزان والمدخلات والمبرّر.
- الخلافات تُصعَّد إلى حزمة القرار التنفيذية
  (`docs/EXECUTIVE_DECISION_PACK.md`).

## الواجهات
| المدخلات | المخرجات | المالكون | الإيقاع |
|---|---|---|---|
| سجلات Proof، Capital، Unit Performance | نتيجة أولوية لكل فرصة | رئيس الديوان | شهري + عند الحدث |
| نتائج الأولوية | نطاق القرار + الفعل | الرئيس التنفيذي + المدير المالي | شهري |
| نطاق القرار | إعادة توزيع الدلاء | الرئيس التنفيذي + المدير المالي | شهري |
| خطّة الدلاء | التوظيف والإنفاق والوقت | الرؤساء | شهري |

## المقاييس
- **تغطية التقييم** — نسبة الفرص النشطة بنتيجة حالية (الهدف: 100%).
- **استقرار التقييم** — نسبة النتائج المتحرّكة > 15 نقطة خلال 30 يوماً
  (الهدف: < 20%).
- **انضباط الإيقاف** — نسبة الفرص < 40 المُوقَفة خلال 30 يوماً
  (الهدف: 100%).
- **انجراف الدلو** — التباين عن الحصة المستهدفة شهرياً (الهدف: ≤ 3 نقاط).
