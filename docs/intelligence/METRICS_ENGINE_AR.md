# محرّك المقاييس — الذكاء · الدماغ التشغيلي

**الطبقة:** Intelligence · الدماغ التشغيلي
**المالك:** رئيس البيانات
**الحالة:** مسودة
**آخر مراجعة:** 2026-05-13
**النسخة الإنجليزية:** [METRICS_ENGINE.md](./METRICS_ENGINE.md)

## السياق
يُحوّل محرّك المقاييس قيود السجلات إلى مؤشرات تستطيع طبقة القرار التحرّك
عليها. لكل سجل مجموعة مقاييس مُختارة، ولكل مقياس تعريف ووحدة وإيقاع
وقاعدة قرار. هذا الملف هو الكتالوج لما تراقبه Dealix ولماذا. بدون كتالوج
موحّد، تنحرف الشركة إلى لوحات مرتجلة يُحسب فيها المؤشر ذاته بثلاث طرق،
فتفقد القرارات الموثوقية. راجع `docs/BUSINESS_KPI_DASHBOARD_SPEC.md`
للوحة المؤشرات المُسطّحة، و`docs/FINANCE_DASHBOARD.md` للسطح المالي،
و`docs/AI_OBSERVABILITY_AND_EVALS.md` لتعريفات مقاييس تشغيل الذكاء.

## مبادئ المحرّك
- **تعريف واحد لكل مقياس.** لكل مقياس تعريف SQL وحيد في مستودع
  المقاييس. لا أحد يُعيد حسابه بنسخته الخاصة.
- **مُجذَّر في السجلات.** كل مقياس يُحلّ إلى سجل أو أكثر من
  `LEDGER_ARCHITECTURE.md`. لا قراءة من قواعد بيانات التطبيقات.
- **مُصدَّر.** التعريفات مُصدَّرة؛ أي تعديل يستلزم إعلان طبقة القرار
  ونافذة سماح 7 أيام.
- **مُصنَّف بالإيقاع.** لكل مقياس صنف تحديث — مستمر، ساعي، يومي،
  أسبوعي — حسب حساسية قاعدة القرار.
- **مرتبط بقرار.** كل مقياس موجود لأن قاعدة قرار تحتاجه. المقاييس
  اليتيمة تُهجَّر.

## الكتالوج حسب السجل

### مقاييس AI Run
المصدر: AI Run Ledger.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| AI cost per project | sum(`cost_usd`) حسب `project_id` | ساعي | تجاوز الميزانية → التحويل لطبقة أرخص |
| AI cost per workflow | sum(`cost_usd`) حسب `agent_id` + وسم سير العمل | يومي | الربع الأعلى → جري تحسين |
| QA pass rate | نسبة `eval_verdict=pass` | يومي | < 90% → إيقاف الوكيل وفتح Eval |
| Schema failure rate | نسبة `schema_valid=false` | مستمر | > 2% خلال 7 أيام → إيقاف |
| Model fallback rate | نسبة تشغيلات النموذج الاحتياطي | يومي | > 15% مستدام → إعادة تدريب المُوجّه |
| High-risk run count | عدد التشغيلات عالية الخطورة | ساعي | > 0 → موافقة مطلوبة |

### مقاييس الحوكمة
المصدر: Audit Ledger.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| Blocked actions | العدد حسب مرجع السياسة | يومي | ارتفاع → مراجعة سياسة |
| Approval delays | الوسيط من `approval_required` إلى `approval_granted` | يومي | > 24 ساعة → تصعيد |
| PII flags | عدد أحداث `pii_detected` | مستمر | > 0 → مراجعة حوكمة |
| Source attribution coverage | نسبة المخرجات بمصدر قابل للتحقّق | يومي | < 95% → فحص المصدر |
| Audit coverage | نسبة أفعال الوكلاء المُدقَّقة | يومي | < 100% → حظر النشر |
| Policy violations | عدد فحوصات الحوكمة الفاشلة | مستمر | > 0 → حادثة |

### مقاييس الإثبات
المصدر: Proof Ledger.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| Proof packs per project | عدد أحداث الإثبات حسب `project_id` | أسبوعي | < 2 → مراجعة تسليم |
| Proof events per project | إجمالي الإثباتات المُعتمدة من العميل | أسبوعي | ≥ 3 عبر ≥ 3 مشاريع → ترقية |
| Value categories covered | `proof_type` مميّز لكل خط خدمة | أسبوعي | < 3 → تنويع الإثبات |
| Client-approved proof count | إثباتات `client_approved=true` | أسبوعي | اتجاه → قائمة دراسات حالة |
| Proof-to-retainer conversion | فوز احتفاظ / حِزم إثبات مُسلّمة | شهري | < 30% → تصحيح العرض |

### مقاييس رأس المال
المصدر: Capital Ledger.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| Assets per project | العدد حسب `project_id` | أسبوعي | < 2 → مراجعة تسليم |
| Reusable assets ratio | القابل / الإجمالي | شهري | < 60% → دفع التمنتج |
| Playbook updates | عدد أحداث `playbook_updated` | أسبوعي | لا شيء → فجوة ذاكرة |
| Feature candidates | المرشّحون المفتوحون في سجل التمنتج | أسبوعي | أعلى 3 → خارطة طريق |
| Market-safe insights | عدد الأصول بـ `market_safe=true` | شهري | > 0 → قائمة محتوى |

### مقاييس المنتج
المصدر: Productization Ledger + Capital Ledger + AI Run Ledger.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| Manual hours saved | الساعات المُقدَّر توفيرها من الوحدات المُطلَقة | شهري | أقل من المتوقّع → دفع تبنّي |
| Feature reuse rate | نسبة الوحدات المستخدَمة في ≥ 3 مشاريع | شهري | < 50% → إيقاف أو تقليص نطاق |
| Internal tool usage | المستخدمون النشطون لكل أداة داخلية | أسبوعي | < 50% من الفريق → تبنّي |
| Module adoption | معدّل التبنّي الخارجي | شهري | < 20% → إعادة تغليف |
| Delivery time reduction | الوسيط مقابل المرجع | ربعي | < 10% → مراجعة عملية |

### مقاييس وحدات العمل
المصدر: Unit Performance Ledger + ضم متعدّد السجلات.

| المقياس | التعريف | الإيقاع | قاعدة القرار |
|---|---|---|---|
| Unit revenue | sum(`revenue`) لكل `unit_id` | شهري | دون الهدف → خطة إيراد |
| Unit margin | نسبة هامش الوحدة | شهري | < الهدف → مراجعة تكلفة |
| Unit QA | متوسط `qa_score` | شهري | < الهدف → تدريب تسليم |
| Unit proof count | العدد من Proof Ledger | شهري | < 2 لكل مشروع → انضباط إثبات |
| Unit retainer count | عدد `retainer_won` | شهري | اتجاه نزولي → مراجعة تسعير/ICP |
| Unit product maturity | درجة التمنتج 0-100 | ربعي | ≥ 70 → مراجعة ترقية |
| Unit playbook maturity | درجة تغطية الـPlaybook 0-100 | ربعي | ≥ 70 → مراجعة ترقية |

## خريطة الإيقاع
| الصنف | أمثلة | المُحرّك |
|---|---|---|
| مستمر | Schema failure rate، PII flags | عند كل حدث ذي صلة |
| ساعي | AI cost per project، High-risk runs | دفعة ساعية |
| يومي | QA pass rate، Blocked actions | 02:00 الرياض |
| أسبوعي | Proof packs، Capital assets | الإثنين 06:00 |
| شهري | Unit margin، Product adoption | الأول 06:00 |
| ربعي | درجات نضج الوحدة | أول كل ربع 06:00 |

## دورة حياة المقياس
1. **اقتراح** — يقدّم المالك التعريف + قاعدة القرار.
2. **تحقّق** — يفحص رئيس البيانات الجدوى وتكلفة الضم.
3. **نشر** — يستقر التعريف في مستودع المقاييس، مُصدَّراً.
4. **ربط** — تربط طبقة القرار القاعدة.
5. **مراجعة** — مراجعة ربعية؛ تُهجَّر المقاييس اليتيمة.

## المقاييس
- **حداثة المقياس** — نسبة المقاييس ضمن SLO إيقاعها (الهدف: ≥ 99%).
- **انحراف التعريف** — عدد المستهلكين الذين يحسبون مقياساً خارج المحرّك
  (الهدف: 0).
- **عدد المقاييس اليتيمة** — مقاييس بلا قاعدة قرار (الهدف: 0).
- **زمن إعادة الحساب** — p95 لكل صنف.
