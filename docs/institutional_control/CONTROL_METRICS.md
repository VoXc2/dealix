# Institutional Control Metrics

Measure the governance, not just the output.

## 1. Governance metrics

- `% data sources with Source Passport`
- `% AI runs logged`
- `% outputs with governance status`
- `% client outputs with QA`
- `% external actions with approval`
- `PII flags detected`
- `unsafe actions blocked`
- `audit coverage`
- `proof pack completion rate`

## 2. Institutional thresholds

| Metric | Threshold | Decision |
| --- | --- | --- |
| AI run logging | < 100% | Not enterprise-ready |
| External-action approval | < 100% | No external automation |
| QA average | < 85 | Do not scale |
| Proof pack rate | < 100% | No case studies |
| Capital assets per project | < 1 | Agency risk |

## 3. Operating discipline

- Thresholds are tracked monthly.
- Breaches are logged in the Decision Log as `HOLD` decisions until resolved.
- Coverage gaps are owned by named operators.
- Threshold changes require a Decision Log entry; silent redefinition is forbidden.

## 4. Anti-patterns

- Reporting only the metrics that look favorable.
- "Roughly 100%" as a substitute for 100%.
- QA averages drifting downward without a paged response.
- Threshold redefinition for a board meeting.

## 5. The principle

> Institutional metrics are leading indicators of trust, not lagging indicators of revenue.
