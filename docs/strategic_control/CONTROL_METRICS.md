# Control Metrics

Measure **leadership**, not just performance. Control metrics are the early signals that the firm is actually leading, not merely shipping.

## 1. The metrics

- Data sources with Source Passport %
- AI runs logged %
- Outputs with governance status %
- Client-facing outputs with QA %
- Proof packs completed %
- Projects with capital assets %
- Manual steps productized %
- Retainer conversion %
- Business unit maturity %
- Market language adoption %

## 2. Decision thresholds (doctrine)

| Metric | Threshold | Decision |
| --- | --- | --- |
| Governance coverage | < 70% | Do not scale |
| QA average | < 85 | Fix delivery before expansion |
| Proof pack rate | < 100% | Do not claim value publicly |
| Capital assets per project | < 1 | Agency-risk flag |
| Repeated manual steps | < 3 | Do not build a feature yet |

## 3. Why these thresholds

They are the **lines below which the firm fools itself**. Operating below them is not a slow descent — it is a structural problem that compounds.

## 4. Operating discipline

- Thresholds are tracked monthly.
- Breaches are recorded in the Decision Log as `HOLD` decisions until resolved.
- Coverage gaps are owned by named operators, not by “the team.”
- Threshold changes require a Decision Log entry; silent redefinition is forbidden.

## 5. Anti-patterns

- Reporting only the metrics that look good.
- Letting QA average drift downward without a paged response.
- Treating proof pack rate < 100% as “acceptable for this quarter.”
- Threshold redefinition for a board meeting.

## 6. The principle

> What you fail to measure honestly, you eventually pay for twice.
