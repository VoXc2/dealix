# Control Domains

Seven domains define what Dealix actually controls. Without these, AI output becomes risk.

## 1. Data Control

- Source Passport
- Allowed Use
- PII Status
- Sensitivity
- Retention
- AI Access
- External Use

Without Data Control, any AI output may be dangerous.

## 2. AI Control

- LLM Gateway
- Model Router
- Prompt Registry
- Output Schema
- Eval Hook
- Cost Guard
- AI Run Ledger

Prevents lock-in to one model or one prompt style. Forces every AI call through the same governed surface.

## 3. Agent Control

- Identity
- Owner
- Allowed Inputs
- Allowed Tools
- Forbidden Actions
- Autonomy Level
- Audit
- Decommission Rule

Agent behavior emerges from execution paths. Without identity and scope, an agent fleet is ungovernable.

## 4. Workflow Control

- Workflow Owner
- Inputs
- Outputs
- Approval Path
- QA
- Proof Metric
- Operating Cadence

AI is allowed only inside a governed workflow.

## 5. Claim Control

- No claim without proof
- No guaranteed sales claims
- No fake proof
- No inflated Arabic marketing language

## 6. Channel Control

- Email: draft only
- WhatsApp: only with consent or an existing relationship
- No cold WhatsApp automation
- No LinkedIn automation
- No scraping engine

## 7. Capital Control

Every project must create:

- 1 Trust Asset
- 1 Product or Knowledge Asset
- 1 Expansion Path

## 8. The rule

A capability that touches one of these domains without the corresponding control is rejected. A new capability that introduces a new domain extends the list explicitly — silently expanding the surface is forbidden.

## 9. Failure modes

- Treating these domains as suggestions.
- Letting BU-specific exceptions accumulate.
- Adding capabilities that bypass a domain in the name of speed.
- “Just-this-once” deviations becoming permanent practice.
