## 1. Onboarding Checklist (يوم 0 → يوم 1)

### قبل البدء (يوم 0)
- [ ] تم استلام الدفع وتأكيده (bank transfer / Moyasar)
- [ ] تم إرسال رسالة ترحيب رسمية عربية خلال 2 ساعة
- [ ] تم مشاركة رابط Dealix Portal
- [ ] تم إرسال DPA للتوقيع
- [ ] تم جدولة اجتماع Kick-off (30 دقيقة)

### اجتماع Kick-off (يوم 1، 30 دقيقة)
- [ ] تعريف العميل بـ Sprint SOP (ماذا يتوقع كل يوم)
- [ ] جمع بيانات onboarding: pipeline، ICP، 3 صفقات حالية
- [ ] تحديد نقطة التواصل الرئيسية (مؤسس / مدير)
- [ ] توضيح draft_only policy: "أنت توافق على كل شيء قبل الاستخدام"
- [ ] إرسال ملخص Kick-off خلال ساعة

### رسالة ترحيب (مسودة)
```
مرحباً [الاسم]،

يسعدني أبلغك إن Sprint الإيرادات بـ Dealix بدأ رسمياً!

هذا ما سيحصل في الـ 7 أيام القادمة:
✓ يوم 1: تشخيص شامل لعملياتك
✓ يوم 2-3: تحديد أفضل 5 فرص في pipeline
✓ يوم 4-5: 5 مسودات رسائل مخصصة لمراجعتك
✓ يوم 6: Proof Pack أولي
✓ يوم 7: تسليم نهائي + خطة 30 يوم

تذكّر: لا يُرسَل أي شيء بدون موافقتك المباشرة.
أنت صاحب القرار دائماً.

reference: [DLX-XXXX]

[توقيع]
```

## 3. SLA الدعم (Support SLA)

| نوع الطلب | وقت الرد | وقت الحل |
|-----------|---------|---------|
| سؤال عاجل (مسودة توقفت) | < 2 ساعة | < 4 ساعات |
| طلب تعديل مخرج | < 4 ساعات | < 24 ساعة |
| استفسار عام | < 8 ساعات | < 48 ساعة |
| طلب تقرير إضافي | < 24 ساعة | < 72 ساعة |
| شكوى / تصعيد | < 1 ساعة (إقرار) | < 24 ساعة (خطة) |

**ساعات الدعم:** أحد–خميس 8 ص–6 م | جمعة–سبت: محدود

## 4. إشارات مخاطر التجديد (Renewal Risk Signals)

### 🔴 خطر مرتفع (تدخل فوري خلال 24 ساعة)
- [ ] لم يرد العميل على آخر تقرير أسبوعي
- [ ] رفض > 3 مسودات متتالية بدون ملاحظات
- [ ] أرسل رسالة "هل هذا مفيد فعلاً؟"
- [ ] لم يحجز اجتماع شهر 2 بعد نهاية sprint

### 🟡 خطر متوسط (متابعة خلال 72 ساعة)
- [ ] معدل رد على مسودات نزل < 50%
- [ ] تأخر في تقديم بيانات onboarding أسبوع 2+
- [ ] طلب "تجميد مؤقت"
- [ ] لم يستخدم Portal لأسبوعين

### 🟢 صحي (لا تدخل مطلوب)
- [ ] رد على 80%+ من المسودات
- [ ] طلب تقارير إضافية
- [ ] شارك Proof Pack مع أشخاص آخرين
- [ ] سأل عن مستوى Managed Ops

---

## 5. إشارات Upsell / Downgrade

### إشارات Upsell (اعرض الخطوة التالية)
- "هل يمكن تغطية قطاع آخر؟" → عرض Data-to-Revenue Pack
- "نريد هذا بشكل شهري" → عرض Managed Revenue Ops
- "أريد تقارير يومية" → عرض Executive Command Center
- طلب تدريب فريق → عرض حزمة team onboarding (مخصص)

### إشارات Downgrade (تعامل مع توقع)
- "الميزانية ضاقت هذا الربع" → عرض Sprint بدلاً من Managed Ops
- "لا أحتاج هذا الشهر" → اعرض pause (إذا كان مناسباً)
- "يحتاج فريق أكبر" → أجّل حتى يكون جاهزاً

## 6. أسباب الـ Churn الشائعة

| السبب | الوقاية |
|-------|---------|
| "لم أجد وقتاً للتطبيق" | قلل المطلوب منه + اجعل Sprint أكثر async |
| "النتائج لم تكن ما توقعت" | إدارة توقعات أوضح في Kick-off |
| "الثمن كثير على قيمة" | تأكد من توثيق كل proof event |
| "انتهت الميزانية" | اعرض Sprint بدلاً من المتابعة |
| "وجدنا بديل أرخص" | ذكّر بالفرق: عربي، PDPL، draft approval |

## 7. مقاييس نجاح العميل (Success Metrics)

### مقاييس كمية
| المقياس | هدف Sprint | هدف Managed (شهري) |
|---------|-----------|-------------------|
| مسودات موافق عليها | ≥ 4 من 5 | ≥ 15 من 20 |
| وقت رد العميل | < 24 ساعة | < 48 ساعة |
| proof events موثقة | ≥ 3 | ≥ 5 |
| رضا العميل | ≥ 4/5 | ≥ 4/5 |
| اجتماع تسليم تم | نعم | نعم |

### مقاييس نوعية
- هل العميل يشعر "بالوضوح" بعد التقارير؟
- هل يشارك المخرجات مع فريقه؟
- هل يسأل عن خطوات التوسع؟
