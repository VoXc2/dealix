# Dealix Governed AI Operations Standard (D-GAOS)

> Large enterprises do not trust a personal method. They trust a **standard** that is teachable, auditable, and repeatable. Dealix moves from Method to Standard.

Fourteenth strategic layer. Typed surfaces live in `auto_client_acquisition/standards_os/`.

## 1. The standard

**Name:** Dealix Governed AI Operations Standard (D-GAOS).

**Goal:** turn AI from random use into operating capability: clear data, defined workflow, AI assistance, human approval, runtime governance, Proof Pack, monthly improvement.

**Short form:** `Data → Workflow → AI → Governance → Proof → Operating Cadence`

## 2. Nine sub-standards

These make Dealix teachable, audit-able, and partner-able.

## 3. Capability Diagnostic Standard

7 capabilities × 0–5 maturity → output: Current Score, Target, Transformation Gap, Data Readiness, Governance Risk, Recommended Sprint, Expected Proof, Retainer Path.

Rule: no serious AI project without a Capability Diagnostic.

## 4. Data Readiness Standard

Score on source clarity, completeness, duplicates, missing fields, PII status, allowed use, format quality, update frequency. Tiers: 85+ ready, 70–84 cleanup, 50–69 diagnostic only, <50 readiness sprint first.

## 5. Source Passport Standard

Canonical schema (see `docs/sovereignty/SOURCE_PASSPORT_STANDARD.md` and `institutional_control_os/source_passport.py`). Five enforcement rules: no passport = no AI; no allowed use = internal only; PII unclear = redact/block; external action = approval required; unknown source = no outreach.

## 6. Runtime Governance Standard

Decision vocabulary `ALLOW`, `ALLOW_WITH_REVIEW`, `DRAFT_ONLY`, `REQUIRE_APPROVAL`, `REDACT`, `BLOCK`, `ESCALATE`. Fail-closed default is `DRAFT_ONLY`. 8 pre-action questions per call.

## 7. AI Agent Control Standard

Agent Card with `allowed_tools`. MVP allows autonomy 0–3 only.

## 8. AI Output QA Standard

QA dimensions: accuracy, source support, schema validity, Arabic quality, business usefulness, governance status, claim safety, actionability. Tiers: ≥90 client-ready, 80–89 review, 70–79 revise, <70 reject.

## 9. Proof Pack Standard

14-section Proof Pack v2 (see `proof_architecture_os/proof_pack_v2.py`). 8-component proof score (total 100). Tiers: 85+ case, 70–84 sales support, 55–69 internal, <55 weak.

## 10. Operating Cadence Standard

Weekly / Monthly / Quarterly cadence requirements. Rule: no retainer without cadence, no cadence without owner, no owner without metric, no metric without proof.

## 11. Capital Creation Standard

Each project must produce 1 Trust Asset + 1 Product/Knowledge Asset + 1 Expansion Path.

## 12. Certification System (companion: `docs/certification/`)

5 levels (Aware → Operator → Implementer → Certified Partner → Strategic Partner). Each: training + assessment + sample work + QA review + governance test + renewal.

## 13. Certification Exam — 4 tests

Knowledge / Case simulation / Governance test / QA review.

The Governance Test rejects partners who would propose cold WhatsApp automation; the correct answer is: classify PII, check passport, block cold WhatsApp, draft-only if consent basis, approval gate, log decision.

## 14. Partner Certification Gate

Passes certification + accepts Partner Covenant + submits sample delivery + passes QA + accepts audit + uses approved messaging. Suspension on unsafe claims, skipping Proof Pack, bypassing QA, misusing data, selling prohibited automation.

## 15. Public Standard Page

Publicly published Dealix Method page makes the market learn the language. Discipline: every claim sources to Proof; every refusal sources to doctrine.
