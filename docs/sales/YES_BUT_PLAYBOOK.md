# Yes, But Playbook (Safe Alternatives)

When the client asks for something **risky** or **vague**, do not only refuse — **redirect** to an allowed, valuable path.

| Client request | Yes, but response |
|----------------|-------------------|
| “WhatsApp للكل cold” | نعم نجهّز **مسودات** وجداول أولوية؛ **الإرسال** فقط بعد موافقة وسياسة تواصل واضحة؛ **لا cold WhatsApp** تلقائي. |
| “Scraping للسوق” | نبني **بحثًا** من مصادر **مسموحة** أو بيانات **تزودوننا بها**؛ لا scraping افتراضي بلا إطار قانوني/تعاقدي. |
| “نضمن مبيعات” | نضمن **مخرجات تشغيلية** (تقارير، ترتيب، حوكمة، proof pack)؛ **لا ضمان** نتائج سوق خارجة عن سيطرتنا. |
| “كل شيء Enterprise من اليوم” | نبدأ **Diagnostic** ثم **Sprint** ثم **Pilot** قبل توسيع AI OS — حتى لا نغرق في تخصيص بلا أساس. |
| “Automate LinkedIn DMs” | لا نقدم أتمتة واسعة مخالفة للسياسة؛ نركز على **محتوى drafts** وعمليات **موافقة**. |
| “Use any leaked list” | لا؛ مصادر يجب أن تكون **مبررة**؛ غير ذلك **research-only** أو رفض. |
