# Client Education — How to Buy AI Implementation

Use in **discovery** and **proposal** framing. Positions Dealix as an operator, not a tool vendor.

## Good AI projects need

1. **Clear business problem** — one primary metric
2. **Usable data** — exports, lawful basis, owners
3. **Process owner** — who decides and approves
4. **Human approval** — especially for external channels
5. **Quality review** — scored outputs before client sees them
6. **Measurable output** — proof pack, not vibes
7. **Governance** — PDPL, claims, retention, audit where needed

## Bad AI projects start with

- Random tools or “GPT for everything”
- No data owner
- No workflow definition
- No success metric
- No approval path for customer-facing text
- “Just integrate AI” without scope
