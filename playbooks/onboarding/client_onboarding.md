# Client Onboarding Playbook

## Objective

Onboard a client into a governed, traceable, and measurable Dealix operating workflow.

## Onboarding Steps

1. Collect tenant profile and business goals.
2. Configure users, roles, and RBAC mapping.
3. Connect approved data and execution systems.
4. Register enabled agents and permissions.
5. Configure workflow and approval rules.
6. Run pilot with supervised execution.
7. Review eval and business impact report.
8. Sign operational readiness and handoff.

## Onboarding Evidence

- tenant configuration record
- RBAC mapping record
- integration validation logs
- first successful governed workflow trace
- first evaluation report
- acceptance signoff

## Exit Criteria

Onboarding is complete only if:
- first end-to-end governed workflow succeeds,
- no cross-tenant or approval bypass issue occurs,
- rollback path is documented and tested.
