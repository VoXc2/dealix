الـ Multi-Tenancy

كل صف في كل جدول يحمل tenant_id (foreign key لـ tenants table). Isolation enforced على ثلاث مستويات:

  1. Middleware: api/middleware/tenant_isolation.py يستخرج tenant من JWT claims أو admin context قبل أي query
  2. ORM: الـ query patterns كلها تفرض where(Model.tenant_id == current_tenant_id)
  3. DB (production): Row-Level Security policies على Postgres

White-label theming (R6) عبر tenant_themes table (migration 007). Custom CSS variables تُحقن server-side قبل theme.css. كل tenant يستطيع override: palette، fonts، logo URL، display name، custom domain.

الـ Data Flow — Customer Inbound Reply (golden path)

1. Meta WhatsApp → POST /api/v1/webhooks/whatsapp
                   ├── verify x-hub-signature-256
                   └── parse Meta payload (text/image/audio)

2. Pipeline (auto_client_acquisition/pipeline.py)
                   ├── IntakeAgent: normalize + dedupe
                   ├── PainExtractorAgent: LLM extract intents
                   ├── ICPMatcherAgent: score against tenant ICP
                   ├── QualificationAgent: gate decision
                   └── CRMAgent: persist + sync

3. AI Workforce (/api/v1/ai-workforce)
                   ├── Select agent from 12-agent registry
                   ├── Generate Arabic reply candidate
                   └── Compute confidence score

4. Approval Engine
                   ├── If confidence > threshold AND policy allows: queue auto-send
                   ├── Otherwise: queue for human approval (Decision Passport entry)
                   └── Customer sees in /api/v1/approval-center

5. Decision Passport
                   ├── Record agent_id + version + evidence_level
                   ├── PDPL Art. 18 audit log entry
                   └── ZATCA-compliant if it's an invoice action

6. WhatsApp out (Meta Cloud API)
                   ├── Compliance gates (consent, quiet hours, opt-out)
                   ├── Send via integrations/whatsapp.py
                   └── Update tenant metrics

الـ Deploy

Production: Railway (Postgres + Redis + Python service) في region me-south-1 (Bahrain GCC).

Deploy flow: merge to main → Railway auto-deploys → migrate via Alembic pre-deploy hook → health check on /healthz → traffic shifted. Runbook في DEPLOY_RUNBOOK.md.

Rollback: Railway UI redeploy of last-green takes ~30s. If migration corrupted data: alembic downgrade -1 after backup verification.

Observability: Sentry (with PII scrubber) + Langfuse (LLM costs) + OpenTelemetry (OTel exporter to vendor of choice).
