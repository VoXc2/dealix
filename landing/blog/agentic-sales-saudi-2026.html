Agentic sales in 2026 — what's actually production-ready

12 May 2026 · Sami Assiri

The agent stack moves faster than the conferences can keep up. Here's the short list of what holds up in production at Dealix today, after running it across hundreds of Saudi tenant workflows.

1. LangGraph for state, not LangChain for chains

Chains break the second a step requires retry, fan-out, or human-in-the-loop. A state graph with checkpointed state in Postgres is the right primitive. Use the official langgraph.checkpoint.postgres — don't roll your own.

2. Computer Use for the long tail of RPA

Filing a ZATCA invoice through the Fatoorah portal, looking up a tender on Etimad — the long tail of "this thing has a web UI but no API" is what Computer Use cleans up. Constrain it with a tight system prompt + cost guard + screenshot diff.

3. MCP makes Dealix a tool inside the customer's editor

The Model Context Protocol turns every Dealix API into a tool Claude / Cursor / ChatGPT can call directly. dealix/mcp/server.py auto-registers every skill in the catalogue. Customers don't have to import our SDK — they just point their MCP-enabled IDE at our endpoint.

4. Promptfoo + Ragas for evaluation, not for "vibes"

Every PR that touches an agent runs the eval against a golden set. Score deltas are posted to the PR. This is the single hardest discipline to adopt and the single largest quality improvement once you do.

5. Lakera + NeMo Guardrails — prompt-injection defence isn't optional

Once any agent has tool-call ability + customer input, you have a prompt-injection surface. Lakera at the input boundary, NeMo Guardrails on the dialog, output validators on the response.

6. Cost guards turn the LLM bill into a budget

A per-request hard cap (default $0.50) and a per-tenant daily cap save you from runaway prompts. Trips downgrade to the cheaper model and ping the founder via Knock.
