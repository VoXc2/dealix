> **الغرض:** الصفحة الرئيسية للمسوّق — نقطة البداية لكل قرار تسويقي وتجاري  
> **المستخدمون:** فريق التسويق، المبيعات، القيادة التجارية  
> **آخر تحديث:** مايو 2026

## 1. جملة التموضع الرسمية (لا تُعدَّل)

> **"Dealix هي نظام تشغيل مؤسسي للإيراد والصفقات والشراكات والاعتمادات والتنفيذ، يحوّل القرار إلى تنفيذ قابل للتدقيق."**

هذه الجملة هي الـ anchor المرجعية — كل نسخة تسويقية، كل عرض، كل رسالة تُشتق منها وتعود إليها.

## 2. الأعمدة الثلاثة

| العمود | المعنى التشغيلي | الرسالة للعميل |
|--------|----------------|----------------|
| **نرفع سرعة القرار** | الموافقات التي تستغرق أسابيع تنتهي في ساعات — بسلاسل اعتماد واضحة وتدقيق لحظي | "قراراتك لا تنتظر بريداً إلكترونياً أو اجتماعاً طارئاً" |
| **نرفع جودة التنفيذ** | كل صفقة، كل شراكة، كل عقد — موثّق بمسار تدقيق كامل لا يمكن تعديله | "فريقك ينفّذ بدقة وليس بتخمين" |
| **نخفض الفوضى والمخاطر** | نُلغي تشتت الموافقات بين Excel وWhatsApp وSlack ونجمعها في سجل واحد قابل للتدقيق | "لا مفاجآت — كل قرار موثّق وكل انحراف مرئي" |

## 3. ماذا نبيع / ماذا لا نبيع

### ✅ ما نبيعه

| المنتج/الوحدة | الوصف في جملة |
|---------------|---------------|
| **Revenue OS** | نظام تشغيل الإيراد: خط مبيعات قابل للتدقيق من أول اتصال حتى التجديد |
| **Partnership OS** | نظام تفعيل الشراكات: اكتشاف الشريك ← تأهيله ← تفعيله ← تقييم أدائه ببطاقة نقاط |
| **Pricing & Approval Control** | ضبط التسعير والخصومات: كل خصم يمر بمسار اعتماد موثّق — لا تسرب في الهامش |
| **Executive/Board Operating Layer** | حزمة التشغيل التنفيذية: ملخص أسبوعي آلي للقيادة — القرارات المعلقة، الانحرافات، الفرص |
| **Trust & Evidence Layer** | طبقة الثقة والدليل: audit trail بـ hash chain، توافق PDPL، جاهزية GCC الكاملة |

### ❌ ما لا نبيعه (وما لا نقوله أننا إياه)

| ما لا نكونه | لماذا |
|-------------|-------|
| **CRM** (Salesforce / HubSpot) | نحن لا نخزّن جهات الاتصال — نُشغّل القرارات فوق أي CRM موجود |
| **Chatbot** أو **Chat widget** | لسنا أداة محادثة — نحن نظام تنفيذ وتدقيق |
| **Copilot** أو **AI Assistant** | لا نقترح — نُنفّذ ونوثّق ونُعيد التحكم للقيادة |
| **BI / Tableau / Power BI** | لسنا أداة رسم لوحات — نحن نُنتج القرار لا نعرض البيانات فحسب |
| **RPA / UiPath** | لسنا أتمتة للمهام الميتة — نحن تنسيق حي للصفقات والاعتمادات |
| **"منصة AI"** بلا تفاصيل | هذا وصف مبهم محظور — نُذكر ما يفعله النظام بالضبط |

## 4. ملفات العملاء المثاليين (ICP Deep-Dives)

### ICP-1: شركات B2B SaaS بصفقات متوسطة إلى كبيرة

| البُعد | التفاصيل |
|--------|----------|
| **الحجم** | 20–500 موظف، MRR بين 200K–5M ر.س |
| **آلية الشراء** | صفقات Enterprise تستلزم موافقات متعددة (Legal, Finance, C-suite) |
| **الألم الحقيقي** | الخصومات تخرج بلا توثيق — المندوب يعطي 40% خصم بدون اعتماد رسمي. التجديدات تُفوَّت لأن لا نظام يُنبّه قبل 90 يوماً من انتهاء العقد. |
| **الفرصة** | توحيد مسار الخصومات + مسار التجديد + تقرير الإيراد الأسبوعي للمؤسس |
| **الوتد** | "خصم واحد غير موثّق يضيّع هامشك — Dealix يجعل كل خصم قابلاً للتدقيق" |
| **ACV المتوقع** | 60K–240K ر.س/سنة |
| **القناة المفضلة** | LinkedIn + Email |
| **ملف القطاع** | [SaaS Playbook](./SECTOR_PLAYBOOKS_AR/saas.md) |

## 5. بطاقات القطاعات

### بطاقة: SaaS

```
القطاع:     SaaS B2B (الخليج)
الألم:      خصومات بلا توثيق + تجديدات مفوّتة + لا رؤية تنفيذية
الوتد:      Revenue OS + Pricing Control
الاتصال:   المؤسس / VP Sales / CFO
الإثبات:   عرض حي: مسار خصم → اعتماد → إغلاق في 7 دقائق
```

### بطاقة: Banking & Financial

```
القطاع:     بنوك وشركات تمويل وتأمين (SAMA-regulated)
الألم:      audit trail غير كامل + قرارات مشتتة + امتثال PDPL معقد
الوتد:      Trust & Evidence Layer + GCC Readiness
الاتصال:   CISO / Chief Compliance Officer / COO
الإثبات:   وثيقة أمنية + عرض audit export
```

### بطاقة: Large Enterprise

```
القطاع:     مؤسسات +500 موظف (متعددة الفرق)
الألم:      فوضى الاعتمادات + تقارير يدوية متأخرة + لا رؤية موحدة
الوتد:      Executive Control Room + Approval OS
الاتصال:   CEO / COO / VP Operations / CFO
الإثبات:   عرض الحزمة التنفيذية الأسبوعية الآلية
```

### بطاقة: Distribution & Supply

```
القطاع:     شركات توزيع وسلاسل إمداد (B2B)
الألم:      عقود الموزعين بلا تتبع + شراكات غير مقيّمة + ترقيات صامتة
الوتد:      Partnership OS + Scorecard
الاتصال:   VP Commercial / Commercial Director
الإثبات:   بطاقة نقاط الشريك الحي
```

### بطاقة: Government & Semi-Gov

```
القطاع:     جهات حكومية وشبه حكومية (Etimad)
الألم:      إثبات الامتثال للجهات الرقابية + audit دوري + قرارات غير موثّقة
الوتد:      Trust & Evidence Layer + PDPL Compliance
الاتصال:   مدير تقنية المعلومات / مدير الشراء / مستشار قانوني
الإثبات:   تصدير تقرير audit كامل بضغطة واحدة
```

## 7. قواعد الاستخدام الإلزامية

1. **لا تتجاوز CLAIMS_REGISTRY_AR.md** — كل ادعاء في أي مواد تسويقية يجب أن يكون في حقل "موافق عليه" في السجل.
2. **جملة التموضع ثابتة** — لا تختصرها أو تعيد صياغتها في المواد الرسمية.
3. **الأعمدة الثلاثة أولاً** — ابدأ كل عرض أو وثيقة بالأعمدة الثلاثة قبل الحديث عن الميزات.
4. **ICP أولاً** — كل رسالة يجب أن تُصنَّف لـ ICP محدد — لا رسائل عامة.
5. **[PENDING PILOT DATA]** — الوسم الوحيد المسموح به للبيانات غير المثبتة.
