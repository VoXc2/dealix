## 1. تعريف القطاع

| البُعد | التفاصيل |
|--------|----------|
| **الوصف** | بنوك تجارية، شركات تمويل، تأمين، FinTech regulated |
| **الجهات الرقابية** | SAMA (بنك مركزي سعودي)، SDAIA، NCA، ZATCA |
| **الحجم** | جميع الأحجام — المُحدِّد هو الخضوع للرقابة |
| **دورة البيع** | 12–24 أسبوع — RFP رسمية، لجان تقييم |
| **أولوية الربحية** | متوسطة–عالية: ACV كبير لكن دورة بيع طويلة |

## 2. نقاط الألم الرئيسية

| الألم | الظهور الفعلي | الأثر |
|-------|---------------|-------|
| **audit trail غير كامل** | القرارات المالية تمشي بإيميلات وواتساب — لا سجل رسمي قابل للتحقق | مخاطر SAMA + نتائج تدقيق سلبية |
| **امتثال PDPL معقد** | بيانات العملاء تنتقل بين أنظمة متعددة بدون consent موثّق | مخاطر غرامات PDPL (SDAIA) |
| **موافقات معقدة متعددة** | كل قرار مالي يمر بـ 4–6 مستويات — العملية بطيئة وغير شفافة | تأخير الصفقات + مخاطر compliance |
| **تقارير audit يدوية** | عند كل مراجعة SAMA — الفريق يجمع أدلة من أنظمة متفرقة | أسابيع عمل يدوي لكل مراجعة |
| **Data residency** | البيانات قد تخرج من KSA — مشكلة مع NCA وSAMA | قيود تشغيلية ومخاطر قانونية |

## 3. صنّاع القرار

| الشخصية | الدور | ما يهمه | كيف تصل إليه |
|---------|-------|---------|--------------|
| **Chief Compliance Officer** | الحارس — قراره إيقاف أو تمرير | SAMA compliance، audit readiness، PDPL | تواصل رسمي + وثيقة compliance |
| **CISO** | يُقيّم الأمن والبيانات | Data residency، تشفير، penetration testing | Technical briefing |
| **COO** | يريد كفاءة تشغيلية | تسريع الموافقات، تقليل العمل اليدوي | Executive pitch |
| **Head of IT / CTO** | يملك التكامل التقني | API، compatibility، لا downtime | Technical session |
| **CEO / Board** | القرار الاستراتيجي النهائي | مخاطر regulatory، تنافسية، تكلفة | C-suite pitch |

## 6. نقاط الإثبات للتأكيد عليها

```
✅ PDPL مُدمج في التصميم من اليوم الأول — ليس إضافة لاحقة
✅ hash chain audit trail — لا يمكن تعديله بعد التسجيل
✅ خيار hosting داخل KSA (Enterprise)
✅ تشفير AES-256 في الراحة + TLS 1.3 في النقل
✅ data subject rights مُطبّقة: وصول، حذف، تنقل
✅ وثيقة التصميم الأمني متاحة للمراجعة (تحت NDA)
❌ لا ادعاء SOC 2 أو ISO 27001 حالياً — في الخارطة
```

## 7. إرشادات التسعير

| النوع | النطاق | ملاحظات |
|-------|--------|---------|
| بنوك صغيرة / FinTech | 150–300K ر.س/سنة | يُشمل data residency setup |
| بنوك متوسطة | 300–600K ر.س/سنة | يشمل SLA مُعزَّز + compliance reports |
| بنوك كبيرة / مجموعات | 600K–2M ر.س/سنة | Custom — يحتاج تفاوض مع Board |
| لا تبدأ محادثة تسعير قبل تأهيل CCO وCISO |

```

الاشتراط:  موافقة CCO + CISO قبل انطلاق الـ Pilot
```
