## 1. تعريف القطاع

| البُعد | التفاصيل |
|--------|----------|
| **الوصف** | مؤسسات ذات فرق متعددة (+500 موظف)، تشغيل معقد، اعتمادات كثيرة |
| **الأنشطة** | تطوير عقاري، تجزئة، مجموعات صناعية، شركات تكنولوجيا كبيرة |

| **دورة البيع** | 8–20 أسبوع — لجنة شراء، IT، قانوني، مشتريات |
| **أولوية الربحية** | عالية — ACV كبير، فرص expansion كبيرة |

## 2. نقاط الألم الرئيسية

| الألم | الظهور الفعلي | التأثير |
|-------|---------------|---------|
| **فوضى الاعتمادات** | 15 فريق يُرسل موافقاتهم عبر WhatsApp وإيميل — لا أحد يعرف أين القرار | قرارات تتأخر أسابيع، فرص تُفقد |
| **تقارير يدوية للقيادة** | فريق كامل يُعدّ تقرير الأحد 2–4 ساعات — يصل بيانات الجمعة | القيادة تعمل بمعلومات قديمة |
| **لا رؤية إيراد موحّدة** | 5 خطوط أعمال — كل واحد تقريره الخاص — لا مجموع موحّد | CFO لا يرى الصورة الكاملة |
| **شراكات غير مُفعَّلة** | 20+ شريك استراتيجي — نصفهم لم يُشغَّل بشكل كامل | إيراد indirect غير محقق |
| **قرارات دورة الشراء** | كل شراء كبير يدور بين 6+ موافقين — لا أحد يتابع أين توقف | إهدار وقت الفريق والموردين |

## 6. نقاط الإثبات للتأكيد عليها

```
✅ حزمة تنفيذية أسبوعية تلقائية — لا عمل يدوي
✅ كل قرار له مسار اعتماد موثّق — لا يُفقد في الإيميلات
✅ لوحة CEO/CFO — بيانات في الوقت الفعلي
✅ API-first — لا استبدال لـ SAP أو ERP
✅ Partnership scorecard — لكل شريك أداء موثّق
[PENDING PILOT DATA]: ساعات توفير التقارير الأسبوعية
```

## 8. استراتيجية الدخول متعدد الخيوط (Multi-threading)

```
خيط 1 (من الأعلى): CEO/CFO — رسالة ROI والسيطرة
خيط 2 (من الوسط): COO/VP Ops — رسالة الكفاءة التشغيلية
خيط 3 (من التقنية): Head of IT — رسالة الأمان والتكامل

لا تعتمد على خيط واحد — في Enterprise، الـ champion ينتقل دائماً
```

## 9. نطاق Pilot المقترح (Enterprise)

```
النطاق:     فريق تجاري واحد (10–20 مستخدم) + الحزمة التنفيذية للـ COO
Workflows:  موافقات العقود التجارية + التقرير الأسبوعي التلقائي
المقاييس:
  - وقت إعداد التقرير الأسبوعي (قبل/بعد)
  - وقت دورة الموافقة على عقد (قبل/بعد)
  - رضا COO عن جودة البيانات في لوحة القيادة
المدة:       6–8 أسابيع
الاشتراط:   sponsor من C-suite (COO أو CFO) ليس فقط Middle management
```
