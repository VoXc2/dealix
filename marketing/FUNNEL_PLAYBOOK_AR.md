## نظرة عامة على القمع

```
TOFU (أعلى القمع) ─── الوعي والتعليم
  ↓ CTA: "تعلّم المزيد / تحميل / قراءة"

MOFU (وسط القمع) ─── الاهتمام والتقييم
  ↓ CTA: "احسب ROI / قارن / احجز عرضاً"

BOFU (أسفل القمع) ─── القرار والتحويل
  ↓ CTA: "ابدأ Pilot / تحدث مع فريقنا / وقّع"
```

## القسم الثالث: BOFU — 4 أصول أسفل القمع

> **الهدف:** تحويل المهتم الجاد إلى قرار شراء أو Pilot.  
> **القنوات:** email direct، اجتماع مباشر، proposal

---

### BOFU-01: وثيقة نطاق Pilot (Pilot Scope Template)

```
النوع:         PDF احترافي يُرسَل بعد العرض الثاني
المحتوى:
  - الهدف من الـ Pilot (بالاتفاق مع العميل)
  - الفريق المشارك + الأدوار
  - النطاق المحدود (workflow واحد أو فريق واحد)
  - معايير النجاح (3 مقاييس محددة)
  - الجدول الزمني (4–8 أسابيع)
  - Checkpoint صريح للتقييم
  - شروط الإيقاف إذا لم تتحقق المعايير
  - الالتزامات المتبادلة
CTA:         "نبدأ؟ وقّع وثيقة الـ Pilot"
```

---

### BOFU-02: خطة التنفيذ (Implementation Plan)

```
النوع:         وثيقة تقنية + تجارية مشتركة
المحتوى:
  الأسبوع 1:  إعداد بيئة Pilot + API integration assessment
  الأسبوع 2:  تكوين workflow الأول + تدريب الفريق الأساسي
  الأسبوع 3:  تشغيل حقيقي + monitoring مكثّف
  الأسبوع 4:  مراجعة البيانات الأولى + تعديلات
  الأسبوع 6:  تقرير checkpoint — قياس النتائج مقابل المعايير
  الأسبوع 8:  قرار التوسع أو الإيقاف
CTA:         "أرسل لنا بيانات البنية التقنية لنُكمل خطة التنفيذ"
```

---

### BOFU-03: ملخص أمني (Security Brief)

```
النوع:         PDF للمراجعة التقنية (IT + Security)
المحتوى:
  - بنية الأمان: TLS 1.3، AES-256، tenant isolation
  - PDPL compliance: data subject rights، consent log، retention policy
  - Data residency: خيارات hosting داخل KSA
  - Audit trail: hash chain، tamper-proof
  - Incident response: عملية الاستجابة للحوادث
  - ما لا نملكه بعد: SOC 2 في خارطة الطريق 2026–2027
تحذير:      راجع مع CLAIMS_REGISTRY_AR.md قبل الإرسال
CTA:         "لجلسة تقنية مع CISO/IT — تواصل مع فريق Solutions"
```

---

### BOFU-04: نموذج طلب العرض (Demo Request + Brief)

```
النوع:         Typeform أو صفحة landing مخصصة
المدخلات:
  - اسم الشركة والقطاع
  - الحجم (موظفين / إيراد تقريبي)
  - المشكلة المحددة (dropdown)
  - الأنظمة الحالية (CRM؟ ERP؟)
  - التوقيت (مدى الإلحاح)
  - المسؤول عن القرار
بعد الإرسال: تأكيد فوري + حجز موعد في تقويم فريق المبيعات
CTA:         → [HANDOFF_TO_SALES_AR.md](./HANDOFF_TO_SALES_AR.md) للـ MQL criteria
```
