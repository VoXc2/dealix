## 1. رسائل واتساب — فتح بارد (5 قوالب قطاعية)

> **مبادئ واتساب:**
> - رسالة واحدة — لا تُرسل عدة رسائل متتالية فوراً
> - لا تطلب شيئاً في الرسالة الأولى — اكسر الجليد
> - لهجة خليجية طبيعية — ليست رسمية جداً ولا عامية مبالغاً
> - لا opt-out = لا إرسال (PDPL متطلب)

```

السؤال ما يخص البيع — أبحث عن شركات SaaS تعاني من هذا الموضوع بالذات 
لأشاركهم شيئاً بنيناه.
```

## 3. رسائل لينكدإن (5 طلبات + 5 متابعة)

### طلبات التواصل

**LI-CON-01: SaaS**
```
[الاسم]، أرى شركتكم تنمو في مجال [SaaS/B2B]. 
أعمل على حلول ضبط مسار الموافقات والإيراد المؤسسي — 
أودّ التواصل لتبادل الأفكار إذا تسمحون.
```

**LI-CON-02: بنوك ومالية**
```
[الاسم]، خبرتكم في [المجال المالي] لفتت انتباهي. 
نعمل على ملف compliance وaudit trail للجهات الخاضعة لـ SAMA — 
أودّ التواصل إذا كان الموضوع يهمكم.
```

**LI-CON-03: Enterprise**
```
[الاسم]، أتابع توسع [الشركة] — ماشاءالله. 
أعمل على نظام القرار التنفيذي للمؤسسات الكبيرة. 
أودّ التواصل للتعرف أكثر.
```

**LI-CON-04: توزيع**
```
[الاسم]، خبرتكم في إدارة شبكات التوزيع مثيرة للاهتمام. 
عندي أفكار عن أتمتة تقييم الشركاء — أودّ تبادل الرأي إذا تسمحون.
```

**LI-CON-05: حكومي**
```
[الاسم]، أقدّر عملكم في [المجال الحكومي]. 
نعمل على حلول الامتثال الرقمي PDPL للجهات الحكومية — 
يسعدني التواصل.
```

---

### متابعة بعد القبول

**LI-FU-01: بعد قبول الطلب (48 ساعة)**
```
شكراً [الاسم] على القبول.

كنت أتساءل — في [اسم الشركة]، موافقات الصفقات الكبيرة كيف تمشي؟ 
هل عندكم workflow رسمي ولا يمشي عبر القنوات المعتادة؟

فضولي حقيقي — لأن كثير من الشركات في مجالكم تجد هذا تحدياً.
```

**LI-FU-02: بعد رد إيجابي**
```
يسعدني أنكم تفكرون في هذا الموضوع.

عندي 15 دقيقة أُريكم فيها كيف يبدو نظام Dealix في مسار مثل مسار شركتكم.
هل الأسبوع القادم يناسبكم؟ أو أرسل لي timing يناسبكم.
```

## 4. رسائل SMS (3 قوالب)

> **ملاحظة:** SMS للمتابعة والتذكير فقط — ليس للتواصل البارد الأول  
> **القاعدة:** لا إرسال بدون opt-in مسبق (PDPL)

### SMS-01: تذكير بموعد عرض

```
[الاسم]، مرحباً — تذكير بموعد عرض Dealix غداً [اليوم] الساعة [الوقت].
رابط الاجتماع: [الرابط]
للتأجيل: رد على هذه الرسالة أو تواصل عبر [رقم/إيميل]
```

### SMS-02: متابعة ما بعد العرض

```
[الاسم]، شكراً على وقتكم اليوم. 
أرسلت ملخص العرض وخطوات Pilot المقترحة على إيميلكم.
لأي سؤال — رد على هذا الرقم أو تواصل معي مباشرة.
```

### SMS-03: تذكير قبل انتهاء Pilot

```
[الاسم]، الـ Pilot ينتهي خلال 7 أيام. 
يسعدنا مراجعة النتائج معكم قبل القرار النهائي — 
هل 30 دقيقة هذا الأسبوع مناسبة؟
```

## 5. تسلسل المتابعة (4 لمسات في 14 يوماً)

> استخدم هذا التسلسل لكل ليد لم يرد على التواصل الأول

### اليوم 1 — الاتصال الأول (واتساب أو إيميل)

استخدم القالب المناسب للقطاع من القسم 1 أو 2 أعلاه.

---

### اليوم 3 — متابعة: زاوية مختلفة

**النسخة A (إيميل)**
```
الموضوع: Re: [اسم الشركة] — زاوية أخرى

[الاسم]،

ما أبي أكرر نفس الرسالة — خذوا هذا بدلاً عنها:

أهم سؤال يطرحه CFO كل آخر شهر على فريق المبيعات في شركات مثل شركتكم:
"وين راحت الصفقات اللي كانت في pipeline الشهر الماضي؟"

Dealix يُجيب على هذا السؤال تلقائياً — بدون تقرير يدوي.
١٥ دقيقة؟

[الاسم]
```

**النسخة B (واتساب)**
```
[الاسم]، أعرف تعبت من رسائل المبيعات 😅

أرسل لكم شيئاً واحداً مفيداً — مقال كتبناه عن تسرب هامش الخصومات 
في شركات SaaS الخليج. لو ما فيه value — لن أرسل مرة ثانية.

هل تسمحون؟
```

---

### اليوم 7 — محتوى: قيمة مجانية

```
الموضوع: [اسم الشركة] — 3 أسئلة يجب أن يعرف إجابتها مسؤول الإيراد

[الاسم]،

لا أطلب ردك — فقط أشارككم ثلاثة أسئلة نطرحها على كل VP Sales قبل اقتراح أي حل:

١. في آخر ربع سنوي — كم خصم أُعطي بدون وثيقة موافقة رسمية؟
٢. كم عقد تجديد ستنتهي صلاحيته خلال 90 يوماً — وهل فريقكم ينبّه لها تلقائياً؟
٣. لو CFO سأل عن أي صفقة بعينها اليوم — كم دقيقة لتجمع الملف الكامل؟

إذا الإجابات ما تبدو واضحة — هذا يعني فرصة فعلية.

[الاسم]
```

## 6. قواعد استخدام القوالب

| القاعدة | التفاصيل |
|---------|----------|
| **خصّص دائماً** | غيّر [الاسم] و[اسم الشركة] — لا ترسل قالباً حرفياً |
| **لا ادعاءات محظورة** | راجع [CLAIMS_REGISTRY_AR.md](./CLAIMS_REGISTRY_AR.md) قبل الإرسال |
| **توقيت الإرسال** | واتساب: 9–11 صباحاً أو 2–4 مساءً فقط |
| **لا رسائل متتالية** | انتظر على الأقل 72 ساعة بين رسالتين للشخص نفسه |
| **opt-in للـ SMS** | لا SMS بدون إذن مسبق — PDPL متطلب |
| **سجّل كل رسالة** | في نظام CRM أو قاعدة بيانات المتابعة |
