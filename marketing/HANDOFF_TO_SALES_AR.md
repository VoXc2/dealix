## 1. تعريف MQL (Marketing Qualified Lead)

الليد يُصبح MQL عندما يُحقق **3 من 5** من المعايير التالية:

| # | المعيار | الوزن | كيف يُقاس |
|---|---------|-------|-----------|
| 1 | **ICP fit**: قطاع + حجم يتطابقان مع ICP-1/2/3 | عالي | يدوي أو scoring |
| 2 | **Intent signal**: تفاعل مع محتوى MOFU/BOFU (تحميل، حضور webinar، زيارة صفحة demo) | عالي | Analytics |
| 3 | **Contact quality**: عنوان وظيفي يشير لصلاحية (VP+، Director+، مؤسس) | متوسط | LinkedIn/CRM |
| 4 | **Pain match**: أبدى اهتماماً بمشكلة تتطابق مع ألم ICP (موافقات، audit، إيراد) | عالي | من conversation أو form |
| 5 | **Timing signal**: استجاب لتواصل في آخر 14 يوماً أو طلب عرضاً | عالي | CRM |

## 2. البيانات الإلزامية عند كل تسليم

> **لا تسليم** بدون اكتمال هذه الحقول في بطاقة الليد

| الحقل | الوصف | إلزامي؟ |
|-------|-------|---------|
| **اسم الشركة** | الاسم الكامل الرسمي | ✅ |
| **القطاع** | (SaaS / بنوك / Enterprise / توزيع / حكومة) | ✅ |
| **الحجم التقريبي** | موظفين أو إيراد سنوي تقريبي | ✅ |
| **الألم المحدد** | الجملة التي قالها الليد أو استنتاج واضح | ✅ |
| **مستوى الإلحاح** | (فوري / 1–3 أشهر / استكشافي) | ✅ |
| **صانع القرار** | الاسم والمنصب والدور في القرار | ✅ |
| **Wedge** | الدخول المقترح: أي منتج/وظيفة للبدء به | ✅ |
| **نطاق ACV المقدّر** | (صغير <100K / متوسط 100K–300K / كبير 300K+) | ✅ |
| **الاعتراضات المسبقة** | ما سُمع من مخاوف حتى الآن | مهم |
| **مصدر الليد** | (LinkedIn / email / WhatsApp / إحالة / حدث) | ✅ |
| **آخر تفاعل** | التاريخ + ملخص ما قيل | ✅ |
| **الأنظمة الحالية** | CRM؟ ERP؟ Approval system؟ | مهم |

## 3. قواعد عدم التسليم (Disqualification Rules)

**لا تُسلَّم MQL** في أي من الحالات التالية:

```
🚫 الـ Wedge غير محدد — "مهتمين بشكل عام" لا يكفي
🚫 لا صلة بـ ICP — حجم الشركة أو القطاع بعيد جداً عن ICP-1/2/3
🚫 لا صانع قرار محدد — الاتصال مع جهة لا تملك الصلاحية
🚫 Timing = "ربما السنة القادمة" بدون signal إضافي
🚫 لا ألم محدد — "نريد نعرف فقط" بدون مشكلة فعلية ذُكرت
🚫 المؤسسة في قائمة "لا نبيع لها" (competitor، خارج GCC بدون استثناء)
```

## 4. معايير قبول المبيعات (SQL — Sales Accepted Lead)

المبيعات تقبل الـ MQL وتُحوّله لـ SQL عند التحقق من:

| المعيار | كيف يُتحقق |
|---------|------------|
| **صانع القرار مُؤكَّد** | تحدث معه المبيعات بالفعل — ليس مُفترَضاً | 
| **Wedge واضح** | المبيعات توافق على نقطة الدخول المقترحة |
| **الألم مُؤكَّد** | صانع القرار اعترف بالمشكلة بكلامه |
| **لا مانع تقنية واضح** | IT أو security لم يرفض مبدئياً |
| **التوقيت معقول** | الـ Pilot يمكن أن يبدأ خلال 60 يوماً |

## 6. عملية التسليم — خطوة بخطوة

```
الخطوة 1:  التسويق يُكمل بطاقة الليد في CRM (كل الحقول الإلزامية)
           ↓
الخطوة 2:  التسويق يُرسل إشعار تسليم على قناة المبيعات + رابط البطاقة
           ↓
الخطوة 3:  AE يراجع البطاقة ويُقرر: قبول → SQL / رفض مع سبب / طلب معلومات إضافية
           ↓ (خلال 24 ساعة)
الخطوة 4:  إذا مقبول → AE يتواصل مع الليد مباشرة ويُعرّف نفسه
           ↓
الخطوة 5:  AE يُحدث CRM بنتيجة التواصل الأول
           ↓
الخطوة 6:  كل أسبوع — مراجعة pipeline بين التسويق والمبيعات
```

## 7. تصنيف الـ Wedge (نقطة الدخول)

كل ليد يجب أن يُخصَّص له **Wedge واحد للبدء** — لا "نبيع كل شيء"

| الـ Wedge | يناسب | الـ Demo الأول |
|-----------|-------|----------------|
| **ضبط الخصومات** | SaaS + أي شركة بمندوبين | Demo 1 |
| **الحزمة التنفيذية الأسبوعية** | CEO/CFO في Enterprise | Demo 3 |
| **audit trail PDPL** | Regulated sectors | Demo 4 |
| **Partnership Scorecard** | توزيع + Enterprise | Demo 2 |
| **موافقات سريعة** | أي ICP مع ألم "القرارات تتأخر" | Demo 1 أو 3 |

## 8. التعامل مع MQL المرفوضة

إذا رفضت المبيعات MQL — التسويق يُسجّل السبب ويُقرر:

| سبب الرفض | إجراء التسويق |
|-----------|---------------|
| "لا صانع قرار" | أعِد إلى nurture sequence + ابحث عن contact مناسب |
| "لا ألم محدد" | أضف لـ MOFU content sequence لمدة 30 يوم |
| "التوقيت بعيد" | أضف لـ "re-engage 90 days" sequence |
| "خارج ICP" | أوقف — لا resources على leads خارج ICP |

---

## 9. التقرير الأسبوعي المشترك (Marketing & Sales Sync)

كل أسبوع — الاثنين — جلسة 30 دقيقة:

```
التسويق يُقدّم:
  - عدد MQL هذا الأسبوع + المصدر
  - MQL رُفضت + السبب
  - أداء المحتوى + الـ funnel

المبيعات تُقدّم:
  - SQL accepted هذا الأسبوع
  - demo/pilot في المسار
  - أهم اعتراضات جديدة سُمعت (→ تُضاف لـ OBJECTION_HANDLING_AR.md)

المشترك:
  - هل الـ MQL quality تتحسن؟
  - هل هناك قطاع يُنتج leads أجود؟
  - هل يوجد bottleneck في الـ handoff؟
```
